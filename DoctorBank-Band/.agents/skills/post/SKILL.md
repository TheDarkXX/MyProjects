# /drbpost — DoctorBank-Band Post Writer

> **Slash Command:** `/drbpost [style] [หัวข้อ]`
> สั่งให้ AI เขียนโพสต์ขายอาหารเสริมสำหรับ DR.BANK
> พร้อม Score 20 เกณฑ์ (200 คะแนน) + Viral Suggestion ทุกครั้ง

```

```

## 1. Cold Start Protocol — อ่านก่อนเขียนทุกครั้ง

```
AI MUST อ่านไฟล์ต่อไปนี้ตามลำดับ ก่อนเขียนโพสต์:
1) memory.md                              → Iron Rules 18 ข้อ
2) Brand/brand-memory.md                  → Brand identity, Tone, Compliance
3) Products/[สินค้า]/PRODUCT_MEMORY.md     → สูตร, สารสำคัญ, USP, ข้อจำกัด
4) ไฟล์ Skill นี้ (SKILL.md)               → กฎกลาง + Style Router
5) Blueprint ของ Style ที่เลือก            → สูตรเขียนทั้งหมด (Persona, Tone, Formula, Tricks, Formatting จบในไฟล์เดียว)
```

> [!CAUTION]
> **Iron Rules ที่ห้ามละเมิดเด็ดขาด:**
> - Rule 9: ห้ามสับสน "Mg L-Threonate" กับ "L-Theanine"
> - Rule 11: ห้ามใช้คำว่า "หมอ" "แพทย์" ในการขายสินค้า
> - Rule 12: ตรวจสอบ Dose ในสูตร vs งานวิจัยก่อนอ้างอิง
> - Rule 14: ยึดฐาน "1 แคปซูล" เท่านั้น ห้ามหลอนใส่สารที่ไม่มี
> - Rule 17: ห้ามใช้คำเคลมเสี่ยง อย. (รักษา, หายขาด, การันตี)

## 2. Style Router — ระบบเลือกสไตล์

> [!IMPORTANT]
> **กฎสำคัญ: แต่ละ Style มีไฟล์ Blueprint เป็นสูตรเขียนครบจบในไฟล์เดียว (แกะมาจากเพจต้นแบบแต่ละเพจ)**
> เมื่อ User สั่ง `/drbpost [style] [หัวข้อ]` → AI MUST ไปอ่าน **Blueprint** ของ Style นั้นก่อนเขียนโพสต์
> ถ้า User ไม่ระบุ Style → ใช้ Default Style

### Style Registry

| Style ID | ต้นแบบจากเพจ | Blueprint | Default? |
|---|---|---|---|
| `winbiz` | BestWinbiz | `Content/Writing/Styles/Health_Supplement_Winbiz_Style/Blueprints/Health_Supplement_Writing_Blueprint.md` | ✅ |

> **เพิ่ม Style ใหม่:** clone จากเพจใหม่ → สร้างโฟลเดอร์ใน `Content/Writing/Styles/[ชื่อ Style]/` → ใส่ Blueprint ใน `Blueprints/` → เพิ่มรายการในตารางนี้

### วิธี Route

```
เมื่อ User สั่ง: /drbpost [style_id] [หัวข้อ]

1. แยก style_id จากคำสั่ง
   - ถ้าไม่ระบุ → ใช้ Default (winbiz)
   - ถ้าระบุแต่ไม่ตรงกับ Style Registry → ถาม User
2. ไปอ่าน Blueprint ในโฟลเดอร์ Style นั้น:
   → Blueprint คือสูตรเขียนทั้งหมด (Persona, Tone, Formula, Tricks, Formatting, Checklist)
   → ไม่ต้องหาเพิ่มที่อื่น Blueprint จบครบในตัว
3. อ่าน Examples/*.md เป็นตัวอย่างอ้างอิงเพิ่มเติม (ถ้ามี)
4. เขียนโพสต์ตามสูตรใน Blueprint
5. ตรวจ Compliance (Section 3 ของ Skill นี้)
6. ให้ Score (Section 4 ของ Skill นี้)
```

## 3. Compliance — คำต้องห้ามทางกฎหมาย (ใช้ร่วมทุก Style)

```
❌ ห้ามอ้างว่า "รักษาโรค" / "หายขาด" / "ป้องกันโรค"
❌ ห้ามใช้คำว่า "ยา" กับอาหารเสริม
❌ ห้ามอ้างชื่อหมอ/โรงพยาบาลโดยไม่ได้รับอนุญาต
❌ ห้าม Before/After ที่เกินจริง
❌ ห้ามใช้คำว่า "การันตีผล 100%"
❌ ห้ามใช้คำว่า "หมอ" "แพทย์" ในการขายสินค้า
❌ ห้ามใช้ "แก้นอนไม่หลับ" "หลับทันที" "ไม่มีผลข้างเคียง"
```

**คำ Safe ที่ใช้แทน:**

| ❌ ห้ามใช้ | ✅ ใช้แทน |
|---|---|
| รักษา | ดูแล / เสริมสร้าง |
| หายขาด | รู้สึกดีขึ้น / เปลี่ยนแปลง |
| ป้องกันโรค | ช่วยดูแลร่างกาย |
| ต้องกิน | อาจลองดู / เป็นทางเลือก |
| ดีที่สุด | เป็นหนึ่งในตัวเลือกที่ดี |
| การันตี | ส่วนตัวผมรู้สึกว่า / ลูกค้าหลายคนบอกว่า |
| แก้นอนไม่หลับ | สนับสนุนการพักผ่อน |
| ผ่าน อย. แน่นอน | (ห้ามพูดจนกว่าจะผ่านจริง) |

## 4. Scorecard — 20 เกณฑ์ (200 คะแนนเต็ม) — ใช้ร่วมทุก Style

> **AI ต้องให้คะแนนทุกครั้งหลังเขียนโพสต์เสร็จ** แสดงเป็นตารางพร้อมเหตุผล + Viral Suggestion

### 🎣 HOOK & ATTENTION (หยุดนิ้ว)
| # | เกณฑ์ | คำอธิบาย |
|---|---|---|
| 1 | **HOOK** | 2 บรรทัดแรกมีตัวเลข/ความขัดแย้ง/ความลับ ที่ต้องอ่านต่อ |
| 2 | **CURIOSITY GAP** | สร้างช่องว่างความอยากรู้ (บอกว่ามีอะไร แต่ยังไม่บอกว่าคืออะไร) |

### 📖 READABILITY (อ่านง่าย)
| # | เกณฑ์ | คำอธิบาย |
|---|---|---|
| 3 | **SIMPLE** | ภาษาง่าย คนทั่วไปเข้าใจ ไม่ต้องกูเกิลศัพท์ |
| 4 | **RHYTHM** | จังหวะบรรทัดสั้น-ยาวสลับกัน ไม่น่าเบื่อ |

### 🎭 EMOTION & STORY (อารมณ์)
| # | เกณฑ์ | คำอธิบาย |
|---|---|---|
| 5 | **SCENE** | มีฉากที่ผู้อ่านเห็นตัวเองในนั้น (ไม่ใช่แค่ข้อมูล) |
| 6 | **FEELING** | อ่านแล้วรู้สึกอะไรบางอย่าง (กลัว/โล่ง/ตาสว่าง/มั่นใจ) |
| 7 | **ARC** | มี Emotional Arc (เจ็บ → เข้าใจ → โล่ง → มั่นใจ) |

### 🔥 BUILD-UP & WOW (ดำเนินเรื่อง)
| # | เกณฑ์ | คำอธิบาย |
|---|---|---|
| 8 | **TENSION** | มีจุดที่สร้างความตึงเครียด/อยากรู้ (ก่อนเฉลย) |
| 9 | **REFRAME** | มีจุดพลิกมุม "อ้าว! ไม่เคยคิดแบบนี้" |
| 10 | **PAYOFF** | อ่านจบแล้วรู้สึกว่า "คุ้มที่อ่าน" |

### 📚 CONTENT QUALITY (เนื้อหา)
| # | เกณฑ์ | คำอธิบาย |
|---|---|---|
| 11 | **COMPLETE** | ครอบคลุม Key Points สำคัญ |
| 12 | **WHY** | ทุกข้อมี "ทำไม" / "เพราะ" อธิบายกลไก |
| 13 | **DATA** | มีตัวเลข/สถิติ/งานวิจัย อ้างอิงได้ |
| 14 | **LOGIC** | มีเหตุ → ผล ไม่พูดลอยๆ |

### 📣 VIRAL & ENGAGEMENT (ไวรัล)
| # | เกณฑ์ | คำอธิบาย |
|---|---|---|
| 15 | **COMMENT BAIT** | มีคำถาม/ชวนตอบ ที่คนอยากพิมพ์ |
| 16 | **SHARE TRIGGER** | มีเหตุผลที่คนจะแชร์ ("ต้องบอกเพื่อน!") |
| 17 | **TAG MOMENT** | มีจุดที่คนอยากแท็กเพื่อน/แฟน/แม่ |
| 18 | **SAVE WORTHY** | มีเนื้อหาที่คนอยากเซฟไว้ดูทีหลัง |

### 💰 SELL & CTA
| # | เกณฑ์ | คำอธิบาย |
|---|---|---|
| 19 | **SOFT SELL** | ขายไม่เกิน 20% + มีทางเลือกธรรมชาติก่อน |
| 20 | **CTA CLEAR** | CTA ชัดเจน ทำตามง่าย (พิมพ์คำเดียว/ทักแชท) |

### วิธีตัดเกรด
```
90-100% (180-200) = 🔥 Viral Masterpiece — พร้อมโพสต์เลย!
75-89%  (150-179) = ✅ ดีมาก — โพสต์ได้ อาจปรับเล็กน้อย
60-74%  (120-149) = ⚠️ พอใช้ — ควรปรับก่อนโพสต์
ต่ำกว่า 60% (<120) = ❌ ยังไม่พร้อม — ต้องปรับหลายจุด
```

## 5. Version Control — กฎเขียนเพิ่ม ไม่เขียนทับ (ใช้ร่วมทุก Style)

> [!CAUTION]
> **กฎเหล็ก: เมื่อเขียน Version ใหม่ ให้เขียนต่อท้ายเสมอ ห้ามเขียนทับ Version เดิม**

```
แต่ละ Version ต้องมี:
1. หัวข้อ: "# ตัวอย่างโพสต์ V[N]: [ชื่อ]"
2. คำอธิบาย: "*ปรับจาก V[N-1] — [สิ่งที่เปลี่ยน]*"
3. Changelog block:
   ```text
   [เปลี่ยน] ...
   [เพิ่ม] ...
   [ตัด] ...
   ```
4. เนื้อหาโพสต์เต็ม
5. Score ตาราง 20 เกณฑ์
6. Viral Suggestion
```

## 6. Workflow — ขั้นตอนการเขียนโพสต์

```
เมื่อ User สั่ง /drbpost [style] [หัวข้อ]:

1. อ่าน Cold Start files (memory.md → brand-memory.md → PRODUCT_MEMORY.md)
2. Style Router: ดึง Blueprint ของ Style ที่ระบุ
   - ถ้าไม่ระบุ Style → ใช้ Default (winbiz)
   - ถ้า Style ไม่อยู่ใน Registry → ถาม User
3. ถาม User (ถ้าไม่ระบุ):
   - หัวข้อ/สารอาหาร?
4. อ่าน Blueprint ของ Style นั้น (สูตรเขียนครบจบในไฟล์เดียว)
5. อ่าน Examples ของ Style นั้น เป็นตัวอย่างอ้างอิง (ถ้ามี)
6. เขียนโพสต์ตามสูตรใน Blueprint
7. ตรวจ Compliance (Section 3 — คำต้องห้าม + Iron Rules)
8. ให้ Score 20 เกณฑ์ + Viral Suggestion (Section 4)
9. ถ้า Score < 75% → แนะนำจุดปรับปรุง
10. ถ้า User ขอแก้ → เขียน Version ใหม่ต่อท้าย (Section 5 — ไม่เขียนทับ)
```

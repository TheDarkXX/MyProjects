# /clonepost — Page Writing Style Reverse Engineering

> **Slash Command:** `/clonepost [path-to-file]`
> แกะ + วิเคราะห์สไตล์การเขียนจากโพสต์ของเพจใดก็ได้
> ผลลัพธ์คือ Writing Blueprint ระดับ Clone Kit พร้อมใช้

---

## วิธีใช้

```
/clonepost C:\path\to\posts.csv
/clonepost C:\path\to\posts.xlsx
/clonepost C:\path\to\posts.txt
```

### รองรับไฟล์หลายแบบ

| Format | วิธีอ่าน |
|---|---|
| **CSV** | อ่านคอลัมน์ `text` (หรือคอลัมน์ที่มีเนื้อหาโพสต์) |
| **Excel (.xlsx/.xls)** | เปิดด้วย script หรืออ่าน sheet แรก หาคอลัมน์เนื้อหาโพสต์ |
| **TXT** | แต่ละโพสต์คั่นด้วยบรรทัดว่าง 2+ บรรทัด หรือ separator (`===`, `---`) |

> คอลัมน์เสริมที่มีประโยชน์: `reactions`, `comments`, `shares`, `timestamp`, `postType`
> ถ้าไม่มีคอลัมน์ engagement → ใช้ความยาวโพสต์เป็นตัวเรียงแทน

### ขั้นตอนที่ 0: จัดประเภทโพสต์ (Post Classification)

```
จากไฟล์ทั้งหมด:
1. อ่านเนื้อหาโพสต์ทุกแถว
2. นับบรรทัดจริงของแต่ละโพสต์ (ตัดตาม line break)
3. จัดประเภทโพสต์ทั้งหมดเป็น 4 กลุ่ม:

   📝 Long-form     (≥ 50 บรรทัด)  → วิเคราะห์ทุกโพสต์
   📋 Medium-form   (20-49 บรรทัด) → วิเคราะห์เฉพาะที่มี engagement สูง
   💬 Short-form    (5-19 บรรทัด)  → สแกนหา pattern/วลีซ้ำ
   📌 Micro         (< 5 บรรทัด)   → ข้ามไป

4. เรียงตาม engagement (reactions + comments + shares) จากมากไปน้อย
5. สรุปจำนวน:

   | ประเภท | จำนวน | วิเคราะห์ |
   |---|---|---|
   | Long-form (≥50) | [N] | ✅ ทั้งหมด |
   | Medium-form (20-49) | [N] | ⚡ เฉพาะ TOP engagement |
   | Short-form (5-19) | [N] | 🔍 สแกน pattern |
   | Micro (<5) | [N] | ⏭ ข้าม |
```

> [!IMPORTANT]
> **เน้นวิเคราะห์ Long-form เป็นหลัก** เพราะเป็นตัวชี้สไตล์การเขียนที่ชัดที่สุด
> แต่ถ้า Long-form มีน้อย (< 3 โพสต์) → ดึง Medium-form มาวิเคราะห์เพิ่ม
> Short-form ใช้เสริมข้อมูล pattern/วลี ไม่ใช้เป็นตัวหลัก

### ขั้นตอนที่ 2: PART 2 — DNA ของสไตล์การเขียน

วิเคราะห์ 2 มิติ:

#### 2.1 Writing Persona (ตัวตนในการเขียน)
```
สิ่งที่ต้องหา:
- บทบาท: เขาวาง Position ตัวเองเป็นอะไร? (ครู, พี่ชาย, เพื่อน, ผู้เชี่ยวชาญ, คนธรรมดา?)
- "ไม่ใช่": เขา "ไม่ใช่" อะไรอย่างชัดเจน? (ไม่ใช่ Guru, ไม่ใช่พ่อค้า?)
- สรรพนาม: ใช้ "ผม" "เรา" "พี่" หรืออะไร?
- จุดยืน: มี "แบรนด์ตัวเอง" ในฐานะคนเขียนไหม? (เช่น "แบรนด์กบฏ")
- ระดับภาษา: ทางการ, กึ่งทางการ, บ้านๆ, สแลง?
```

เขียนเป็นตาราง:
```
| มิติ | ลักษณะ |
|---|---|
| **บทบาท** | [ค่าที่พบ] |
| **ไม่ใช่** | [ค่าที่พบ] |
| **สรรพนาม** | [ค่าที่พบ] |
| **จุดยืน** | [ค่าที่พบ] |
| **ระดับภาษา** | [ค่าที่พบ] |
```

#### 2.2 Signature Elements (ลายเซ็นที่จำได้)
```
สิ่งที่ต้องหา:
- คำ/วลีที่ใช้ซ้ำทุกโพสต์ (เช่น ลงท้ายว่า "ครับ" ทุกประโยค?)
- ลายเซ็นปิดท้าย (เช่น "พี่เบส แบรนด์กบฏ")
- วลี Transition ที่ใช้ประจำ (เช่น "ตรงนี้แหละครับ", "เห็นไหมครับ")
- Series/Hashtag ที่ใช้ซ้ำ (เช่น #วิชาหาเงิน)
```

### ขั้นตอนที่ 3: PART 3 — TONE & EMOTION

วิเคราะห์ 4 มิติ:

#### 3.1 Tone Spectrum
```
สิ่งที่ต้องหา:
- อ่านแต่ละโพสต์ แบ่งเป็นช่วง: Hook, ชี้ปัญหา, Reframe, สอน, ยกตัวอย่าง, สรุป, CTA
- แต่ละช่วงใช้ Tone ยังไง? (กระตุ้น, จี้จุด, อ่อนโยน, มั่นใจ, เป็นกันเอง, หนักแน่น, อบอุ่น?)
- สร้างตาราง Tone Spectrum:

| ช่วงของโพสต์ | Tone | ตัวอย่างจริง |
|---|---|---|
| Hook | [Tone] | "[Quote จริงจากโพสต์]" |
| ชี้ปัญหา | [Tone] | "[Quote จริง]" |
| ... | ... | ... |
```

#### 3.2 Emotional Arc
```
สิ่งที่ต้องหา:
- ทุกโพสต์ยาวมี "เส้นอารมณ์" ซ่อนอยู่ไหม?
- แต่ละขั้นทำให้ผู้อ่านรู้สึกอะไร?
- สร้าง Emotional Arc:

📉 ขั้น 1: [ชื่อ] (เช่น Recognition)
   → ผู้อ่านรู้สึก: "[...]"
📉 ขั้น 2: [ชื่อ] (เช่น Pinch)
   → Quote จริง: "[...]"
📈 ขั้น 3: [ชื่อ] (เช่น Relief)
   → "[...]"
...จนจบ
```

#### 3.3 Emotion Injection (เทคนิคฉีดอารมณ์)
```
สิ่งที่ต้องหา:
- ใส่เสียงลูกค้า/ผู้อ่านจริงๆ ไหม? (" " quotes)
- ตั้งคำถาม rhetorical ที่ไม่ตอบทันทีไหม?
- เปรียบเทียบ 2 ความรู้สึกคู่ขนานไหม?
- มีวงเล็บ Insight ทิ้งท้ายหมวดไหม?

เขียนเป็นตาราง:
| เทคนิค | วิธี | ตัวอย่างจริง |
|---|---|---|
```

#### 3.4 Tone Rules (กฎที่ไม่เคยเปลี่ยน)
```
สิ่งที่ต้องหา:
- สิ่งที่ "ไม่เคยทำ" (เช่น ไม่เคยดุ, ไม่เคยอวด, ไม่เคยกดดัน)
- สิ่งที่ "ทำเสมอ" (เช่น ใช้ "ไม่ผิดครับ" ก่อน Reframe)
```

### ขั้นตอนที่ 4: PART 4 — AUDIENCE

วิเคราะห์ 3 มิติ:

#### 4.1 กลุ่มเป้าหมายที่ระบุชัดในเนื้อหา
```
สิ่งที่ต้องหา:
- เขาเรียกกลุ่มเป้าหมายว่าอะไรบ้าง?
- คัดทุกคำเรียกมาลิสต์ (เช่น "เจ้าของกิจการ", "คนวัย 30+", "แม่ลูกอ่อน")
```

#### 4.2 วิธีเรียกผู้อ่าน (Addressing the Reader)
```
สิ่งที่ต้องหา:
- ใช้ "คุณ" ตรงๆ ตอนไหน?
- ใช้ "เรา" (รวมตัวเอง) ตอนไหน?
- ใช้ "หลายคน" (ไม่ชี้หน้า) ตอนไหน?
- ใช้ตำแหน่ง/อาชีพ เรียกตอนไหน?

เขียนเป็นตาราง:
| วิธี | ตัวอย่าง | ใช้เมื่อ |
|---|---|---|
```

#### 4.3 วิธีทำให้ผู้อ่านรู้สึก "เขาพูดกับเรา" (Identification)
```
สิ่งที่ต้องหา:
- เขียนอาการที่คนอ่านเจอจริงไหม? (ลิสต์)
- ใช้ "อาจ" "บางครั้ง" เพื่อไม่กดดันไหม?
- ยกย่องผู้อ่านก่อนสอนไหม?
- แต่ละเทคนิคมีตัวอย่างจริง
```

### ขั้นตอนที่ 5: PART 5 — VOCABULARY & WORD USAGE

วิเคราะห์ 4 มิติ:

#### 5.1 Signature Vocabulary (คำที่ใช้ซ้ำ)
```
แบ่งเป็นกลุ่ม:
🔵 คำเปิดเรื่อง (Opening Words) — คัดทุกวลีเปิดโพสต์
🟡 คำพลิกมุม (Reframe Words) — คัดทุกวลี Reframe
🟢 คำยกตัวอย่าง (Example Words) — คัดทุกวลียกตัวอย่าง
🔴 คำย้ำประเด็น (Emphasis Words) — คัดทุกวลีย้ำ
🟣 คำสรุป (Closing Words) — คัดทุกวลีปิดท้าย
```

#### 5.2 การผสมภาษา
```
- ใช้คำอังกฤษแบบไหน? (ศัพท์เฉพาะ? ปนในประโยค?)
- มีกฎอะไรบ้าง? ✅ ใช้ / ❌ ไม่ใช้
- วิธีใส่คำอังกฤษ (Hashtag? คู่กับไทย? ชื่อ Framework?)
```

#### 5.3 Word Frequency (คำที่ใช้บ่อยสุด)
```
นับความถี่คำสำคัญจากทุกโพสต์ เรียงจากมากไปน้อย TOP 10:
| อันดับ | คำ | ความถี่ | หน้าที่ |
```

#### 5.4 ลูกเล่นการใช้คำพิเศษ
```
สิ่งที่ต้องหา — รูปแบบประโยคที่ใช้ซ้ำ:
- "ไม่ใช่... แต่คือ..." (Negation-Revelation)?
- "ต่อให้... ก็..." (Concession)?
- "ก่อนที่... ต้อง... ก่อน" (Sequential Logic)?
- "อย่ารีบ... ก่อนที่..." (Don't Rush Pattern)?
- Hashtag ฝังในเนื้อหา?
- รูปแบบอื่นๆ ที่พบ?

แต่ละลูกเล่น ต้องมี:
1. ชื่อลูกเล่น
2. สูตร (Pattern)
3. ตัวอย่างจริงจากโพสต์
4. จุดประสงค์
```

### ขั้นตอนที่ 6: PART 6 — WRITING TRICKS (เทคนิคลูกเล่น)

```
สิ่งที่ต้องหา:
สแกนทุกโพสต์ เทียบกับ Trick Bank ด้านล่าง ถ้าพบ Trick ไหน → บันทึก + ยก Quote จริง

Trick Bank (ตรวจทุกข้อ):
□ Trick: Reframe the Problem (พลิกมุมปัญหา)
□ Trick: Before vs After (เปรียบก่อน-หลัง)
□ Trick: Analogy / Metaphor (เปรียบเทียบ)
□ Trick: Cascading Short Lines (ประโยคสั้นซ้อน)
□ Trick: Bracketed Maxim (คำคมในกรอบ [ ])
□ Trick: Parenthetical Insight (วงเล็บคม ( ))
□ Trick: Rhetorical Question Chain (โซ่คำถาม)
□ Trick: Repetition with Variation (ซ้ำแต่เปลี่ยนมุม)
□ Trick: Emotional Dialogue (บทสนทนาอารมณ์/เสียงในหัว)
□ Trick: Internal Monologue Quote (ใส่เสียงลูกค้า/ผู้อ่าน)
□ Trick: Case Study Embed (ฝัง Case Study)
□ Trick: Gift + Comment CTA (ของขวัญแลก Comment)
□ Trick: อื่นๆ ที่พบเพิ่มเติม (ตั้งชื่อเอง)

แต่ละ Trick ที่พบ ต้องเขียน:
### 🎯 Trick [N]: "[ชื่อ]" ([คำอธิบายสั้น])
**สูตร:**
[Pattern template]
**จากโพสต์จริง:**
[Quote ตัวอย่างจริง]
> **จุดประสงค์**: [ทำไมถึงใช้ + ผลที่ได้]
```

### ขั้นตอนที่ 7: PART 7 — โครงสร้างการเรียบเรียง (Post Structure)

#### 7.1 Macro Structure
```
สิ่งที่ต้องวิเคราะห์:
- แบ่งโพสต์เป็น Opening / Body / Closing → สัดส่วนเป็น % เท่าไหร่?
- วาด Box diagram:
  ┌──────────────────────┐
  │ Opening — [X]%        │
  │ [หน้าที่]              │
  ├──────────────────────┤
  │ Body — [X]%           │
  │ [หน้าที่]              │
  ├──────────────────────┤
  │ Closing — [X]%        │
  │ [หน้าที่]              │
  └──────────────────────┘
```

#### 7.2 Opening (ขึ้นต้น) — รายละเอียด
```
สิ่งที่ต้องวิเคราะห์:
- โครงสร้างละเอียด: [บรรทัดที่ 1-2] HOOK, [3-5] CONTEXT, [6-15] PAIN, [16] เส้นคั่น
- เทียบ Opening ของทุกโพสต์ → หา Pattern ที่ซ้ำ
- สร้างตาราง HOOK types:
  | แบบ | ตัวอย่าง |
  |---|---|
  | สัญญาผลลัพธ์ | "[...]" |
  | ท้าทายความเชื่อ | "[...]" |
  | ตัวเลข + คุณค่า | "[...]" |
  | สถานการณ์เฉพาะ | "[...]" |
  | อื่นๆ | "[...]" |
```

#### 7.3 Body (ส่วนกลาง) — รายละเอียด
```
สิ่งที่ต้องวิเคราะห์:
- มี Loop structure (ส่วนย่อยที่ซ้ำ) ไหม?
- วาด Loop template:
  ┌─────────────────┐
  │ SEPARATOR        │
  │ KEY POINT        │
  │ DETAILS          │
  │ EXAMPLE          │
  │ (INSIGHT)        │
  │ SEPARATOR        │
  └─────────────────┘
      ↓ ซ้ำ [N] ครั้ง

- กฎจังหวะ:
  ✅ ทุก [N] บรรทัด ต้องมี [อะไร]?
  ✅ ทุก [N] หมวด ต้องมี [อะไร]?

- การจัดลำดับเนื้อหา:
  ลำดับที่ 1: [...]
  ลำดับที่ 2: [...]
  ...
```

#### 7.4 Closing (ปิดท้าย) — รายละเอียด
```
สิ่งที่ต้องวิเคราะห์:
- โครงสร้าง: [สรุป] → [ภาพอนาคต] → [CTA] → [Hashtag]?
- CTA มีกี่แบบ? (Follow, Comment, Sell?)
- ยก Quote จริงของ Closing ที่ดีที่สุด
```

### ขั้นตอนที่ 10: PART 10 — POST FORMATTING RULES

> [!IMPORTANT]
> ส่วนนี้ต้อง **วัดจริง** จากข้อมูลโพสต์ — ไม่ใช่เดา

#### 10.1 ความยาวบรรทัด (Line Length)
```
สิ่งที่ต้องวัด:
- นับจำนวนตัวอักษรต่อบรรทัดของทุกโพสต์ที่วิเคราะห์
- แบ่งเป็น 4 ช่วง: 1-15, 16-30, 31-60, 61+ ตัวอักษร
- คำนวณสัดส่วน %
- คำนวณค่าเฉลี่ย + ค่ากลาง

เขียนเป็นตาราง:
| ช่วง | จำนวน | สัดส่วน | บทบาท |
|---|---|---|---|

กฎทอง:
✅ [สรุปจากข้อมูลจริง]
```

#### 10.2 การตัดบรรทัด & เว้นบรรทัด
```
สิ่งที่ต้องวิเคราะห์:
- กฎ "1 บรรทัด = 1 ความคิด" ใช้ไหม?
- มีบรรทัดว่างระหว่างประโยคไหม?
- ตัดบรรทัดตรงไหน? (ก่อน "แต่"? ก่อน Quote? ก่อนข้อสรุป?)
- ประโยค impact อยู่เดี่ยวไหม?

แต่ละกฎมีตัวอย่าง ❌ ผิด vs ✅ ถูก
```

#### 10.3 การตัดคำ (Word Breaking)
```
สิ่งที่ต้องวิเคราะห์:
- ตัดที่ "รอยต่อความคิด" ไม่ใช่ตามจำนวนตัวอักษร
- สร้างตารางจุดตัดที่ดี:
  | จุดตัดที่ดี | ตัวอย่าง |
  |---|---|

- สิ่งที่ไม่ควรทำ (ตัดกลางประโยค, ตัดกลางวลี)
```

#### 10.4 สัญลักษณ์คั่น (Separators)
```
สิ่งที่ต้องวัด:
- ใช้เส้นคั่นแบบไหนบ้าง? (━, ─, ═, —, ___)
- แต่ละแบบใช้กี่ครั้ง?
- ใช้เมื่อไหร่?
- จังหวะการวาง (ทุกกี่บรรทัด?)

เขียนเป็นตาราง:
| สัญลักษณ์ | ชื่อ | จำนวนที่พบ | ใช้เมื่อ |

Pattern การใช้ (เช่น Pattern A: คั่นหมวด, Pattern B: ครอบ Maxim, ...)
```

#### 10.5 Emoji Usage
```
สิ่งที่ต้องนับ:
- จำนวน Emoji ที่พบในโพสต์ยาว = [N] ตัว
- ถ้า 0 → นี่คือ Design Choice สำคัญ อธิบายเหตุผล
- ถ้ามี → Emoji ไหนที่ใช้? ใช้ตรงไหน? ใช้กี่ตัว/โพสต์?
- สัญลักษณ์ที่ใช้แทน Emoji (ถ้ามี)
```

#### 10.6 สัญลักษณ์ที่ใช้จริงทั้งหมด
```
สร้างตาราง:
| สัญลักษณ์ | หน้าที่ | ตัวอย่าง | ความถี่ |

+ รายการสัญลักษณ์ที่ "ไม่เคยใช้"
```

#### 10.7 Visual Rhythm (จังหวะทางสายตา)
```
สิ่งที่ต้องวิเคราะห์:
- วาด Visual Rhythm diagram (กล่องแสดงจังหวะสั้น-กลาง-ยาว)
- สรุปสูตรจังหวะ: เช่น "สั้น สั้น สั้น → กลาง → สั้น → [เส้นคั่น] → ซ้ำ"
- กฎ "ไม่เกิน N" (บรรทัดยาวติดกันไม่เกินกี่บรรทัด?)
```

#### 10.8 ความยาวรวมของโพสต์
```
เขียนเป็นตาราง:
| ประเภท | ความยาว | จำนวนเส้นคั่น | จำนวน Maxim |
|---|---|---|---|
```

#### 10.9 Hashtag Formatting
```
สิ่งที่ต้องวิเคราะห์:
- จำนวน Hashtag ทั้งหมด
- ตำแหน่งที่วาง (ฝังในเนื้อหา? ท้ายโพสต์? ใน Maxim?)
- กฎ Hashtag (ไทย/อังกฤษ? กี่ตัว/โพสต์?)
- Hashtag ท้ายโพสต์ — เรียงลำดับ (Brand > Series > Topic > Product > Persona)
```

#### 10.10 สูตร Format สำเร็จรูป
```
สร้าง Copy-Paste Format Template ที่ใช้ได้กับทุกหัวข้อ
ยาว ≥ 30 บรรทัด ใส่ comment กำกับ
```

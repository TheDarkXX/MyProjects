# Skill: `/cons` (Consensus MCP Research)

**Command:** `/cons`
**Trigger:** เมื่อผู้ใช้พิมพ์คำสั่ง `/cons` ตามด้วยหัวข้อวิจัยที่ต้องการค้นหา
**Description:** ใช้ Consensus MCP Server ในการค้นหางานวิจัยวิชาการและเปเปอร์ที่มีการ Peer-Review จากฐานข้อมูลกว่า 200 ล้านฉบับของ Consensus เพื่อนำมาสนับสนุนเนื้อหาหรือหา Insight

## ⚙️ Workflow (การทำงาน)

1. **ตรวจสอบคำค้นหา (Query):** 
   - ดึงหัวข้อที่ผู้ใช้ต้องการค้นหาหลังจากคำสั่ง `/cons`
   - หากผู้ใช้พิมพ์แค่ `/cons` ให้ถามผู้ใช้กลับว่าต้องการค้นหางานวิจัยเรื่องใด

2. **เรียกใช้ MCP Tool:** 
   - ใช้เครื่องมือ `call_mcp_tool` ของ IDE
   - **ServerName:** `consensus`
   - **ToolName:** `search`
   - **Arguments:** `{ "query": "<user_research_topic>" }`

3. **การจัดรูปแบบผลลัพธ์ (Formatting Results):**
   - เมื่อได้รับผลลัพธ์จาก Consensus ให้ดึงเปเปอร์ที่เกี่ยวข้องที่สุดมาแสดงผล
   - บังคับการอ้างอิง: ทุกครั้งที่มีการอ้างถึงงานวิจัย **ต้องใส่ Inline Citation** เสมอ เช่น `[1]`, `[2]`
   - สรุปเนื้อหาสำคัญ (Abstract / Key Findings) ให้ผู้ใช้อ่านเข้าใจง่าย
   - ส่วนท้ายของการตอบกลับ (References): ต้องมีรายการอ้างอิงเปเปอร์ทั้งหมดที่ใช้ พร้อมรายละเอียดดังนี้:
     - `[1] [Paper Title](url) (Authors, Year, Journal, Citations)`
     - ⛔ **ห้ามดัดแปลง ย่อ หรือสร้าง URL ใหม่** ให้ใช้ URL ดั้งเดิมจากผลลัพธ์ของ Tool เท่านั้น!
   - ⛔ **กฎเหล็กของ Consensus:** หากในผลลัพธ์มีข้อความแจ้งเตือนเกี่ยวกับการใช้งาน, โควต้า, หรือการอัปเกรด (Sign-up, upgrade, usage message) **จะต้องคัดลอกข้อความเหล่านั้นมาแสดงไว้ตอนท้ายสุดแบบคำต่อคำ (word-for-word)** เสมอ

4. **ระบบ Fallback (กรณีที่ IDE ยังไม่ได้โหลด Tool):**
   - หาก `call_mcp_tool` แจ้งเตือนว่า `tool search is not enabled for server consensus` หรือไม่พบ Tool
   - ให้แจ้งผู้ใช้ว่า **"ระบบมีการแคช Auth Token ไว้เรียบร้อยแล้ว แต่ IDE ยังจำสถานะเดิมอยู่ รบกวนผู้ใช้ปิดแล้วเปิด AG IDE ใหม่อีกครั้งเพื่อเริ่มการเชื่อมต่อครับ"**

## RAW ARTIFACT BACKUP

### 1. Content Posting API

| ฟีเจอร์ | รองรับ? | รายละเอียด |
|---|:---:|---|
| ภาพ Cover | YES | video: video_cover_timestamp_ms / photo: photo_cover_index |
| แปะตะกร้าสินค้า | NO | Official API ไม่มี → ต้องใช้ Browser Automation |
| Caption | YES | title หรือ description field |
| ใส่เพลงฮิต | LIMITED | auto_add_music เท่านั้น ระบุ Music ID ไม่ได้ |
| ตั้งเวลาโพสต์ | NO | ต้องสร้าง Cron Job เองบน Server |

Video Analytics (Business API เสริม):
- Scope: video.list + video.analytics
- ได้: Views, Likes, Comments, Shares, Audience Insights
- ได้เฉพาะบัญชีที่ Auth ให้แอปเราเท่านั้น

### 2. TikTok Shop API (Seller API)

Base URL: https://open-api.tiktokglobalshop.com
Auth: OAuth 2.0 via x-tts-access-token
Version: v202309

2.1 Product API: POST/GET /product/202309/products
- สร้าง/แก้ไข/ลบสินค้า, SKU & สต็อก Real-time, Category Tree, Global Products

2.2 Order API: GET /order/202309/orders
- ดึงออเดอร์ filter (สถานะ/วันที่/buyer ID), รายละเอียดครบ
- จัดการ: ยืนยัน, ยกเลิก, Split Shipment
- Webhooks Real-time ทุก Status Change

2.3 ข้อมูลผู้ซื้อ (Privacy)
- ได้: Recipient Name, Shipping Address, Phone, Buyer ID (internal)
- ไม่ได้: TikTok Username, Email, Payment Method, ประวัติซื้อนอกร้าน
- เหตุผล: PDPA/GDPR

2.4 Logistics API: POST /logistics/202309/shipments
- สร้าง Shipment, Shipping Label, อัปเดต Tracking, สถานะ Real-time, Return/Refund

2.5 Analytics API (สำคัญมาก)
- GET /analytics/202409/shop_videos/performance (Scope: data.shop_analytics.public.read)
  → Views, GMV จากวิดีโอ, SKU Orders, Units Sold, CTR ของตะกร้า
- Product-level: GMV/ออเดอร์/คืนสินค้า รายสินค้า/รายวัน/รายเดือน
- Shop Overview: Revenue by Channel (วิดีโอ vs Live vs Shop Tab)

2.6 Finance API
- Settlement Reports, Transaction History, Commission & Fee, Statement IDs

2.7 Affiliate API
- สร้าง Collaboration Campaign, ดึง Performance รายชื่อ Creator
- Promotion: Creator Exclusive Price, Flash Deals

2.8 Customer Service API
- ดึง Inquiries, Reply ผ่าน API (Auto-reply Bot), Dispute/Return

2.9 Webhooks: ORDER_STATUS_CHANGE, PRODUCT_STATUS_CHANGE, PACKAGE_UPDATE, RETURN_STATUS_CHANGE

สรุปตาราง:
| หัวข้อ | ทำได้? | หมายเหตุ |
|---|:---:|---|
| ยอดขายรายสินค้า GMV | YES | รายสินค้า/รายวัน |
| วิดีโอ Analytics | YES | shop_videos endpoint |
| รายชื่อผู้ซื้อชื่อจริง | NO | Privacy Policy |
| ที่อยู่ผู้รับสินค้า | YES | เพื่อ Logistics เท่านั้น |
| สถานะจัดส่งทุก Order | YES | ครบทุกสถานะ |
| Settlement/ยอดเงิน | YES | Finance API |
| Affiliate Performance | YES | ต้องขอสิทธิ์พิเศษ |
| Customer Messages | YES | Customer Service API |
| Stock/Inventory | YES | Real-time sync |
| Discount/Flash Deal | YES | Promotion API |

### 3. Marketing API (Ads)
- Campaign/AdGroup/Ad: สร้าง/จัดการ/ปิด
- Creative: Batch manage วิดีโอ/รูป + Performance
- Audience: Custom Audiences
- Reporting: Performance at scale
- Advanced: Split Test API
- SDKs: Python, Java, JavaScript

## Key Decisions

D1: แปะตะกร้า Official API ทำไม่ได้ ระบบในตลาดใช้:
1. Browser Automation RPA (เสี่ยงแบน)
2. Reverse-Engineered Private API (เทา-เสี่ยงสูง)
3. TikTok Shop Native Auto-Post ใน Seller Center (ทางการ 100%)

D2: รายชื่อผู้ซื้อ ได้แค่ Recipient Name + Address สำหรับ Logistics เท่านั้น

# Discord Bot AG-Level Intelligence Upgrade (Omega Mode)

## Context
Boss requested a plan to upgrade the existing Discord Bot (which currently relies on single-shot LLM calls and limited hardcoded `brain-query.sh` commands) to have AG-level capabilities. This means giving the bot access to the filesystem, database, system commands, and the ability to autonomously loop through tool-use logic.

## Analysis & Decision
The existing system is constrained by a lack of an autonomous tool-calling loop. We proposed a 3-phase architecture named **"Omega Mode"**:
1. **System Knowledge Injection:** Giving the bot full context about the VPS layout, PM2 services, and DB schema.
2. **Tool-Use Agent (Core):** Implementing a Gemini Function Calling loop that allows the bot to iteratively read files, execute commands, and fetch API data before answering.
3. **Self-Healing Integration:** Proactive monitoring and auto-fixing capabilities.

This represents a major architectural shift. The plan outlines the creation of an `Omega Agent Router` and an `Omega Tool Loop` to safely execute actions on the VPS.

# Discord Bot → AG-Level Intelligence Upgrade

## ปัญหาที่ต้องแก้

ปัจจุบัน Discord Bot ของเราเก่ง**แค่ในขอบเขตที่ hardcode ไว้** — มันรู้แค่สิ่งที่ system prompt บอก + Brain API context ที่ inject ให้ก่อน AI call แต่ AG (Antigravity) เก่งกว่ามากเพราะมี **tools** — มันอ่านไฟล์ได้, รัน command ได้, SSH ได้, แก้โค้ดได้, deploy ได้

## Gap Analysis: Discord Bot vs AG

| ความสามารถ | Discord Bot ปัจจุบัน | AG (Antigravity) |
|---|---|---|
| **รู้ข้อมูลระบบ** | ❌ แค่ brain context (tasks, memories, notes) | ✅ อ่านไฟล์ได้ทุกไฟล์, list directory, grep |
| **แก้โค้ด** | ⚠️ แค่ Fast-Edit ผ่าน hydra | ✅ edit ไฟล์ตรงๆ, สร้างไฟล์ใหม่ |
| **รัน command** | ❌ ไม่ได้ (ยกเว้น hardcoded scripts) | ✅ รัน command อะไรก็ได้ |
| **ดู logs / debug** | ❌ ไม่ได้ | ✅ อ่าน log, ดู error output |
| **PM2 / service control** | ⚠️ มี restart API แต่ bot เรียกไม่ได้ | ✅ restart, reload, view logs |
| **Git operations** | ❌ ไม่ได้ | ✅ commit, push, pull |
| **System health check** | ❌ ไม่ได้ (watchdog แยกต่างหาก) | ✅ เช็ค CPU, RAM, disk, processes |
| **Self-healing** | ⚠️ Hydra + Watchdog แยกกัน | ✅ เข้าใจปัญหาแล้วแก้ได้ทันที |
| **Knowledge base** | ❌ แค่ system prompt | ✅ อ่าน KI, memory, docs ได้ |

## สิ่งที่ AG ทำได้ที่ Bot ทำไม่ได้ (Root Cause)

**AG มี Tool-Use Architecture** — มันไม่ใช่แค่ "ถาม AI แล้วแสดงคำตอบ" แต่ AI สามารถ:
1. **เลือกใช้ tool** ที่เหมาะสม (file read, command exec, grep, etc.)
2. **ดู output** แล้วตัดสินใจขั้นต่อไป
3. **วนลูป** จนกว่าจะเสร็จ (multi-step reasoning)

Discord Bot ปัจจุบัน = **Single-shot**: ถาม 1 ที → ตอบ 1 ที → จบ

## Proposed Architecture: "Omega Mode" 🐙

แทนที่จะทำ bot ใหม่ เราจะ **upgrade #dev-agent channel** ให้เป็น "Omega Mode" — agent ที่มี tool-use capability เต็มรูปแบบ

```
User Message (Discord #dev-agent or #hydra-core)
        │
        ▼
┌──────────────────────────┐
│ Omega Agent Router       │
│ (detect: need tools?)    │
├──────────────────────────┤
│ NO tools needed          │ YES tools needed
│ → Normal AI chat         │ → Enter Omega Loop
│   (fast, cheap)          │
└──────────────────────────┘
        │                          │
        ▼                          ▼
  Regular Reply           ┌─────────────────────┐
                          │ Omega Tool Loop      │
                          │ (max 10 iterations)  │
                          │                      │
                          │ 1. AI picks tool     │
                          │ 2. Execute tool      │
                          │ 3. Feed result back  │
                          │ 4. AI decides next   │
                          │ 5. Repeat or finish  │
                          └─────────────────────┘
                                   │
                                   ▼
                            Final Reply to User
```

> [!IMPORTANT]
> **Risk Levels ใช้กฎ R0/R1/R2 จาก AGENTS.md:**
> - **R2 (safe)**: Bot ใช้ได้เลย ไม่ต้องถาม
> - **R1 (costly to reverse)**: Bot ใช้ได้ แต่ต้องบอก user ก่อนว่าจะทำอะไร
> - **R0 (irreversible)**: Bot ต้องถาม user ก่อน + รอ confirm

### 2.2 Omega Agent Loop

#### [NEW] `discord-bot/lib/omega-agent.js`
Multi-step agent loop ที่ใช้ Gemini function calling:

```javascript
export async function runOmegaAgent(userMessage, systemContext, channelId) {
  const messages = [{ role: 'user', content: userMessage }];
  const MAX_ITERATIONS = 10;
  
  for (let i = 0; i < MAX_ITERATIONS; i++) {
    const result = await callAIWithTools(systemPrompt, messages, OMEGA_TOOLS);
    
    if (result.type === 'text') {
      return result.content; // Final answer
    }
    
    if (result.type === 'tool_call') {
      // Execute tool
      const toolResult = await executeTool(result.tool, result.args);
      // Feed result back to AI
      messages.push({ role: 'tool', content: toolResult });
    }
  }
  
  return '⚠️ Omega Agent: ถึง iteration limit แล้วครับ';
}
```

## User Review Required

> [!IMPORTANT]
> **เลือก Strategy:** มี 2 แนวทาง:
>
> **A) "Omega Channel" (แนะนำ)** — สร้าง channel ใหม่ `#omega` ที่เป็น super-agent เฉพาะ แยกจาก dev-agent ปกติ ข้อดีคือ ไม่ disturb agent อื่น + ค่อยๆ เพิ่ม capability ได้
>
> **B) "Upgrade Dev-Agent"** — เพิ่ม Omega mode เข้า dev-agent ที่มีอยู่ ข้อดีคือ ไม่ต้องสร้าง channel ใหม่ แต่อาจซับซ้อนกว่า

> [!WARNING]
> **Security Concern:** Bot จะมีสิทธิ์รัน shell command บน VPS ดังนั้น:
> - ต้องมี command whitelist ที่เข้มงวด
> - ต้องมี path whitelist (ห้ามเข้าถึง `/etc/`, `/usr/`, `/var/`)
> - R0 operations (git push, file delete, service stop) ต้องถาม user ก่อนเสมอ
> - ทุก tool execution ต้อง log ไว้

> [!CAUTION]
> **Token Cost:** Omega Agent ใช้ multi-turn (avg 3-5 AI calls per question) ดังนั้นค่า token จะสูงกว่า normal chat ~3-5x
> แนะนำใช้ Gemini 2.5 Flash (ฟรี/ถูก) เป็น default model

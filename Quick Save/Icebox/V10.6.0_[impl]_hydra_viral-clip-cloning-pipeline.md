# 🚀 Implementation Plan: Viral Clip Cloning Pipeline

## 📋 What & Why
บอสมี workflow ที่ทำซ้ำเป็นประจำ: ค้นหาคลิป viral ใน FB Reels -> ดาวน์โหลด -> แกะ pattern คลิปเองด้วยตา -> เขียน image + motion prompt -> นำไป gen คลิปใหม่ใน Veo 3.1 / Grok / GPT. กระบวนการนี้กินเวลา 30-60 นาทีต่อคลิป.

**Solution:** เราจะใช้ AI ทำแทน 100% ในส่วนของการดาวน์โหลดและการวิเคราะห์วิดีโอ โดยใช้ `yt-dlp` และ Native Google API (`gemini-2.5-flash` ผ่าน Google AI Studio) ซึ่งรับไฟล์ .mp4 ได้โดยตรง และประหยัดค่าใช้จ่ายที่สุด (ประมาณ 0.02 บาท/คลิป) โดยไม่ต้องวุ่นวายกับการแตกสับเฟรมด้วย `ffmpeg` และจะยิงข้อมูล Prompt Table กลับไปที่ **Google Sheets** อัตโนมัติเพื่อให้ง่ายต่อการ Copy.

## 🛠️ Proposed Changes (4 Phases)

### Phase 1: MVP — `!clone <url>`
1. ติดตั้ง `yt-dlp` บน VPS
2. สร้าง `scripts/off-peak-agents/viral-clip-analyzer.js`
   - รับ URL
   - Download `.mp4`
   - Upload ผ่าน Google File API & วิเคราะห์ด้วย `gemini-2.5-flash`
   - LLM Pass 2 ดัดแปลงเป็น character Doctor Skill
3. สร้าง `scripts/off-peak-agents/lib/sheets-output.js` เพื่อ push ข้อมูลเข้า Google Sheets ที่มีอยู่
4. เพิ่มคำสั่ง `!clone <url>` ใน `discord-bot/lib/ai.js`

### Phase 2: Auto-Scraper (FB Reels)
1. ติดตั้ง Apify FB Reels Scraper (ใช้เครดิตฟรี $5/month)
2. สร้างตาราง `viral_target_pages` ใน Database (กำหนดเพจเป้าหมาย และขั้นต่ำยอดวิว)
3. รันเช็คคลิปใหม่รายสัปดาห์

### Phase 3: Off-Peak Cron
1. ผูกระบบเข้ากับ `scripts/cron-auto-planner.js` ช่วงตี 1 - ตี 5
2. ถ้าเจอคลิปน่าสนใจ -> Download -> Analyze -> ส่งแจ้งเตือนสรุปใน Discord และ Google Sheets

### Phase 4: Brain App UI
1. สร้างหน้า Dashboard สำหรับ "Clip Analyzer" บน Brain App
2. เพิ่มช่อง Input URL, ปุ่มกดวิเคราะห์ และแสดงตารางผลลัพธ์บน UI

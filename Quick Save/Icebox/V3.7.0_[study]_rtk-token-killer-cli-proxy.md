## 📌 Executive Summary

RTK คือ **CLI proxy** ที่เขียนด้วย Rust ทำหน้าที่ **ดักจับ output จาก command ต่างๆ** (git, ls, test runners, linters, etc.) แล้ว **compress/filter** ก่อนส่งเข้า LLM context window → **ลด token consumption 60-90%** โดยไม่สูญเสียข้อมูลสำคัญ

## ⚠️ ข้อจำกัดและจุดอ่อน

### 1. 🔒 Bash Hook Only — ไม่ครอบคลุม Built-in Tools

> [!WARNING]
> Hook ทำงานเฉพาะ **Bash tool calls** เท่านั้น  
> Claude Code built-in tools เช่น `Read`, `Grep`, `Glob` **ไม่ผ่าน hook** → ไม่ถูก compress

สำหรับ Antigravity/ระบบเรา ที่ใช้ **built-in tools** เป็นหลัก (view_file, grep_search, list_dir) → RTK **จะไม่ช่วยอะไรกับ tools พวกนี้เลย**

### 2. 🖥️ Windows Support — ยังไม่สมบูรณ์

- มี pre-built binary สำหรับ Windows (`rtk-x86_64-pc-windows-msvc.zip`)
- แต่ hook system ออกแบบมาสำหรับ **Bash/Unix** เป็นหลัก
- ระบบเราใช้ Windows (PowerShell) → hook integration อาจไม่ smooth

### 3. 📉 Savings ขึ้นอยู่กับ Workflow

- ถ้า session เน้น **code review / conversation** → savings ต่ำมาก
- ถ้า session เน้น **test, build, git** → savings สูงมาก (89% average)
- Agent ที่ทำ SSH commands บน VPS → ได้ประโยชน์เต็มที่

### 4. 🔄 Telemetry Enabled by Default

- เก็บ anonymous usage metrics วันละครั้ง
- Opt-out ได้ด้วย `RTK_TELEMETRY_DISABLED=1`
- ไม่เก็บ source code, file paths, secrets

### 5. 📦 Binary Size & Compilation

- ต้อง compile Rust (เพราะ crates.io มี project ชื่อซ้ำ "rtk")
- แนะนำ `cargo install --git` แทน `cargo install rtk`
- หรือ download pre-built binary

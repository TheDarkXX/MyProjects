### Proposal 2: mnfst/manifest (Smart Model Router)
- **Verdict:** ⏸️ **ICEBOX**
- **Reasoning:** 
  The user explicitly chose to ICEBOX this because our current configuration (`model-registry.js`) is already well-optimized and highly controlled. 
  
  **AG's architectural opinion on why this is the right call:**
  1. **Black Box Risk:** A "smart" router like Manifest uses a black-box 23-dimension scorer. It might wrongly classify a critical, complex data-extraction task (which we rely on for strict JSON validation) as "simple" and route it to a cheaper, less capable model. This would break our entire pipeline.
  2. **100% Deterministic Control:** Our current system allows us to explicitly define which agent gets which model. We prioritize predictability and stability over minor cost optimizations.
  3. **VPS Resources:** Running an additional Docker container for Manifest would consume precious memory and CPU on our VPS, which is already heavily utilized by the ai-gateway, PM2 processes, Discord bots, and Postgres.

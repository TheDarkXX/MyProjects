### Core Concept
AI-powered video editing via coding agents (Claude Code, Codex, Hermes, etc.). Drop raw footage → chat with AI agent → get `final.mp4`. Features: auto filler-word removal, color grading, subtitle burning, Manim/Remotion animation overlays, self-evaluation loop at every cut boundary, and session memory persistence via `project.md`.

**Key Architecture:** LLM never *watches* the video — it reads a 12KB transcript (ElevenLabs Scribe word-level timestamps + speaker diarization) and only generates visual filmstrips on-demand at decision points. Pipeline: `Transcribe → Pack → LLM Reasons → EDL → Render → Self-Eval`.

### Star Scores

| Dimension | Score | Reasoning |
|---|---|---|
| 💎 Value | 5/10 | We create TikTok/FB content (Doctor Skill pipeline). AI video editing could supercharge content production. BUT — we don't currently handle raw video footage; our pipeline is text→image→caption, not video editing. |
| 🔗 Compatibility | 3/10 | Python-based (uv/pip), requires ffmpeg + ElevenLabs API. Our stack is Node.js on VPS. Would need entirely separate Python runtime. Doesn't integrate with our existing Discord bot or content pipelines. |
| ⚠️ Risk | 4/10 (moderate) | ElevenLabs API dependency = paid service. ffmpeg on VPS = resource-heavy. The "agent with shell access" model could conflict with our PM2/Git-based deployment. |
| 🚀 Growth | 6/10 | Video content is huge for social media growth. If we eventually move to short-form video (TikTok/Reels), this could be transformative. But that's a future need, not current. |
| ⏱️ Effort | 3/10 (high effort) | Major new subsystem: Python runtime, ffmpeg, ElevenLabs subscription, new agent skill registration. 10+ files, entirely new pipeline. |

| Self-eval loop at cut boundaries — matches our consensus/audit philosophy | Requires ElevenLabs paid API |
| Session memory via `project.md` — similar to our `memory/` pattern | We don't produce raw video footage currently |
| 4.9k stars, by browser-use team (credible) | Heavy VPS resource requirements (ffmpeg) |

# Wafer.ai Pricing Evaluation (GLM-5.1 & Qwen3.5)

## 📌 Context
User requested an evaluation of a "$10 Starter Plan" with "1,000 requests / 5hr window" offering Qwen3.5-397B and GLM-5.1 models.
The goal was to see if it covers the entire agentic system (Hydra, cron, bots) and whether it's worth implementing.

## 🔍 Investigation Findings
1. **Provider Identified:** Wafer.ai (`https://pass.wafer.ai/v1`)
2. **Pricing Correction:** The plan is actually **$10 per week ($40/month)**, not $10/month.
3. **Capabilities:**
   - Exposes an OpenAI-compatible endpoint.
   - `GLM-5.1` is exceptionally strong for agentic engineering (SWE-Bench Pro 58.4%).
   - `Qwen3.5-397B-A17B` is extremely fast and capable for chat/reasoning.

## 📊 Usage Analysis (Real Telemetry)
A custom script (`check-usage.py`) was run against `public/data/discord-usage.json` (ai-gateway telemetry) to assess true usage:

- **Average Volume:** ~41 calls per day.
- **Peak Volume:** 302 calls per day (April 22).
- **Total Cloud API Tokens (All Time):** ~2.1M Input / 164K Output.
- **Rate Limit Check:** 1,000 req/5hr is *vastly* more than our current peak. Rate limits are not a concern.

| Monthly cost | ~$0.25 | **$40.00** |

## 🛑 Decision: ICEBOXED
**Financial Verdict: NOT RECOMMENDED at current volume.**
Even though MiniMax M2.7 is paid, our usage is so low that we are spending less than $1/month. Upgrading to a $40/month fixed plan is a massive 16,000% cost increase for capacity we aren't using.

**Re-evaluation Trigger:**
If Hydra is updated to run *heavy* daily continuous evolution loops (e.g., thousands of calls per day where per-token pricing would exceed $40/mo), the Wafer.ai pass becomes extremely cost-effective. Until then, stay on the Pay-As-You-Go OpenRouter/Kie architecture.

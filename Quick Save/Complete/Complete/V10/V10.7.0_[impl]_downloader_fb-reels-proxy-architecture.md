# FB Reels Scraper Extension & VPS Proxy Architecture

## 1. Problem Statement
The user wanted to extract Facebook Reels directly from the `/reels` grid, including URLs and view counts, and then download them as MP4s. Due to Facebook's new DASH (Blob Streaming) implementation, direct `fetch()` for MP4s on the client side fails. Later, the user also wanted to automatically sync full metadata to a Google Sheet without waiting.

## 2. Solution: VPS Proxy Streaming
To bypass Facebook's protection without exposing the user to manual terminal commands, we implemented a **Proxy Downloader Architecture** on the VPS.

### A. The Backend (VPS)
Created a new route `routes/downloader.js` that bypasses standard auth (requires a simple query key `key=doctorbank2026`).
- It receives a `?url=` parameter for streaming downloads via `yt-dlp -o -`.
- It receives a `POST /sync-sheets` payload to trigger a fire-and-forget background job.
- The background job loops URLs, calls `yt-dlp --dump-json`, formats metadata (Reactions, Duration, Title, Dates, Views), and pushes to Google Sheets using existing OAuth credentials.
- It automatically creates new Tabs based on the uploader name.
- Dedup Logic: New IDs are appended. Existing IDs are skipped unless they are > 7 days old, in which case their views and reactions are updated.
- Notifies Pinky in Discord upon URL receipt and completion.

### B. The Frontend (Chrome Extension)
Created `fb-reels-scraper` extension running natively in the browser.
- **UI:** A floating Sidebar built into `content.js`.
- **Auto Scroll:** Added a button to auto-scroll to the bottom, wait 2 seconds, and re-scan until no new reels are found for 5 consecutive tries.
- **Direct Download:** `📥 โหลด (VPS)` button linking directly to the proxy.
- **Sync to Sheets:** `📊 Sync to Google Sheet` button that sends URLs to the VPS and immediately frees up the UI.

## 3. Code Modifications

- `routes/downloader.js`: Created streaming proxy and background sheet sync.

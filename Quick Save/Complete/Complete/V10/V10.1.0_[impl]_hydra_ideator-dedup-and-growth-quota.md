# 🚀 Implementation Plan: Ideator Dedup & Growth Quota

## 📌 Context (Compiled Truth)

### Context & Root Cause
Ideator has been generating duplicate defensive patches (e.g., 4 proposals for PM2 crash loops) instead of "Steve Jobs" moonshot ideas. 
1. **Dedup Failure:** The SQL check `WHERE (dedup_key = ? OR category = ?)` was failing because `item.category` from `intelligence_pool` ('recurring_error') doesn't match `evolution_log.category` ('optimization').
2. **Maintenance Trap:** Ideator prioritizes highest urgency pool items (which are usually Inspector bug reports). It only runs Innovation Mode if the pool is empty.

## 🛠️ Proposed Changes

### 1. Fix Deduplication Logic (`scripts/hydra/ideator.js`)
- **Action:** Improve the `findingDedupKey` to use `findingData.affected_component` if available, falling back to `item.title`. This prevents slight title variations from bypassing the check.
- **Action:** Fix the SQL query to strictly check `dedup_key` without the flawed `OR category = ?` clause.

### 2. Enforce Growth Quota (`scripts/hydra/ideator.js`)
- **Action:** Limit the bug-fix pool processing from 3 down to 2 (`LIMIT 2`).
- **Action:** Refactor the "Innovation Mode" block so that if the `activePersona` is `moonshot` or `bold`, it ALWAYS generates 1 Growth Proposal at the end of its run, regardless of how many bugs it just fixed.

## 💻 Code Snippets

### Change 1: Better Dedup Key
```javascript
const findingData = (() => { try { return JSON.parse(item.raw_data); } catch (e) { return {}; } })();
const affectedComp = findingData.affected_component;
const findingDedupKey = affectedComp ? `${item.category}:${affectedComp}` : item.title;

const activeSameCategory = db.prepare(`
  SELECT id, title FROM evolution_log 
  WHERE dedup_key = ?
  AND status IN ('proposed', 'approved', 'awaiting_human') 
  AND created_at > datetime('now', '-14 days')
  LIMIT 1
`).get(findingDedupKey);
```

### Change 2: Growth Quota
Instead of exiting early when pool items exist, we ensure Innovation Mode runs:
```javascript
  // Process up to 2 high-urgency pool items...
  let poolQuery = `SELECT * FROM intelligence_pool WHERE status = 'open'`;
  // ... LIMIT 2
  
  // After pool loop finishes:
  const shouldRunInnovation = (activePersona === 'moonshot' || activePersona === 'bold' || poolItems.length === 0);
  
  if (shouldRunInnovation) {
      console.log('[Ideator] Executing Innovation Mode (Growth Quota)...');
      // Pick random evolution_goal and generate Growth Proposal...
```

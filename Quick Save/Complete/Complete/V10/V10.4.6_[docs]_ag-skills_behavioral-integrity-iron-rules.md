# Adding Behavioral Integrity Iron Rules

## 📌 Context (Compiled Truth)
The AI agent system required explicit behavioral boundaries to address structural flaws inherent in LLMs, such as the tendency to hallucinate when context is missing, the sycophantic drive to agree with the user even when dangerous, and the tendency to silently expand the scope of a task ("scope drift").

By adding 5 explicit "Iron Rules" to `AGENTS.md`, we force a behavioral constraint that requires the AI to:
1. Make all assumptions explicit and ask if context is missing (**NO MAGIC**).
2. Gather empirical evidence before claiming a task is done (**VERIFY BEFORE DONE**).
3. Surface concerns and disagree before executing risky R0/R1 commands (**DISSENT**).
4. Track stated goals and flag any scope creep (**SCOPE DRIFT DETECTION**).
5. Classify reversibility of actions and act accordingly (**R0/R1/R2**).

These are now placed immediately after the Karpathy Principles in the core ruleset.

- What's the blast radius if this goes wrong?
- What assumptions are we making?
- What's the reversibility path?
- What are we NOT seeing because of momentum?

AI มี bias ที่จะเห็นด้วยกับเรา ถ้าเราบอก "ทำแบบนี้" มันจะทำเลย ไม่ค่อยเถียง

Track stated goals vs actual execution.
Flag when:
- "Just one more thing" accumulates
- Nice-to-haves get treated as must-haves
- The ask was "fix bug X" but we're now
  "refactoring the entire module"

โกหกที่แยบยลที่สุดของ AI คือ "ผมทำเพิ่มให้นิดนึงนะครับ" แล้ว refactor ครึ่ง codebase

R0 (irreversible) — STOP. Ask before proceeding.
R1 (costly to reverse) — Do it, but tell me why.
R2 (easily reversed) — Just do it. No permission needed.

ไม่ใช่ทุก decision ต้องถาม ไม่ใช่ทุก decision ทำได้เลย

- deploy contract (R0) → ต้องถาม ถอยไม่ได้
- เปลี่ยน API contract (R1) → ทำได้ แต่ต้องบอก

5 rules นี้ไม่ได้ทำให้ AI ฉลาดขึ้น แต่ทำให้มันซื่อสัตย์ขึ้น

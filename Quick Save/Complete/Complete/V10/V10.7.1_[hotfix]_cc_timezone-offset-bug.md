# Command Center Alert Timezone Issue (10.7.1)

## Root Cause
When the user snoozed a bill, the `cc_bill_snooze_` logic calculated the new date in the local timezone but formatted it using `newDate.toISOString().slice(0, 10)`.
`toISOString()` converts the date to UTC. Since the VPS is in `Asia/Bangkok` (UTC+7), a local date like `2026-05-09T00:00:00` becomes `2026-05-08T17:00:00Z` in UTC. 
This shifted the due date backward by 1 day.

## Proposed Changes
- In `discord-bot/index.js`, changed `toISOString()` to explicitly format the date using local year, month, and day.
- In `routes/command-center.js` (`advanceDueDate` function), applied the same fix to prevent subscription renewals from drifting backward by 1 day.
- In `scripts/import-facebook-posts.js`, applied the same fix for consistency.
- In `discord-bot/scripts/cron-alerts.js`, changed the alert emoji from `🔴` to `🔥` for Action Required items.
- In `discord-bot/scripts/cron-alerts.js`, fixed the `daysUntil` calculation for overdue subscriptions. Subscriptions pushed to the overdue list due to an expired `expire_date` will now calculate days passed using `expire_date` rather than the future `next_due` date (resolving the "318d overdue" bug).
- In `routes/command-center.js`, implemented an auto-advance logic for subscriptions where `auto_renew` is enabled. Whenever the `/alerts` endpoint is hit, it transparently advances their `next_due` dates instead of flagging them as overdue.

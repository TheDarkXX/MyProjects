# Routines Dashboard V2: GitHub Calendar & Breakdown

## 1. What was done
- **GitHub Contribution Calendar**: Transformed the old flat 30-day flexbox heatmap into a proper 7-row CSS Grid calendar (Mon-Sun) displaying 90 days of history. Implemented `Stripe Purple` and `Magenta` brand tokens for the 4-level activity intensities.
- **Per-Routine Breakdown**: Added a detailed breakdown view for individual routines.
  - Added `GET /api/routines/stats/breakdown` to fetch 28-day history grouped by routine.
  - Implemented dot-timelines to show the last 28 days of completions (● vs ○) per routine.
  - Added gradient progress bars for completion rates.
- **KPI Adjustment**: Replaced the "Paused Routines" stat (which provided little value) with a "30-Day Rate" percentage, showing global completion efficiency over the past month.
- **Encoding Bugfixes**: Fixed widespread Unicode corruption (emoji turning into `??` or `เน ยŸโ€ เธ…`) caused by local PowerShell file encoding overrides during development by building targeted Node.js script string replacements.
- **Tab Save State**: Added `localStorage` caching to remember whether the user was viewing the Dashboard or My Routines grid upon refreshing the page.

## 2. Architecture Changes

### Backend
- Added `GET /api/routines/stats/breakdown` to `routes/routines.js`. Iterates through active routines and queries the `tasks` table for their respective last 28 days of history.

### Frontend
- **Calendar Logic**: Shifted from simple array iteration to calculated week-offsets to align the first column properly with the starting day of the week, enabling a clean 7-row layout. Added automatic scroll-to-right to keep the current week in view.
- **Breakdown Logic**: New `renderBreakdown()` function consuming the breakdown API endpoint. Uses DOM creation and reverse-mapped HTML strings to generate timeline dot indicators.

# Routines Dashboard V2 — Per-Routine Breakdown & Enhanced Visualizations

## Background

We already shipped Dashboard V1 with:
- Hero Section (Today's progress + mini-cards)
- Stats Strip (4 KPIs)  
- Weekly Completion Bar Chart (Chart.js)
- Streak Leaderboard (Top 5)
- 30-Day Completion Heatmap (flat grid)

The user wants **per-routine breakdown** and richer visualizations, inspired by the GitHub Contributions calendar style.

## Proposed Changes

### 1. GitHub-Style Contribution Calendar (replace current flat heatmap)

**Current:** 30 cells in a flat row — hard to read, no day-of-week context  
**New:** A proper 7-row × N-column grid (Mon–Sun rows, weeks as columns), exactly like GitHub's contribution graph

- 4 opacity levels using `Stripe Purple` (#533afd) at 0.15 / 0.35 / 0.65 / 1.0
- Day labels on left (Mon, Wed, Fri)
- Tooltip on hover showing "May 2: 4/5 done (80%)"
- Legend strip at bottom: "Less ░░ ▒▒ ██ More"

### 2. Per-Routine Breakdown Section (NEW — below heatmap)

A card-per-routine view showing individual performance:
Each routine row shows:
- **Title + streak badge** (right-aligned)
- **Completion % progress bar** — gradient using Stripe Purple
- **28-day dot timeline** — filled circles (done ●) vs empty (missed ○), newest on right
- Clicking a routine card could expand or navigate to its full history

### 3. Overall Completion Rate Gauge (NEW — add to Stats Strip)

Replace the "Paused" stat (low-value) with a more useful **30-Day Completion Rate**:
- Shows the overall % of all routine-tasks completed in last 30 days
- Displayed as a bold percentage number with trend arrow (↑ or ↓ vs last 30 days)

# Routines Dashboard V2 Hotfix

## Issue
The calendar grid and breakdown sections were completely invisible on the dashboard. This was caused by three overlapping issues:
1. **CSS Collapse:** The grid cells had no explicit `min-width` / `min-height`, causing flex/grid to squash them to 0px.
2. **Timezone UTC Offset Bug:** The loop used `.toISOString()` which shifted times by -7 hours, creating a mismatch with SQLite local time strings.
3. **ReferenceError Halt:** The variable `rtWeeklyChartInstance` was undefined globally. Checking it threw a ReferenceError, which caused the `Promise.all` chain to abort, preventing the heatmap and breakdown render functions from ever being called.

## Fix
- Declared `let rtWeeklyChartInstance = null;` globally in `routines.js`.
- Added timezone compensation (`getTimezoneOffset()`) before extracting the ISO string.
- Injected explicit CSS boundaries and increased baseline background opacity to `0.06`.

## 🔬 Timeline & Debugging Log
### Update 2 — 22:00
- **Cache Hell Detected:** Even though the server cache-busting string (`v=...`) was updated, the browser aggressively cached the `index.html` file itself. So the old cached HTML loaded the old cached JS.
- **Fix:** Added `Cache-Control: no-cache` headers directly in `server.js` for all HTML files to prevent aggressive browser caching.
- **Refactor:** Rewrote `renderCalendar` function to use raw `var` and `+` string concatenation to eliminate any parsing/template-literal issues in older browsers, ensuring bulletproof rendering.
- **UI Tweaks:** Increased opacity for empty calendar squares (`.rt-cal-cell.level-0` and default background) to `0.08` so the grid is visible even when empty.

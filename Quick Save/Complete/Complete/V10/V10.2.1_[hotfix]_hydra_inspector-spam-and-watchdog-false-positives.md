# 🚀 V10.2.1: Inspector Spam & Pipeline Block Fix

## 📋 What & Why
**Category:** hotfix | **Risk:** high
**Root Cause:**
บอสเจอปัญหาว่า "ทำไมรายงานแบบเดิมทุกๆเวลา ไม่แก้ไขซักที แทนทีจะส่งาน หรืออัปเดทตัวเอง" (Inspector ส่งแจ้งเตือนปัญหา Watchdog 2 จุดและ 5 จุดที่โตได้ซ้ำๆ ทุกวันโดยไม่แก้ไข)

จากการตรวจสอบพบ 3 ปัญหาซ้อนทับกัน:
1. **Inspector Spam:** `inspector.js` ทำการ query หาปัญหาทั้งหมดและเก็บลง `allFindings` แม้ว่าปัญหาจะถูกข้าม (dedup) หรือถูกบล็อคไม่ให้ใส่ pool แล้วก็ตาม ส่งผลให้รายงานส่งเข้า Discord ยังคงแสดงปัญหาเก่าๆ ที่รอแก้ไขอยู่ตลอด
2. **Ideator Pipeline Block:** `ideator.js` ทำการ Hardcode สถานะตอน insert ข้อมูลลง DB เป็น `'awaiting_human'` ทุกครั้ง ไม่ว่า `risk_level` จะเป็น safe แค่ไหนก็ตาม ทำให้ `evolver.js` ที่คอยรอหยิบสถานะ `'proposed'` ไป auto-deploy ไม่เคยได้ทำงาน! ทำให้ Success Rate ตกต่ำ และไม่มีการ "อัปเดทตัวเอง" อัตโนมัติตามที่บอสหวัง
3. **Watchdog False Positives:** `cron-tracker-check` ทำการเช็คว่ามี scheduler run ไหมใน 48-72 ชม. แม้ว่าระบบจะฉลาดพอที่จะบอกได้ว่าช่วงนี้เป็นวันหยุด (ไม่ได้เสีย) แต่มันก็ยังคง return `passed: false` เสมอ ทำให้เกิด False Positive ขึ้นบ่อยมาก

## 🛠️ Proposed Changes & Implementation
| File | Action | Risk |
|------|--------|------|
| `scripts/hydra/inspector.js` | แก้ไขให้ `writeToPool` คืนค่า boolean นำมาดึงเฉพาะ `newFindings` เพื่อส่ง Discord Report ลด Spam | safe |
| `scripts/hydra/ideator.js` | แก้ไขลอจิกจาก Hardcode เป็น `initialStatus = risk_level === 'high' ? 'awaiting_human' : 'proposed'` | high |
| `scripts/watchdog/watchdog.js` | แก้ไขบรรทัดที่ดักจับกรณี no-job ให้คืนค่า `passed: !schedulerLikelyBroken` แทนการ fix false ตลอด | safe |

# Hydra Evolution Pipeline Cleanup & Ideator Fixes

## 📌 Context (Compiled Truth)
This conversation tackled three separate but related issues regarding the Hydra Evolution Pipeline:

1. **Dashboard UI Clutter:** The active persona columns (Bold Builder, Pragmatic Fox, Moonshot Eagle) were displaying all tasks regardless of status, causing a massive backlog of `deployed`, `rejected`, and `expired` items to clog the UI. 
   - **Decision:** Modified the backend API (`routes/hydra.js`) to strictly filter these columns to only show `proposed` and `awaiting_human` tasks. The archived items are now correctly confined to the collapsed sections at the bottom.

2. **Silent Ideator / Missing Piggyback Files:** The Ideator appeared to stop working. The investigation revealed that:
   - "Safe" proposals successfully bypassed Discord alerts (intentional Noise Reduction feature).
   - However, when the Ideator generated a proposal with a Thai title, the `replace(/[^a-z0-9]/gi, '-')` regex stripped all Thai characters, resulting in an invalid filename `----------------.md` and failing to write the piggyback file (`ENOENT` error).
   - **Decision:** Updated the regex to `/[^a-z0-9ก-๙]/gi` to support Thai Unicode characters, and added safety checks for leading/trailing dashes and empty strings.

3. **Ignored Pool Items (Missing Categories):** The `intelligence_pool` had many pending items categorized as `infrastructure`, `frontend`, `backend`, `security`, and `api_health`. Because the `Pragmatic Fox` persona's `poolFilter` used a strict `IN (...)` whitelist that omitted these categories, they were completely ignored.
   - **Decision:** Changed the `Pragmatic Fox` pool filter to use `NOT IN ('innovation_trigger','external','content','dependency_upgrade','skill_gap')` (which are Bold's categories). This transforms Pragmatic Fox into a true "catch-all" worker, ensuring that any new or unconventional category created by the Inspector will be automatically processed.

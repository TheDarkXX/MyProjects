# 🚀 Evolution Hotfix: Ideator Vision Context Fix

## 📋 What & Why
**Category:** `hotfix` | **System:** `Hydra Ideator`

**The Problem (The "Blind Spot"):**
The Hydra Ideator kept generating duplicate proposals for standard infrastructure (Circuit Breakers, Token Caching) that already existed in the system (e.g., `lib/ai-call.js`). 
- When we investigated why the Ideator was ignoring existing code, we discovered that `loadContextForFinding()` only injected source code if the Finding explicitly specified an `affected_component` that was a `.js` file.
- Since Gateway 504 Timeouts don't specify `affected_component`, Ideator flew completely blind.
- Furthermore, we found that Innovation Mode completely bypassed `loadContextForFinding()` and used a hardcoded string as context.
- Finally, the Ideator was only scanning `Quick Save/Complete/` to avoid duplicates, meaning it had no idea what was rejected or iceboxed in the past.

**Our Initial Flawed Fix (Band-aid):**
We initially added overly rigid rejection patterns (`circuit-breaker/fallback` ban) to `rejection-patterns.jsonl`. This forced the AI to "stop thinking" about these topics, which goes against the core philosophy of OpenClaw.

## 🛠️ Proposed Changes & Implementation

1. **Reverted the Band-aid Fix:**
   - Removed the rigid `circuit-breaker` and `redis-cache` bans from `data/hydra-memory/ideator/rejection-patterns.jsonl`.
   - Cleaned up corrupted null bytes in the JSONL file.

2. **Core Infra Injection (The Real Fix):**
   - Modified `scripts/hydra/ideator.js` to define a `CORE_INFRA_FILES` array (`lib/ai-call.js`, `lib/heartbeat.js`, `lib/hydra-memory.js`).
   - `loadContextForFinding()` now **always** reads and injects these files so the Ideator knows exactly what infrastructure already exists.

3. **Complete Quick Save Context:**
   - Updated the directory scanning logic in `loadContextForFinding()` to scan `['Complete', 'Rejected', 'Icebox', 'DeepFreeze']`.
   - The context injection now clearly tells the AI to not propose fixes for these as they are already "solved or rejected".

4. **Innovation Mode Context Fix:**
   - Replaced the hardcoded string context in Innovation Mode with a proper call to `loadContextForFinding()`, giving it the full history and core infra context.

## 🔬 Timeline & Debugging Log
- **2026-04-29**: The user executed `/evo` which resulted in rejecting 6 duplicate proposals for circuit breakers and caching.
- **2026-04-29**: The user questioned why Ideator was still blind despite a previous fix.
- **2026-04-29**: I audited `ideator.js` and discovered that the context injection only fired if `affected_component` was a file path.
- **2026-04-29**: I implemented the `CORE_INFRA_FILES` injection and removed the band-aid rejection patterns.
- **2026-04-29**: The user asked if it would read past rejected works. I realized it only read `Complete`, so I expanded it to read `Rejected`, `Icebox`, and `DeepFreeze`.
- **2026-04-29**: During the full pipeline audit, I discovered the Innovation Mode context bug and fixed it.

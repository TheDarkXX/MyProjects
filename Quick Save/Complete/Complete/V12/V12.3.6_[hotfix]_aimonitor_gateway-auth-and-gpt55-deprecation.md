## Context
The user asked why the AI monitor dashboard showed consecutive red bars for GPT models (`AutCodex: GPT 5.1`, `AutCodex: GPT 5.4`, `OpenClaw: GPT-5.5`).

## Investigation
1. **Initial Symptoms**: Found timeouts and "session file locked" errors in `ai-status.json`.
2. **Previous "Fix"**: In a recent conversation, an agent tried to fix gateway auth errors in `cron-ai-monitor.js` by adding `envCopy.OPENCLAW_GATEWAY_URL = '';`. This bypassed the gateway and forced OpenClaw into embedded mode.
3. **Gateway State**: Discovered the actual `openclaw-gateway` process had been running for 14 days and was hanging. 
4. **Embedded Mode Failure**: Running `openclaw agent` in embedded mode fails because the models `gpt-5.1`, `gpt-5.4`, `gpt-5.5` are not mapped correctly in embedded mode, throwing: `The 'gpt-5.1' model is not supported when using Codex with a ChatGPT account.`
5. **Gateway Verification**: After passing the correct token (`GATEWAY_AUTH_TOKEN`) to the gateway instead of bypassing it, the gateway still timed out. Restarted the gateway via `openclaw gateway restart`.
6. **Underlying OpenAI Issue**: The underlying issue was that OpenClaw `v2026.4.2` did not properly support the new `gpt-5.5` mappings for Codex OAuth. As the user researched, updating to a version newer than `2026.4.22` is required.

## Actions Taken
- ✅ Removed `envCopy.OPENCLAW_GATEWAY_URL = '';` bypass from `cron-ai-monitor.js`.
- ✅ Added `envCopy.OPENCLAW_GATEWAY_TOKEN = process.env.GATEWAY_AUTH_TOKEN;` to properly authenticate pings via the gateway.
- ✅ Updated OpenClaw CLI globally from `v2026.4.2` to `v2026.5.6`.
- ✅ Ran `openclaw doctor --fix` to repair obsolete `openclaw.json` config schemas.
- ✅ Used Node script to replace all deprecated `openai-codex/gpt-5.1` occurrences in `openclaw.json` with `openai-codex/gpt-5.5`.
- ✅ Set `openai-codex/gpt-5.4` as the fallback model using `openclaw models fallbacks`.
- ✅ Restarted `openclaw-gateway` process on the VPS (`openclaw gateway restart`).
- ✅ Ran `cron-ai-monitor.js` manually: All GPT models (5.1, 5.4, 5.5) now report **OK (Green)**.

## Changelog
* **2026-05-07: Added OpenClaw Model & Architecture Deep Dive**
  * **Supported Models**: Confirmed OpenClaw v2026.5.6 now fully maps `openai-codex/gpt-5.5`, `gpt-5.4`, and older 5.2/5.3 mini models. GPT-5.1 is officially blocked by OpenAI.
  * **Thinking Modes**: Verified OpenClaw supports passing `requestShaping.thinking` with levels `off` (Instant), `low`, `medium`, and `high` which dynamically controls the OpenAI o1/o3/v5 reasoning budget. Default is `medium`.
  * **Hybrid Architecture Explanation**: Clarified to user that non-GPT models (Gemini, DeepSeek, Qwen) bypass OpenClaw completely and connect directly to our own `kie-proxy`. ONLY GPT-5.4/5.5 are routed through OpenClaw Gateway due to the necessity of scraping the ChatGPT Plus web session.
  * **Image Generation Timeout & Fallback Bug**: Attempted to generate a Thai infographic via CLI `openclaw agent`. Discovered that DALL-E 3 generation taking 30-45 seconds causes a Gateway timeout. When this timeout occurs, OpenClaw automatically triggers its fallback model, which we had set to `gemini-2.5-flash`. Since Gemini cannot return images natively in this chat pipeline, the CLI silently fails/hangs. Conclusion: Image generation MUST be done via Discord Bot which handles long-polling and Webhooks properly.
  * **Token Tracking via AI Gateway**: Confirmed that `ai-gateway` successfully intercepts OpenClaw's internal `tiktoken` heuristic estimations (since ChatGPT web doesn't send exact tokens) and logs them correctly to our DB.

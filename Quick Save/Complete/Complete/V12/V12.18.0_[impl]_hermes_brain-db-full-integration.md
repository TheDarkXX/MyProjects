## 🔑 การตัดสินใจสำคัญ (Key Decisions)

### ทำไมใช้ mcp-sqlite ไม่ใช้ API Bridge?
- mcp-sqlite ให้ Honey เข้าถึง DB ได้ **100% โดยตรง** (SELECT, INSERT, UPDATE)
- ไม่ต้องสร้าง middleware อีกชั้น → ลด latency + ลดความซับซ้อน
- Safety ควบคุมได้ผ่าน SOUL.md Safety Rules + Hook Script audit trail

### ทำไม terminal.cwd ต้องเป็น /root/brain-app?
- Honey ทำงานในบริบทของ Brain App เป็นหลัก
- เวลาสั่ง `pm2 list`, `node scripts/...` ไม่ต้อง cd ก่อนทุกครั้ง

### Pinky vs Honey — การแบ่งงาน
- **AG (Antigravity):** CTO/Lead Developer — เขียนโค้ด, สร้างฟีเจอร์, วาง Architecture, Deploy
- **Honey (Hermes):** System Admin/Data Analyst — เฝ้าเซิร์ฟเวอร์, ดึงข้อมูล, จัดการ DB, ส่ง Report
- **Pinky:** Generalist chatbot — ไม่มีสิทธิ์เข้าถึง DB หรือ Terminal โดยตรง
- **ไม่ควรให้ Honey เขียน/แก้โค้ด** เพราะ: (1) แก้ไฟล์ Production โดยตรง → Git Conflict, (2) ไม่ผ่าน bump-cache.js → Cache Hell, (3) ไม่มี IDE tools แบบ AG

### Image Gen (FAL.ai) — ยังไม่ได้เปิด
- Hermes ใช้ FAL.ai สำหรับ Image Generation → **ต้องมี `FAL_KEY`**
- ตอนนี้ `.env` ไม่มี FAL_KEY → Honey ยังใช้ Image Gen ไม่ได้
- ราคา: FLUX Schnell ~$0.003/ภาพ, FLUX Pro ~$0.05/ภาพ
- Free tier: $10 credits ตอนสมัคร

#!/bin/bash
# log-to-brain.sh — Post-tool-call hook
# Logs every terminal command Honey runs to ag_logs table in Brain DB

INPUT=$(cat)

TOOL_NAME=$(echo "$INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('tool_name',''))" 2>/dev/null)
TOOL_INPUT=$(echo "$INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(json.dumps(d.get('tool_input',{}))[:500])" 2>/dev/null)

if [ "$TOOL_NAME" = "terminal" ]; then
    DATE=$(date +%Y-%m-%d)
    SAFE_INPUT=$(echo "$TOOL_INPUT" | sed "s/'/''/g")
    sqlite3 /root/brain-app/db/brain.db "INSERT OR IGNORE INTO ag_logs (date, pc_source, conversation_id, summary) VALUES ('$DATE', 'honey-hermes', 'discord-gateway', 'Terminal: $SAFE_INPUT')" 2>/dev/null
fi

echo '{"action": "continue"}'

### Update (2026-06-06) — Codex OAuth vs Higgsfield (Image Generation Decision)

**Context:** บอสพบปัญหาฝั่ง Openclaw ไม่สามารถแนบรูป (Image-to-Image) เพื่อเจนภาพผ่าน `GPT Image 2` ได้ และสอบถามว่าหากจะ "Tear off Openclaw" แล้วย้ายให้ Hermes ทำแทน 100% จะสามารถอัปเดต Hermes ให้รองรับ Codex OAuth ตัวนี้ได้ไหม

**Investigation:**
1. สาเหตุที่ Openclaw แนบรูปไม่ได้: Openclaw ยัดรูปเป็น Base64 (`image_url: dataUrl`) ส่งตรงเข้า `/backend-api/codex/responses` (Bypass Endpoint) ซึ่งฝั่ง ChatGPT ปฏิเสธ เพราะระบบจริงต้องอัปโหลดผ่าน `/backend-api/files` ก่อนเพื่อเอา `file_id`
2. ฝั่ง Hermes: การจะแฮค `openai-codex` provider ใน Hermes ให้รองรับฟีเจอร์นี้ ต้องเขียนระบบจำลองการอัปโหลดไฟล์ผ่าน Cloudflare และจำลองการทำงานของ GPT-4o (Vision) เพื่อแปลงรูปเป็น Text ก่อนส่งให้ DALL-E 3 ซึ่งมีความเสี่ยงสูงที่จะโดน OpenAI แบนระบบทั้งหมด

**Key Decision (Architecture Shift):**
- **REJECTED:** การนำ Codex OAuth Image Gen (Bypass) ไปใส่ใน Hermes
- **APPROVED:** หากจะ Tear off Openclaw การเจนภาพแบบมี Reference (Image-to-Image) ให้ใช้ **Higgsfield (ผ่าน Hermes)** เป็นตัวหลักแทน
- **เหตุผล:** 
  1. Higgsfield (`nano_banana_2`, `gpt_image_2`) รองรับการดึงรูปภาพ (Reference) อย่างถูกต้อง
  2. ปลอดภัย 100% ไม่เสี่ยงโดนบล็อค Token 
  3. ทำงานผ่าน Discord ของ Hermes ได้ทันที แค่แนบรูปแล้วพิมพ์สั่ง

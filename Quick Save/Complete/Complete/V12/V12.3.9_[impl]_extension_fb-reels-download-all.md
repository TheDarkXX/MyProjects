# FB Reels Scraper - Download All & DB History Architecture

## 🎯 Objective
ผู้ใช้ต้องการปุ่ม "Download All" ในหน้า Extension โดยใช้หลักการเดียวกับปุ่ม Download (VPS) แต่รันดาวน์โหลดทีละรายการเรียงลงมาจนครบ (Sequential Queue) รวมถึงต้องการระบบ History ที่เรียกดูย้อนหลังได้จาก Database บน VPS แทนที่จะพึ่งพา Google Sheets เพียงอย่างเดียว

## 🧠 Implementation Details

### 1. Download Queue Architecture (Extension)
เปลี่ยนจากการยิง Request โหลดพร้อมกันมาเป็น Queue System โดยใช้ `chrome.downloads`:
- สร้างฟังก์ชัน `startDownloadAll(type)` รองรับทั้ง `scan`, `manual`, `history`
- ใช้ `chrome.downloads.download({ saveAs: false })` คู่กับ `chrome.downloads.onChanged.addListener` เพื่อดักจับ Event `state.current === 'complete'`
- หน่วงเวลา 1.5 วินาทีระหว่างแต่ละไฟล์เพื่อไม่ให้โหลดเซิร์ฟเวอร์หนักเกินไป
- เพิ่ม Auto-save ให้แท็บ Auto Scan (`scanDataArray`) เพื่อป้องกันข้อมูลหายเมื่อปิด Sidepanel

### 2. SQLite Database History (VPS)
สร้าง Dual-Write Architecture (ยิงปืนนัดเดียวได้นกสองตัว):
- เพิ่มตาราง `fb_reels_history` ใน `db/init.js`
- ป้องกันข้อมูลซ้ำด้วย `PRIMARY KEY (id)` และ `ON CONFLICT(id) DO UPDATE SET views=excluded.views` (UPSERT)
- ในระหว่างที่ยิง `/api/downloader/sync-sheets` ระบบจะบันทึกข้อมูลลง Database ทันที (พร้อมๆ กับที่รอ Batch เพื่อยิงขึ้น Google Sheets) ทำให้ข้อมูลซิงก์ตรงกัน 100%

### 3. History Tab & API
- สร้าง API `GET /api/downloader/history` ใน `routes/downloader.js` 
- เพิ่มแท็บ `🕒 History` ใน `sidepanel.html` 
- เพิ่มฟังก์ชัน `fetchHistory()` ใน `sidepanel.js` ให้แสดง 100 รายการล่าสุด และมีปุ่ม Download All ให้โหลดคลิปเก่าซ้ำได้ทันที

# Hydra Batch Hotfix Implementation

## 📌 Context (Compiled Truth)
The system was experiencing multi-layered operational issues identified during a Hydra/Oracle audit. Six critical hotfixes and guardrails were approved to restore stability and secure the environment:
1. **Discord Database Integrity:** Prevented the bot from loading stale/corrupted databases by enforcing `fileMustExist` on the canonical VPS path and falling back smoothly.
2. **Viral Planner Infinite Loop:** Fixed a 404 URL resolution error combined with a 0-second scheduling delay that caused 106% CPU load.
3. **SQLite Double Quotes:** Repaired query regressions where `datetime("now", "localtime")` was being evaluated as a column name.
4. **FB Token Guard:** Protected the publishing queue from being exhausted by expired token requests by validating the token before initializing the Graph API upload.
5. **Preflight Syntax Gate:** Hooked into the VPS `post-receive` script to strictly `node --check` all API logic before PM2 restarts, preventing crash-loops on syntax errors.
6. **Agent Shell Injection (RCE):** Hardened the openclaw CLI execution endpoint by using `execFileAsync` rather than unprotected string concatenations.

# 🍯 Honey Hermes LINE Bot Integration

## 📌 Context (Compiled Truth)
บอสต้องการให้ Honey Hermes (ที่เป็น Cloud Dev Agent & DBA แบบ 24/7 บน Discord) สามารถคุยผ่านช่องทาง LINE OA ได้ (รวมถึงใช้ใน Group LINE ร่วมกับครอบครัว) 

**Architecture Discovery & Final Decision:**
- **Native Hermes CLI (One-shot):** ค้นพบว่า Hermes รองรับ One-shot Execution ผ่าน CLI (`-z`) และการแยก Session (`-c`) 
- **Implementation:** สร้าง Webhook บน `brain-app` (ที่รันบน VPS ตัวเดียวกับ Hermes) เพื่อรับ HTTP POST จาก LINE
- **Execution:** นำ LINE User ID (หรือ Group ID) ไปใช้เป็น Session Name แล้วยิงคำสั่ง: `hermes -z "USER_TEXT" -c "line_user_12345"`
- **Benefit:** ไม่ต้องตั้ง Server ใหม่, ได้ Memory ประจำตัวผู้ใช้/Group 100% และดึงความสามารถเต็มๆ ของ Native Hermes มาตอบผ่าน LINE ได้ตรงๆ

**UX Optimization (Loading Animation):**
- เนื่องจาก Hermes Native ใช้เวลาบูตระบบ 2-4 วินาที (อ่านกฎ, เช็คประวัติ ฯลฯ) จึงใช้ LINE API `chat/loading/start` เพื่อแสดงสถานะ "กำลังพิมพ์..." ให้ผู้ใช้เห็นทันทีที่ส่งข้อความมา เพื่อลดความรู้สึกว่าแชทบอทตอบช้า

**Auth Resolution:**
- แก้ปัญหา Codex API (GPT-5.4) Token หมดอายุ โดยเพิ่ม Script Sync Token จาก `~/.codex/auth.json` (ที่ล็อกอินใหม่) เข้าสู่ `~/.hermes/auth.json` โดยตรง เพื่อให้ Hermes กลับมาใช้งาน GPT ได้สมบูรณ์

## 🔬 Timeline & Debugging Log
- **Phase 1:** สร้าง Webhook `/api/webhook/line` และเขียนโค้ด `routes/line-hermes.js` เชื่อมต่อ Native CLI
- **Phase 2:** พบปัญหา Hermes ไม่ตอบกลับบน LINE ทดสอบแล้วพบว่า Token Authentication ของฝั่ง Codex (GPT-5.4) หมดอายุ (`HTTP 401: Your authentication token has been invalidated`)
- **Phase 3:** สั่งรัน `codex login --device-auth` ใหม่เพื่อได้ Refresh Token แต่ Hermes ยังคงหา Token ไม่เจอ จึงเขียนสคริปต์ Node ไป Force Sync นำข้อมูลจาก `~/.codex/auth.json` เข้าไปแทนที่ใน `~/.hermes/auth.json` ทำให้ Hermes ใช้งานได้ปกติ
- **Phase 4:** อัปเดต `LINE_CHANNEL_SECRET` และ `LINE_CHANNEL_ACCESS_TOKEN` ให้ถูกต้อง
- **Phase 5 (Optimization):** เติมระบบส่ง Loading Animation "กำลังพิมพ์..." (POST API `chat/loading/start`) เพื่อปรับปรุง UX ขณะรอ Hermes Boot up ตัวเอง

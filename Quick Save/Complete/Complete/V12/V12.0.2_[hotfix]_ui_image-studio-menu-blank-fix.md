# V12.0.2 Hotfix: UI Main Menu & Image Studio Blank Fix

## 📌 Context
User reported two issues:
1. "หน้า blank": The Image Studio page is blank, which looks like a regression.
2. "เหมือนปุ่ม Main menu จะหายไปหลายปุ่มเลย": Many icons in the main rail disappeared.

## 🔍 Investigation & Root Cause
1. **Missing Main Menu Icons**: 
   - Found that `PAGE_TO_CATEGORY` in `app.js` was misconfigured during the recent Data Tables update. `'data-tables'` was incorrectly mapped to `'workspace'`, causing the UI router to switch to the wrong sidebar panel and drop the contextual menu options. `'image-studio'` was completely missing from the map.
2. **Blank Image Studio**: 
   - The `<div id="page-image-studio">` container was present, but `initImageStudio()` was never being called if the user navigated directly to `https://brain.doctorbankonline.com/#image-studio`. The app router executed before `image-studio.js` finished loading, resulting in a blank container.

## ⚡ Execution
- **`app.js`**: Re-mapped `'data-tables': 'data-tables'` and added `'image-studio': 'image-studio'` to `PAGE_TO_CATEGORY` to fix the sidebar router.
- **`image-studio.js`**: Added a direct execution hook at the bottom of the script (`if (window.location.hash.replace('#', '') === 'image-studio') { setTimeout(() => initImageStudio(), 100); }`) to ensure it initializes even on direct URL load.
- **Cache-Buster & Deploy**: Executed `node bump-cache.js` and pushed to VPS, triggering the deployment hook which restarted the server.
- **Verification**: Dispatched `browser_subagent` to test the live production site. It confirmed all rail icons are back and the Image Studio renders perfectly.

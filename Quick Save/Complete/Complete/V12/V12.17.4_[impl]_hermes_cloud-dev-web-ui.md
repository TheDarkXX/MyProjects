## Background & Decision

The user's local PC restricts continuous development when turned off. A previous plan (V4.5.0) rejected migrating the whole system to the Hermes ecosystem. However, installing Hermes *alongside* the existing Node.js stack on the VPS solves the problem of "dev without PC" by acting as an always-on Cloud Dev Agent.

Key architectural shifts in this plan:
1. **No WSL2 on PC:** PC users access the Hermes Web UI (`hermes dashboard`) via a reverse proxy (Port 80/443 -> 9119) directly on the VPS.
2. **ChatGPT Pro OAuth:** Utilizing Hermes' built-in OAuth proxy to use the user's ChatGPT Pro subscription without per-token API costs.
3. **Database Protection:** 5-layer security including MCP Read-Only mode, Approval Gates, and automated SQLite WAL backups every 6 hours.

## Implementation Plan Overview

1. **Phase 1: VPS Backend & Security Prep**
   - Create isolated `/root/hermes-workspace`.
   - Implement `backup_brain.sh` and set cron for DB backups.
   - Stub `hermes-brain-bridge.js` for safe DB writing.
2. **Phase 2: Hermes Installation & ChatGPT OAuth**
   - Install Hermes Agent on VPS.
   - Configure ChatGPT Pro OAuth and setup fallbacks (Kie-proxy).
3. **Phase 3: Brain DB Integration**
   - Connect `mcp-sqlite` for read-only access.
   - Supply system context via `MEMORY.md`.
4. **Phase 4: Discord Gateway**
   - Setup a dedicated Hermes Discord Bot to handle tasks on the go.
5. **Phase 5: Web UI Dashboard**
   - Run `hermes dashboard` as a service.
   - Route via Nginx for PC access.

## ⚖️ Honest Risk Assessment

### ✅ ทำไมแนวคิดนี้เข้ากับระบบเราได้ดี
1. **ฐานแข็งแกร่ง** — Memory Layer, Tool Registry, Budget Guard, Agent Factory ครบแล้ว
2. **Single VPS** — ไม่ต้อง Kubernetes, Docker, microservices. SQLite + Node.js + cron พอ
3. **Proven Patterns** — เราใช้ "Adopt Concepts, Not Framework" มาตลอด (Paperclip, GBRAIN) → ทำแบบเดิมได้
4. **12 Agents แล้ว** — ถึง threshold ที่ orchestration เริ่มคุ้มค่า

### ⚠️ ความเสี่ยงที่ต้องระวัง

| Risk | ระดับ | แผนรับมือ |
|---|---|---|
| **Over-engineering** — ทำ CEO Agent ซับซ้อนเกิน | 🔴 สูง | เริ่มจาก hardcoded routing rules ก่อน → ค่อยใส่ AI routing ทีหลัง |
| **Token cost explosion** — CEO + Dept agents = เรียก LLM หลายตัวต่อ request | 🟠 กลาง | Budget Guard มีอยู่แล้ว + CEO ใช้ cheap model (30B) สำหรับ routing |
| **VPS resource** — 2GB RAM อาจไม่พอ ถ้ารันหลาย agent พร้อมกัน | 🟡 ต่ำ | agent เป็น cron ไม่ได้ resident — รันแล้วจบ ไม่กิน RAM ค้าง |
| **Coupling risk** — dept เชื่อมกันมาก = พังทีเดียวหมด | 🟠 กลาง | Graceful degradation: ถ้า dept หนึ่งพัง, อีก dept ยังทำงานได้ |
| **Premature abstraction** — สร้าง abstraction ก่อนที่จะมี use case จริงๆ | 🔴 สูง | **ต้องมี concrete use case ก่อนสร้าง** — ไม่สร้าง "just in case" |

### 🎯 คำแนะนำสำคัญ: Start with ONE concrete flow

> **อย่าสร้าง "บริษัท AI" ทั้งหมดในครั้งเดียว**

เริ่มจาก 1 flow ที่เจ็บปวดจริง เช่น:

ถ้า flow นี้ทำงานได้ดี → ค่อยขยายไปฝ่ายอื่น
ถ้าทำไม่ได้ → architecture ผิด ต้องแก้ก่อนขยาย

## 🧠 Section 1: Model Strategy — ใช้ ChatGPT Pro OAuth (Free API!)

**Hermes รองรับ ChatGPT Pro OAuth** ทำให้ใช้งานได้ "ฟรี" ไม่เสียค่า API per-token 

### วิธีตั้งค่า ChatGPT Pro กับ Hermes
1. รันคำสั่ง `hermes model` บน VPS
2. เลือก Provider เป็น **OpenAI-Codex (OAuth)**
3. Hermes จะให้ Link เพื่อ Login ผ่าน Browser บน PC
4. Hermes จะดึง Session มาใช้รันเบื้องหลังบน VPS
5. **แถม:** Hermes มีฟีเจอร์ Proxy เปิดให้ใช้ API ฟรีจาก Tools อื่นๆ ได้ด้วย

### Fallback Chain (ป้องกันจังหวะ Session หลุด)
1. **Primary:** ChatGPT Pro (OAuth) — ฟรี!
2. **Fallback 1:** Gemini 2.5 Flash (ผ่าน `kie-proxy` 18810) — ถูกมาก
3. **Fallback 2:** DeepSeek V3.2 (OpenRouter) — ถูก

## 🔒 Section 2: Security — ป้องกัน Brain DB พัง

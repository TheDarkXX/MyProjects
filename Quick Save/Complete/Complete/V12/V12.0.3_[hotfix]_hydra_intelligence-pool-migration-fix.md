# V12.0.3: Hydra Intelligence Pool Migration Fix

## 📌 Problem Context
- The user reported that the **Awakened (Hydra) Dashboard** was showing `...` for all metrics and columns.
- The user also noticed that their "old data" in the pool (usually tens of items) had completely disappeared.
- **Root Cause 1:** The `migrate-v3-1.js` schema update for `intelligence_pool` had been run locally but was **never applied to the VPS production database** (`db/brain.db`). The API `fetchEvoSummary()` threw a `no such table: intelligence_pool` error which stalled the UI.
- **Root Cause 2:** Once the table was created, it started empty (0 items). The old "Pool" items were originally stored in `evolution_log` with status `finding` or `ideated`. Because of the V12 architecture split, these old items did not automatically migrate to the new `intelligence_pool` table.

## 🛠️ Actions Taken

1. **VPS Database Schema Fix:**
   - SSH'd into the VPS and executed the SQLite DDL to create the `intelligence_pool` table.
   - Added the missing `persona` and `pool_item_id` columns to `evolution_log`.
   - Result: The Awakened Dashboard API successfully executed without silent errors.

2. **Data Restoration & Migration:**
   - Discovered 94 historical items in `evolution_log`. 43 of these were `expired` by the auto-cleanup cron, which is expected behavior.
   - 15 items were still "active" (3 `finding` + 12 `ideated`).
   - Wrote and executed a custom Node.js script (`tmp_migrate_pool.cjs`) on the VPS to migrate these 15 legacy items into the new `intelligence_pool` table.
   - Mapped `raw_data` to the new columns (`urgency_score`, `impact_score`, `novelty_score`, `evidence`, etc.) and set them to status `open`.
   - Updated the old `evolution_log` rows to `status = 'processed'` and linked their `pool_item_id` so they do not duplicate.

3. **Validation:**
   - Tested the `/api/hydra/evolution-pipeline` API. All 19 deployed, 43 expired, and 10 rejected items properly returned to their respective collapsed sections.
   - The "OPEN ITEMS" in the Pool correctly reflected the migrated active items, restoring the UI to its fully populated state.

\`\`\`javascript
const Database = require('better-sqlite3');
const db = new Database('/root/brain-app/db/brain.db');

try {
  const activeItems = db.prepare(\`
    SELECT id, title, category, status, raw_data, created_at 
    FROM evolution_log 
    WHERE status IN ('finding', 'ideated')
  \`).all();

  const insertPool = db.prepare(\`
    INSERT INTO intelligence_pool (
      source, category, title, urgency_score, impact_score, novelty_score,
      raw_data, evidence, affected_component, layer, status, created_at, expires_at
    ) VALUES (
      ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, datetime(COALESCE(?, datetime('now')), '+14 days')
    )
  \`);

  let count = 0;
  
  db.transaction(() => {
    for (const item of activeItems) {
      let rd = {};
      try { rd = JSON.parse(item.raw_data || '{}'); } catch(e){}
      
      const source = String(rd.source || (item.status === 'finding' ? 'legacy_finding' : 'legacy_ideated'));
      const category = String(item.category || 'system');
      let ev = rd.evidence || rd.description || item.title || 'no evidence';
      if (typeof ev === 'object') ev = JSON.stringify(ev);
      const evidence = String(ev);
      
      const affected = String(rd.affected_component || 'unknown');
      const layer = String(rd.layer || 'core');
      
      const urgency = Number(rd.urgency_score) || (item.status === 'finding' ? 60 : 40);
      const impact = Number(rd.impact_score) || 50;
      const novelty = Number(rd.novelty_score) || 50;
      
      const rawData = item.raw_data ? String(item.raw_data) : null;
      const createdAt = item.created_at ? String(item.created_at) : new Date().toISOString();

      const params = [
        source, 
        category, 
        String(item.title || 'Untitled'), 
        urgency, 
        impact, 
        novelty,
        rawData, 
        evidence, 
        affected, 
        layer, 
        'open', 
        createdAt, 
        createdAt
      ];
      
      const res = insertPool.run(params);
      
      db.prepare(\`UPDATE evolution_log SET pool_item_id = ?, status = 'processed' WHERE id = ?\`).run(res.lastInsertRowid, item.id);
      count++;
    }
  })();

  console.log(\`Successfully migrated \${count} items to intelligence_pool.\`);

} catch (e) {
  console.error('Migration error:', e);
}

# Production deploys can restart brain-app onto syntactically invalid JavaScript without a preflight gate

## 📌 Context (Compiled Truth)
PM2 reloads the app even if the new code has syntax errors, bringing down the production server during fast deploy loops.
Evaluation Score: 4.5/5.0
Impact: 5/5 | Feasibility: 4/5 | Urgency: 4/5 | Alignment: 5/5
Verdict: APPROVE

Plan: Add a deploy-hook preflight before PM2 reload: run `node --check` or an ESM import smoke test over changed server/route files plus a minimal `/api/health` post-reload verification. If preflight fails, abort reload and record an ops_event with the failing file/line.

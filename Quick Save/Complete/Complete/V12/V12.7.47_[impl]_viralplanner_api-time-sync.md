# Viral Planner API Time Sync

## 📌 Context (Compiled Truth)
The user realized that if they edited the scheduled time of a video in Meta Business Suite, the local Viral Planner would fall out of sync. To maintain our "Native FB Schedule" architecture without data discrepancy, we added `scheduled_publish_time` to the Graph API fields requested during the `/mc/planner/sync-analytics` poll. 
If the time fetched differs from the local `scheduled_at` by more than 1 minute, the system updates the local database. If the API returns error code 100 (Object does not exist), the system assumes the Reel was deleted on FB and safely updates the status to `rejected` with an appropriate error message, allowing the user to review or reschedule it later.

# V12.7.48 [hotfix] Hydra: Discord Guard & Downloader Crash Fix

## 📌 Context (Compiled Truth)
This implementation resolved two critical issues flagged by the Oracle meta-agent in the May 19, 2026 batch review:

1. **Discord Empty Message Guard (EVO-715):** The Discord bot was blindly passing AI outputs into `channel.send()`. When the AI only generated `brain-query.sh` commands with no conversational text, the `cleanReply` stripper reduced the payload to an empty string, triggering `DiscordAPIError[50006]` and burning webhook interactions. 
   **Decision:** Added a guard in `sendLongMessage` that detects empty/whitespace content and substitutes a fallback string (`_[AI did not generate any text update]_`).

2. **Sync-Sheets Queue Regression (EVO-7224):** The background downloader queue for Facebook Reels was crashing completely when an undefined variable was hit, due to a missing outer `catch` block mechanism that preserves state.
   **Decision:** Wrapped the `processNextJob` execution block in an improved `catch` that intercepts the crash, safely saves the failed state into `global.failedJobsQueue`, and moves on to the next job, rather than executing `activeJob = null` unconditionally and stalling the queue. Also added a missing `.catch()` to the failure notification to prevent secondary throws from permanently locking the queue loop.

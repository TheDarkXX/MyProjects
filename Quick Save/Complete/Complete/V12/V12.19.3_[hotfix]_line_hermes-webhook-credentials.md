# V12.19.3 — Fix LINE Hermes Webhook Missing Credentials

## 📌 Context (Compiled Truth)

### Root Cause
`routes/line-hermes.js` returns HTTP 500 when `LINE_CHANNEL_SECRET` or `LINE_CHANNEL_ACCESS_TOKEN` is absent from `.env`. The PM2 environment for brain-app does not have these variables set, yet LINE webhook traffic is actively hitting `/api/webhook/line`.

### Impact
- Real LINE user/group messages are accepted (HTTP 200) but then silently fail
- Users think their message was received but get no response — trust-destroying
- Hermes runs asynchronously via `execAsync` for accepted messages, so failures are invisible

### Implementation Plan

**Option A (Preferred): Add credentials to PM2 env**
1. SSH into VPS, check if LINE credentials exist anywhere (`.env`, ecosystem config)
2. If credentials exist → add to PM2 ecosystem.config.js and restart brain-app
3. If credentials don't exist → proceed to Option B

**Option B: Disable route until configured**
1. Add a startup check in `line-hermes.js`: if credentials missing → log warning, mount a stub route that returns 503 with clear message
2. Don't silently accept webhooks that will fail

**Both options also need:**
3. Add bounded Hermes timeout (e.g., 30s) so `execAsync` doesn't hang forever
4. Add fallback LINE error reply: if Hermes fails, send a LINE message back saying "Sorry, I'm having trouble right now"

# New SQLite errors show code is using double-quoted `now` as an identifier instead of a time literal

## 📌 Context (Compiled Truth)
SQL errors are occurring because `"now"` is used with double quotes (treating it as a column) instead of single quotes (as a literal).
Evaluation Score: 3.75/5.0
Impact: 3/5 | Feasibility: 5/5 | Urgency: 4/5 | Alignment: 3/5
Verdict: APPROVE

Plan: Search recent route/worker SQL for `datetime("now")`, `date("now")`, `strftime(...,"now")`, or raw `"now"` date expressions. Replace with single-quoted SQLite literals such as `datetime('now','localtime')` or parameterized timestamps, and add a small smoke test for the affected endpoint/job.

# Viral Planner Modal Controls & Timezone Fix

## 📌 Context
Following the massive UX optimization for the Viral Planner, the user requested further workflow streamlining by moving key controls (Status Dropdown & Next 3 Available Slots) directly into the Video Preview Modal. This allows for a "zero-click-exit" workflow where the user can watch a video, schedule it, and click Next directly within the overlay. During implementation, two bugs were identified and fixed: the video navigator not respecting the table's visual sort order, and the `datetime-local` input displaying UTC hours instead of Local Time.

## Issues Found & Root Causes

### 1. Calendar Icon □ (Schedule Column)
**Location:** Every row in Viral Planner table → Schedule column  
**Root Cause:** The `<input type="datetime-local">` HTML element renders a native browser calendar picker icon. On some Windows systems/browsers, this icon renders as □ (missing glyph).  
**Also affected:** Line 1475 — `📅` emoji for posted items with schedule date  
**Fix:** Style the datetime-local input to hide native icon + add custom styled wrapper

### 2. Missing Captions (4 items)
**IDs:** 162, 166, 167, 178  
**Files:** `2023-10-27-S05_v1`, `2024-05-15-S01_v1`, `2024-05-16-S03_v1`, `2024-04-23-MIR01_v1`  
**Root Cause:** These were early imports before the auto-caption pipeline was built  
**Fix:** Bulk auto-fill button using existing Gemini endpoint — all 4 have thumbnails ✅

### 3. No Video Preview
**Root Cause:** Thumbnails display as static images. No video player modal exists.  
**Data:** All 49 items have `video_url` on VPS → video playback is fully possible ✅

### 4. Sync Analytics Can't Work
**Root Cause:** The 17 "posted" items have `fb_post_id = NULL`. The sync endpoint requires `fb_post_id` to query Facebook Graph API video insights.  
**Why:** Items were likely published manually via Meta Business Suite, not via the "📤 Send" button.  
**Fix Options:**
- A) Manual: User pastes fb_post_id for each posted video
- B) Auto-match: Search FB page videos by caption text, match back to DB
- C) Accept reality: Only future posts sent via UI will have analytics

### 5. Missing Thumbnails (11 items)
**IDs:** 197-209 (old "FULL" format files like `S01_FULL_v2_REVIEW_20260506.mp4`)  
**Root Cause:** These files have `.mp4` on VPS but no `.jpg` thumbnail  
**Fix:** Generate thumbnails server-side using ffmpeg (extract first frame)

## 📝 Compiled Truth & Technical Decisions
- **Zero-Exit Workflow:** The UI now dynamically injects a `vp-video-controls` div below the caption in `vpRenderVideoPlayer`. Because `vpSetSchedule` and `mcPlannerUpdateStatus` operate synchronously on the cached `vpAllJobs` array before re-rendering the table, the modal can safely call `vpRenderVideoPlayer()` immediately after a button click to refresh its own state without needing an API refresh.
- **Visual Sync:** The table rendering engine (`vpRenderContent`) now populates a global array `window.vpCurrentVisualJobs` exactly in the order items are rendered (including groupings and sorts). `vpShowVideo` references this array, meaning clicking "Next" in the modal exactly matches the item directly beneath it in the UI table.
- **Timezone Safety Protocol:** Standardized that all `scheduled_at` modifications must save the exact `Date.toISOString()` representation (which includes `Z`). The frontend now uses `vpFormatLocalInput` to extract the correct local timezone offset before feeding the string into the `<input type="datetime-local">`.

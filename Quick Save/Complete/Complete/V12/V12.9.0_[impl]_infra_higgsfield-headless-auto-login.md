# 🚀 Higgsfield Headless Auto-Login Flow

## 📌 Context
Higgsfield CLI credentials (`~/.config/higgsfield/credentials.json`) expire every 14-30 days. When this happens, the background daemon `higgsfield-cloud-gen` fails to generate images.
We needed a way to fully automate the re-login process on the VPS without manual SSH intervention.

## 💡 Solution
Created `higgsfield-auto-login.cjs` which is triggered by `higgsfield-auth-watchdog-vps.cjs` when API refresh fails.

1. **CLI Trigger:** Spawns `higgsfield auth login`.
2. **URL Capture:** Parses stdout to extract `https://higgsfield.ai/device?code=XXXX`.
3. **Puppeteer Navigation:** Opens the URL using a persistent profile (`userDataDir: '/root/brain-app/puppeteer-profile'`).
4. **Email & Password Injection:** Clicks "Continue with Email", fills `HIGGSFIELD_EMAIL` and `HIGGSFIELD_PASSWORD` from `.env`.
5. **OTP Fallback:** If the page asks for an OTP (email verification), it polls `/root/brain-app/otp.txt` for 2 minutes. Once the user drops the code in the file, Puppeteer types it in.
6. **Authorization:** Clicks the "Connect" button.
7. **Daemon Restart:** The watchdog verifies `credentials.json` is populated and restarts `pm2 restart higgsfield-cloud-gen`.

## 🛡️ Persistence
Because Puppeteer uses a `userDataDir`, the browser session is cached on the VPS. The OTP step is only required **ONCE** for the profile to be trusted. Future expirations will likely skip the OTP step and just click "Connect" instantly.

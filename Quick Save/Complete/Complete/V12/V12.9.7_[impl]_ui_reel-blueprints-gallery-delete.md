# Unified Gallery, Multi-Select Downloads, and Blueprint Deletion Flow

## 📌 Context (Compiled Truth)
- **Goal:** The user requested:
  1. Combining all generated image assets in the main image preview gallery of the detail view for simple review.
  2. Adding "Download Active" and "Download Selected" buttons, with checkboxes on the gallery thumbnails to enable multi-select downloading.
  3. Adding a "Delete" button in the list view row to delete a blueprint.
  4. Implementing V2 enhancements: strict sequence sorting (First Frame → Grid → Storyboard → Hooks → Scenes), Prev/Next detail navigator, separating Download/Delete/Regenerate toolbar buttons, auto-scroll active thumbnails on navigation, and a modal for scene/storyboard regeneration with editable prompts and auto-attached references.
- **Design Decisions:**
  - Build custom native HTML modals for delete confirmation and image regeneration to fully avoid browser `confirm()` or `alert()` popups.
  - Implement sequential image downloads with a 300ms delay to prevent browser blockages of multi-downloads.
  - Populate the unified gallery with grid, storyboard, scene images, and hook variation images.
  - Create a backend Hono delete route `DELETE /api/reel-blueprints/:id` that cleans up database rows and the filesystem.
  - **Regenerate Architecture:** Spawning `higgsfield` CLI directly on the VPS (`POST /api/reel-blueprints/regenerate-scene/:id`) in a child process is the best practice. The Holyman reference (`/root/brain-app/Holyman_Ref.jpeg`) and storyboard images reside on the VPS, avoiding uploading megabytes of image references from the browser.
  - **Auto-attachment:** Scene/Hook regeneration automatically binds style and character by sending `--image <storyboard> --image <holyman>` to `higgsfield`. Storyboard regeneration binds character by sending `--image <holyman>`.

## User Review Required

> [!IMPORTANT]
> **Regenerate Architecture Decision (Best Practice):**
> 
> There are two approaches for Regenerate:
> 
> **Option A — Browser calls VPS API → VPS daemon runs Higgsfield CLI (Recommended ✅)**
> - The **UI sends a POST request** to a new VPS route `POST /api/reel-blueprints/regenerate-scene/:id`.
> - The **VPS backend** runs `higgsfield generate create gpt_image_2 ...` in a child process (same pattern as `generate_cloud_images.cjs`).
> - Result URL is saved to DB immediately, and a JSON response is returned.
> - **Why this is best practice:** Single source of truth. Holyman ref file lives on VPS at `/root/brain-app/Holyman_Ref.jpeg`. No need to transfer a 2MB reference over the network. The storyboard is also already on VPS. Execution is server-side, so the browser tab can be closed and it still finishes.
> 
> **Option B — Browser calls Higgsfield directly → updates DB separately**
> - ❌ Requires uploading Holyman ref from browser. Complex. Fragile.
> 
> **We go with Option A.**

> [!IMPORTANT]
> **Regenerate UX Flow (from the user's perspective):**
> 1. In the Gallery, tick one or more images (Scene, Hook, or Storyboard) → click **"🔄 Regenerate Selected"**
> 2. A modal opens with an **Edit Prompt textarea** for each selected image:
>    - **For Scenes/Hooks:** Pre-filled with the original `image_prompt`. Shows a badge: `📎 Auto-attached: Storyboard + Holyman Ref`.
>    - **For Storyboard:** Pre-filled with the massive full-length `storyboard_prompt` from the database. Shows a badge: `📎 Auto-attached: Holyman Ref`.
> 3. Clicking "Generate" sends the request to VPS. A spinner shows "Generating... (1/N)".
> 4. As each image completes, the Gallery refreshes in-place.
> 
> *(Note: If you regenerate the Storyboard, any scenes you regenerate AFTER that will automatically use the newly generated Storyboard as their reference!)*

> [!WARNING]
> **Bulk Delete:** Deleting a *scene image* from a blueprint doesn't delete the scene itself — it only nullifies `generated_image_url` and removes the file. The scene's prompt data remains intact for future regeneration.

## Proposed Changes

### Backend

#### [MODIFY] [reel-blueprints.js](file:///c:/My%20Claw/Openclaw-VPS/routes/reel-blueprints.js)

**New Route 1: `POST /regenerate-scene/:id`**

Accepts JSON body:
```json
{
  "scene_number": 2,
  "prompt": "Edited prompt text here...",
  "type": "scene"           // "scene" | "hook" | "storyboard"
}
```

Backend flow:
1. Look up the blueprint row. Parse `scenes_json`.
2. Get `storyboardPath = row.ai_storyboard_image_path` and `holymanPath = '/root/brain-app/Holyman_Ref.jpeg'`.
3. Build the injected prompt: `"Create this image for panel N. Use Image 1 as a reference for the overall style. Use Image 2 as a strict reference for the character's appearance. {user_edited_prompt}"`.
4. Spawn `higgsfield generate create gpt_image_2 --aspect_ratio 9:16 --image <storyboard> --image <holyman> --wait` with the prompt piped via stdin.
5. Parse the returned URL.
6. **For `type: "scene"`:** Update `scenes[scene_number-1].generated_image_url = newUrl`, also update `scenes[scene_number-1].image_prompt = prompt` (so the prompt edit is persisted). Save updated `scenes_json` to DB.
7. **For `type: "hook"`:** Parse `hook_variations_json`, update the hook text and URL at the given index. Save back.
8. **For `type: "storyboard"`:** Spawn `higgsfield generate create gpt_image_2 --aspect_ratio 9:16 --image <holyman> --wait` with the edited massive prompt. Download to disk, update `ai_storyboard_image_path`.
9. Return `{ success: true, url: newUrl }`.

**New Route 2: `POST /delete-images/:id`**

Accepts JSON body:
```json
{
  "items": [
    { "type": "scene", "scene_number": 2 },
    { "type": "hook", "index": 0 },
    { "type": "storyboard" }
  ]
}
```

Backend flow:
1. For each item, nullify the corresponding URL in the JSON and delete the file from disk.
2. Save updated JSON back to DB.
3. Return `{ success: true }`.

```js
window.bpRegenerateSelected = function() {
  const checked = document.querySelectorAll('.bp-gallery-checkbox:checked');
  const items = Array.from(checked).map(cb => {
    const idx = parseInt(cb.dataset.index);
    return window._bpGalleryItems[idx];
  }).filter(item => item.type === 'scene' || item.type === 'hook' || item.type === 'storyboard');
  
  if (items.length === 0) {
    showToast('Select scene or hook images to regenerate', '⚠️');
    return;
  }
  
  // Build modal with one editable prompt per item
  const overlay = document.createElement('div');
  overlay.className = 'modal-overlay';
  overlay.id = 'bp-regen-modal';
  overlay.style.display = 'flex';
  overlay.onclick = (e) => { if (e.target === overlay) overlay.remove(); };

  overlay.innerHTML = `
    <div class="modal" style="max-width:600px; background:var(--surface-1); border:1px solid var(--border); box-shadow:0 30px 45px -30px rgba(50,50,93,0.25); border-radius:8px;">
      <div class="modal-header"><h3 style="margin:0; color:var(--accent);">🔄 Regenerate ${items.length} Image(s)</h3></div>
      <div class="modal-body" style="max-height:60vh; overflow-y:auto;">
        <div style="display:flex; gap:8px; margin-bottom:16px; padding:8px; background:var(--surface-3); border-radius:4px; font-size:12px; color:var(--text-2);">
          <span>📎 Auto-attached: <strong>Storyboard</strong> + <strong>Holyman Ref</strong></span>
        </div>
        ${items.map((item, i) => `
          <div style="margin-bottom:16px;">
            <label style="font-size:12px; color:var(--text-3); display:block; margin-bottom:4px;">${item.title}</label>
            <textarea id="bp-regen-prompt-${i}" class="mc-input" style="min-height:80px; font-size:13px;">${escapeHtml(item.promptText || '')}</textarea>
          </div>
        `).join('')}
      </div>
      <div class="modal-footer">
        <button class="modal-btn-cancel mc-btn mc-btn-ghost" onclick="document.getElementById('bp-regen-modal').remove()">Cancel</button>
        <button id="bp-regen-go-btn" class="modal-btn-save mc-btn mc-btn-primary" onclick="window.bpRunRegenerate()">Generate</button>
      </div>
    </div>`;
  document.body.appendChild(overlay);
  
  window._bpRegenItems = items;
};
```

Execution function:
```js
window.bpRunRegenerate = async function() {
  const items = window._bpRegenItems;
  const btn = document.getElementById('bp-regen-go-btn');
  btn.disabled = true;
  
  for (let i = 0; i < items.length; i++) {
    btn.innerText = `Generating... (${i+1}/${items.length})`;
    const prompt = document.getElementById(`bp-regen-prompt-${i}`)?.value || items[i].promptText;
    
    const body = { prompt };
    if (items[i].type === 'scene') {
      body.type = 'scene';
      body.scene_number = items[i].sceneNumber;
    } else if (items[i].type === 'hook') {
      body.type = 'hook';
      body.hook_index = items[i].hookIndex;
    } else if (items[i].type === 'storyboard') {
      body.type = 'storyboard';
    }
    
    const res = await api(`/reel-blueprints/regenerate-scene/${window._bpCurrentId}`, { method: 'POST', body });
    if (!res || !res.success) {
      showToast(`Failed: ${res?.error || 'Unknown'}`, '❌');
    } else {
      showToast(`${items[i].title} regenerated`, '✅');
    }
  }
  
  document.getElementById('bp-regen-modal')?.remove();
  mcOpenReelBlueprint(window._bpCurrentId); // Refresh the detail view
};
```

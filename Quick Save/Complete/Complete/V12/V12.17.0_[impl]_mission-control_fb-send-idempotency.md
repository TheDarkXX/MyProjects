# EVO-5433: Mission Control Facebook send idempotency

## 📌 Context (Compiled Truth)
Oracle critical finding: Mission Control Facebook send action is not idempotent and is retrying already-scheduled jobs while new jobs remain stuck in `uploading`. 
Impact: 3.75/5 (ทำให้ระบบ publish ทั้งหมดเสถียรขึ้นมาก — ตัดปัญหาที่ user เจอซ้ำๆ หลายรอบ). 
Feasibility: 4/5 (Straightforward state-guard + idempotent return).

**Action Plan:**
1. Make `/api/mc/planner/:id/send-to-facebook` state-aware.
2. If a job is already `scheduled` or `posted` with fb_post_id, return 200 with the existing post metadata instead of 400.
3. Only transition to `uploading` after validation passes.
4. On any upload/schedule failure, immediately set `publish_status='failed'` with error_message.
5. Disable or relabel the UI send button for `scheduled`, `posted`, and `uploading` rows.

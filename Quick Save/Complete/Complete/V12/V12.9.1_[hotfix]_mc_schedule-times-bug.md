# Hotfix: Schedule Times Bug in Viral Planner Pro

## The Problem
When attempting to save the "Set Time of Viral Planner Pro", the user encountered a `Failed to update times` error. Upon debugging the `/planner/schedule-times` POST endpoint in `routes/mission-control.js`, it was discovered that the ES Module codebase attempted to dynamically invoke `require('fs')` and `require('path')`. Since `require` is not available in an ES module, this threw a `ReferenceError` resulting in a 500 API response. A similar silent error was also present in `getScheduleTimes()`.

## The Solution
Replaced all inline calls to `require('fs')` and `require('path')` with the top-level standard `fs` and `path` imports that were already available in the file.

- **2026-05-24**: Investigated UI error popup and located POST `/mc/planner/schedule-times` endpoint mapping.
- **2026-05-24**: Discovered the `require is not defined` ReferenceError.
- **2026-05-24**: Implemented fix in `routes/mission-control.js`.
- **2026-05-24**: Deployed to VPS.

## Overview
This study captures the entire visual DNA extraction and prompt engineering process for the viral "The Reach of Jesus" video series. The goal is to automate the storyboard and prompt generation pipeline using Custom GPTs or Gemini Gems.

## 🎯 The 2-Phase Production Workflow

Based on an analysis of over 100 viral videos, we distilled the production formula into a strict 2-Phase pipeline. 

### PHASE 0: THE ULTIMATE PRODUCTION BLUEPRINT (ASSET LOCK)
Before generating any storyboard scenes, you MUST generate a single 16:9 reference image that contains all visual assets. This guarantees perfect continuity for Video AI.
The Blueprint must contain:
1. **The Holy Man:** (White tunic, red waist belt, red shoulder cloth straight down)
2. **Primary Subject (Before):** (e.g., Crying pregnant soldier)
3. **Primary Subject (After):** (e.g., Smiling pregnant soldier)
4. **Secondary Subject (Before):** (e.g., Skeleton)
5. **Secondary Subject (After):** (e.g., Living soldier husband)
6. **Scene (Before):** (e.g., Grey bombed-out city)
7. **Scene (After):** (e.g., Lush green blooming city)

### PHASE 1: THE 6-SCENE STORYBOARD (M1-S2)
Once the Blueprint is established, generate the 6 specific scenes using the Blueprint as the core visual reference.
1. **M1 (Hook):** Visual + Sound interrupt. First frame must stop the scroll.
2. **M2 (Walk-in):** Jesus enters the scene to help.
3. **S1 (Tears):** Emotion amplifier inserted between M2 and M3. Extreme CU of Jesus crying.
4. **M3 (Transform):** The miracle moment. Before → Light → After.
5. **M4 (Ending):** Resolution. Embrace, peaceful ending.
6. **S2 (Reaction):** The healed subject's joy.

The workflow is: **Image Gen → Veo 3.1 (8s video per scene) → CapCut Assembly**.

# 🎬 The Reach of Jesus — 6-Scene Production Blueprint

## 📐 The 6-Scene Formula

```
┌─────────────────────────────────────────────────────────┐
│  SCENE FLOW (Total: ~20-25s after CapCut trim)          │
│                                                         │
│  M1 ─── M2 ─── S1 ─── M3 ─── M4 ─── S2               │
│  Hook   Walk   Tears  Trans  End    React              │
│  3-4s   3-5s   2-3s   4-6s   3-5s   2-3s              │
│                                                         │
│  M = Main Scene    S = Support Scene (emotion boost)    │
└─────────────────────────────────────────────────────────┘
```

| Scene | Code | Role | Duration | Description |
|-------|------|------|----------|-------------|
| **M1** | HOOK | Visual + Sound interrupt | 3-4s | First frame must **stop the scroll**. Stunning visual, curiosity, or shock. Establish subject & setting. |
| **M2** | WALK-IN | Jesus enters | 3-5s | Holy man walks into frame, discovers the problem, reaches out. Movement = forward walk toward subject. |
| **S1** | TEARS | Emotion amplifier | 2-3s | **Extreme CU** of Jesus's face — sadness, tears rolling down. Inserted **between M2 and M3** to build empathy before the miracle. |
| **M3** | TRANSFORM | Before → Light → After | 4-6s | The miracle moment. Divine light effect → environment/subject transforms. **Must feel smooth, no jarring cuts.** This is the money shot. |
| **M4** | ENDING | Resolution | 3-5s | Embrace, hug, peaceful together. Empathy, happiness, warmth. The "amen" moment. |
| **S2** | REACTION | Subject alive | 2-3s | The healed subject's face/action/activity after the miracle. Joy, gratitude, new life. Inserted **after M4** as the emotional payoff coda. |

### CapCut Assembly Order
```
[M1: 8s Veo clip → trim to 3-4s]
[M2: 8s Veo clip → trim to 3-5s]
[S1: 8s Veo clip → trim to 2-3s]   ← emotion insert
[M3: 8s Veo clip → trim to 4-6s]   ← keep longer, this is the hero
[M4: 8s Veo clip → trim to 3-5s]
[S2: 8s Veo clip → trim to 2-3s]   ← final coda
```

For each analyzed video, auto-generate 3 variations:
- **Same story, different subject** (monkey → kitten, duckling → lamb)
- **Same subject, different setting** (meadow → beach, desert → snow)
- **Same arc, amplified emotion** (stronger hook, more dramatic transform)

**5. CapCut timing guide**
Auto-generate a cut sheet: which second to trim each 8s Veo clip to match the target duration per scene. Could even output an EDL/timeline file.

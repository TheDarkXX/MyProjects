# V12.1.0 [impl] Media Pipeline Skills & ReviewChat Upgrades

## 📌 Context (Compiled Truth)
During a session to batch-rename 18 AI-generated video files based on visual content (first frame), the AI blindly attempted `ffmpeg` and `browser_subagent` without explaining the plan, causing user confusion. We established the "Plan First" protocol in `memory.md` to prevent silent fumbling. To streamline the workflow, we created the `/vdobatch` skill (which automates frame extraction, AI viral caption generation, draft table generation, renaming, and Content_Tracker.md appending) and the `/smartdl` skill (a boilerplate generator for Chrome Extensions that download videos alongside sidecar `.txt` prompts to preserve metadata).

**Viral AI Caption Generation (The Viral Balance Formula):**
The `/vdobatch` skill incorporates an automated Master Prompt to generate captions. The secret to Facebook Reels virality is an "Emotional Bridge" and a strict length constraint.
- **Hook:** Story hook (First 1-sentence).
- **Bridge:** Emotional validation (e.g., "True love still exists.").
- **CTA:** Soft interaction request (e.g., "Drop an AMEN if you believe 👇").
- **Constraint:** Must be < 125 characters before hashtags so the CTA is fully visible before the "See more" truncation.

**Ultimate Auto Web Integration:**
Because the user's files reside on a local Windows machine (`P:` drive) while the Brain App dashboard runs on a Cloud VPS, we architected a 2-way sync bridge. Local `/vdobatch` executions will POST captions to the Cloud Database. A new panel in Mission Control will allow 1-click newline-preserved copying and a "Mark Posted" trigger. A local background skill (`/syncposted`) will fetch POSTED statuses from the API and physically move the `.mp4` files locally.

### docs/skills/vdobatch.md
```markdown
# 🧠 Skill: `/vdobatch` (or `/videobatch`)

## Objective
To completely automate the ingestion, analysis, renaming, and organization of newly generated AI video files into the standard production folder structure.

## Execution Steps

1. **Target Identification:**
   - If the user provides a folder path, use it.
   - Otherwise, ask the user for the folder containing the raw `.mp4` video files.

2. **Setup Automation (First Frame Extraction):**
   - Write and execute a Python script using OpenCV (`cv2`) to extract the very first frame of every `.mp4` file in the folder.
   - Save these frames as `.jpg` images alongside the video files.

3. **Visual Analysis:**
   - Use the `view_file` tool to inspect the generated `.jpg` previews.
   - Analyze the visual content (e.g., characters, setting, actions) to determine the narrative context.

4. **Draft Table & AI Caption Generation:**
   - Correlate the visual content with the user's known Story IDs (e.g., S01, A07, S05).
   - Use the "Viral SEO Master Prompt" to generate a hyper-viral caption (Hook + Climax + AMEN CTA + Niche Hashtags) specifically tailored to that Story ID.
   - Format a Markdown table containing: `Original Name` -> `Proposed New Name` -> `Viral Caption`.
   - Ensure the Naming Convention matches: `[StoryID]_[Beat]_[Version]_REVIEW_[YYYYMMDD].mp4`.
   - Present this Draft Table to the user for confirmation.

5. **Batch Execution & Tracking (Post-Confirmation):**
   - Wait for the user's "Confirm" or adjustments.
   - Run a Python script to batch rename all approved files.
   - Automatically generate the standard Production Folder Structure if it doesn't exist (`REVIEW/`, `READY_TO_POST/TIER...`, etc.).
   - Move all the newly renamed `.mp4` files and their `.jpg` previews into the `REVIEW/` folder.
   - **MANDATORY Tracking:** Append the final data (File Name, Story ID, Viral Caption) into a tracking file named `Content_Tracker.md` located in the root of the video folder.
   - Inform the user of completion.

---

## 📝 Viral SEO Master Prompt (Reference)
When generating captions in Step 4, use this exact persona and structure:

> **Act as a Viral SEO & Psychology Copywriter for Facebook Reels, TikTok, and YouTube Shorts.**
> Your task is to generate 5 short, intensely emotional, and highly engaging English captions for a Christian channel named "The Reach of Jesus". The target audience is the USA and Western countries.
> 
> **Core Themes:** Unconditional Love, Tears of Joy, Divine Mercy, Miracles.
> 
> **Strict Rules & Viral Structure:**
> 1. **The 'Stop-Scrolling' Hook:** A 1-sentence story hook (Curiosity or Emotion).
> 2. **The Emotional Bridge:** A short, comforting statement validating the miracle (e.g., "True love and miracles still exist.").
> 3. **The Engagement Trigger:** A soft but compelling CTA (e.g., "Drop an 'AMEN' if you believe! 🙏👇").
> 4. **Length Constraint:** The text BEFORE hashtags must be around 110-130 characters. This ensures the CTA is perfectly visible right above the "See more" cutoff on FB Reels.
> 5. **Format:** `[Hook] 🥺 [Bridge] ✨ [CTA] 🙏👇 [5 Hashtags]`
> 6. **Hashtags:** Use exactly these 5: `#Jesus #FaithInGod #Miracles #USA #ChristianReels`
```

```markdown
# 🧠 Skill: `/smartdl`

## Objective
To generate a Chrome Extension boilerplate (Content Script + Background) designed to extract video URLs and text prompts from AI generation web UIs, ensuring they are downloaded simultaneously as a "Sidecar" pair (`.mp4` + `.txt`).

## Execution Steps

1. **Target Analysis:**
   - Ask the user to provide the target website's HTML snippet for the video card / prompt block.
   - Analyze the DOM structure to locate the video URL (`src`) and the prompt text (`innerText`).

2. **Extraction Logic (Content Script):**
   - Draft `content.js` to inject a "Smart Download" button into the DOM near each generated video.
   - When clicked, the script must traverse the DOM, extract the video URL and text prompt, and send a message to the Background Script.

3. **Download Pipeline (Background Script):**
   - Draft `background.js` leveraging `chrome.downloads.download()`.
   - The script must trigger the video download and simultaneously create a Blob for the text prompt, triggering a second download with the exact same base filename (Sidecar pair).

4. **Delivery:**
   - Provide the complete code for `manifest.json`, `content.js`, and `background.js` in a clear, copy-pasteable format.
   - Provide brief instructions on how the user can load this Unpacked Extension into Chrome.
```

```markdown
# 🚀 The Ultimate Auto: Content Publishing Pipeline

To achieve the "Most Auto" setup where the Cloud Web Dashboard (Brain App) syncs perfectly with your Local Windows Filesystem (P: Drive), we need a 2-way bridge architecture.

- Create a new UI panel (e.g., "🎥 Media Pipeline") in Brain App Mission Control.
- **One-Click Copy:** A dedicated "Copy Caption" button that copies the exact format (with `<br>`/newlines) so you can paste it directly into Facebook without reformatting.
- **Mark as Posted:** A simple checkbox or button that updates the database status to `POSTED` instantly.

### 3. Phase 3: Auto File Mover (Cloud -> Local Trigger)
- **Option A (Full Auto):** Fetch FB Graph API to check if uploaded.
- **Option B (Simple):** Manual check off in the web dashboard.
- A local background script checks the API every 5 minutes and automatically moves physical files from `REVIEW/` to `POSTED/FACEBOOK/` when the status is updated.
```

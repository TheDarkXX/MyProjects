# Graceful Shutdown Drain for LINE Hermes (V12.22.4)

## 📌 Context (Compiled Truth)
Deploy restarts are currently killing in-flight LINE Hermes child processes (SIGTERM) after the webhook has already accepted the user message, resulting in silent failures and dropped user messages. The solution is to implement graceful shutdown tracking: when receiving SIGINT/SIGTERM, the system should stop accepting new LINE work and wait for active Hermes executions to complete (up to a drain timeout) before exiting.

## Recommended Action
Add graceful shutdown tracking for LINE Hermes child processes: on SIGINT/SIGTERM stop accepting new LINE work, wait for active Hermes executions up to a short drain timeout, and only then exit. For safer shipping, decouple LINE webhook from Hermes execution by enqueueing accepted events and processing them from a worker so deploy restarts do not silently drop user replies.

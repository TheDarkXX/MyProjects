# V12.7.39: AI Monitor Layout Fix

## 1. Problem Statement
The user reported that the AI Monitor bars were visually "messed up" and extremely stretched. Upon investigation, it was discovered that the newer GPT/AutCodex models were returning error messages (such as `not specified`) that contained double quotes. 

Because the AI Monitor injects the `tooltipMsg` directly into an HTML attribute without escaping double quotes:
```html
<div class="ai-bar err" onmousemove="showAiTooltip(event, 'Outage: "not specified"')"></div>
```
The double quote closed the `onmousemove` attribute prematurely, which caused the rest of the JavaScript logic to be parsed as raw text content inside the HTML element. This text forced the CSS flexbox container to stretch excessively, ruining the layout.

## 2. Solution
In `brain-app-public/aimonitor.js`, modified the tooltip payload injection to escape both single quotes (for JS strings) and double quotes (for HTML attribute encapsulation):

```javascript
let tooltipMsg = isOk ? `<b>${dateStr}</b><br/>Latency: ${pt.l}ms<br/>(Avg: ${Math.round(avgLatency)}ms)` : `<b>${dateStr}</b><br/>Outage: ${pt.e || 'Timeout'}`;
let safeTooltip = tooltipMsg.replace(/'/g, "\\'").replace(/"/g, "&quot;");
barsHtml += `<div class="ai-bar ${cls}" onmousemove="showAiTooltip(event, '${safeTooltip}')" onmouseleave="hideAiTooltip()"></div>`;
```

# Internal Auth Completion (V12.22.3)

## 📌 Context (Compiled Truth)
This plan merges three related EVOs from the Oracle addressing cascading regressions from the localhost auth bypass removal (V12.22.1):
1. **EVO-50**: Local cron scripts (viral analytics, cc statements) are failing with 401 Unauthorized because they call internal API endpoints without the `GATEWAY_AUTH_TOKEN`.
2. **EVO-5907**: LINE Hermes calls the semantic search API locally without the `GATEWAY_AUTH_TOKEN`, causing silent DB context loss.
3. **EVO-843**: `routes/auth.js` has a hardcoded fallback token if `GATEWAY_AUTH_TOKEN` is unset. This must be removed to fail-closed, as production relies on the env var being set.

**Implementation Plan:**
- Create a shared `internalFetch` helper or pass `Authorization: Bearer process.env.GATEWAY_AUTH_TOKEN` to cron scripts and LINE Hermes API calls.
- Remove the fallback token string `ZIvyWp4BTqcX2Gm1aDHR7lwz0i8PrVqug5KWBX53wqI` from `routes/auth.js`.
- Add a startup check in `server.js` or `auth.js` to ensure `process.env.GATEWAY_AUTH_TOKEN` is present, else exit immediately.

# 🐝 Honey LINE Memory, Auth & Prompt Hotfixes (V12.20.6)

## Root Cause 1: LINE Memory Save Bug
บอสรายงานว่า Honey ไม่รู้เลขบัตรประชาชนของน้ำชา (ตอบว่าไม่มีข้อมูล) 
เมื่อบอสเช็คโค้ดฝั่ง LINE (Fast Path) ใน `routes/line-hermes.js` พบว่าตอนที่ Honey รับคำสั่งให้จดจำข้อมูล และทำการสร้าง Command `brain-query.sh add-memory ...` ออกมานั้น โค้ดเดิมทำการ `replace` ลบคำสั่งนี้ทิ้งออกไปเฉยๆ เพื่อไม่ให้รกหน้าแชท LINE **โดยไม่ได้นำคำสั่งนั้นไป Execute จริง** ทำให้ทุกอย่างที่ Honey พูดว่า "จดให้แล้วค่ะ" บนแชท LINE ไม่เคยถูกส่งเข้าสู่ฐานข้อมูลเลย

## Proposed Changes 1
### [NEW] `discord-bot/lib/command-parser.js`
- แยกส่วน (Refactor) ฟังก์ชันที่ใช้จับและ Execute คำสั่ง `brain-query.sh` ออกมาจาก `discord-bot/index.js` เป็นโมดูลใหม่ที่ใช้ร่วมกันได้ ทั้งฝั่ง Discord และ LINE
### [MODIFY] `discord-bot/index.js`
- ลบฟังก์ชัน `executeAICommands`, `findTaskByName`, `findBillByName` ทิ้ง (ย้ายไป `command-parser.js`)
- ทำการ `import { executeAICommands } from './lib/command-parser.js'`
### [MODIFY] `routes/line-hermes.js`
- นำเข้า `executeAICommands` มาใช้ใน block ของการตอบแชทเพื่อให้ดักจับและรันคำสั่ง `brain-query.sh` ก่อนจะส่งตอบกลับเข้า LINE แบบสวยงาม

## Changelog: Internal API Auth Fix (401 Unauthorized)

### Root Cause 2
บอสรายงานว่า Pinky บน Discord แจ้งเตือน:
`⚠️ ตอนนี้ Brain Database ยังเชื่อมต่อไม่ได้ค่ะ`
`🔧 Error: Brain API 401: {"error":"Unauthorized","login":"/login.html"}`

ก่อนหน้านี้มีการแก้ไข Security ใน `routes/auth.js` โดยยกเลิกการ Bypass `Host` header สำหรับการเรียก API ภายใน (Localhost) ทำให้คำสั่ง `fetch` จากบอททุกตัวใน VPS จะต้องส่ง `Authorization` header ไปด้วย แต่โค้ด `discord-bot` เก่ายังไม่ได้อัปเดต

### Proposed Changes 2
#### [MODIFY] `discord-bot/lib/brain.js`
- เพิ่ม header `Authorization: Bearer <GATEWAY_AUTH_TOKEN>` ลงในฟังก์ชัน `callBrainAPI` ทุก request
#### [MODIFY] `discord-bot/lib/command-parser.js`
- เพิ่ม header `Authorization` ลงใน `fetch` ของ `findTaskByName` และ `findBillByName`
#### [MODIFY] `discord-bot/lib/ai.js`
- เพิ่ม header `Authorization` ในการเรียก `/api/knowledge/search` เพื่อดึงข้อมูลเข้าสู่บริบท AI
#### [MODIFY] `discord-bot/index.js`
- เพิ่ม header `Authorization` ให้กับ Interactive Buttons สำหรับ `pay-bill` และ `pay-subscription`

## Changelog: Honey Prompt Number Parenthesis Fix

### Root Cause 3
บอสสังเกตว่าเมื่อถามเลขบัตรประชาชนบน LINE, Honey ตอบว่า:
`เลขบัตรประชาชนพี่น้ำชาคือ 1101400810113 ค่ะ (1101400810113) 💳`
สาเหตุมาจากใน System Prompt ของ Honey มีคำสั่งบังคับว่า: *"ตอบเป็นภาษาไทยทั้งหมดเสมอ ห้ามสลับไทยคำอังกฤษคำ แต่ให้ใส่วงเล็บภาษาอังกฤษต่อท้ายเพื่อความสวยงามและครบถ้วน"* LLM จึงแปลตัวเลขเป็นตัวเลขและใส่วงเล็บต่อท้าย

### Proposed Changes 3
#### [MODIFY] `discord-bot/prompts/honey.md`
- แก้ไขบรรทัดที่เป็นกฎของภาษา ให้มีการยกเว้นตัวเลข บัญชี หรือเลขบัตร ไม่ต้องใส่วงเล็บซ้ำ: `แต่ให้ใส่วงเล็บภาษาอังกฤษต่อท้ายเพื่อความสวยงามและครบถ้วน (ยกเว้นตัวเลข บัญชี หรือเลขบัตรต่างๆ ไม่ต้องใส่วงเล็บซ้ำ) เช่น:`

# FB Reels Scraper - Download All Architecture

## 🎯 Objective
ผู้ใช้ต้องการปุ่ม "Download All" ในหน้า Extension (ทั้ง Auto Scan และ Manual Collect) โดยใช้หลักการเดียวกับปุ่ม Download (VPS) แต่รันดาวน์โหลดทีละรายการเรียงลงมาจนครบ (Sequential Queue)

## 🧠 Smart & Best Practice Logic

การทำระบบ Download All ทีละไฟล์บนเว็บเบราว์เซอร์มักจะมีปัญหา (Popup Blocker, เบราว์เซอร์ค้าง, หรือเซิร์ฟเวอร์โดนยิง Request รัวๆ จนแบน) 

**วิธีที่ดีที่สุด (Best Practice) คือการทำใน Chrome Extension Sidepanel** โดยใช้ `chrome.downloads` API แทนการเปิดแท็บใหม่:

1. **Queue Architecture (ระบบคิว):**
   - สร้าง Array คิวของ URL ที่ต้องการดาวน์โหลดทั้งหมด
   - ใช้ `async/await` ควบคู่กับ Promise ที่ไปผูกกับ `chrome.downloads.onChanged`
   
2. **Execution Flow (การทำงาน):**
   - เมื่อกดปุ่ม "Download All" ระบบจะเปลี่ยนสถานะ UI เป็น "กำลังดาวน์โหลด 1/10"
   - สั่ง `chrome.downloads.download({ url: vpsUrl, filename: "FB_Reels/reel_id.mp4", saveAs: false })`
   - รอจนกว่า Event Listener จะส่งสัญญาณว่า `state === 'complete'`
   - หน่วงเวลาเล็กน้อย (Delay 1-2 วินาที) เพื่อไม่ให้ VPS หรือเน็ตโหลดหนักเกินไป
   - วนลูปเริ่มดาวน์โหลดไฟล์ต่อไป

3. **UI/UX Feedback:**
   - เพิ่มแถบ Progress หรืออัปเดต Badge บนแต่ละรายการใน Sidepanel ว่า (⏳ Downloading -> ✅ Downloaded)
   - หากเจอ Error ให้ข้ามไปโหลดไฟล์ถัดไปและแจ้งเตือน

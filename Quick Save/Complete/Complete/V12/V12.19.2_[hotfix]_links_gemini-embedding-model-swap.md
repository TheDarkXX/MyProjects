# V12.19.2 — Fix Broken Gemini Embedding Model in Knowledge Links

## 📌 Context (Compiled Truth)

### Root Cause
`routes/links.js` line 35 hard-codes `models/text-embedding-004` for the local Google proxy at `127.0.0.1:18810`. This model has been retired from `v1beta` and returns a "not found" error on every call.

### Impact
- `/api/links/embed` → 500 error to UI
- `/api/links/embed-all` → silently produces 0 new embeddings (catches per-note failures)
- `/api/links/semantic-search` → 500 error to UI
- **All knowledge semantic search is dead.**

### Implementation Plan

1. **Identify supported model:** Check the local Google proxy at `127.0.0.1:18810` for available embedding models (likely `text-embedding-005` or `embedding-001`).
2. **Update `routes/links.js`:** Replace the hard-coded model string with a configurable value (env var or central config).
3. **Add graceful 503 fallback:** When embedding fails with a model-not-found error, return HTTP 503 with `{ error: 'Embedding model unavailable', model: '...', details: '...' }` instead of an unclassified 500.
4. **Test:** Hit `/api/links/embed` with a test note, verify vector generation works.

# Fix Unknown Models & Add Global Paid Tokens Filter

## What Was Done
1. **Fix Unknown Models in Analytics**:
   - Added missing entries to `MODEL_PRICING` in `openclaw.js` (e.g. DeepSeek V3.2, GLM-4.7, GPT-5.5, GPT-5.4 Mini).
   - Added fallback logic for older sessions missing the `modelProvider` field to derive it from `mp.note`.
   - Updated `ai-gateway.js` and `discord-bot/lib/ai.js` to properly track and pass `modelProvider` in new sessions.

2. **Global "Paid Tokens Only" Filter**:
   - Added a UI checkbox in the `.rpt-filter-bar` of `index.html`.
   - Checkbox defaults to `checked` (`true`) to focus the dashboard on actual resource expenditure.
   - Filter logic applies at the top of `initOpenclawPage()` before any charts or stats are rendered.
   - When checked, models with `free: true` (ChatGPT Pro, AI Studio Gemini, BytePlus GLM) are stripped from `allSessions` so they don't bloat the token volume metrics.
   - Verified that `DEFAULT_PRICING` correctly has `free: true`, so unrecognized models don't falsely inflate paid stats.

# Global Paid Tokens Filter

Make the "Paid Tokens Only" toggle apply globally across the entire OpenClaw Analysis page rather than just the Daily Chart.

## Rationale
Currently, the "Paid Tokens Only" toggle only affects the Daily Token Usage bar chart. The user suggested making it a global filter for the entire page. Doing so aligns the entire dashboard with actual resource expenditure, which is often the primary metric of interest. 

## User Review Required

> [!WARNING]
> Making this global means that when "Paid Tokens Only" is checked:
> - **Agent Intelligence (Leaderboard)**: Will ONLY show tokens from paid models. Agents using free models (like GPT-5.4 via ChatGPT Pro) will appear to have 0 tokens.
> - **Model Usage (Donut Chart)**: Free models will completely disappear from the chart.
> - **Hero Stats**: The "Tokens Today", "7-Day Avg", and "30-Day Total" will drop significantly as they will exclude free tokens. 
>
> Are you okay with the free models completely disappearing from the leaderboards and charts when the toggle is active? (You can always uncheck it to see the raw total).

## Proposed Changes

### `brain-app-public/openclaw.js`
Move the filtering logic to the top of `initOpenclawPage()` so it applies to all downstream functions.

#### [MODIFY] openclaw.js
- Read the `ocl_daily_paid_only` from localStorage at the start of `initOpenclawPage()`.
- Filter `allSessions` and `mergedSessions` immediately before passing them to any render functions:
```javascript
  const paidOnly = localStorage.getItem('ocl_daily_paid_only') !== 'false';
  let activeMergedSessions = mergedSessions;
  let activeAllSessions = allSessions;

  if (paidOnly) {
    activeMergedSessions = mergedSessions.filter(s => !getModelPricing(s.model).free);
    activeAllSessions = allSessions.filter(s => !getModelPricing(s.model).free);
  }
```
- Remove the local filtering logic from `renderOclDailyBars` since it will now receive pre-filtered data.

### `brain-app-public/index.html`
Move the checkbox from the "Daily Token Usage" chart header up to the main Page Header or the Filter Bar, so it's clear that it applies to the whole page.

#### [MODIFY] index.html
- Move the toggle to the `.rpt-filter-bar` next to the date range pills.
- Remove it from the Daily Token Usage chart header.

## Verification Plan
1. Toggle the filter to ON: Verify the Hero stats token counts drop, the Model Usage donut chart has no free models, and the Agent Leaderboard only counts paid tokens.
2. Toggle the filter to OFF: Verify all free tokens return across all charts.

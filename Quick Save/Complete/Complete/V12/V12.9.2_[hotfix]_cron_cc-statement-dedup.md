# Fix KBank CC Statement Re-Processing

## Context & Root Cause
The cron job `cron-cc-statements.js` searches for emails with `newer_than:30d`. It then relies on the local `/api/cc/credit-cards` API to populate `uniqueBanks` to skip processing if a statement for the *current* month has already been imported.
However, because KBank's statement for the current month hasn't arrived yet (e.g., we are in May, but the last imported statement was for April), `uniqueBanks` does not contain KBank for May.
As a result, the search query finds the *April* statement (which is still `<30d` old), and the script re-processes it with AI (costing tokens) every single day. The DB layer correctly deduplicates the entries (`0 inserted, X skipped`), but the script still sends a Discord notification every day saying `0 new, X skipped`.

## Implementation (The Fix)
Introduced email-level deduplication by keeping track of processed Gmail Message IDs in a local JSON file (`discord-bot/data/processed_cc_emails.json`).

1. **Load Processed IDs:**
   Before the email loop, load the array of processed message IDs from `processed_cc_emails.json` (if it exists).
2. **Skip Known Emails:**
   Inside the loop over `data.messages`, check if `msg.id` is in the processed array. If it is, log a message (`⏭️ ข้ามอีเมล {msg.id} (เคยประมวลผลไปแล้ว)`) and `continue`.
3. **Save Processed IDs:**
   After successfully calling `processCreditCardStatement`, push `msg.id` to the array, keep the last 100 entries to prevent the file from growing indefinitely, and save it back to `processed_cc_emails.json`.

This approach ensures that we NEVER re-parse the same email twice, completely eliminating the daily Discord spam and saving AI token costs.

## Credit Card Arrival Dates Insight
Also queried the historical data to understand when credit card statements usually arrive. The results match the user's observation:
- **Bangkok Bank (BBL):** Exactly on the **20th** of every month.
- **Krungsri:** Mostly on the **19th** (sometimes 20th or 22nd) of every month.
- **CardX (SCB):** Roughly on the **26th or 27th** of every month.
- **KBank:** Roughly on the **27th or 28th** of every month.

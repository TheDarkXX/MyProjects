# GLM 4.7 AI Monitor Removal

## 📌 Context (Compiled Truth)
The AI Monitor logic (`discord-bot/scripts/cron-ai-monitor.js`) automatically removes models from the JSON status file only when they are no longer defined in the hardcoded `MODELS` array. It does not auto-remove actively monitored models that are failing (red). This behavior prevents the system from permanently abandoning models during a temporary outage, instead enforcing a 429 cooldown. Since GLM 4.7 was persistently problematic and not expected to recover, it required a manual code-level removal.

## 📦 RAW ARTIFACT BACKUP (Iron Rule)
*(No artifacts were generated in this session. All modifications were done directly inside `cron-ai-monitor.js` based on the user's report.)*

## 🔬 Timeline & Debugging Log
- **2026-05-29:** User reported that GLM 4.7 was still showing in the monitor despite a previous expectation of auto-removal when red.
- Investigated `cron-ai-monitor.js` and confirmed the actual behavior (auto-remove only works for models absent from `MODELS`).
- Removed `bp-glm-4.7` from `MODELS` array.
- Code changes deployed and cache bumped successfully.

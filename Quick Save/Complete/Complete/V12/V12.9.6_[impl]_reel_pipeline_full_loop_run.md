# 🎬 Full Loop Storyboard & Scene Generation Run

## 📌 Context (Compiled Truth)
- **Goal:** The user requested running the full image generation loop (storyboard, scenes, and all hook variations) for all blueprints in the system. The operation must run entirely on the cloud VPS so they can turn off their local machine.
- **Execution Strategy:**
  1. We use the background daemon script `generate_cloud_images.cjs` running on the VPS under PM2 (`higgsfield-cloud-gen`, process ID `11`).
  2. The daemon iterates through all 6 records (IDs 49, 47, 46, 43, 34, 31) in `/root/brain-app/db/brain.db`.
  3. For each record, if the storyboard is missing, it runs Higgsfield `gpt_image_2` (9:16 aspect ratio).
  4. It sequentially runs image generation for each scene using the storyboard as Image Ref 1 and `/root/brain-app/Holyman_Ref.jpeg` as Image Ref 2.
  5. It generates Hook Variations (A-C) using the same dual references.
  6. The database columns `scenes_json` and `hook_variations_json` are updated progressively after each image is generated to prevent loss of progress.
  7. Images are backed up locally in the `/root/brain-app/public/blueprints/` folder.

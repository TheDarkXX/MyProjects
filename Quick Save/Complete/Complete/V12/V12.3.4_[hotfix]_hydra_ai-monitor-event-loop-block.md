## 📋 Compiled Truth & Timeline

### The Issue
After deploying `V12.2.0_[impl]_hydra_oracle-rebuild-completion.md`, the AI Monitor completely broke. The dashboard showed timeouts (`ERR: spawnSync /bin/sh ETIMEDOUT`) and fetch timeouts (`The operation was aborted due to timeout`) for all models.

### Root Cause
In `discord-bot/scripts/cron-ai-monitor.js`, the models are pinged concurrently using `Promise.all(promises)`. However, the implementation for `openclaw-cli` models (GPT-5.1, GPT-5.4, GPT-5.5) used `execSync` with a timeout of 20000ms. 
Because `execSync` is synchronous, it blocks the main JS thread. 3 OpenClaw models timing out sequentially blocked the event loop for 60 seconds. This prevented the asynchronous `fetch` requests (for Openrouter, Kie, etc.) from resolving or handling their own timeouts gracefully, causing them to abort abruptly when the event loop unblocked.

### Actions Taken
1. Replaced `execSync` with an asynchronous `execAsync` (using `util.promisify(exec)`) in `pingModel` for `openclaw-cli` provider in `cron-ai-monitor.js`.
2. Replaced `execSync` with `execAsync` for the background Oracle retry command at the bottom of the script.
3. The openclaw agent then failed with "Gateway url override requires explicit credentials". Fixed by setting `envCopy.OPENCLAW_GATEWAY_URL = ''` before executing the child process, which prevented `dotenv` inside the CLI from incorrectly loading the gateway URL override from the `.env` file.
4. `openai-codex/gpt-5.5` was taking ~56 seconds to respond, which caused the new `execAsync` to hit its 20s timeout and fail silently. Increased the `execAsync` timeout to 60000ms and the CLI timeout to 45000ms.
5. Discovered that **Oracle (`scripts/hydra/oracle.js`)** was also failing for the exact same 3 reasons (event loop blocking, OPENCLAW_GATEWAY_URL bleed, and locked session files from timeouts).
6. Applied the same `execAsync` and `envCopy.OPENCLAW_GATEWAY_URL = ''` fix to `oracle.js`.
7. Cleared stale `.jsonl.lock` files in `/root/.openclaw/agents/main/sessions/` which were causing the agent to fallback.
8. Deployed the fixes to VPS, restoring full functionality to the AI Connectivity Matrix dashboard and confirming GPT-5.5 is "green". Verified Oracle completes a full delta run successfully.

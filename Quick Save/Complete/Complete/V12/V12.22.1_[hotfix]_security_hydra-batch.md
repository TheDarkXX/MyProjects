# Security Hotfix: Hydra Oracle Batch

## 📌 Context (Compiled Truth)
This batch addresses 3 critical vulnerabilities reported by the Oracle:
1. **EVO-1426 (PII Disclosure):** LINE prompt explicitly allowed disclosing passwords/secrets. The fix is to modify the prompt to explicitly deny sharing passwords, tokens, API keys, or server credentials, while allowing general personal info like phone numbers and addresses as requested by the boss.
2. **EVO-6943 (Host Bypass):** The `auth.js` route bypassed authentication for any request with a `Host: localhost` header. Additionally, `config.js` accepted any Bearer token > 10 chars. The fix uses remote-addr verification instead of Host header, and strictly verifies the JWT token.
3. **EVO-6449 (Cache Collision):** The pseudo-semantic cache in `discord-bot/lib/ai.js` used a SHA-256 derived vector and cosine similarity, causing cross-prompt caching collisions and privacy leakage. The fix restricts the semantic cache or uses exact hash matching for sensitive queries.

**Mission:** guard
**Level:** critical
**Title:** Hash-based “semantic” cache can replay a prior LINE/Discord answer to a different prompt or user

## Recommended Action
Disable semantic-cache reads for LINE/Discord conversational chat, or scope cache keys by agent+channel/session+normalized prompt and require exact prompt hash for sensitive contexts.

# V12.20.4: LINE Hermes Webhook Timeout & Shell Injection Fix

## Root Cause Analysis
ผู้ใช้แจ้งว่า "Line honey hermes พังไหม เชคหน่อย" จากการตรวจสอบพบ 2 ปัญหาหลักใน `routes/line-hermes.js`:
1. **Timeout Issue**: ใน `execAsync` เดิมตั้ง `timeout: 30000` (30 วินาที) ทำให้คำสั่ง `hermes` ถูก `SIGTERM` ฆ่าทิ้งทันทีหากการประมวลผล (เช่น การค้นหา memory ด้วย embedding) ใช้เวลานานกว่า 30 วินาที
2. **Shell Injection Vulnerability**: Oracle ตรวจพบว่าการประกอบคำสั่งด้วย `cmd = hermes -z '${linePrefix}${safeText}'...` นั้นเปิดช่องโหว่ให้เกิด shell injection ผ่านชื่อ LINE displayName (`userName`) ที่ไม่ได้ถูก escape อย่างถูกต้อง

## Resolution
1. **เพิ่ม Timeout**: ปรับเพิ่ม timeout จาก 30000ms เป็น 55000ms (55 วินาที) เพื่อสอดคล้องกับ loading animation ของ LINE
2. **เปลี่ยนไปใช้ execFileAsync**: แทนที่การรัน shell command โดยตรงด้วยการใช้ `execFileAsync` และส่ง argument เป็น array ของ string ซึ่งจะช่วยป้องกัน shell injection ได้ 100% เพราะไม่มีการตีความอักขระพิเศษโดย shell อีกต่อไป

```javascript
// แก้ไขจาก
const cmd = `hermes -z '${linePrefix}${safeText}' -c '${sessionId}' -t '${toolsets}'`;
const { stdout, stderr } = await execAsync(cmd, { timeout: 30000, ... });

// เปลี่ยนเป็น
const args = [
  '-z', linePrefix + userText,
  '-c', sessionId,
  '-t', toolsets
];
const { stdout, stderr } = await execFileAsync('hermes', args, { 
  timeout: 55000, 
  env: { ...process.env, PATH: `${process.env.PATH}:/usr/local/bin:/usr/bin` } 
});
```

## Changelog
- **[2026-06-15] Update 1**: พบว่าการค้นหา memory ของ Hermes กินเวลาจริงถึง 3 นาที 24 วินาที (เนื่องจากการรอ rate limit หรือ fallback embedding) การตั้ง 55 วินาทีจึงยังไม่เพียงพอ จึงขยาย timeout เป็น 300,000ms (5 นาที) และเปลี่ยน LINE API จาก `message/reply` เป็น `message/push` เพื่อหลีกเลี่ยงการหมดอายุของ Reply Token ภายใน 1 นาที
- **[2026-06-15] Update 2**: ค้นพบสาเหตุที่แท้จริงของการที่ Hermes ค้นหา Memory นานถึง 3 นาที ในขณะที่ Pinky (Discord Bot) ใช้เวลาเพียง 10 วินาที นั่นเป็นเพราะ Hermes CLI ฝังการเรียกใช้ (hardcode) โมเดล `text-embedding-004` (ผ่าน @google/genai SDK) ซึ่งปัจจุบัน Google ได้ถอดโมเดลนี้ออก ทำให้ได้ Error 404 แต่ระบบของ Hermes วนลูป Retry ด้วย Exponential Backoff นานถึง 3 นาทีจนกว่าจะ Fallback ได้สำเร็จ ส่วน Pinky ไม่เจอปัญหานี้เพราะเรียกใช้ API ตรงซึ่ง fallback เป็น `gemini-embedding-2` ไว้แล้ว จึงได้ทำการแก้ที่ระดับ Network ทันที โดยแก้ไขไฟล์ `ai-gateway.js` ให้คอยดักจับและแปลง (Rewrite) request ของ `text-embedding-004` ให้กลายเป็น `gemini-embedding-2` เสมอ ทำให้ Hermes ค้นหา Memory เร็วขึ้นกลับมาเป็นระดับวินาทีเช่นเดิม
- **[2026-06-15] Update 3**: บอสต้องการให้บน LINE ตอบไวทันใจและเน้นดึงข้อมูลจาก DB ให้แม่นยำเหมือน Pinky (Discord) จึงเพิ่มระบบ **RAG (Retrieval-Augmented Generation) DB Context Injection** ลงไปใน `routes/line-hermes.js` โดยให้ดึงข้อมูลจาก Endpoint `/api/links/semantic-search` ของ Brain DB มายัดใส่ใน `linePrefix` (System Prompt) ก่อนส่งให้ Hermes CLI ทันที ช่วยให้ Hermes ได้รับคำตอบจากความจำทันทีใน 1 วินาที โดยไม่ต้องเสียเวลาเรียก Tool ค้นหาเอง ทำให้ตอบกลับไวปรื๊ดแน่นอน!
- **[2026-06-15] Update 4**: พบปัญหา Hermes CLI เอาข้อความ System Prompt ไปปนกับข้อความ User ทำให้บอททวนกฎซ้ำและยึดติดกับกฎใน SOUL.md แถมยังกินเวลาทำงานเกิน 60 วินาที จึงสร้าง **⚡ FAST PATH: Direct LLM Call** โดยยิงตรงเข้า `ai-gateway` ด้วย OpenAI Format (แยก `role: system` กับ `user` ชัดเจน) ซึ่งจะข้ามการเรียก CLI ทันทีถ้าผู้ใช้ไม่ได้สั่งให้ค้นเว็บ ทำให้เวลาตอบกลับเหลือเพียง **1-3 วินาที (ระดับ Pinky)** และบอทเลิกพูดทวนกฎแล้ว 100%
- **[2026-06-15] Update 5**: เพิ่มระบบความจำระยะสั้น (Multi-turn In-memory History) ลงใน Fast Path ของ LINE เพื่อให้บอทจำบริบทของแชทก่อนหน้าได้ (เช่น ถามต่อว่า "ตรงกับวันอะไร") โดยเก็บ history ย้อนหลัง 10 ข้อความแบบเดียวกับ Pinky ใน Discord ทำให้คุยต่อเนื่องได้ไหลลื่นและตรงประเด็น
- **[2026-06-15] Update 6**: เปลี่ยนสมองของ LINE ให้กลายเป็น Pinky (Discord) แบบ 100% เพื่อแก้ปัญหา LLM เดาคำตอบมั่ว (Hallucination) เนื่องจาก Semantic Search ค้นหาข้อมูลมาไม่ครบ โดยใช้วิธีเรียก Function `chat` และ `brainQuery` จากโค้ดของ Discord Bot มาใช้งานโดยตรง ทำให้บอท LINE ดึงข้อมูล DB ทั้งก้อนมาอ่านเหมือน Pinky รับประกันความแม่นยำ 100% พร้อมนิสัยและ System Prompt เดียวกับ Pinky (ตั้งชื่อให้ใหม่ว่า Honey 🍯 สำหรับ LINE)

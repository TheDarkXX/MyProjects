# V12.0.0 [impl] AG-Skills — Smart Versioning v2

## 📌 Context (Compiled Truth)
The previous versioning strategy caused rapid version inflation (V1 to V11 in 37 days). Root causes included:
1. `Rollover Rule` auto-bumped Major every 10 features.
2. Hydra EVOs generated by `/evo` and `/dev` that were rejected, iceboxed, or deepfreezed consumed version numbers without shipping anything.
3. Studies and designs consumed `.1` bumps instead of patch bumps.

**The Solution (Smart Versioning v2):**
We implemented Semantic Versioning correctly without digit ceilings (0-99+ range allowed for minor).
- `[study]`, `[design]`, `[docs]`, `[hotfix]` now only bump Patch (+0.0.1).
- `[impl]`, `[infra]` bump Minor (+0.1.0) infinitely (no rollover at 10).
- **Major auto-bumps** only when hitting 20 shipped impls or a major architectural shift.
- **EVO / STUB exception:** Unapproved proposals are saved as `EVO-{id}_[study]_{component}_{desc}.md` and do not consume a version number.

## 📚 Research: SemVer Best Practices

ผลการ research ยืนยัน:

> **SemVer ไม่มี limit สำหรับ digit** — Minor/Patch เป็น non-negative integer ได้ไม่จำกัด (0-99, 0-999 ฯลฯ)

| Project | Current Version | Minor Range |
|---------|-----------------|-------------|
| **Linux Kernel** | 6.12.x | minor 0-12+ |
| **Node.js** | 22.x | major 22+ |
| **React** | 19.x | major 19+ |
| **VS Code** | 1.96.x | minor 96+ |
| **TypeScript** | 5.7.x | minor 0-7+ |

**สรุป:** แอปใหญ่ๆ ใช้ unlimited minor range ทั้งนั้น ไม่มี rollover ที่ 10

## 🧠 Smart Major Auto-Bump Rule (ออกแบบใหม่)

แทน manual decision, ใช้กฎ auto ที่ smart:

### Major Bump Trigger (Auto — ทำเองไม่ต้องถาม)

| Trigger | Condition | Example |
|---------|-----------|---------|
| **Cumulative Milestone** | เมื่อ **shipped `[impl]` + `[infra]` รวม 20 ตัว** นับจาก major ล่าสุด → auto bump major | V12.0 → ... → V12.19.x (20 impls shipped) → V13.0.0 |
| **New System Component** | เมื่อเพิ่ม **component ใหม่ที่ไม่เคยมีมาก่อน** (เช่น ระบบใหม่สด) → ถ้า impact สูง AG bump major | เพิ่ม "payment gateway" component ใหม่ |
| **Architectural Breaking** | เปลี่ยน architecture ที่ affect ≥3 components พร้อมกัน | Rewrite AI Gateway + Hydra + Deploy pipeline |

## 📋 Full Versioning Rules v2 (จะแทนที่กฎเก่าใน AGENTS.md)

```markdown
### Versioning Strategy v2 (Format: Major.Minor.Patch)

**Patch (+0.0.1):** Hotfixes, UI tweaks, docs, study, design
- `[hotfix]`, `[design]`, `[study]`, `[docs]`, `[spike]`

**Minor (+0.1.0):** New shipped features, infrastructure changes  
- `[impl]`, `[infra]` ที่ ship แล้ว

**Major (Auto — Smart Milestone):**
- AG ต้อง count จำนวน shipped `[impl]` + `[infra]` นับจาก major version ปัจจุบัน
- เมื่อ **ครบ 20 ตัว** → auto bump major
- หรือเมื่อเพิ่ม **system component ใหม่ที่ยังไม่เคยมี** + impact สูง
- หรือ **architectural breaking change** ที่ affect ≥3 components

**Minor/Patch ไม่มี rollover limit:**
- ใช้ได้ไม่จำกัด (V12.25.3, V12.99.7 ฯลฯ)
- ไม่มีการ rollover minor เป็น major อัตโนมัติ

**Hydra EVOs (ไม่กิน version):**
- Proposals ที่ยังไม่ approved ใช้ format: `EVO-{id}_[study]_{component}_{desc}.md`
- เฉพาะเมื่อ ✅ APPROVE เท่านั้นจึง assign version `V{x.y.z}_[impl]_...`
```

# V12.7.38 [hotfix] Viral Planner Sync & UI Fixes

## 1. Facebook Reel Sync Failure
**Issue:**
The Viral Planner `sync-analytics` endpoint was consistently failing for scheduled jobs that were successfully published. The logs indicated `(#100) The value must be a valid insights metric` and `(#200) read_insights permission missing`.

**Root Cause:**
All videos from the system are uploaded via the `/video_reels` edge. Facebook Graph API does not support standard `video_insights` metrics (like `total_video_views`) for Reels in the same way, or it requires different endpoints. Querying `video_insights` for a Reel throws an error, causing the sync logic to skip the status update, leaving the job permanently in "Scheduled" status.

**Solution:**
Refactored the `sync-analytics` endpoint to query the basic video node directly (`?fields=status,views,comments.summary(true)`). 
- `views` and `comments` are natively available on the Reel object.
- Replaced `video_insights` with this direct query.
- Ensured it parses `data.status.publishing_phase.publish_status === 'published'` before transitioning the system status from `scheduled` to `posted`.

## 2. Viral Planner UI Time Display Confusion
**Issue:**
The user was confused by the `datetime-local` input in the "Scheduled" column, as it displayed e.g., `07:00` for a slot scheduled at 19:00 local time, making it look like 7:00 AM (past) instead of 7:00 PM (future).

**Root Cause:**
The input `type="datetime-local"` was constrained to `width: 130px`. On Chrome/Windows with US locale, this was just narrow enough to hide the `AM/PM` suffix, making a 12-hour formatted time look like a 24-hour time in the morning.

**Solution:**
- Increased the width of the `.vp-datetime` input from `130px` to `150px` in `vpRenderTable`.
- Ensured a `min-width: 150px` in the Publish Wizard as well.
- This allows the browser to fully render the `AM/PM` designator, eliminating the confusion.

## 3. Dedicated URL Route for Viral Planner Pro
**Issue:**
Viral Planner Pro was previously tightly coupled with Mission Control (`#mc`), requiring users to navigate to Mission Control first and then open the planner. It did not have its own direct bookmarkable URL.

**Solution:**
- Added `#viral-planner` to the `validPages` list in `app.js`.
- Configured routing logic in `navigateTo()` so that `#viral-planner` automatically mounts the `page-mc` container, updates the Sidebar/Breadcrumbs to correctly highlight "Viral Planner Pro", and immediately invokes `mcShowPlanner()`.
- Updated `index.html` navigation button to use `navigateTo('viral-planner')` instead of a custom script execution.

## 4. UI/UX: 24hr Time Format & Modal Auto-Close Fix
**Issue:**
- Time selectors (`datetime-local`) were showing 12hr AM/PM format dependent on the user's OS locale (e.g. Windows).
- Clicking schedule inside the Video Modal caused the modal to abruptly close because the parent container (`page-mc`) was entirely re-rendered by `vpRender()`.
- Users needed a state to mark jobs that are not yet ready (e.g., pending re-edit).

**Solution:**
- Implemented a 24hr format overlay label `div` over an invisible (`opacity: 0`) `datetime-local` input, guaranteeing consistent 24hr `DD/MM/YYYY HH:mm` presentation across all environments.
- Decoupled `vp-video-modal` and `vp-image-modal` from the `vpRender()` string template, migrating them to `vpInitModals()` which mounts them directly onto `document.body`. This prevents `vpRender()` from destroying the active modal DOM nodes when state updates occur.
- Added `🛠️ Incomplete` to the pipeline tabs and dropdowns, wired to `publish_status = 'incomplete'`.

## 5. Send All Workflow & Smart Organize (V12.7.46 - V12.7.48)
**Issue:**
- "Send All" was ignoring items that used the new Custom Cover feature.
- Users couldn't easily spot which items were pending a schedule.
- The confirmation modal for Send All was invisible.

**Solution:**
- **Smart Organize:** Implemented `⭐ Smart Organize` as the default sorting logic. It floats scheduled (but unpublished) jobs to the top ordered chronologically, then falls back to `viral_score` sorting for unpublished items.
- **Row Highlighting:** Added dynamic row styling in `vpRenderTable`. Jobs with a schedule get a purple tint (`rgba(168, 85, 247, 0.15)`) and a bold left-border (`#a855f7`) to immediately distinguish them.
- **Send All Logic Fix:** Updated the `readyJobs` filter to ensure `(j.thumbnail_url || j.custom_thumbnail_url)` passes and explicitly rejected `incomplete` status jobs from batch processing.
- **Modal CSS Fix:** Added inline layout and `z-index` overlay properties to the `vpConfirm` modal, since the expected `.vp-modal` class was not defined in the CSS stylesheets.

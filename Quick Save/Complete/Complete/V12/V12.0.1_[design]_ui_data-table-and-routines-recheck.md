## 🔵 Feature 1: ระบบตาราง Data (Excel-Like Tables)

### แนวคิด
- เพิ่มระบบตาราง **แยกออกมาจาก Task** — ไว้จดข้อมูลแบบตาราง
- UI เป็น **Excel-like** เหมือนตาราง 🏠 Life/Tasks ที่มีอยู่แล้ว
- แยก Project ได้ (เพิ่ม/ลบ Project)

### Use Case ตัวอย่าง
- จดตาราง "สร้างเพจ Facebook ใหม่" → columns: ชื่อเพจ, วันที่สร้าง, วิธีสร้าง, จำนวนเพจ
- จดตาราง "รายรับ-รายจ่าย" → columns: วันที่, รายการ, จำนวนเงิน
- จดตาราง "Content Plan" → columns: หัวข้อ, platform, status, วันที่ publish

### Tasks
- [x] ออกแบบ DB schema สำหรับ custom tables (table_name, columns, rows, project_id)
- [x] API endpoints: CRUD tables, CRUD rows, CRUD columns, CRUD projects
- [x] UI: สร้างหน้า Data Tables — ไอคอนใหม่ใน Primary Sidebar ใต้จรวด 🚀
- [x] UI: Secondary Sidebar — แสดงรายชื่อ Projects (เพิ่ม/ลบ/เลือก Project)
- [x] UI: Table editor — inline edit, add/edit/delete rows & columns
- [x] UI: Excel-like features — column resize, sort, filter, frozen header
- [x] UI: Basic Templates — 5 templates ให้เลือกตอนสร้าง Project ใหม่
- [x] UI: Right-click context menu สำหรับ rows (Edit, Duplicate, Delete, Move)

### 🔗 Multi-Table & Relations — ประเมินความซับซ้อน

> **คำถาม:** ควรทำแบบ Airtable (หลาย table เชื่อมกัน) หรือเริ่มแบบง่ายก่อน?

**ข้อเสนอ: เริ่มแบบ "Single Table per Project" ก่อน (Phase 1)**
- Phase 1: 1 Project = 1 Table (เหมือน Google Sheets ง่ายๆ) ← **ทำก่อน**
- Phase 2 (Future): เพิ่ม Multi-Table per Project + Linked Records (Relations)
- เหตุผล: ถ้าทำ Relations ตั้งแต่แรกจะซับซ้อนมาก (schema, UI, migration)
  และ use case ปัจจุบันยังไม่ต้องการ. เริ่มง่ายก่อนแล้วค่อยขยาย

## 🤔 Design Decisions (อัปเดตแล้ว)

1. **Data Table อยู่หน้าไหน?**
   - **ตำแหน่งไอคอน (Primary Sidebar):** อยู่ใต้รูปจรวด 🚀 (เมนูใหม่สำหรับ Data Table โดยเฉพาะ)
   - **โครงสร้าง (Secondary Sidebar):** พอกดไอคอนแล้ว กรอบเมนูย่อย (เหมือน Mission Control) จะแสดงรายชื่อ **Projects**
   - **ฟีเจอร์ Project:** สามารถ **เพิ่ม/ลบ Project** ได้ในแถบนี้เลย 

2. **Column types รองรับอะไรบ้าง?** (อ้างอิง ClickUp / Notion / Airtable)
   - **พื้นฐาน:** Text, Number, Date, Checkbox, URL
   - **จัดกลุ่ม:** Select (Single Dropdown), Tags (Multi-select), Status (สีตาม design system)
   - **เงิน:** Currency (Number + สัญลักษณ์ ฿/$, JetBrains Mono)
   - **(Phase 2):** Rating, Formula, File Attachment, Linked Record (Relations)

3. **จัดการ Routine Task ที่ค้างสะสม (Overdue)**
   - **ตัดสินใจแล้ว:** ใช้ Option A (Auto-Skip งานเก่าของเมื่อวานทิ้งไปเลย)

4. **Multi-Table Complexity — เริ่มอย่างไร?**
   - **ตัดสินใจ:** Phase 1 = Single Table per Project (ง่าย ใช้ได้เลย)
   - Phase 2 = Multi-Table + Relations (เมื่อต้องการจริงค่อยเพิ่ม)

# Data Tables UI/UX Overhaul (Notion/Airtable Level)

เป้าหมายคือการยกระดับ Data Tables ให้มีความรู้สึกพรีเมียม (Premium Feel) และมีฟังก์ชันการใช้งานเทียบเท่า Top Apps อย่าง Notion หรือ Airtable ตามที่คุณหมอต้องการ

## User Review Required

มีรายละเอียดการออกแบบ UI และพฤติกรรม (Behavior) ที่ต้องรบกวนคุณหมอพิจารณาก่อนเริ่มลงมือเขียนโค้ดครับ

## Open Questions

1. **สีของ Select (Tags):** สำหรับคอลัมน์ประเภท Select (เช่น Status: Active, Pending) คุณหมออยากให้ระบบสุ่มสีให้แบบอัตโนมัติ (เช่น Notion) หรือให้เป็นสีโทนเดียว (Monochrome) ดูกลืนกับธีม Dark Mode ของเราครับ?
2. **การลบ/แก้ไข Column:** เนื่องจากเราใช้ ag-Grid แบบ Free Version ซึ่งไม่รองรับ Header Context Menu แบบคลิกขวาโดยตรง ผมจะใช้ท่า **"ปุ่มไข่ปลา (⋮) หรือลูกศร (▼) บนหัวคอลัมน์"** เมื่อคลิกซ้ายจะเด้งเมนู Edit/Delete/Rename ขึ้นมาแทน ซึ่งเป็นวิธีที่ Airtable ใช้ แบบนี้โอเคไหมครับ?

## Proposed Changes

### 1. Visual & Aesthetics (แก้ปัญหาเส้นตาราง & UI)
- **Grid Lines & Contrast:** เขียน CSS Overrides ทับ `ag-theme-alpine-dark` เพื่อให้เส้นขอบเซลล์ (Border) ชัดเจนขึ้น (เช่น `rgba(255,255,255,0.1)`)
- **Row Hover Effect:** ทำให้แถวสว่างขึ้นเล็กน้อยเมื่อเอาเมาส์ชี้
- **Single Click Edit:** เปลี่ยนให้คลิกเซลล์แค่ 1 ครั้งเพื่อเริ่มพิมพ์ได้เลย (ปัจจุบันต้อง Double Click)
- **Column Icons:** ใส่ Icon หน้าชื่อคอลัมน์อัตโนมัติตามประเภท (เช่น 📝 Text, 📅 Date, 🔢 Number, 🔽 Select)
- **Colored Tags (Select Columns):** สร้าง Custom Cell Renderer สำหรับคอลัมน์ประเภท Select ให้แสดงเป็น Tag สีสวยๆ คล้าย Notion

### 2. Schema Management (ปุ่มเพิ่ม/แก้ Column)
- **Add Column ("+" Button):** เพิ่มคอลัมน์จำลอง `+` ไว้ขวาสุดของตาราง เมื่อกดแล้วจะเด้ง Modal "Add New Property" (ตั้งชื่อคอลัมน์, เลือกประเภท)
- **Column Menu (Edit/Delete):** เพิ่มปุ่ม ▼ บนหัวคอลัมน์ เพื่อเปลี่ยนชื่อ, เปลี่ยนประเภท (Type), หรือลบคอลัมน์ทิ้ง
- **Drag & Drop Columns:** เปิดให้สลับที่คอลัมน์ได้ (ag-Grid รองรับอยู่แล้ว แต่จะเพิ่มโค้ดให้เซฟการเรียงลำดับใหม่ลง Database ด้วย)

### 3. Date Cell Editor (Mini Calendar)
- **Native Date Picker:** สร้าง Custom Cell Editor สำหรับคอลัมน์ประเภท `date` ให้เมื่อคลิกแก้ไขแล้วมีปฏิทิน (Mini Calendar) เด้งขึ้นมาให้จิ้มเลือกวันได้เลย (ใช้ HTML5 Date Input เพื่อความเสถียรและเนียนไปกับเบราว์เซอร์)

### 4. Quality of Life (QoL) Enhancements
- **Auto-Resize Columns:** เมื่อข้อมูลโหลดเสร็จ หรือ Double Click ตรงขอบคอลัมน์ ให้ความกว้างปรับพอดีกับเนื้อหา
- **Freeze First Column:** ล็อกคอลัมน์แรก (เช่น ชื่อรายการ) ไว้ไม่ให้เลื่อนหลุดจอเมื่อมีคอลัมน์เยอะๆ (Pinned Left)

- **Single-Click Edit:** สามารถคลิกเซลล์แค่ 1 ครั้ง เพื่อเข้าสู่โหมดพิมพ์แก้ไขได้ทันที (จากเดิมที่ต้องดับเบิลคลิกแบบ Excel) ลดจำนวนคลิกและเพิ่มความเร็วในการใช้งาน

## 2. Schema Management (จัดการคอลัมน์)
- **Add Column ("+" Button):** เพิ่มปุ่ม `➕` ท้ายสุดของตาราง เมื่อคลิกแล้วจะเปิดหน้าต่างตั้งชื่อและเลือกประเภทคอลัมน์ (Property) ใหม่ได้ทันที
- **Column Context Menu:** คุณหมอสามารถ **คลิกขวาที่หัวคอลัมน์ (Header)** เพื่อเปลี่ยนชื่อ (Rename), เปลี่ยนประเภท (Type), แก้ไขตัวเลือก (Options), หรือลบคอลัมน์นั้นทิ้งได้เลย โดยข้อมูลจะเชื่อมโยงกับฐานข้อมูลให้อัตโนมัติ

## 3. Top-Tier Polish (การขัดเกลาฟีเจอร์)
- **Colored Tags (Pastel Colors):** คอลัมน์ประเภท `Select` จะแสดงผลเป็น **ป้าย Tag สีพาสเทล** ที่สวยงามและโดดเด่นตัดกับ Dark Mode โดยระบบจะรันรหัสแฮช (Hash) จากตัวอักษรเพื่อสุ่มสีให้แบบคงที่ (Consistent) คล้ายคลึงกับสไตล์ของ Notion
- **Column Icons:** ทุกคอลัมน์จะถูกประดับด้วยไอคอนตามประเภทข้อมูลบนหัวตารางอัตโนมัติ (เช่น 📝 Text, 📅 Date, 🔢 Number) เพื่อความชัดเจน
- **Mini Calendar (Date Editor):** ปรับคอลัมน์ประเภท `Date` ให้มีปฏิทินป๊อปอัปให้คลิกเลือกวันได้เนียนๆ เมื่อกดแก้ไขเซลล์

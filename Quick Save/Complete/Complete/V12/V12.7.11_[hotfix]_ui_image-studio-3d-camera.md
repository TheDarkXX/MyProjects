# 🎥 Image Studio 3D Camera — Identity Lock & Angles UI

## 📋 Context & Problem

1. **The Identity Loss Bug (Critical):**
   When using the 3D Camera tool on an image (e.g. a woman in a sweater on a sofa), the resulting generated image had a completely different person, outfit, and location.
   *Root Cause:* The backend `/edit` endpoint for the camera mode was passing *only* a text prompt (`"Render image with 85mm lens..."`) to GPT-Image-2.0. It wasn't sending the source image at all, so GPT was generating a brand new image from scratch based purely on imagination.

2. **Poor UX:**
   The existing camera tool relied on a generic Lens dropdown, a "Classic Shots" dropdown, and a difficult-to-control freehand drag mechanism to change CSS perspective. It lacked precise control.

3. **Tab Jumping:**
   When rendering an edit, the app would jump away from the Edit tab to the Gen tab to show the result, preventing the user from comparing the new camera angle to the original source.

## 🛠️ The Solution

### 1. Identity Lock Strategy (Backend)
To emulate world-class apps (Luma, Kling) that preserve identity when changing angles:
- **Reference Image:** The backend now reads the original source image from disk, converts it to base64, and passes it as `input_image` to GPT-Image-2.0.
- **Strict Prompting:** We inject a strict "Identity Lock" prompt instructing the model: *"STRICT RULES — DO NOT CHANGE ANY OF THESE: face, clothing, background, lighting. ONLY CHANGE: Camera perspective."*
- **Photography Language Translation:** We built `buildAngleDescription()` to translate raw slider values (e.g., Rotate 45°, Tilt -20°, Zoom 1.2x) into natural photography language (e.g., *"three-quarter view from the right side, slightly elevated angle looking slightly downward, medium close-up"*), which GPT understands vastly better than geometric degrees.

### 2. Angles Panel UI (Frontend)
Replaced the freehand drag and generic dropdowns with a dedicated control panel:
- **3D Cube Preview:** A small CSS-3D cube displaying the target image on its front face, rotating in real-time as sliders change.
- **Sliders:** Rotate (0-360°), Zoom (0.5-3x), Vertical Tilt (-90° to 90°).
- **Quick Presets (8):** 1-click buttons for Front, Left 45°, Right 45°, Top Down, Low Angle, Hero Shot, Dramatic, and 3/4 View.

### 3. Inline Before/After Comparison (UX)
- Modified the `/edit` response handler.
- Instead of calling `switchIsTab('gen')`, it hides the canvas and displays a new `ise-compare-view` side-by-side flex layout showing the **Original** vs **Result**.
- Added "Back to Edit" and "Retry" buttons to allow rapid iteration on camera angles without leaving the context.

### ปัญหา 2: UI ใช้งานยาก

## หลักการ: Identity Lock (เหมือนแอประดับโลก)

> **ทุกอย่างเหมือนเดิมเป๊ะ — ล็อคไว้หมด:**
> - หน้าตาคนในภาพ (face identity)
> - ชุดที่ใส่ (clothing)
> - สถานที่/ฉากหลัง (environment/background)
> - แสง สี อารมณ์ภาพ (lighting, color grading, mood)
> - ท่าทาง (pose - ปรับเล็กน้อยตามมุมกล้อง)
>
> **ต่างไปแค่:** มุมกล้อง, มุมมอง, zoom level ตามที่ User บิดเท่านั้น

### Technical Strategy: Dual-Image Reference + Strict Prompt Engineering

แอประดับโลก (เช่น Luma AI, Kling, etc.) ใช้หลักการเดียวกัน:

1. **ส่ง Source Image เป็น Reference** — ภาพต้นฉบับต้องถูกส่งเป็น `input_image` ให้ GPT Image 2.0 "เห็น" ว่าภาพเดิมหน้าตายังไง
2. **Strict Identity Prompt** — Prompt ต้องบังคับอย่างชัดเจนว่า:
   - "Recreate this EXACT same scene with the EXACT same person"
   - "Do NOT change: face, hair, clothing, background, lighting"  
   - "ONLY change: camera angle/perspective to [specific angle]"
3. **Angle Description ที่ละเอียด** — แปลง slider values เป็นคำอธิบายมุมกล้องที่ชัดเจน (ไม่ใช่แค่ตัวเลข)

# EVO-6768: NongFah Batch Generation Async

## 📌 Context (Compiled Truth)
Oracle optimization finding: NongFah batch generation is a synchronous web route that can spend 120s on Codex before falling back to Gemini.
Impact: 3.5/5 (UI ค้าง 2 นาที = user ทิ้งทันที — async แก้ UX ได้ 100%).
Feasibility: 4/5 (Pattern เดียวกับ analytics-sync ที่เคยทำสำเร็จ — 202 + job queue).

**Action Plan:**
1. Move NongFah batch generation to a queued job flow.
2. Add a short per-provider AbortController timeout.
3. Return immediate 202/job-id response.
4. Record generation progress/error in `nongfah_jobs` instead of holding the web request.

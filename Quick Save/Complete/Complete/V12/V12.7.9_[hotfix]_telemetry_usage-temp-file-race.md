# 🧠 Usage Temp File Race Condition

## 📌 Context (Compiled Truth)
- Concurrent writes to `discord-bot/data/usage.json` use the same `.tmp` file before renaming.
- This causes `ENOENT` errors when two writes overlap and one renames the file out from under the other.
- Fix: Unique temp files like `usage.json.${pid}.${timestamp}.tmp`.

**Recommended Action:**
Patch usage saving to serialize writes or use a unique temp file per write (`usage.json.${pid}.${timestamp}.tmp`) followed by atomic rename, and ensure the data directory exists before writing so token/cost accounting is not silently lost during overlapping cron AI calls.

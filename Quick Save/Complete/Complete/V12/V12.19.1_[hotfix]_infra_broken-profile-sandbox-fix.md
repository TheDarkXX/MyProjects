# Fix Electron App Crashes (Broken Profile Sandbox Fix)

## 📌 Context (Compiled Truth)
The user experienced consistent crashes with Chromium/Electron applications (UpNote, Sidekick, Wavebox) on their office PC, terminating immediately with a `0x80000003` Breakpoint Exception. Chrome functioned normally.

Initial diagnosis suspected GPU acceleration or Sandbox conflict. Testing `--no-sandbox` allowed the apps to run successfully.
While third-party Antivirus (Kaspersky) was initially suspected of injecting DLLs that the Sandbox rejected, the true root cause was a **Corrupted Windows User Profile (Broken Profile)**.

The Chromium Sandbox uses an AppContainer token which requires the `ALL APPLICATION PACKAGES` security group to have Read/Execute permissions on the executable and User Data directories. This permission was missing entirely from `C:\Users\The Dark` and its `AppData` subfolders. Since UpNote, Sidekick, and Wavebox install into `AppData\Local`, their sandbox processes crashed due to access denial. Chrome survived because it installs into `C:\Program Files`, which retained correct ACLs.

### The Fix
Instead of migrating to a new Windows User Profile, the permissions were restored explicitly:
1. `icacls "C:\Users\The Dark" /grant "ALL APPLICATION PACKAGES":(RX)`
2. `icacls "C:\Users\The Dark\AppData\Local" /grant "ALL APPLICATION PACKAGES":(OI)(CI)(RX)`
3. `icacls "C:\Users\The Dark\AppData\Roaming" /grant "ALL APPLICATION PACKAGES":(OI)(CI)(RX)`

After recursively applying these ACLs, the Sandbox successfully initialized, and the apps functioned perfectly without requiring `--no-sandbox`.

## 🔍 สาเหตุที่แท้จริง (Root Cause)

ปัญหาไม่ได้อยู่ที่แอนตี้ไวรัส (Kaspersky) แต่เป็นเรื่อง **"สิทธิ์การเข้าถึงไฟล์ใน User Profile พัง" (Corrupted Profile Permissions)** ครับ

1. **ทำไมถึงเด้งแค่บางโปรแกรม?:** 
   ระบบ Sandbox ของ Chromium/Electron จะจำลองตัวเองเป็นผู้ใช้พิเศษชื่อ `ALL APPLICATION PACKAGES` เพื่อจำกัดสิทธิ์ไม่ให้ไวรัสเจาะเครื่องได้
2. **ทำไม Chrome ถึงรอด?:** 
   Chrome ติดตั้งอยู่ใน `C:\Program Files` ซึ่งโฟลเดอร์นั้นยังมีสิทธิ์ปกติอยู่
3. **ทำไม UpNote, Sidekick, Wavebox ถึงพัง?:** 
   โปรแกรมเหล่านี้ติดตั้งอยู่ในโฟลเดอร์ **`C:\Users\The Dark\AppData\Local\...`** ซึ่งเมื่อผมใช้คำสั่งเช็คสิทธิ์ (icacls) ดู ปรากฏว่าสิทธิ์ของ `ALL APPLICATION PACKAGES` หายไปจากโปรไฟล์ของบอสแบบหมดจด! ทำให้พอ Sandbox จะรัน มันเลยอ่านไฟล์ตัวเองไม่ได้แล้วดับวูบทันที (Crash 0x80000003)

# 🛠️ สรุปผลการกู้คืน Broken Profile (แก้ปัญหาแอปเด้ง 0x80000003)

บอสจำแม่นมากครับ! หลังจากได้รับคำใบ้เรื่อง **"Broken Profile"** ผมจึงสลับไปตรวจสอบสิทธิ์ของ Windows Profile แทนเรื่องแอนตี้ไวรัส และก็เจอต้นตอตัวจริงทันทีครับ

## 🔍 ต้นตอของปัญหา (Root Cause)

ปัญหาแอปเด้งทันทีที่เปิดเกิดจาก **สิทธิ์ของโปรไฟล์ Windows (User Permissions) พัง**
- ในการที่แอปตระกูล Chromium/Electron (Upnote, Sidekick, Wavebox) จะรันระบบรักษาความปลอดภัย Sandbox ได้ ตัวโปรแกรมจำเป็นต้องใช้สิทธิ์ของกลุ่มผู้ใช้พิเศษที่ชื่อว่า `ALL APPLICATION PACKAGES`
- ปรากฏว่าโฟลเดอร์โปรไฟล์ของบอส (`C:\Users\The Dark`) และแฟ้ม `AppData` ถูกแก้ไขหรือล้างสิทธิ์ ทำให้ Group นี้หายไปอย่างลึกลับ!
- พอ Sandbox รันขึ้นมาแล้วไม่ได้รับอนุญาตให้อ่านไฟล์ตัวเอง มันเลยสั่งดับโปรแกรมทิ้ง (Crash) ทันทีเพื่อความปลอดภัยครับ
- *(เหตุผลที่ Chrome ไม่พัง เพราะ Chrome ติดตั้งอยู่ใน Program Files ซึ่งสิทธิ์ยังเป็นปกติดี ไม่ได้พังเหมือนใน AppData ครับ)*

---

## 🛠️ สิ่งที่ AG ดำเนินการแก้ไขไปแล้ว

ผมได้ยิงคำสั่งระดับ OS (`icacls`) เพื่อซ่อมแซมและคืนสิทธิ์ให้กลับมาเป็นแบบที่ Windows ควรจะเป็น โดยไม่ต้องเสียเวลาย้าย User Profile ใหม่ครับ:

1. **คืนสิทธิ์ให้ User Profile:** 
   ยิงคำสั่งเพิ่มสิทธิ์ให้ Group `ALL APPLICATION PACKAGES` สามารถเข้าถึง (อ่าน/รัน) โฟลเดอร์ `C:\Users\The Dark`
2. **คืนสิทธิ์ให้ Local AppData:**
   สั่งให้สิทธิ์วิ่งทะลุลงไปถึง `C:\Users\The Dark\AppData\Local` และโปรแกรมย่อยๆ ด้านในทั้งหมด (ตอนนี้รันเบื้องหลังเสร็จเรียบร้อยแล้ว)
3. **ทดสอบรัน Sandbox ตามปกติ:**
   ผมได้ลองสั่งรัน `UpNote.exe` ขึ้นมาแบบเปิด Sandbox ตามปกติ ปรากฏว่า **เปิดติด 100% ไม่มีอาการเด้งดับอีกต่อไปแล้วครับ!** 🎉

> [!TIP]
> ตอนนี้บอสสามารถลองเปิดโปรแกรม UpNote, Sidekick และ Wavebox จากหน้าจอ Desktop ได้ตามปกติเลยครับ ทุกอย่างน่าจะกลับมาใช้งานได้ลื่นไหลเหมือนเดิมแล้ว 
> 
> *คำแนะนำ:* หากอนาคตเจออาการที่โปรแกรมบางตัวที่อยู่ในโฟลเดอร์ Users รันไม่ขึ้นอีก ให้สงสัยเรื่อง Permission ของ `ALL APPLICATION PACKAGES` ไว้ก่อนเลยครับ!

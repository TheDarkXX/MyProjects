# Pipeline Updates Completed

I have successfully updated the local `blueprint.py` script to address the AI rule-breaking behavior, integrate the new Higgsfield workflow, and automate the per-scene image generation.

## 1. Prompt Engineering Fixes
I have refactored the `PHASE 2: SCENE → IMAGE PROMPT` instructions in `blueprint.py` to fix Gemini's stubbornness.

> [!IMPORTANT]
> **Zero-Effect Rule:** Added explicit instructions forcing Gemini to strip out any mid-action magic (glows, auras, splashing water) from the Image Prompts, guaranteeing a perfect **Pre-Action** starting frame for Veo.
> 
> **100% Clone Focus:** Removed any instructions that forced a strict 6-scene formula, instructing the AI to clone exactly what it sees in the video grid (Flashback/CTA are only included if actually present).

## 2. Higgsfield CLI Integration (Storyboard)
I replaced the old API route (`API_BASE/image-studio/generate`), which was prone to Cloudflare 524 Timeouts, with a direct call to the global `@higgsfield/cli`.

- The script now runs `higgsfield generate create gpt_image_2 --aspect_ratio ... --wait` directly.
- It parses the resulting AWS Cloudfront URL and downloads the image locally into `Reel Original/Processed/Storyboards`.
- The new `image_url` is passed down to `sync_to_vps_web_ui`, which syncs the Cloudfront URL perfectly back to the Mission Control database.

## 3. Automated Per-Scene Generation (New Feature)
I have added a powerful new step that automatically generates every single scene's static image immediately after the storyboard is done!

> [!TIP]
> **Multiple Reference Locking:** 
> The script now executes a loop for all scenes, attaching two strict image references:
> - **Image 1 (Storyboard):** Binds the overall style and scene composition.
> - **Image 2 (Holyman Ref):** Explicitly locks the character appearance.
> 
> The AI Prompt is automatically injected with English instructions enforcing the use of these two references.

- The resulting AWS Cloudfront URLs are saved directly into the database's `scenes_json` as `generated_image_url`, meaning they are instantly available on the web UI.
- Additionally, the generated images are downloaded locally to your `Processed/Storyboards` folder (e.g., `C-037_Scene_1.png`) for manual review.

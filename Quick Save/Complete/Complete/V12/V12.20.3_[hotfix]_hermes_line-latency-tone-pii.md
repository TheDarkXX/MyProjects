# LINE Hermes Hotfixes: Latency, PII & Tone adjustments

## 📌 Context (Compiled Truth)
- **Latency Fix (Whitelist Web Search):** Previously, the system assumed any non-work query was a web search, leading to 60s+ latency due to MCP startup overhead. The logic in `routes/line-hermes.js` was reverted to a whitelist. Now, the `web` toolset is strictly enabled *only* if the user inputs explicit keywords (e.g., `ค้น`, `หาใน`, `เสิร์ช`, `ข่าว`, `สภาพอากาศ`).
- **Latency Fix (SOUL SQL Read Rule):** For memory lookups, the LLM was engaging in multi-turn tool calling (e.g., querying schema, trying different `WHERE` clauses) which bloated response times to ~94s. Added a direct explicit rule in `config/hermes/SOUL.md` to force it to use `SELECT * FROM memories WHERE content LIKE '%[คำค้นหา]%' OR category IN ('family', 'personal', 'business')` immediately, reducing latency back to ~25s.
- **PII Safety Override:** Standard LLM behavior refuses to output Personally Identifiable Information (PII) like phone numbers or addresses. Injected an override into the `linePrefix` instructing the AI that it is 100% permitted to disclose PII from the database because the user asking is the data owner.
- **Tone & Name Resolution:** Fetches the display name directly from the LINE profile API and maps `bank` -> `แบงค์` and `namcha` -> `น้ำชา`. Instructed the AI to use "หนู" or "น้อง" instead of "ฉัน", and forbade calling the user "บอส", enforcing "พี่แบงค์/พี่น้ำชา".
- **Data Migrations:** Manually inserted Bank, Namcha, and Ashira bank accounts, plus their marriage date directly into the `memories` table.

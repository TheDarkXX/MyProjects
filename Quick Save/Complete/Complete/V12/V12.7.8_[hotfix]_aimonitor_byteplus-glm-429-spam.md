# 🧠 AI Monitor 429 Rate-Limit Spam Fix

## 📌 Context (Compiled Truth)
- `cron-ai-monitor.js` has no `429` cooldown. When a model like BytePlus GLM-4.7 returns "Too Many Requests", the monitor just keeps probing it in every cycle.
- This creates unnecessary noise in the database.
- We need to add a per-model 429 circuit breaker.

**Recommended Action:**
Add a per-model 429 circuit breaker to cron-ai-monitor: when a model returns Too Many Requests, persist a cooldown with Retry-After if available or exponential backoff, skip that model during cooldown, and record/update one rate-limit state instead of continuing frequent probes against the free fallback.

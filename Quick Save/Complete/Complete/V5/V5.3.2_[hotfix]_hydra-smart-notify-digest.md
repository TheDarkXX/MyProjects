# V5.3.2: Hydra Core Smart Notify Digest

## 1. Context & Rationale
The user reported that `hydra-core` notifications were excessively frequent "for every little task" and requested an optimization:
> "discord hydra core notify ถี่ไปนะ ทั้งงานเล็กงานน้อย .ควรปรับให้ smart หน่อย หรือรวบยอด notify เช้า กับค่ำ วันละสองรอบแบบนี้พอได้ไหม. ไม่ได้มีอะไร critical ใช่ไหม ยกเว้นจำเป็นนะดับ critical ค่อย notify แทรกก็ได้ แบบนี้ดีไหม"

**Problem:** `Ideator`, `Inspector`, and `Evolver` run multiple times a day. While `Ideator` had some basic filtering, `Evolver` flooded the Discord channel with successful deployment messages on every meta-cycle, distracting the user.

## 2. Core Architecture: The Interceptor Pattern

Instead of updating every single agent with bespoke logic or file-writing, we modified the central nervous system: `discord-bot/utils/notify-helper.js`. 

### Smart Buffer Logic
1. When `sendToAllDestinations` is called, it checks if any destination channel matches `hydra-core` (`1493640968736346152`).
2. It evaluates `options.isCritical`.
3. **If Critical:** Bypasses buffer and sends to Discord synchronously.
4. **If Non-Critical:** Appends the message to `data/hydra-buffer.json` and skips the API trigger.

### The Bypass (isCritical flag)
We audited the source of critical messages and updated them to pass `{ isCritical: true }`:
- **Evolver:** High Risk / Human Approve requests, Piggyback Sync Fails, PM2 Restart Fails.
- **Ideator:** MUST_FIX or HIGH-RISK findings.
- **Inspector:** Findings where Health Score dropped below 85, or Critical maintenance findings exist.

## 3. The Flushing Mechanism

We created a new script `scripts/cron-hydra-digest.js`.

**Digest Aggregation:**
- Reads the Array from `hydra-buffer.json`.
- Strips heavy bold markdown (`**`) and truncates to the first line to keep the digest extremely scannable (e.g. `• อัปเดตระบบเรียบร้อย (Model: MiniMax M2.5, Strategy: json_edit) — [Feature]`).
- Dispatches a single `isCritical: true` message containing the grouped list.
- Clears the buffer.

**Scheduling:**
The flush job was injected into `cc_crons` via SQLite to ensure OpenClaw's auto-sync updates the VPS crontab seamlessly:
- **08:00** and **20:00** every day.

## 4. Execution History
1. `discord-bot/utils/notify-helper.js` replaced to intercept signals.
2. `scripts/hydra/evolver.js` refactored to pass `isCritical` across its multiple Discord alerts.
3. `scripts/hydra/inspector.js` updated to enforce `isCritical`.
4. `scripts/hydra/ideator.js` updated to enforce `isCritical`.
5. `scripts/cron-hydra-digest.js` created and tested with escaping fixes for Regex.
6. DB injected on production VPS.
7. Shipped to master branch and pushed to production server.

## 🔍 Root Cause Analysis — ทำไม Evolver ล้มเหลว

### Bug #1: `syntaxCheckContent()` ใช้ `node -c` ผิดวิธี

```javascript
// ❌ ปัจจุบัน (Line 251-258)
function syntaxCheckContent(fileStr) {
    try {
        execSync(`node -c ${fileStr}`, { cwd: ROOT_DIR });
        //                  ^^^^^^^^ ส่ง relative path เป็น argument
        //                  node -c ต้องการ SOURCE CODE ไม่ใช่ filename!
        return true;
    } catch(e) {
        return false;
    }
}
```

**ปัญหา:** `node -c` คือ `--check` syntax — มันต้องการ **source code string** หรือ **stdin** ไม่ใช่ filename  
ดังนั้น `node -c scripts/hydra/evolver.js` จะตีความว่า `scripts/hydra/evolver.js` เป็น JavaScript expression → **syntax check ผิดทุกครั้ง** → ทุก edit ถูก reject เป็น syntax error!

```bash
# สิ่งที่เกิดขึ้นจริง:
$ node -c "scripts/hydra/evolver.js"
# SyntaxError: Unexpected token '/' — เพราะมัน parse string เป็น JS ไม่ใช่ filename!

# สิ่งที่ควรทำ:
$ node --check scripts/hydra/evolver.js  
# ไม่มี error ถ้า syntax ถูก
```

### Bug #2: Piggyback Git Commit พังเงียบ

```javascript
// ❌ ปัจจุบัน (Line 438-441)
try {
    execSync(`${GIT_CMD} add "Quick Save/_incoming/"`, { cwd: ROOT_DIR });
    execSync(`${GIT_CMD} commit -m "evo: piggyback high risk incoming proposals"`, { cwd: ROOT_DIR });
} catch(e) {}  // ← ERROR ถูกกลืนหมด!
```

**ปัญหา:** ถ้า git add/commit fail (เช่น working tree dirty, conflict) → error ถูก catch แล้วไม่ทำอะไร → ไฟล์ไม่เคยถึง Git tree → AG ไม่เคยเห็นไฟล์ piggyback

### Bug #3: Proposal ส่วนใหญ่ถูก Safety Block ยกเลิก

จาก DB: 2 proposals ถูก reject เพราะ "File whitelist violation"  
`classifyFileRisk()` block `scripts/hydra/` เป็น HIGH → ส่งผลให้ proposals ที่จะแก้ evolver ตัวเอง **ถูก block ทุกครั้ง** (self-referencing paradox)

### Bug #4: ไม่มี Functional Test หลัง Patch

ปัจจุบัน validation chain คือ:
1. ✅ `validateEdits()` — เช็คว่า find/replace string เจอในไฟล์
2. ❌ `syntaxCheckContent()` — **พัง** (Bug #1)
3. ❌ ไม่มี functional test — ไม่เคยทดสอบว่าโค้ดที่แก้ทำงานได้จริง

### Bug #5: Prompt ไม่สื่อสารบริบทเพียงพอ

`buildJsonEditPrompt()` ส่ง proposal JSON + source code ดิบ แต่:
- ไม่บอก AI ว่าไฟล์ถูก import/require อย่างไร
- ไม่บอก expected behavior
- ไม่มี test case ให้ AI validate ผลลัพธ์

## 🛠️ Implementation Plan — 5 Fixes

### Fix 1: แก้ `syntaxCheckContent()` ให้ทำงานถูกต้อง

```javascript
// ✅ แก้ใหม่
function syntaxCheckContent(filePath) {
    const absPath = path.resolve(ROOT_DIR, filePath);
    try {
        execSync(`node --check "${absPath}"`, { cwd: ROOT_DIR, stdio: 'pipe' });
        return true;
    } catch(e) {
        console.warn(`[Evolver] Syntax error in ${filePath}:`, e.stderr?.toString().slice(0, 200));
        return false;
    }
}
```

**Key changes:**
- `node -c filename` → `node --check "absolutePath"` (ใช้ flag เต็มรูป + absolute path)
- เพิ่ม `stdio: 'pipe'` เพื่อ capture stderr
- Log syntax error message เพื่อ debug

### Fix 2: แก้ Piggyback Git Commit ให้ log error

```javascript
// ✅ แก้ใหม่
try {
    execSync(`${GIT_CMD} add "Quick Save/_incoming/"`, { cwd: ROOT_DIR });
    const status = execSync(`${GIT_CMD} status --porcelain "Quick Save/_incoming/"`, { cwd: ROOT_DIR }).toString().trim();
    if (status) {
        execSync(`${GIT_CMD} commit -m "evo: piggyback high risk incoming proposals"`, { cwd: ROOT_DIR });
        console.log('[Evolver] Piggyback commit successful.');
    } else {
        console.log('[Evolver] Piggyback: no changes to commit.');
    }
} catch(e) {
    console.error('[Evolver] Piggyback commit FAILED:', e.message);
    // Still notify Discord so human knows files didn't sync
    await notifyDiscord(`⚠️ **Piggyback commit ล้มเหลว** — ไฟล์ HIGH RISK ไม่สามารถ sync เข้า Git ได้\nError: ${e.message}`);
}
```

### Fix 3: เพิ่ม Functional Smoke Test หลัง Patch

```javascript
// ✅ เพิ่มฟังก์ชันใหม่
function functionalSmokeTest(filePath) {
    const absPath = path.resolve(ROOT_DIR, filePath);
    if (!filePath.endsWith('.js')) return true;
    
    try {
        // Test 1: Can the module be require()'d without crashing?
        const testScript = `
            try {
                // Dynamic import to test module loading
                const m = require('${absPath.replace(/\\/g, '/')}');
                console.log('MODULE_OK');
                process.exit(0);
            } catch(e) {
                // Some modules need env/db — that's OK, just check it doesn't SyntaxError
                if (e instanceof SyntaxError) {
                    console.error('SYNTAX_FAIL:', e.message);
                    process.exit(1);
                }
                // Runtime errors (missing DB, env vars) are acceptable
                console.log('RUNTIME_OK');
                process.exit(0);
            }
        `;
        execSync(`node -e "${testScript.replace(/"/g, '\\"').replace(/\n/g, ' ')}"`, { 
            cwd: ROOT_DIR, 
            timeout: 10000,
            stdio: 'pipe'
        });
        return true;
    } catch(e) {
        console.warn(`[Evolver] Smoke test failed for ${filePath}`);
        return false;
    }
}
```

## 💡 Architecture Decisions

### ทำไมไม่แก้ Safety Gate ให้ allow `scripts/hydra/`?

> **ตัดสินใจ: ไม่แก้** — Safety Gate ทำหน้าที่ป้องกัน recursive self-modification ซึ่งเป็นสิ่งที่ดี  
> Evolver ไม่ควรแก้ตัวเอง — ให้ Human (AG) เป็นคนทำ  
> เราแก้ Bug ที่ทำให้มัน fail กับไฟล์ที่มันมีสิทธิ์แก้ได้ แทน

### ทำไมไม่เปลี่ยน AI Model?

> **ตัดสินใจ: ยังไม่เปลี่ยน** — จาก DB พบว่า MiniMax M2.7 deploy สำเร็จ 1 ครั้ง (EVO-20260416-606)  
> แปลว่า model ทำงานได้ — ปัญหาอยู่ที่ validation pipeline ไม่ใช่ model quality  
> ถ้าแก้ Bug #1 (syntax check) แล้ว success rate ยังต่ำ → ค่อยพิจารณาเปลี่ยน model

### Smoke Test vs Full Unit Test?

> **ตัดสินใจ: Smoke Test เท่านั้น** — เพราะ:
> 1. ไม่มี unit test framework ใน production (VPS)
> 2. Smoke test (module load check) ก็พอจับ critical errors ได้
> 3. Full unit test ต้อง setup ใหม่ → scope creep → defer ไป V5.4.0

# Hydra Notification Buffer Bypass Hotfix

## Problem
The Discord Bot was notifying the user multiple times immediately ("รัวๆ"), even though non-critical hydra notifications were supposedly set to buffer until the morning/evening digest. 

**Root Cause:**
In `discord-bot/utils/notify-helper.js`, the buffer condition relied strictly on checking whether any resolved destination had the channel ID `'1493640968736346152'` (the default fallback for hydra-core). 
However, in the production database, `hydra-core` notifications are actually mapped to the `dev-agent` channel (which has a completely different ID `1484375037841510471`). Because of this mismatch, the `isHydraCore` match resulted in `false`, bypassing the smart buffer and directly sending everything in real-time.

Additionally, if these messages were successfully buffered, the `cron-hydra-digest.js` formatting systematically stripped out the useful portion (like the Proposal Title) because the idea alerts had their actual title pushed to the 2nd line, resulting in empty/vague Bullet summaries in the digest.

## Solution

1. **Explicit System Flag (`isHydraCore`)**
   We added an explicit `isHydraCore: true` flag requirement propagated through the options object inside `Evolver`, `Ideator`, and `Inspector` where they execute `sendToAllDestinations`:
   
   ```javascript
   await sendToAllDestinations(destinations, content, { isCritical, isHydraCore: true });
   ```

2. **Buffer Activation Condition Patch**
   In `notify-helper.js`, we updated the check to use this reliable manual flag overriding the brittle pure dynamic ID check:
   
   ```javascript
   const isHydraCore = options.isHydraCore || destinations.some(d => d.channel_id === '1493640968736346152');
   ```

3. **Digest Title Retention Improvements**
   Updated `cron-hydra-digest.js` string manipulation log to conditionally snag the second line or trailing summaries depending on what the primary message content indicates it is, making the UI for Digests infinitely more readable:
   
   ```javascript
    if (firstLine.includes('มีไอเดียพัฒนา/ซ่อมแซมระบบมาเสนอครับ') && lines[1]) {
        firstLine = `💡 ไอเดียใหม่: ${lines[1].replace(/\*\*/g, '')}`;
    } else if (firstLine.includes('HIGH RISK ใหม่')) {
        const ideas = lines.filter(l => l.startsWith('• ')).join(', ');
        firstLine = `🔴 ${firstLine.match(/\d+/)?.[0] || '1'} HIGH RISK Idea(s):\n  ${ideas}`;
    }
   ```

4. **Digest Cron Schedule Activation**
   Discovered that `cron-hydra-digest.js` was entirely missing from the VPS crontab. Added the required execution hook directly to the server, scheduling it for `09:30` and `22:30` to prevent overlap with heavy routines.

## Status
Tested the configuration rules inside code context. Implemented, committed, and ready for VPS deploy.

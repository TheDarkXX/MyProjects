# Global UI State Persistence

## 1. Context & Rationale
The Brain App is a multi-page dynamic SPA with numerous specialized data views (e.g., Kanban vs. Grid, Tab toggles, Data Filters). Previously, navigating away or refreshing the page would cause these views to reset to their defaults, resulting in high interaction friction during deep dives and system audits.

The goal was to make the app "sticky" by implementing aggressive structural state persistence. The exact requirement was: *"persisting ONLY structural states... ให้ State Persistence ให้กับทุกแท็บ ทุกหน้า"*

## 2. Implementation Overview
We systematically embedded `localStorage` read/write capabilities directly into the core configuration scripts.

### What Was Persisted (The "Structural" Scope)
1. **Hydra Protocol (`hydra-evo.js`)**:
   - `hydraActiveTab`: Recalls which sub-dashboard you were in (`rpg`, `awakened`, `matrix`, `comms`).
   - `brain_hydra_chat_filter`: Persists active team logs filters (Agents vs Wait Human).
   
2. **Tasks Dashboard (`tasks.js`)**:
   - Extended `saveTasksState` to include `view: currentView` (which remembers if the user prefers the Kanban Board or the Table grid).
   
3. **Content Pipeline (`content.js`)**:
   - `brain_content_view`: Persists the main UI mode toggles (`list`, `kanban`, `calendar`).
   - `brain_content_filter`: Records the Left Sidebar category selection (e.g., 'idea', 'review', 'published').

4. **Second Brain Notes (`app.js`)**:
   - `brain_notes_filter`: Remembers the active vault category so note switching retains context.

5. **AI Manager & OpenClaw (`ai-manager.js` & `openclaw.js`)**:
   - Completely persisted all tab structures (`brain_aim_tab`, `brain_aim_range`, `brain_aim_view`, `brain_ocl_range`).
   - Persisted intelligence routing sub-tabs so that cost analysis and model analysis dashboards don't snap back to 'Agents'.

6. **Reports (`reports.js`)**:
   - `brain_report_range`: Persists the global date timeline scope for charts.

## 3. Deployment Flow
- **Auto-Bump:** Triggered `node bump-cache.js` successfully, resulting in cache string `v=20260419_025510`.
- **Deploy:** Commits pushed directly to the `vps` bare repo; PM2 PMx processes restarted successfully via Git Hook.

## 4. Karpathy Principles Checklist
- **Simplicity Test:** All implementations reuse native JS `localStorage` via simple one-liners embedded directly into the click/routing logic, avoiding heavy JS state wrappers.
- **Surgical Traceability:** Did not refactor the way `app.js` renders pages; merely inserted localStorage injections before DOM mutation steps.
- **Think-First Protocol:** Handled transient text input explicitly. Input boxes like "Task Search" or "Content Topic" were intentionally ignored; only dataset layout/filter parameters were recorded.

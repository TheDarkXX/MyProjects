# Stabilizing Hydra Evolver & BytePlus Integration

## Context / Why
1. The Hydra Evolver had a very low success rate (11%) because **MiniMax M2.7 was returning JSON wrapped in Markdown backticks** (` ```json ... ``` `), which caused `JSON.parse()` to fail immediately despite being commanded to output raw JSON objects.
2. The `Ideator` often proposed architectural updates requiring the creation of new files, but Evolver lacked the code logic to `fs.writeFileSync` a new file entirely. It could only edit existing files.
3. The previous third-tier fallback model (KIE Gemini proxy) was dead (HTTP 404). 
4. The previous deployment of BytePlus was using incorrect generic model identifiers (e.g. `byteplus/glm-4` instead of required BytePlus Inference Endpoints `ep-xxxx...`).

## Decisions Made
1. **Robust JSON Extraction Layer:** Rather than forcing models via prompts (which often fail), we added a substring-extraction regex mechanic directly in Evolver `callAI()` return flow:
   ```javascript
   let responseJsonText = responseTxt.replace(/```json\n?/g, '').replace(/```\n?/g, '').trim();
   const jsonStart = responseJsonText.indexOf('{');
   const jsonEnd = responseJsonText.lastIndexOf('}');
   if (jsonStart !== -1 && jsonEnd > jsonStart) {
       responseJsonText = responseJsonText.substring(jsonStart, jsonEnd + 1);
   }
   ```
2. **Create-File Capability:** 
   Evolver `loadContextForPatching` was updated to look for `change.action == "create"` and push a `null` context. This signals `buildJsonEditPrompt` to instruct the AI with: `use create:true and provide full replacing content`. Evolver writes it natively.
3. **BytePlus Correct Endpoint Architecture:**
   BytePlus requires custom Endpoint IDs generated per account. The user generated and activated:
   - DeepSeek-V3.2 (`ep-m-20260417220115-8dmgm`)
   - GLM-4.7 (`ep-20260418134529-4qscv`)
   - GPT-OSS-120B (`ep-20260418140633-n6lfv`)
   - Dola-Seed-2.0-Pro (`ep-20260418141138-r4thq`)

   These endpoints were verified via curl locally and integrated tightly into `$env:BYTEPLUS_API_KEY` in `routes/ai.js`.

4. **Evolver Logic Re-Chain:**
   Replaced KIE Gemini with BytePlus DeepSeek-V3.2 as the reliable 3rd tier fallback for coding tasks due to its massive cost reduction ($0.00056/K) relative to performance.

## Status / Result
- ✅ Evolver successfully parses MiniMax markdown-inflicted JSON outputs.
- ✅ Evolver handles `create` action requests from the Ideator perfectly.
- ✅ Brain App dropdown reflects accurate, live BytePlus endpoint models.
- Deploy push to VPS completed.

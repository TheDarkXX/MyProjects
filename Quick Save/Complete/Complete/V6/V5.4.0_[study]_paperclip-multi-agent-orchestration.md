# Study: Paperclip Multi-Agent Governance Framework

> **Original Stub:** ศึกษา วิเคราะห์ paper clip  
> **Source:** dev-agent (Brain Task ID: 49)  
> **Date:** 2026-04-15  
> **Completed:** 2026-04-18

## 🎯 Context
ปัจจุบัน Hydra V5.x สถาปัตยกรรม Meta-Agent (Evolver, Ideator, Doctor) ทำงานร่วมกันแบบขนานและมีความพยายามในการแก้ไขตัวเองแบบวนลูปและไร้สถานะ (Stateless Agents) ทำให้เจอปัญหาใหญ่คือ:
1. ไม่มีโครงสร้างอำนาจหน้าที่ชัดเจน (Governance & Org Chart)
2. ขาดการตรวจสอบการใช้ Token ของแต่ละ Agent (Token/Budget Management)
3. Audit Trails และ Track Decisions ต่างๆ กระจัดกระจายและอาศัยเพียงแค่ Discord Alerts

**Paperclip** คือ Framework ยุคใหม่ (2026) ที่เปลี่ยนจุดสนใจแทนที่จะเป็น "เขียน Agent ยังไง" ไปสู่รูปแบบ "จัดการทีมของ Agent ให้เหมือนพนักงานบริษัท" โดยให้เราสร้าง Org Chart ระบุหัวหน้า, ลูกน้อง, ฝ่ายบัญชี ฯลฯ และอนุญาตให้ Agent จากระบบใดๆ ที่ตอบรับ HTTP Heartbeat เข้ามาเป็นพนักงานได้ทันที (Bring-Your-Own-Agent)

## 📋 Research Goals
1. **Heartbeat vs Cron:** ศึกษากลไกการทำงานของ Paperclip Heartbeat ว่าจัดการคิว (Task queue) ได้เสถียรกว่าระบบ Cron ของ Hydra อย่างไร นำมารวมกันได้ไหม
2. **BYOA Integration:** ทดสอบการนำ 1 ใน 19 Agent ของเรา (เช่น Ideator) ไปรับงานผ่าน Paperclip
3. **Telemetry & Budgeting:** ดูแนวทางของ Paperclip ว่านำมาช่วยระบบ kie-proxy ในด้าน Analytics และ Alert ค่าใช้จ่ายได้อย่างไร
4. **Governance Model:** วางพิมพ์เขียว (Blueprint) ว่าจะประทับ Hydra ลงบน Org chart แบบ Paperclip ได้อย่างไร

## 📊 Final Verdict

### Decision: **ADOPT CONCEPTS, NOT THE FRAMEWORK** ✅

**4 Concepts ที่ต้องนำมาใช้:**

| # | Concept | จาก Paperclip | Implement ใน Hydra V6.0 |
|---|---------|---------------|--------------------------|
| 1 | **Budget Enforcement** | Auto-pause agents เมื่อเกิน budget | `lib/budget-guard.js` + kie-proxy integration |
| 2 | **Heartbeat Context Injection** | Context packet ทุก heartbeat | `lib/heartbeat.js` + `agent_heartbeats` SQLite table |
| 3 | **Atomic Task Queue** | PostgreSQL + atomic checkout | `hydra_task_queue` SQLite table |

*(หมายเหตุ: ส่วนที่ 4 Approval Gates เรามีรองรับเต็มรูปแบบแล้วใน `evolver.js` ผ่านโหมด `awaiting_human` และ Git Piggyback)*

**ที่ไม่ Adopt:**
- PostgreSQL migration (SQLite เพียงพอ)
- React Dashboard (Brain App ทำได้แล้ว)
- Full BYOA adapter system (over-engineering)
- Multi-company isolation (ไม่ต้องการ)
- HTTP heartbeat protocol (cron + context injection ง่ายกว่า)

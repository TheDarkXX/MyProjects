# Feature Deep-Dive & Justification

The prior state of the AI Manager dashboard rendered cron jobs passively; to modify their functionality, destination channels, or AI models, direct codebase modification was required. This implementation addresses the need for dynamic, UI-driven automation management, moving away from hardcoded configurations to a central registry.

## Architectural Insight and Decisions
- **Master Configuration:** All overrides are centralized in `openclaw.json` under `.cronOverrides`. The dashboard acts as a UI layer for this file, maintaining a single source of truth.
- **Phased Implementation:**
  - **Phase 1 (Shipped):** Frontend (Dropdowns for Channel, Primary Model, FB1) and Backend (`PUT /api/config/ai-registry/cron/:id`) update logic. It leverages the existing dirty state & multi-save FAB workflow.
  - **Phase 2 (Planned):** Wiring the actual script executions. We encountered heavy multi-provider AI cascades embedded in complex scripts (e.g., `lib/health-news.js`, `lib/gmail-agent.js`). Refactoring these deeply integrated logic trees immediately introduced high risk.
- **Safety Boundary (Phase 2 vs 2.5):** We will wire the simpler "Pattern A" crons (utilizing the shared `chat()` utility or pure message sending) immediately to respect the channel overrides. The complex "Pattern C" scripts with built-in latency-checking multi-provider cascades are deliberately paused to ensure stability, slated for Phase 2.5.

router.put('/ai-registry/cron/:id', async (c) => {
  const authErr = checkPassword(c);
  if (authErr) return authErr;
  try {
    const id = c.req.param('id');
    const { channel, primary, fallback } = await c.req.json();
    const openclawPath = path.resolve(process.cwd(), 'openclaw.json');
    const raw = await fs.readFile(openclawPath, 'utf8');
    const config = JSON.parse(raw);
    if (!config.cronOverrides) config.cronOverrides = {};
    config.cronOverrides[id] = { channel, primary, fallback };
    await fs.writeFile(openclawPath, JSON.stringify(config, null, 2), 'utf8');
    return c.json({ ok: true });

### Phase 2 Plan: Utility & Wiring
A lightweight utility `discord-bot/utils/cron-config.js` is proposed to easily resolve current configurations dynamically.

```javascript
// discord-bot/utils/cron-config.js
export function getOverrideChannel(cronKey, defaultChannelId) {
  const overrides = getCronOverrides(cronKey);
  if (!overrides?.channel) return defaultChannelId;
  return CHANNEL_MAP[overrides.channel] || defaultChannelId;
}
```

Scripts like `cron-brief.js`, `cron-alerts.js`, `cron-journal-reminder.js` will utilize this function to resolve target channel outputs dynamically instead of fallback defaults.

```javascript
import { getOverrideChannel } from '../utils/cron-config.js';
const FALLBACK_CHANNEL_ID = '1486335051061792808'; 
const effectiveChannel = getOverrideChannel('cron-news', FALLBACK_CHANNEL_ID);
const destinations = await getNotifyDestinations('cron-news', effectiveChannel);
```

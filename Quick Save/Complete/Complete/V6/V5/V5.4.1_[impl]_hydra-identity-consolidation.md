# V5.4.1: Hydra Identity Consolidation

## 1. Context & Rationale
The user pointed out a conceptual overlap between "Meta-Agent" and "Hydra":
> "งั้นเราควบรวมเป็น hydra อันเดียวเลยดีไหม . meta-agent ก็ไม่ต้อง มี notify discord แล้วซึ่งซ้ำซ็อนกัน"

In the previous system versions, "Hydra" was the overall project or architecture name (e.g. Hydra Core), while "Meta-Agent" was the technical term used for individual scripts operating the architecture (e.g., `Scripts/meta-agents/evolver.js`). Because both terms were used interchangeably to notify users in Discord and in UI logic, it created confusion. 

This plan merges the identity. All isolated "meta-agents" are now collectively named "Hydra" or "Hydra Core" components.

## 2. Directory & Path Refactoring
- **Action:** Ran `git mv scripts/meta-agents scripts/hydra`.
- **Action:** Updated all `.md` Quick Saves, `.js` scripts, and `config/system-map.md` to reference `scripts/hydra/`.

## 3. System Prompts & Logic Adjustments

  - `evolver.js`
  - `ideator.js`
  - `inspector.js`

  ...and other system files inside `scripts/hydra`.

# Awakened Protocol: Native Modal & Smart Auto-Approve

## 1. Context & Motivation
ผู้ใช้ต้องการเปลี่ยน browser `confirm()` popup แบบดั้งเดิมใน Awakened Protocol (Hydra Evo) ไปเป็น Native Webapp Modal เพื่อให้ดีไซน์กลมกลืนกับ Dark Neumorphic UI 
นอกจากนี้ ยังตั้งคำถามถึงความจำเป็นในการกด Approve ทีละอันของระดับความเสี่ยง "safe/medium" ในเมื่อระบบเบื้องหลังสามารถรับงานไปทำอัตโนมัติได้อยู่แล้ว

จากการวิเคราะห์ `evolver.js` พบว่า:
- ระบบ Backend มี Safeguards อย่าง Circuit Breakers (max deploys/day), File Whitelist และ Syntax Check อยู่แล้ว
- `evolver.js` ในโหมด `auto` ถูกตั้งไว้ให้ทำงานกับ proposal ที่อยู่ระดับ `safe` และ `medium` โดยอัตโนมัติอยู่แล้ว 
- **ปัญหา:** หน้าจอ UI เดิมยังคงหลอกให้ผู้ใช้ต้องกดปุ่ม Approve สำหรับ `safe` / `medium` ทั้งๆ ที่ Evolver หยิบไปทำเองอยู่ดีถ้าไม่กดอะไร ทำให้เกิดความซ้ำซ้อนและ UX ที่ไม่สอดคล้องกับความจริงเบื้องหลัง

## 2. Alternatives Considered & Architectural Decisions

### Decision 1: Create a 3-Tier Approval UI
- **Alternative:** ใช้ AI สร้าง Approver Agent มาตัดสินใจ
  - **Verdict:** ไม่ចាំเป็น เพราะการให้ AI สร้าง Proposal แล้วให้ AI อีกตัวอนุมัติ เป็น Circular Logic สุดท้ายถ้าพังก็พับอยู่ดี และทาง Evolver มี whitelist ป้องกันอยู่แล้ว
- **Implementation:** 
  - `safe` & `medium`: แสดง badge `🚀 Auto-Approved` ใน Kanban และ Feed ทันที
  - `high`: แสดงปุ่ม `📋 Review Details` ซึ่งกดแล้วจะ Pop up Modal เตือนว่าต้องตัดสินใจอิมพลีเมนต์เอง เพราะ AI Evolver ไม่แตะต้องไฟล์สำคัญ 

### Decision 2: Evolver Core Safety
- **Issue:** ได้รับคำเตือน `Patch ไม่สำเร็จ (Rollback)` สำหรับ Proposal ที่พยายามแก้ไฟล์โครงสร้างอย่าง `evolver.js` ตัวการคือ AI Ideator พยายามจะให้ระบบ optimize โค้ด evolver เพื่อเพิ่มอัตราความสำเร็จ แต่พอ Evolver พยายามแทรกโค้ดตัวเองแล้วเกิด Syntax Error จึงโดนบล็อคและ Rollback แต่กลายเป็นว่าสถานะค้างเติ่งอยู่ในระบบเพราะติด state `approved`
- **Verdict & Fix:** การปล่อยให้ Meta-Agents อัพเดทระบบตัวเองแบบไร้การควบคุมนั้นจะนำไปสู่ความเสี่ยงสูงสุด (System Break) จึงต้องทำการเพิ่ม File Whitelist ไปแบนการแก้ไข `.js` ตัวเอง
- **Solution Code (`evolver.js` & `ideator.js`):**
  ```javascript
  // evolver.js - isFileSafeToAutoModify()
  if (b.startsWith('scripts/hydra/')) {
       if (b.endsWith('evolver.js') || b.endsWith('ideator.js')) return false; // self-modification is HIGH risk
  }
  
  // ideator.js - ideate() loop
  const changes = proposalData.proposed_changes || [];
  for (const c of changes) {
       if (c.file && (c.file.endsWith('evolver.js') || c.file.endsWith('ideator.js'))) {
           proposalData.risk_level = 'high';
           console.warn('[Ideator] Auto-flagged HIGH risk: touched core meta-agent code.');
           break;
       }
  }
  ```

### Decision 3: Resolving "Deadlock" Proposals
- **Issue:** Proposal ที่สถานะเป็น `approved` (เนื่องจากคนเคยกด approve ทิ้งไว้) จะถูกข้ามขบวนการใน `mode = 'auto'` ไปอย่างสิ้นเชิง! 
- **Solution Code (`evolver.js`):**
  ```javascript
  proposals = db.prepare(`SELECT * FROM evolution_log WHERE status IN ('proposed', 'approved') AND risk_level IN ('safe', 'medium')`).all();
  ```

## 3. UI/UX Changes in the Dashboard

### 3.1. Native Global Modal implementation
สร้าง Global Modal พร้อม CSS `.aw-modal` และ Overlay ที่รองรับหลายโหมด หากเป็นโหมด `high` สีเส้นขอบปุ่มและเงาจะเป็น `#fc2d79` เพื่อสะท้อนความสำคัญสูงสุด ปิดท้ายด้วยการสร้าง Global Utility ไว้ให้เรียกใช้ง่ายๆ:
```javascript
window.showAwakenedModal = function({ title, content, type, confirmText, onConfirm }) { ... };
window.showAwakenedToast = function(msg) { ... };
```

### 3.2. Kanban Feed Fixes & Charts
- แก้ช่อง `PENDING REVIEW` ให้นับเฉพาะ `$risk_level = 'high'` และเปลี่ยนฉลากเป็น `NEEDS REVIEW`
- Social Feed สามารถขยายอ่าน `description` และ `root_cause` ได้แบบ inline ไม่ต้องโหลดหน้าใหม่
- แก้ไขปัญหา Canvas Chart `"API Token Resource Burn"` บีบอัดจนข้อความไหล โดยเปลี่ยน `indexAxis: 'x'` เป็น `indexAxis: 'y'` พร้อม truncate ชื่อบอทที่ยาวเกิน 16 ตัวอักษรและเพิ่ม Canvas Height ขึ้นเป็น 280

### 3.3. Discord Alerts (Shortened)
รวบรวมข้อความแจ้งเตือนที่เยิ่นเย้อใน `#hydra-core` ให้มีความกระชับที่สุด เพื่อเป็น High Signal Only:
```javascript
// BEFORE
⚠️ **HYDRA: Patch ไม่สำเร็จ**\nProposal EVO-2... — Optimize...\nระบบ rollback กลับอัตโนมัติแล้ว

// AFTER
⚠️ Patch Failed — EVO-2... Optimize...
↩️ Rollback สำเร็จ
```

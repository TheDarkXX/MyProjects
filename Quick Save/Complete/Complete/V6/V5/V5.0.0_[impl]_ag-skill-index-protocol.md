# ⚡ AG Skills Index & `/skill` Protocol

## 1. บริบทจากบทสนทนาและการตัดสินใจ (Context & Decision)

**จุดที่ 1:** ผู้ใช้สอบถามว่า *"ตอนนี้ใน ag เราใช้ skill ได้ไหม"* โดยระบุในภายหลังว่า *"ไม่ใช่ skill แบบ clouded code/co work น่ะ"* กล่าวคือไม่ใช่แบบ Claude Code / Cursor / Co-pilot ที่มีฟังก์ชัน Skill ของตัวเองแยกต่างหาก

**จุดที่ 2:** AG ตีความได้ว่าผู้ใช้ต้องการใช้ความสามารถของ AG ในการรับขั้นตอนปฏิบัติ (SOP) จากไฟล์แล้วทำตามโดยอัตโนมัติ ซึ่งฟังก์ชันของเครื่องมือ `view_file` ปัจจุบันมีพารามิเตอร์ `IsSkillFile: true` ซึ่งอนุญาตให้ AG อ่านไฟล์แล้วเปลี่ยนเข้าสู่โหมดการทำตามคำสั่งในไฟล์นั้นเสมือนเป็นทาสก์ทำงานได้โดยตรง

**จุดที่ 3:** ผู้ใช้นำเสนอไอเดีย: *"ถามเฉยๆ ว่าอยากทำ index of skill ไว้เพื่อให้เรียกใช้ได้ง่าย รวมถึง command เช่น /skill อะไรแบบนี้น"*
การตัดสินใจ: เราได้วางโครงสร้าง 2 ส่วน เพื่อรองรับการสั่งงานที่กระชับและเชื่อถือได้
1. **จัดตั้ง Skills Index:** สร้างตารางรวม Skill เอาไว้ที่ `docs/AG_SKILLS_INDEX.md`
2. **ผูก Memory Rule ถาวร:** เขียนกฎลงใน `self-improving/memory.md` ให้ตอบสนองต่อ /skill ทุกครั้ง

**จุดที่ 4:** ผู้ใช้สั่งการ: *"จัดเลย สกิลแรก /doctorskill"*
การตัดสินใจ: เพิ่มแถวแรกใน Index ชี้ทริกเกอร์ `/doctorskill` ไปยังเป้าหมาย `agents/daily-journal/AGENT.md` ซึ่งเป็นบอทสำหรับงาน FB ของหมอสกิล

## 2. โครงสร้างที่นำไปใช้งาน (Implemented Changes)

### 2.1 สร้าง `docs/AG_SKILLS_INDEX.md`
ไฟล์สารบัญสำหรับเป็นจุดศูนย์กลาง (Central Hub) ให้ AG เช็คทุกครั้งที่มีคำสั่ง `/skill`

```markdown
# ⚡ Antigravity (AG) - Skills Index

This document serves as the central directory for AG automated skills.
When the user types a shorthand slash command (e.g., `/doctorskill`), AG will consult this index, locate the target file, and execute the workflow automatically.

## 📋 Available Skills

| Command | Target File / KI | Description |
|---|---|---|
| `/doctorskill` | `agents/daily-journal/AGENT.md` | Executes the Doctor Skill FB daily pipeline (FB Post Idea -> Caption -> Cover Image Concept). |
```

### 2.2 แทรก Protocol ใน `self-improving/memory.md`
เพิ่มกฎใหม่ในตอนท้ายของไฟล์เพื่อให้การสั่งงานเกิดขึ้นอย่างอัตโนมัติ และกำจัดขั้นตอนการถามความสมัครใจ (Permission loop) ที่ไม่จำเป็น

```markdown
---

## RULE: /skill Command Protocol
**Context:** The user wants a quick shorthand command to execute standardized operating procedures or agent workflows natively.
**Rule:** 
1. When the user inputs a command starting with a shorthand alias such as `/doctorskill`, AG MUST immediately parse `docs/AG_SKILLS_INDEX.md` to find the mapped target file.
2. AG MUST use the `view_file` tool with `IsSkillFile: true` on the defined target file.
3. AG MUST execute all instructions inside the target file from start to finish without pausing to ask for permission for ordinary steps, fully executing the workflow.
```

## 3. ทำไมถึงเลือกแนวทางนี้แบบนี้ (Architectural Insight)

- **ทำไมไม่ผูกเป็น Script ภายนอก?:** การให้ AG อ่านไฟล์แบบ `IsSkillFile: true` ทำให้ความยืดหยุ่นในการเขียน SOP (Standard Operating Procedure) เป็นแบบ Natural Language ได้ การดูแลรักษาไฟล์ (Maintain) ง่ายกว่าการเขียน Code Script แข็งๆ เพราะมนุษย์สามารถเข้าไปแก้ `AGENT.md` ได้ตลอดเวลาแบบอ่านรู้เรื่อง และ AG ก็พร้อมจะรัน Workflow ใหม่ตามข้อความนั้นทันที
- **ทำไมต้องมี Skills Index:** หากให้ระบุพาทไฟล์ตรงๆ ยาวๆ ทุกครั้ง ผู้ใช้จะพิมพ์ไม่สะดวก (เช่น `help me run agents/daily-journal/AGENT.md`) การรวมมันไว้ตรงกลางช่วยย่อ Trigger ให้สั้นเหลือเพียง `/doctorskill`
- **สิทธิขาดที่ Rule ใน Memory:** การฝังกฎใน Memory ช่วยเป็นคำมั่นสัญญาให้ระบบรู้ว่า "เมื่อเห็นคำสั่งกลุ่มนี้ ห้ามถาม ไม่ต้องรอ ยิงโหลดไฟล์และรันยาวรวดเดียว" ช่วยลดค่า Token/Roundtrip ในการยืนยันคำสั่งที่ไม่จำเป็นลง

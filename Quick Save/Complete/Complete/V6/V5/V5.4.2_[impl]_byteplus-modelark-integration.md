# 🧠 BytePlus ModelArk Integration

## 📌 Context
User requested adding a new API Key for BytePlus ModelArk (`ark-4652a58f-d6cb-4416-acd5-dc19b296d3eb-6fa79`) to check if it can be used identically to OpenRouter since the ModelArk platform provides free quotas like 500k chat tokens for top-tier foundational models (DeepSeek, Doubao, GLM-4, Moonshot Kimi). 

The goal was to test its OpenAI compatibility and integrate it into the global routing logic of OpenClaw VPS and the `brain-app` dashboard.

## 🛠 Features & Implementations

1. **Studied API Specifications**
   - We used `read_url_content` to fetch BytePlus's OpenAPI compatibility docs from `https://docs.byteplus.com/en/docs/ModelArk/1330626`.
   - Determined that `https://ark.ap-southeast.bytepluses.com/api/v3/chat/completions` works perfectly with the standard OpenAI SDK and HTTP formatting.

2. **Backend Configuration (`discord-bot/lib/ai.js` & `.env`)**
   - Appended `BYTEPLUS_API_KEY` to the `.env` configuration file locally and deployed it via SSH to the VPS (`/root/brain-app/.env`).
   - Defined `byteplus` in the `PROVIDERS` dictionary:
   ```javascript
    byteplus: {
      name: 'BytePlus ModelArk',
      model: 'ep-placeholder',
      baseUrl: 'https://ark.ap-southeast.bytepluses.com/api/v3',
      getKey: () => process.env.BYTEPLUS_API_KEY,
      keyName: 'BYTEPLUS_API_KEY',
    }
   ```

3. **Routing Integration (`routes/ai.js`)**
   - Updated the chat `/api/ai/chat` endpoint to detect BytePlus model formats using `modelId.startsWith('byteplus/')`. 
   - Dynamically strips the `byteplus/` string to correctly forward the real target model (`dp-xxx` or `deepseek-v3`) directly to the BytePlus API endpoint.
   ```javascript
    if (modelId.startsWith('byteplus/')) {
      const bpModel = modelId.replace('byteplus/', '');
      const bpKey = process.env.BYTEPLUS_API_KEY;
      // Normal fetch using standard OpenAI structure...
    }
   ```

4. **Frontend UI Dropdown Populator (`routes/ai.js`)**
   - Mapped the exact specific models requested by the user into the API `GET /api/ai/models` response, so the frontend UI (Dev Lab, Chat) immediately reflects these models with their labels and tier indications:
     - `GLM-4.7 (Z.AI)`
     - `DeepSeek-V3.2`
     - `DeepSeek-V3.1`
     - `DeepSeek-V3`
     - `Deepseek-R1-Distill-Qwen-32b`
     - `Kimi-K2 (Moonshot)`
     - `Seedream-4.5 (ByteDance)` (Note: Image gen listed, though unsupported by our current Chat API endpoint)

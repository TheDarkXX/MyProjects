# Issue
The user noticed two connected bugs on the Awakened Protocol tab:
1. **"ปุ่ม approve ไม่ทำงาน กด แล้วเด้งออก"**: The Approve button didn't seem to work. You'd click it, the modal closes, but nothing changes on the card.
2. **"ทำไมยังไม่ auto markdone"**: Items with `Risk: low` were rendering Manual Approve buttons instead of "Auto-Approved".
3. **Ghost / Hanging Event Listeners**: Clicking once occasionally fired the modal 5+ times.
4. **Auto-Markdone Disconnect**: Completing a task in AG didn't close its lifecycle in the Hydra `evolution_log`.

# Root Cause
1. `risk_level: 'low'` was missing from the `safe` and `medium` check that triggers the `🚀 Auto-Approved` badge.
2. After a manual approval (e.g., clicking Approve), the item state changed to `approved`. The Awakened Protocol UI correctly grouped it under "Proposed", but visually it still rendered the `✅ Approve` & `❌ Reject` buttons because it didn't check if `item.status === 'approved'`. So clicking Approve seemingly did nothing (bouncing back to the exact same visual state).
3. "High Risk" tasks were incorrectly calling the `/approve` route, which only sets status to `approved`. High Risk tasks require manual human validation, meaning once approved they should go straight to `deployed`. Using the same `/approve` API left them stuck.
4. `container.addEventListener('click')` was called indiscriminately on every render.
5. Auto-Markdone was previously a manual UI click because the git deployment pipeline had no awareness of Hydra ID markers.

# Fixes
## 1. UI & Backend Fixes
- Added `item.risk_level === 'low'` to the auto-approval check UI logic in `hydra-evo.js`.
- Added an initial UI check `if (item.status === 'approved')` to render `⏳ Awaiting Deployment...` instead of interactive action buttons, stopping the UI from "bouncing". 
- Created a new backend route `POST /api/hydra/proposals/:id/markdone` inside `routes/hydra.js` which updates `evolution_log` status to `deployed` directly, bypassing the evolver.
- Linked the `aw-btn-approve` for High Risk items to the new `data-action="markdone"` logic with a precise confirmation dialog in `hydra-evo.js`.

## 2. Event Listener Guard
- Identified critical `EventListener` leak on pipeline re-renders. Bound `dataset.listenersBound` checkpoint on the `aw-pipeline-board` container to enforce strict idempotency.

## 3. Server-bound Auto-Markdone Engine (Iron Rule System)
- Engineered `scripts/hydra/vps-auto-markdone.js` to parse `Quick Save/Complete/` recursively.
- It detects `brain_task_id: EVO-<id>` inside the frontmatter and runs an immediate `UPDATE evolution_log SET status = 'deployed'` on the VPS database.
- Hooked this script physically into `/root/repos/brain-app.git/hooks/post-receive`. 
- **Outcome:** Every `git push vps` (whether from AG or human) perfectly syncs the deployment closure back to Hydra automatically exactly when the code lands live.

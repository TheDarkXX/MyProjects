## Objective
The user requested that the Hydra UI saves the selected tab, so that when the browser is refreshed, it automatically returns to the same nested tab instead of resetting to the default view.

## Execution
1. Extracted `switchHydraTab` usage within `brain-app-public/hydra-evo.js`.
2. Stored the selected tab string into `localStorage.setItem('hydraActiveTab', tab)`.
3. Created an auto-restore hook in `hydra-evo.js` attached to `DOMContentLoaded`. It queries `localStorage` on page initialization and triggers `switchHydraTab(savedTab)` if a persistent preference is found and `#hydra-app` exists.
4. Guaranteed non-disruptive loading by injecting the restoration loop behind a brief timeout, guaranteeing DOM setup is fully mature before execution.

## Result
Users now enjoy zero-friction refreshing while investigating specific sub-tabs (like Matrix or Comm logs) on the Awakened Dashboard.

## 1. UI UX Layout Refinements (Main Content & Sidebar)
Before checking data consistency, the dashboard was reorganized to optimize screen real estate:
* **The Grid:** Changed from a rigid grid structure to `.ocl-charts-sidebar-layout` with a `3fr 1fr` configuration.
* **Main Area (75%):** Agent Intelligence, Model Usage (Doughnut), and Session Breakdown stacked on the left side to avoid vertical gap stretching.
* **Right Sidebar (25%):** Moved "System Health" to the top right directly over "Model Deep-Dive", letting the tall Agent-Agent drill-down table expand downwards without breaking the main row constraint.
* **Decluttering:** Cleaned out redundant cost breakdowns ("Market Value", "Actual Cost", "You Saved") from System Health, keeping that info purely inside the Hero Command Strip.

## 2. Session Breakdown Accordion (Native Drilldown)
Replaced the traditional view with an accordion sub-table nested right below the active table rows.
* Expand the parent agent row to query `.sessionLogs` instantly without modal overlay interruptions.
* Display up to 100 historical logs per agent.

## 3. The 4 Data Integrity Fixes (Post-Audit)
During QA, massive data inconsistencies were detected (e.g., total Agent Session counts showing 260 compared to sub-table limits, Discord token misattributions, and Model names placed in Provider columns). A full script trace of `initOpenclawPage` identified 4 major bugs:

* **Bug 1: Provider vs Model Name Mixing**
  * *Issue:* The sub-table was fetching `p.label` (which is the actual model name, e.g., "Gemini 2.5 Pro") and placing it under the new User-Requested "Provider" column.
  * *Fix:* Swapped definitions -> **Provider:** `p.note` / **Model Name:** `p.label`.

* **Bug 2: Session Breakdown Double-Counting**
  * *Issue:* `renderOclSessionBreakdown` was passed `mergedSessions` (OCL + DC streams combined) AND additionally executed a parallel array loop mapping `dcUsage.sessions`. Everything on Discord was essentially doubled (e.g., Tata showing 118 instead of 59, AI Monitor showing 260 instead of 130).
  * *Fix:* Replaced argument `mergedSessions` with strictly `sessions` to ensure `dcUsage.sessions` injection functions as intended without conflict.

* **Bug 3: Provider Tab Discord Hardcoding (`Google AI`)**
  * *Issue:* The `renderOclIntelligence` provider map aggregated all Discord usage under one hardcoded string `google-ai` with model `Gemini 2.5 Flash DC`. This was incorrect, given OpenRouter Discord tools use actual paid tokens.
  * *Fix:* Rewrote the Provider injection loop to actively parse each `dcUsage` session string, dynamically fetching the true provider mappings via `getModelPricing(s.model).note`.

* **Bug 4: The "Missing 256 logs" Illusion (Cron / Discord Batching)**
  * *Issue:* User saw "260 sessions" but the sub-table only produced 4 items limit.
  * *Analysis:* No logs were missing. Server background tasks and Discord handlers batch logs dynamically per hour/day with `log.calls = 65` under a single payload array to prevent SQLite lockups.
  * *Fix:* Opted against Dropdown Pagination. Built a `[Batched Calls: n]` mathematical marker directly in the Sub-table paired with an `(~XX/call)` token average formula for immediate user comprehension. (Polished to styling `10px slate-200 85% opacity`).

## 4. Total Cost Calculation Context Note
Added clear user-facing explanation on how OpenClaw evaluates free versus monetized layers.
* Free integrations (Google AI Studio native, ChatGPT Plus endpoints) calculate natively as `$0.00`.
* IDE agents (Antigravity & Code-GPT) are statically voided from dashboard summation `if (!isIDEAgent(aId))` to prevent distortion of production metric averages.

# Feature Implementation: `/mu` Mockup Skill

## 1. Context & Problem
The user suggested adding a command to produce UI mockups `/mu` so that both the user and AI can align on a shared layout visual before diving into potentially destructive frontend coding. Also, earlier UI refinement on `AI Comms` exposed an issue where the AI natively defaults to dim/grey colors (`#5a5a90`) in dark mode, making timestamps illegible.

## 2. Implementation Strategy / Rationale
### A. Mockup Protocol (`/mu`)
- Created `docs/skills/mu.md` standardizing a 5-step flow.
- The AI will absorb context, formulate a detailed Midjourney-style design prompt ensuring "Premium MyDesign System" traits (glassmorphism, neon accents), and execute the `generate_image` tool.
- Directly mapped `/mu` inside `AG_SKILLS_INDEX.md`.

### B. `/design` Skill Upgrade (Legibility)
- The user correctly deduced that font brightness rules should be enshrined in the root design capability, not just in `/mu`.
- I injected a firm rule into the **Typography & Legibility** audit block inside `docs/skills/design.md`:
  > "Crucially, NEVER sacrifice legibility for aesthetics. Dark mode sub-text, timestamps, and small metadata MUST be bright and luminous (e.g., `#9898c8`), forbidding dim, unreadable greys."

## 3. Post-Implementation
- Successfully synced to `AG_SKILLS_INDEX.md`.
- No server-side functionality was touched, so simple git-hook deployments secure the documentation updates correctly.

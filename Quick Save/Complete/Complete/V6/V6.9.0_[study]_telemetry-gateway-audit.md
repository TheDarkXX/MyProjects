## Solution: Unified AI Gateway

### Architecture

```
┌────────────────────────────────────────────────────────────────┐
│           AI Gateway  (ai-gateway.js)  port :18810             │
│                                                                  │
│  POST /openrouter/v1/...   →  openrouter.ai                    │
│  POST /kie/{model}/v1/...  →  api.kie.ai (with filter logic)   │
│  POST /google/v1/...       →  generativelanguage.googleapis.com │
│  POST /byteplus/v1/...     →  ark.ap-southeast.bytepluses.com  │
│                                                                  │
│  Headers Read:                                                   │
│    X-Claw-Cron   → cronId                                       │
│    X-Agent-Id    → agentId                                      │
│    Authorization → forwarded to provider                        │
│                                                                  │
│  Token Tracking:                                                 │
│    → extracts usage.prompt_tokens + completion_tokens           │
│    → writes to discord-bot/data/usage.json  ← same as kie-proxy │
└────────────────────────────────────────────────────────────────┘
```

### ไฟล์ที่จะสร้าง/แก้

#### Phase 1 — Infrastructure (2 files)

**[NEW] `ai-gateway.js`** — Unified proxy server port 18810
- Route ตาม URL path prefix (`/openrouter/`, `/kie/`, `/google/`, `/byteplus/`)
- นำ Kie.ai payload filter จาก `kie-proxy.js` มาฝังใน `/kie/` route
- Token tracking ใช้ logic เดิมจาก `kie-proxy.js` เขียนลง `usage.json`
- PM2: `pm2 start ai-gateway.js --name ai-gateway`

**[NEW] `lib/ai-call.js`** — Universal client helper

```js
// Usage ในทุก script ที่ migrate:
import { callAI } from '../../lib/ai-call.js';

const text = await callAI({
  provider: 'openrouter',          // 'openrouter' | 'google' | 'kie' | 'byteplus'
  model: 'qwen/qwen-2.5-72b-instruct',
  messages: [...],
  cronId: process.env.CLAW_CRON_ID || 'hydra-ideator',  // auto from env
  agentId: 'ideator',
  // Optional: kieModel for kie provider (e.g. 'gemini-2.5-flash')
});
```

- ยิงไป `http://127.0.0.1:18810/{provider}/v1/chat/completions`
- จัดการ API key ให้เองตาม provider
- Return `string` (content) หรือ throw error

#### Phase 2 — Migration (30 files in batch)

**2A: Hydra System (15 files)**
Pattern เดิม: `for (const tier of tiers) { await axios.post(tier.url, ...) }`
Pattern ใหม่: `for (const tier of tiers) { await callAI({ provider: tier.providerKey, ... }) }`

เพิ่ม `cronId` ตาม agent:
- evolver → `'hydra-evolver'`
- ideator → `'hydra-ideator'`
- inspector → `'hydra-inspector'`
- doctor → `'hydra-doctor'`
- visionary → `'hydra-visionary'`
- strategy-engine → `'hydra-strategy'`
- self-rewriter → `'hydra-self-rewriter'`
- red-team → `'hydra-red-team'`
- knowledge-crystallizer → `'hydra-knowledge'`
- dream-mode → `'hydra-dream'`
- cross-pollinator → `'hydra-cross'`
- anticipation-engine → `'hydra-anticipation'`
- agent-spawner → `'hydra-spawner'`
- agent-fusion → `'hydra-fusion'`
- weekly-evolution-report → `'hydra-weekly-report'`

**2B: Discord Bot System (3 files)**
- `credit-card-agent.js`: `callKieAI()` + `callOpenRouter()` → `callAI({ cronId: 'cron-cc-statements' })`
- `gmail-agent.js`: kie.ai direct + google fallback → `callAI({ cronId: 'cron-gmail-expenses' })`
- `health-news.js`: gemini-direct fallback path only → `callAI({ cronId: 'cron-health-news' })`

**2C: Scripts (7 files)**
- `api-clients.js`: refactor 3 functions → `callAI()` (cronId from `process.env.CLAW_CRON_ID`)
- `reformat-content.js` → `callAI({ provider: 'kie', cronId: 'reformat-content' })`
- `compile-skills.js` → `callAI({ provider: 'google', cronId: 'compile-skills' })`
- `gold-price-monitor/notifier.js` → `callAI({ provider: 'google', cronId: 'gold-price-notifier' })`
- `gold-price-monitor/fetcher.js` → `callAI({ provider: 'google', cronId: 'gold-price-fetcher' })`
- `cron-lily-weekly.js` → `callAI({ provider: 'google', cronId: 'cron-lily-weekly' })`
- `cron-registry/templates.js` → `callAI({ provider: 'google', cronId: 'cron-registry-gen' })`

**2D: Routes (5 files) — Different approach**
> Routes ถูก call จาก Dashboard/Chat UI โดย human ไม่ใช่ automated cron → ใช้ `agentId: 'brain-app'` แทน cronId

- `routes/ai.js` → Add inline token logging ต่อจาก fetch (ไม่ route ผ่าน gateway เพราะ routes นี้ทำงาน real-time)
- `routes/ai-chat.js` → เช่นเดียวกัน
- `routes/knowledge.js` → Log embedding token usage (input only, no output)
- `routes/links.js` → Log embedding token usage
- `lib/brain-ai.js` → `callAI({ provider: 'google', agentId: 'brain-app' })`

#### Phase 3 — Decommission `kie-proxy.js`

เมื่อ migration สมบูรณ์:
- รวม logic ของ `kie-proxy.js` เข้าไปใน `ai-gateway.js` `/kie/` route (ส่วนใหญ่ทำแล้วในข้อ Phase 1)
- อัปเดต `openclaw.json` ให้ชี้ baseUrl มาที่ `:18810/kie/` แทน `:18800/`
- Stop PM2 process `kie-proxy`
- เก็บ `kie-proxy.js` ไว้เป็น archive แต่ไม่ run

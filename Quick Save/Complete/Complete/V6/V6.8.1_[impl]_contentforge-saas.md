# 🚀 V6.8.1 — Content Pipeline Mission Control (ContentForge SaaS)

> **Goal:** สร้างระบบ SaaS แยกออกมาเด็ดขาดจาก Brain App เพื่อเป็น Viral Video Factory รองรับการยิง Prompt ทิ้งไว้ → ระบบไปสุ่มตัวแปร → Gen วิดีโอ (Sora 2) → ตั้งเวลาโพสต์บนช่องหลายๆ Platform แบบ Full Auto

---

## 🏗️ Architecture Design (Standalone SaaS)

ผมได้วิเคราะห์ requirement ทั้งหมด และออกแบบระบบเป็น **Next.js 15 (App Router)** แยกจาก codebase เดิม เพื่อรองรับ SaaS Model

**Tech Stack ที่เลือกใช้:**
1. **Frontend / API:** Next.js 15 (React 19)
2. **Database:** PostgreSQL (ใช้ Drizzle ORM) — *จำเป็นต้องเปลี่ยนจาก SQLite เพื่อรองรับ Multi-tenant concurrency อย่างปลอดภัย*
3. **Queue System:** BullMQ + Redis — เพราะงาน Video Gen ใช้เวลานาน (2-5 นาที) ไม่สามารถทำ Synchronous HTTP Request ได้
4. **Auth / Billing Ready:** ออกแบบ Schema ผูก `user_id` ในทุก Table ตั้งแต่วันแรก

---

## 📦 Database Schema Design (PostgreSQL)

ตัวอย่าง Core Tables ที่ออกแบบไว้:

1. `users` — ระบบบัญชีผู้ใช้ และเครดิตการสร้างวิดีโอ (Token/Credit System)
2. `channels` — บัญชี Social Media แต่ละช่อง (เช่น FB Page: Time Lapse บ้าน, FB Page: สร้างสระว่ายน้ำ) ผูก OAuth Token ระดับช่อง
3. `pipelines` — "สูตรผสมวิดีโอ" เก็บ Prompt Template ที่มีตัวแปร `{{variable}}` เช่น `{{action}}` และ `{{building_type}}` รวมถึงความถี่การโพสต์ (Cron)
4. `contents` — วิดีโอรายตัวที่สร้างสำเร็จ (หรือกำลังสร้าง) เก็บสถานะ queue, generated, posted, failed

---

## 🔄 Core Flow: Video Generator Engine

1. **Scheduler Worker** รันทุกวัน หยิบ Pipeline ที่ถึงเวลา
2. ระบบทำการ **Randomize Variables** ใน Prompt Template 
   - *ตัวอย่าง: `{{action}}` สุ่มได้ "building", `{{building_type}}` สุ่มได้ "glass mansion"*
3. ส่ง Final Prompt เข้า **BullMQ Job Queue**
4. **Video Worker** หยิบ Job ไปยิง **Sora 2 API** (หรือ Seedance) และรอ Process
5. เมื่อเสร็จ นำวิดีโอไปเก็บ (MinIO / R2 / S3) หรือดึงตรงจาก API Provider
6. **Auto-Poster** หยิบ Job ไปโพสต์ลง FB Page ตาม Access Token ที่จดจำไว้ในระดับ Channel

## ⚠️ Important Notes & Constraints

1. **Isolation:** โปรเจคนี้จะไม่อยู่ใน `Openclaw-VPS/brain-app` เพื่อป้องกันการปนเปื้อนของโค้ด เนื่องจากใช้ Tech Stack มิติใหม่ (Next.js vs Hono Vanilla JS) 
2. **VPS Resources:** โน้ตย้ำเตือนว่า การรัน PostgreSQL + Redis บน VPS 185.250.38.247 ที่มี `brain-app` อยู่ อาจจะต้องสังเกต Memory Usage หากตึงเกินไปในอนาคตควรย้าย Database ออกไปนอก VPS
3. **Legacy Pipeliens:** ระบบเก่าอย่าง Doctor Mike, Doctor Skill (FB), Minnie จะยังคงทำงานอยู่ใน `brain-app` ตามปกติ จนกว่าจะตัดสินใจย้ายเข้าระบบใหม่ทั้งหมด

### Problem
The `cron-ai-monitor.js` was triggering notifications too frequently and going back and forth between "Failed" and "Recovered" in rapid succession, which annoyed the user. The alerts were trigged by momentary timeouts and short API stumbles.

### Solution / Implementation
Modified `discord-bot/scripts/cron-ai-monitor.js` to change the thresholds:
- **Alert Trigger (Failure):** from `recentFails >= 3` (15 mins) to `recentFails >= 6` (30 mins).
- **Alert Trigger (Recovery):** from `recentSuccess >= 3` (15 mins) to `recentSuccess >= 6` (30 mins).
- **Recurring Alert:** from `recentFails % 12` (60 mins) to `recentFails % 24` (120 mins).
- **Network Timeout:** `AbortSignal.timeout(30000)` was extended to `60000` (60 seconds) to tolerate occasionally slow model providers.

This significantly cuts down the noise and only alerts when multiple consistent sequential retries confirm an issue, ensuring a true outage condition before reaching out to Telegram and Discord.

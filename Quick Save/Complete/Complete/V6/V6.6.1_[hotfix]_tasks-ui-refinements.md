# Tasks UI Refinements

## Issue description
- The "Completed" collapsible section was missing the timestamp of when a task was actually completed, relying purely on the general "dueDate" field which is meant for the past.
- The `dev-lab` projects (AG tickets, codebase refactoring) were showing up directly alongside Life tasks, making the Personal Dashboard extremely cluttered.

## Implementation Data
- **Completed Timestamp:** The `brain.db` already triggers an SQLite timestamp insertion (`completed_at = datetime('now','localtime')`) natively inside `PUT /api/tasks/:id` when the state flips from `anything` -> `done`. I unpacked `t.completed_at` on the frontend inside `renderDoneSection()` and surfaced it using UI monospace tokens inside `brain-app-public/tasks.js`. Modified `style.css` to increase `.done-row` opacity from 0.45 to 0.8 and `.done-row-title` to `--text-primary` for much better contrast.
- **Filtering Logic:** In `loadTasksData()`, enforced an exclusionary filter that drops any record with `project === 'dev-lab'` IF the project selector is set to "All Projects" (null/empty string). Expanded this behavior to `app.js` -> `initDashboard` where `dev-lab` tasks were previously clogging up the `limit=20` recent load endpoint. Increased limit to 200 and explicitly dropped `dev-lab` from the "Today's Tasks" widget to prevent them from wiping out operation items. 

## Note for `dev-agent`
- `dev-agent` is already natively mapped via `discord-bot/config/channels.js` to dispatch tasks with `defaultProject: 'dev-lab'`. The Piggyback webhook handles the routing just fine. This means creating a new task from Discord via #dev-agent immediately categorizes it as `dev-lab`, keeping it perfectly out of sight from the main `Life/Tasks` UI.

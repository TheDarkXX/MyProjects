# Feature: Dev Agent Smart Intent & Persona Strengthening

## 1. Context & Rationale
After deploying the Fast-Edit Agent (`!edit`), the user noticed the Dev Agent chatbot hallucinating about personal life matters (e.g., answering a question about "checking if 30 days have passed" by referencing the user's children leaving the hospital instead of the codebase limit). 
This occurred because `brainQuery` injects global DB contextual memories, and the user requested a more robust developer persona that disregards non-technical contexts. Additionally, relying on `!edit` prefixed commands creates friction, so the user requested natural language integration for code generation.

## 2. Dev Agent Persona Hardening
Modified `discord-bot/prompts/dev-agent.md`:
- Applied a `CRITICAL IDENTITY RULE` strictly reminding the LLM that it is a Software Engineer.
- Enforced a rule to filter out out-of-context RAG memories (`MEMORIES` / `NOTES`) that contain family or daily life events when answering technical `#dev-agent` queries.

## 3. Smart Code Intent Detection (Natural Language `!edit`)
Modified `discord-bot/index.js`:
- Introduced `detectCodeEditIntent()` which analyzes incoming natural language text in `#dev-agent` for **both** a technical "Target" keyword (e.g., config, code, db, api, script) and an "Action" verb (e.g., change, update, fix, adjust).
- Refactored the `!edit` handling logic into a reusable `handleFastEdit()` asynchronous function.
- Bound `handleFastEdit()` to intercept `#dev-agent` messages. If the intent detector flags `true`, the bot skips generic conversation and pipes the user's exact phrase to `discord-patch-agent.js` as an evolution prompt.
- Retained the actual `!edit` command as a fallback.

## 4. Architectural Behavior Recap
- `#dev-agent` conversations are now purely code-related.
- Casual phrases like "ขยาย time slot cron planner เป็น 30 วัน" trigger a root-level full codebase RAG scan via Evolver (Fast-Edit).
- Casual conversational phrases like "ระบบนี้ทำงานยังไง" resolve via Pinky Conversational LLM normally without triggering code-rewrites.

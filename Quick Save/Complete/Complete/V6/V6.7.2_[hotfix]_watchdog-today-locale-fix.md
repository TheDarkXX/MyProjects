# 🚨 บทวิเคราะห์: การแจ้งเตือนผิดพลาดของ Watchdog "today-has-scheduled"

## 📍 ปัญหาที่พบ (The Symptom)

ผู้ใช้งานแจ้งว่าได้รับรายงานแจ้งเตือน (Alert) จาก Watchdog ทาง Discord โดยมีข้อความว่า:

> **🚨 รายงานจาก Watchdog: Doctor Skill FB Pipeline**
> **🎯 สิ่งที่ควรจะเป็น:** วันนี้หรืออนาคตมีโพสที่ schedule ไว้
> **❌ ปัญหาที่เจอ:** 🚨 ไม่ผ่านเกณฑ์! (ตั้งไว้คือ cnt >= 1 แต่นับได้ 0)
> **🔥 ระดับความรุนแรง:** วิกฤต (CRITICAL)

แต่ว่าเมื่อตรวจสอบหน้า UI ปฏิทินและในฐานข้อมูลจริง (Production Database: `/root/brain-app/db/brain.db` ในตาราง `content_plans`) กลับมี Content ที่เตรียมพร้อมถูกตั้งเวลาไว้ไปจนถึงวันที่ 24 เมษายน จำนวน 11 โพสต์แล้ว

## 🕵️‍♂️ การสืบสวนและทดสอบ (Investigation)

จากการรันตรวจสอบ SQL Query เดียวกับที่ Watchdog ใช้ใน `c:\My Claw\Openclaw-VPS\scripts\watchdog\goals.js`:
\`\`\`sql
SELECT COUNT(*) as cnt FROM content_plans 
WHERE (substr(scheduled_at,1,10) >= '{today}') 
AND status IN ('planned','scheduled','ready','in-process','published','produced','scripted') AND project='doctor-skill'
\`\`\`

พบว่าที่ฝั่ง JS จะทำการแปลงตัวแปร `{today}` ให้เป็นวันที่ปัจจุบันด้วยคำสั่ง:
\`\`\`javascript
const qStr = check.query.replace(/\\{today\\}/g, new Date().toLocaleDateString('sv-SE'));
\`\`\`

**ปัญหาเกิดจากการตีความ String แบบ Alphabetical ใน SQLite**
ถ้าระบบของ VPS (ตอนที่รันโดย PM2 หรือ Cron แบบ Background) อ่านค่า `toLocaleDateString('sv-SE')` แบบ Fallback (เพราะอาจไม่มี Locale รองรับ) มันจะให้ค่าเดือนและวันกลับด้านหรือกลับไปเป็น Format พื้นฐานของ US เช่น \`4/18/2026\`

และเมื่อคำนวณ:
\`\`\`sql
SELECT '2026-04-20' >= '4/18/2026';
\`\`\`
SQLite จะเช็คเรียงตามอักขระ (Alphabetical comparison) โดยพบว่าอักขระตัวแรก \`'2'\` (จาก 2026) **มีค่าน้อยกว่า** อักขระ \`'4'\` (จากเดือน 4) ผลที่ได้จึงกลายเป็น **FALSE**

ทำให้ทุกๆ Content Plan ที่แม้จะตั้งไว้ตามช่วงเวลาที่ถูกต้อง ถูกประเมินค่าว่าหลุดเกณฑ์ทั้งหมด ส่งผลให้นับ Count ได้ \`0\` 

## 🛠️ วิธีแก้ไข (The Fix)

แทนที่จะให้ Javascript เป็นคนแปลและส่ง String \`{today}\` ข้ามเข้าไปใน SQL Query แล้วเสี่ยงกับปัญหา Locale Format เราใช้ความ বাতাসে Native ของ SQLite ในการดึงวันที่ Local Time ของเซิร์ฟเวอร์โดยตรงไปเลย:

**ไฟล์ที่แก้ไข:** `scripts/watchdog/goals.js`

\`\`\`diff
- query: \`SELECT COUNT(*) as cnt FROM content_plans WHERE (substr(scheduled_at,1,10) >= '{today}') AND status IN ('planned','scheduled','ready','in-process','published','produced','scripted') AND project='doctor-skill'\`,
+ query: \`SELECT COUNT(*) as cnt FROM content_plans WHERE (substr(scheduled_at,1,10) >= date('now', 'localtime')) AND status IN ('planned','scheduled','ready','in-process','published','produced','scripted') AND project='doctor-skill'\`,
\`\`\`

## 🚀 ผลสรุป (Outcome)

ทำกระบวนการ *Commit -> Bump-cache -> Push VPS* เพื่อแจกแจงแพตช์ให้ Active Environment นำไปใช้ 
ผลลัพธ์คือ Watchdog ได้หยุดร้องเตือนมั่วซั่ว และระบบ Auto-fix ตัวที่เกี่ยวข้องก็จะหยุดรันซ้ำซ้อนจนพังเป้าหมาย Circuit-breaker (Max auto-fixes per day reached) ด้วยเช่นกัน

# Automated Expense Tracker: Architecture & Implementation Plan

แผนงานนี้ออกแบบมาเพื่อเชื่อมโยงระบบอ่านอีเมลใบเสร็จ/สลิปโอนเงิน (Gmail Agent) เข้ากับระบบฐานข้อมูลหลักของ Command Center (SQLite) และแสดงผลให้คุณจัดการในหน้าเว็บ UI ได้อย่างสวยงามและเป็นระบบครับ

---

## 🏗️ 1. Database Architecture (ฝั่งฐานข้อมูล)
เนื่องจากหน้าเว็บ Command Center ใช้ฐานข้อมูล SQLite เป็นหลัก เราจะสร้างตารางใหม่ชื่อ **`cc_expenses`** ในไฟล์ `brain-app.sqlite` เพื่อเก็บข้อมูลให้เป็นหมวดหมู่ ค้นหาและทำสรุปรายเดือนได้ง่าย (ดีกว่าจดลง Markdown สดๆ แบบที่ Pinky กังวล)

**Schema Design (`cc_expenses`):**
- `id` (Primary Key)
- `date_time` (TEXT) - วัน-เวลาที่เกิดรายการ เช่น `2026-03-28 13:34:44`
- `date` (TEXT) - วันที่ เช่น `2026-03-28` สำหรับ filter รายวัน
- `year_month` (TEXT) - เดือน-ปี เช่น `2026-03` สำหรับทำสรุปรายเดือน
- `name` (TEXT) - ชื่อร้านค้า / ผู้รับเงิน
- `amount` (REAL) - รายจ่าย (Cost)
- `category` (TEXT) - หมวดหมู่ตามชุดมาตรฐาน
- `account` (TEXT) - จากบัญชี (เช่น `KBank-4828`, `K PLUS`)
- `source_ref` (TEXT) - อ้างอิงแหล่งที่มา เช่น `Message-ID` จากอีเมล (กันการบันทึกซ้ำซ้อนเวลาสคริปต์รันซ้ำ)
- `created_at` / `updated_at`

---

## ⚙️ 2. Backend API (`routes/command-center.js`)
เราจะเขียน API Endpoints ใหม่สำหรับให้ **Gmail Agent ยิงข้อมูลเข้ามาบันทึก** และให้ **หน้าเว็บ UI ดึงข้อมูลไปแสดงผล**:
- `GET /api/cc/expenses` — เรียกดูรายการย้อนหลัง (รองรับ Filter ตาม `year_month` หรือ `category`)
- `POST /api/cc/expenses` — รับ Data Payload จากสคริปต์ `cron-gmail.js` เพื่อบันทึกรายจ่ายใหม่ลง DB อัตโนมัติเวลาอ่านเมลเจอ
- `PUT /api/cc/expenses/:id` — สำหรับให้ User กด Save อัปเดตข้อมูลบนหน้าเว็บ (ตั้งค่าหมวดหมู่จาก *ไม่ระบุ* → ตัวอื่น)
- `DELETE /api/cc/expenses/:id` — เผื่อรายการที่ AI อ่านผิดให้ลบทิ้งได้

---

## 🎨 3. Command Center UI (`brain-app-public/`)
- เพิ่มแถบ **Expenses & Finance** ในหน้า Command Center 
- ข้อมูลมาแสดงเป็น Data Table สไตล์มินิมอล มีฟีเจอร์โหลดแบบ Pagination
- รายการไหนระบุไม่ได้ (`ไม่ระบุ`) จะ **ไฮไลต์สีส้มแจ้งเตือน**
- พอคลิกที่แถว จะเด้ง Edit Modal ให้เลือก `Category` หรือแก้ไขข้อมูลได้
- หน้าตาจะโชว์ตัวเลขภาพรวม (Total Expenses of This Month)

---

## 🤖 4. Integration with Gmail AI Agent (`cron-gmail.js`)
1. เปลี่ยน Flow ของสคริปต์กวาดอีเมล (Gmail App) ให้ไปยิง API เข้าระบบ Command Center ด้วย
2. ให้ Gemini เดา Category ตามชุดมาตรฐานที่คุณตั้งไว้ก่อน 
3. พอระบบเซฟเข้า DB แล้ว Agent ค่อยยิงแจ้งเตือนสรุปกระชับลงที่ Discord `#expense-tracker` ตาม Format ที่ Pinky แนะนำ

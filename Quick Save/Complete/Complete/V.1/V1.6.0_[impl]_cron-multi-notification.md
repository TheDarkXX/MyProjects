# Cron Multi-Notification Support Plan

แผนงานอัปเกรดระบบ Command Center ให้รองรับการส่งแจ้งเตือนหลาย Platform/Channel ใน 1 Cron Job รวมถึงออกแบบ UI ให้จัดการง่าย สวยงาม และสามารถเปิด/ปิดแยกรายช่องทางได้

---

## Current State Analysis

### ปัจจุบันเก็บข้อมูลอย่างไร
- `cc_crons` table มี 2 คอลัมน์: `platform TEXT` + `notify_channel TEXT`
- 1 cron = 1 destination เท่านั้น → อยากยิง 2 channel ต้องสร้าง 2 cron jobs
- Scripts (`cron-brief.js`, `cron-news.js`) **Hard-code** CHANNEL_ID ใน env/code

### ไฟล์ที่ต้องแก้ไข
| ไฟล์ | บทบาท |
|------|--------|
| `routes/command-center.js` | Backend API — schema, CRUD, migration |
| `brain-app-public/command-center.js` | Frontend — table, detail panel, edit modal |
| `brain-app-public/style.css` | Styles สำหรับ Destination Builder + Pills |
| `discord-bot/scripts/cron-brief.js` | อ่าน notify_config จาก DB |
| `discord-bot/scripts/cron-news.js` | อ่าน notify_config จาก DB |
| `discord-bot/scripts/cron-gmail.js` | อ่าน notify_config จาก DB |
| `discord-bot/scripts/cron-alerts.js` | อ่าน notify_config จาก DB |
| `brain-app-public/index.html` | Cache-busting version bump |

## Risk Assessment

| ความเสี่ยง | ระดับ | มาตรการ |
|-----------|-------|---------|
| Scripts hard-code CHANNEL_ID → DB ล่มส่งไม่ได้ | 🟠 Medium | Fallback env var เสมอ |
| SQLite ALTER TABLE ใน production | 🟢 Low | ADD COLUMN ปลอดภัย |
| Frontend เปลี่ยนเยอะ → break UI | 🟠 Medium | ทำทีละ Phase, test ทุก step |
| Backward compat กับ old data | 🟢 Low | เก็บ platform/notify_channel ไว้ |
| LINE/Telegram ยังไม่มี sender | 🟢 Low | ใส่ schema ก่อน, ทำ sender ทีหลัง |

## Total Estimated Effort: ~10 ชั่วโมง

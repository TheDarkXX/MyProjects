# Cron Analytics Breakdown — Implementation Plan

## 1. The Problem: "Token Mixing"
Currently, both Discord bots and Cron automation scripts hit `kie-proxy` by authenticating with their `agentId` (e.g., `pinky`, `lily`). 
Because Cron jobs share the same identity as Discord bots, the proxy logs all their usage under a single source identity in `discord-usage.json`. 
When the AI Automation Manager visualizes this, all crons owned by `lily` display the **exact same token usage** because they query the same `agentId` rollup data.

## 2. Infrastructure Goal
Enforce exact 1:1 attribution for backend background jobs so `ai-manager.js` can accurately report costs, tokens, and runs per specific `.js` cron script.

## 3. Implementation Steps

### Phase 1: Custom Telemetry Header (Cron Side)
1. **Target Files**: All files in `/root/cron/` and background tasks (e.g., `cron-news.js`, `cron-alerts.js`, `cron-gmail-expenses.js`).
2. **Action**: Wait until `kie-proxy/index.js` or `openclaw` request utility is identified. We need to inject an HTTP header to every outgoing AI request.
   *Example Header:* `X-Claw-Cron: cron-news`

### Phase 2: Intercept & Log (Proxy Side)
1. **Target File**: `kie-proxy.js`
2. **Action**:
   - Detect the `X-Claw-Cron` header.
   - If present, append it to the log record in `discord-usage.json` (or route it to `openclaw-sessions.json` with a dedicated unique `key`).
   - Example new shape: 
     ```json
     {"date":"2026-04-11","agentId":"pinky","cronId":"cron-news","model":"gemini-2.5-flash", ...}
     ```

### Phase 3: Dashboard Mapping (UI Side)
1. **Target File**: `brain-app-public/ai-manager.js`.
2. **Action**: 
   - Update `renderCronsTab` filtering logic.
   - Stop falling back to `s.agentId === cron.owner`.
   - Instead, do an exact match: `s.cronId === scriptName`.

## 4. Current Status: Parked/Planned
Pending review. Once executed, this will permanently fix the "Token 0" or duplicate grouping issue across multi-cron agents.

# 🤫 V4.8.3 — HYDRA Core Silent Auto-Deploy

## 1. ปัญหา Noise Reduction ในช่องแชท
หลังจากปรับให้ \`medium\` ทำ Auto-Deploy โดยพึ่งพา Safety Layers ของสถาปัตยกรรม HYDRA (ทดสอบ V4.8.2) ตามการร้องเรียนเรื่องคอขวด 
User ก็ได้พิจารณาต่อว่า **"งั้นควรเตือนเฉพาะเท่าที่จำเป็นจะดีกว่าไหม เพื่อไม่ให้ระบบเตือนพร่ำเพรื่อ?"** 

จากการวิเคราะห์ พบว่า Pipeline เดิมทำการแจ้งเตือน 2 รอบเสมอในทุกครั้งที่มีสิ่งให้แก้ไข:
1. `ideator.js`: แจ้งว่าพบเป้าหมายแล้วนะ กำลังนำเสนอการเปลี่ยนแปลง!
2. `evolver.js`: เอางานที่เสนอไปรันสำเร็จ!

เพื่อลดความสแปมและความน่ารำคาญ จึงปรับให้เป็น **Silent Mode** ฝั่งการเสนอไอเดีย

---

## 2. การตัดสินใจเชิง Architecture (Silent Ideation)
เราหยุดระบบแจ้งเตือนแบบ Realtime ใน `ideator.js` โดยให้ตัวแปรพึ่งพางานดังนี้:

- 🟢 **Safe / 🟡 Medium Risk:** ปล่อยให้บันทึก Proposals ลง Database แบบเงียบๆ ไม่มี Alert ใดๆ ไม่ต้องแท็กหา — AI จะเอาไปรันเอง แล้ว `evolver.js` จะแจ้งโผล่มาแค่บรรทัดสั้นๆ ว่าอัปเกรดสำเร็จแล้ว (Success Ping)
- 🔴 **High Risk:** คงการแจ้งเตือนไว้เหมือนเดิม เพราะ AI จะไม่เข้าถึง layer การแก้ไขระบบที่เป็น High Risk ตรงนี้ "จำเป็น" ต้องให้ Human-in-the-loop เข้ามาประเมิน (Actionable Request)

## 3. Code Implementation

เป้าหมายอยู่ที่ `scripts/hydra/ideator.js` ในฟังก์ชั่น `notifyDiscord`:

```javascript
// Discord notification helper
async function notifyDiscord(proposal) {
  try {
    const isHigh = proposal.risk_level === 'high';
    
    // 🔕 NOISE REDUCTION: แจ้งเตือนเฉพาะตอนที่เป็น HIGH risk เท่านั้น
    // Safe/Medium จะทำงานกันเงียบๆ (Silent Auto-Deploy) และให้ Evolver ไปแจ้งทีเดียวตอนเสร็จ
    if (!isHigh) {
        console.log(\`[Ideator] Skipping Discord alert for \${proposal.risk_level} proposal (Silent Mode).\`);
        return;
    }
    
    // ... proceed with alert logic ...
```

## 4. Expected User Experience
- ปกติแล้ว User จะไม่เห็นการแจ้งวิเคราะห์ Root Cause ใดๆ ขึ้นเตือนให้รบกวนเลย
- เมื่อระบบหา Proposal และทดสอบการ Patch เข้า Deploy เสร็จสมบูรณ์ จะมีโผล่ข้อความมารายงานผลสั้นๆ แทน 
- ถ้ามีปัญหาระหว่างทำ (พัง, Code Error, Rollback) ค่อยมีเสียงร้องเตือน!
- หากมีเรื่องยากๆ รุนแรง (High Risk) จะมีแจ้งพร้อมคำอธิบายแบบเดิม

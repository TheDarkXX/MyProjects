# Bill Not Found & Cron Alert Hotfix

## Issue Details
1. **KrungSri Credit Bug**: The parser output `query = "KrungSri Credit"` exactly when parsed from quotes. However, the exact match failed if the casing didn't perfectly match what was in the database, and `.includes()` alone wasn't robust enough. Fixed the parsing by adding `.trim()` and making substring matching bidirectional: `b.name?.toLowerCase().includes(lower) || lower.includes(b.name?.toLowerCase())`.
2. **Recurring Bill Reset Bug**: `discord-bot/lib/brain.js` was using `PUT /cc/bills/:id` to mark a bill as paid. However, this is just a standard update route that doesn't trigger the reset logic for recurring bills (advancing cycle month and resetting `paid=0`). Changed it to use `PUT /cc/bills/:id/paid`.
3. **Daily Alert Clutter**: "Failed Cron Jobs" was wrongly displaying in the daily Command Center alert (`#bill-subscription`), making the personal financial view messy with system dev warnings. Removed this section entirely from `cron-alerts.js`.

## Code Modifications
- **discord-bot/index.js**:
  - `query = (args[0] || argsRaw.replace(/["']/g, '')).trim();`
  - Beefed up `b.name.toLowerCase().includes(lower) || lower.includes(b.name.toLowerCase())` in `findBillByName`.
- **discord-bot/lib/brain.js**:
  - `callBrainAPI('PUT', '/cc/bills/${id}/paid', ...)` instead of `/cc/bills/${id}`.
- **discord-bot/scripts/cron-alerts.js**:
  - Removed block `6. Failed Cron Jobs` and removed its length from `totalAlerts`.

## Verification
- `KrungSri Credit done` will now trigger the proper `pay-bill` regex, strictly match the exact DB item, and update the database via the `/paid` API so it properly sets recurring cycles.
- Tomorrow's Morning Alert will not show cron jobs.

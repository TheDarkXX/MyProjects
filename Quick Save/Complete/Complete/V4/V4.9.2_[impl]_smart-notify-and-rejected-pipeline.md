   - User สงสัยว่างานเหล่านี้คืองานของ Meta Agents (Ideator / Evolver) ไมใช่ของ Hydra เหรอ? และประการที่สองคือ ข้อเสนอ (Proposal) ไปหลงอยู่ที่ไหนเพราะไม่เห็นแสดงบนหน้าเว็บ (Hydra RPG/Awakened Protocol Dashboard) และประการที่สามคือ ทำไมจึง "fail ตลอด"?

   - **คำถาม 2:** ทำไมมันหายไปจากหน้าเว็บ? เพราะ `EVO-20260415-008` ละเมิดสิทธิพยายามแก้โค้ดตัวเองในโพลเดอร์ `scripts/hydra/ideator.js` ทันทีที่ `evolver.js` ตรวจเจอ มันจะปฏิเสธ (Safety Block) และปรับสเตตัสในฐานข้อมูลเป็น `rejected` แต่โค้ดในหน้า UI ดึงมาแค่ `{ key: 'finding' }, { key: 'proposed' }, { key: 'deployed' }, { key: 'rolled_back' }` ทำให้ Proposals ที่ถูก Rejected ล่องหนหายตัวไป
   - **คำถาม 3:** ทำไม Fail ตลอด? เพราะ AI พยายามร่นระยะเวลา โดยการเสนอปรับแก้ระบบตัวเอง (Ideator/Evolver) ซึ่งเซฟตี้ของเราออกแบบบล็อกไว้แต่แรกแล้ว ระบบจึงเตะออกอัตโนมัติ

   - **"web ui ปรับเพิ่มสว่น reject ด้วยดีไหม พร้อมเหตุผล เผื่อ ai บางตัว จะหยิบมา recycle idea"** -> เติมกระดานโชว์สิ่งที่ AI ปฏิเสธออกมาบน UI ให้เห็นภาพรวม

# 🛠️ โค้ดที่แกไขและการตัดสินใจ (Architectural Insights & Decisions)

## Phase 1: การตัดวงจรไล่งับหางตัวเอง (Safety Policy Rules Update)

เพื่อให้ `ideator.js` เลิกยุ่งเกี่ยวกับการแก้โค้ดของตัวเอง เราได้สอดแทรก **"กฎเหล็ก"** เข้าไปใน System Prompt

**1. แก้ไข `scripts/hydra/ideator.js`**
```diff
- "dependency_upgrade": อัปเแกรด model/library ให้นานขึ้น
+กฎเหล็ก: ห้ามเสนอแก้ไฟล์ระบบหลักในโฟลเดอร์ scripts/hydra/ เด็ดขาด (ห้ามยุ่งกับเครื่องรับรู้ของตัวเอง)
 Return เป็น JSON ตาม Format นี้เท่านั้น (ห้ามมีข้อความอื่นรวมอยู่ด้วย):
```

## Phase 2: การจัดการความน่ารำคาญของการแจ้งเตือน (Smart Notify Consolidation)

ระบบเก่า โชว์ ID (เช่น `EVO-20260415-008`), JSON Data บางส่วน และโค้ดแจ้งเตือนแบบดุดัน เรายุบมันให้เป็นภาษาคน (Colloquial Thai)

**1. ปรับปรุง `scripts/hydra/evolver.js`**
เราแก้ `notifyDiscord` ฟังก์ชันทั้งหมดใน Evolver
```javascript
// ก่อน:
await notifyDiscord(`🚨 **Safety Block** — \`${proposalData.id}\` แตะไฟล์ต้องห้าม: \`${violateFiles[0].file}\`\n✖ Auto-deploy ยกเลิก`);
// หลัง:
await notifyDiscord(`🚨 **ระบบเซฟตี้ทำงาน (Safety Block)**\nไม่อนุญาตให้อัปเดตโค้ดนี้ เพราะพยายามจะแก้ไฟล์ต้องห้าม (${violateFiles[0].file})`);

// ก่อน:
await notifyDiscord(`⚠️ **Patch Failed** — \`${proposalData.id}\` ${proposalData.title}\n↩️ Rollback สำเร็จ`);
// หลัง:
await notifyDiscord(`⚠️ **เขียนโค้ดไม่ผ่าน** — ${proposalData.title}\nไม่ต้องห่วง ผมกู้คืนระบบกลับให้เรียบร้อยแล้ว (Rollback)`);

// ก่อน:
await notifyDiscord(`✅ **Deployed** — \`${proposalData.id}\` ${proposalData.title}\nPM2 restart สำเร็จ (${mode === 'auto' ? 'auto' : 'manual'})`);
// หลัง:
await notifyDiscord(`✅ **อัปเดตระบบเรียบร้อย** — ${proposalData.title}\nรีสตาร์ทเซิฟเวอร์ให้แล้ว พร้อมใช้งานครับ 🚀`);
```

**2. ปรับปรุง `scripts/hydra/inspector.js` Prompt**
สั่งให้ Inspector คุยกับบอสด้วยภาษาบ้านๆ:
```javascript
const prompt = `คุณคือ Inspector Meta-Agent หน้าที่ของคุณคือการอ่านข้อมูล Technical Findings และสรุปรายงานส่งบอส
กฎการเขียนรีพอร์ต (สำคัญมาก ห้ามฝ่าฝืน):
- เขียนสั้นๆ กระชับที่สุด เน้นอ่านปุ๊บเข้าใจปั๊บ ว่าเกิดอะไรขึ้น และบอสต้องทำอะไรต่อไป
- ห้ามใส่ Code, JSON, หรือรหัส ID ใดๆ เช่น EVO-12345 หรือ Path ไฟล์ยาวๆ เด็ดขาด (พิมพ์มาแค่ชื่อพาร์ทที่เข้าใจง่ายๆ)
- เน้นภาษาพูด ภาษาบ้านๆ ที่มนุษย์ทั่วไปอ่านเข้าใจ
- มีแค่ Bullet points ง่ายๆ ไม่ต้องอารัมภบท

แยกเป็น 2 ส่วน:
1. 🛠️ จุดที่ต้องซ่อม: (ระบบมีปัญหาอะไร บอกสั้นๆ)
2. 🌱 จุดที่โตได้: (มีโอกาสอะไรให้ระบบเก่งขึ้น เอาแค่อันที่เวิร์คจริงๆ)`
```

## Phase 3: การโชว์ไอเดียที่ถูกตกไปในบอร์ด (Rejected Ideas Pipeline)

เนื่องจาก UI หน้าเก่ายกเลิกการเรนเดอร์ไอเท็มที่ได้คลาส `rejected` เราจึงเพิ่มคอลัมน์และดึง Reason ออกมาจาก API

**1. อัปเดต `routes/hydra.js` (`/api/hydra/evolution-pipeline`)**
ให้ดึงฟิลด์ `deploy_result` ซึ่งก็คือฟิลด์ที่เก็บ Reason ว่าทำไม Evolver หรือคนถึงกด Reject
```javascript
// ก่อน 
const all = db.prepare(`SELECT id, proposal_id, title, category, risk_level, status, proposed_changes, created_at, deployed_at, rolled_back_at FROM evolution_log ORDER BY created_at DESC LIMIT 100`).all();
// หลัง
const all = db.prepare(`SELECT id, proposal_id, title, category, risk_level, status, proposed_changes, created_at, deployed_at, rolled_back_at, deploy_result FROM evolution_log ORDER BY created_at DESC LIMIT 100`).all();
```

**2. อัปเดตฝั่ง UI: `brain-app-public/hydra-evo.js`**
ตั้งค่าแสดงคอลัมน์ Rejected ในอาร์เรย์ `cols` เป็นลำดับสุดท้าย และทำการ Parse `deploy_result` เพื่อนำคลาส `reasonHtml` พ่นออกมาใต้การ์ด
```javascript
        const cols = [
            { key: 'finding',     icon: '🔍', label: 'Findings',  approvable: false },
            { key: 'proposed',    icon: '💡', label: 'Proposed',   approvable: true  },
            { key: 'deployed',    icon: '🚀', label: 'Deployed',   approvable: false },
            { key: 'rolled_back', icon: '⚠️', label: 'Rolled Back',approvable: false },
            { key: 'rejected',    icon: '🗑️', label: 'Rejected',   approvable: false },
        ];
```
และการเขียน `reasonHtml` ใน loop ของการแสดงการ์ดแต่ละใบ:
```javascript
            items.forEach(item => {
                const riskColor = item.risk_level === 'high' ? '#fc2d79' : item.risk_level === 'medium' ? '#fd5514' : '#15be53';
                let reasonHtml = '';
                if (col.key === 'rejected' && item.deploy_result) {
                    try {
                        const dr = JSON.parse(item.deploy_result);
                        if (dr.reason) reasonHtml = `<div style="font-size:0.75rem; color:#fc2d79; margin-top:4px;">${escapeHtml(dr.reason)}</div>`;
                    } catch(e) {}
                }

                html += `<div class="aw-kanban-card${col.key === 'deployed' ? ' success' : ''}${col.key === 'rejected' ? ' rejected' : ''}" ${col.key === 'rejected' ? 'style="border-color: #fc2d79; opacity: 0.8;"' : ''}>
                    <div class="aw-card-title">${escapeHtml(item.title || '(Untitled)')}</div>
                    <div class="aw-card-meta" style="color:${riskColor}">Risk: ${item.risk_level || 'low'}</div>
                    ${reasonHtml}`;
```

# Credit Card "Trust Qwen" Architecture & Pipeline Optimization

## 1. Problems Identified (from V3.8.1 Triple-Guard)

| Problem | Root Cause | Impact |
|---------|-----------|--------|
| SCB timeout | 120s API timeout too short for CardX PDF | Statement not ingested |
| Nemotron dragging | `nemotron-nano-12b-v2-vl` returned 0 items 90% of the time | Wasted tokens, added latency |
| BBL HELD incorrectly | Math Mismatch: sum ฿1005 vs statement ฿149.48 (credit balance of ฿856 carried over) | Data blocked from DB |
| BBL case-sensitivity | Manual override inserted `'bbl'` instead of `'BBL'` | Row invisible in Command Center |
| Redundant cron job | `cron-cc-statements.js` duplicated work already done by `cron-gmail-expenses.js` | Wasted API calls daily |

## 2. Architecture Change: Trust Qwen

### Before (V3.8.1 Triple-Guard)
```
Qwen 72B ─┐
Gemini    ─┼── Promise.allSettled → buildConsensus() → Vote → Save
Nemotron  ─┘    (waited for slowest model, ~2-3 min)
```

### After (V4.4.0 Trust Qwen)
```
Qwen 72B (Primary)
    │
    ├── Smart Validation
    │   ├─ Math OK + Sanity OK → ✅ SAVE instantly (~15-30s)
    │   ├─ Math Mismatch → Call Qwen 32B backup
    │   │   ├─ Both agree → ✅ VERIFIED_BY_BACKUP (special case like BBL credit)
    │   │   └─ Disagree → ⏸️ HELD for manual review
    │   └─ Sanity fail → Call Qwen 32B backup
    └── Primary failed → Qwen 32B takes over entirely
```

### Smart Validation Details
- **Refund-aware math**: Subtracts refund items before comparing sum vs statement_total
- **Date sanity**: All dates within ±2 years of current year
- **Price sanity**: All prices > 0 and < ฿10M
- **Tolerance**: max(2% of total, ฿50)

### Model Config
| Role | Model | Cost | Provider |
|------|-------|------|----------|
| Primary | `qwen/qwen2.5-vl-72b-instruct` | $0.40/$0.80 | OpenRouter |
| Backup | `qwen/qwen2.5-vl-32b-instruct` | $0.20/$0.60 | OpenRouter |

**Why same family?** Same prompt format, same extraction patterns, no need to tune separately. "พี่น้อง Qwen คุมกันเอง"

## 3. Cron Optimization

### Disabled: `cron-cc-statements.js`
- **Was**: `0 12 15-28 * *` (noon daily, 15th-28th of month)
- **Action**: Removed from crontab entirely

### Kept: `cron-gmail-expenses.js` (piggyback trigger)
- **Schedule**: `30 22 * * *` (22:30 daily)
- Already calls `detectCCBank()` → `processCreditCardStatement()` when CC emails found
- Has auto-stop: checks `created_at` in current month, skips if 4 banks already imported
- **Result**: No wasted API calls on non-statement days

### Auto-Stop Verification
```

✅ นำเข้าบิลครบ 4 ใบแล้ว สำหรับเดือนนี้ ข้ามการทำงานรอบนี้ เพื่อประหยัด API
```

## 4. PDF Archive by Statement Month

### Before
```
cc-statements/
├── bbl_2026-04-10_xxx.pdf      ← flat, no organization
├── cardx_2026-04-10_xxx.pdf
├── kasikorn_2026-04-10_xxx.pdf
└── krungsri_2026-04-10_xxx.pdf
```

### After
```
cc-statements/
├── 2026-03/                    ← organized by billing cycle
│   ├── bbl_2026-04-10_xxx.pdf
│   ├── cardx_2026-04-10_xxx.pdf
│   ├── kasikorn_2026-04-10_xxx.pdf
│   └── krungsri_2026-04-10_xxx.pdf
├── 2026-04/                    ← auto-created next month
└── ...
```

**Logic**: `archivePdf()` saves flat first → after AI extracts `statement_month` → `moveToStatementFolder()` moves PDF into the correct subfolder.

### moveToStatementFolder
```js
function moveToStatementFolder(archivedPath, statementMonth) {
  const monthDir = resolve(STATEMENTS_DIR, statementMonth);
  mkdirSync(monthDir, { recursive: true });
  writeFileSync(resolve(monthDir, filename), readFileSync(archivedPath));
  unlinkSync(archivedPath);
}
```

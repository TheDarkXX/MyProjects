# 🌙 Filter Dev Tasks from AI Briefs

## 🐛 The Problem
TaTa (the AI assistant) was pulling in backend developer tasks (e.g., "Review GitHub repo: milla-jovovich/mempalace", "Study rtk-ai/rtk", "Study Kanban board") into the Morning Brief and Daily Review summaries. This clutter made the summary less useful for personal and business tracking.

## 🛠️ The Fix
Added explicit filtering instructions (กฎเหล็ก) into the system prompts that generate these briefs.

1. **`discord-bot/lib/morning-brief.js`**:
   - Added Rule 4 to `generateMorningBrief`: `ห้ามนำงานฝั่ง dev/technical (เช่น ศึกษาเรื่องใหม่ๆ, GitHub, Kanban, OpenClaw, AI proxy, ฯลฯ) มารายงาน ให้คัดออกทั้งหมด ห้ามรายงานเด็ดขาด`
   - Added Rule 2 to `generateDailyReview`: `ห้ามนำงานฝั่ง dev/technical (เช่น ศึกษาเรื่องต่างๆ, GitHub, Kanban, OpenClaw, AI proxy, ฯลฯ) มารายงาน ให้คัดออกทั้งหมด ห้ามรายงานเด็ดขาด`

2. **`skills/templates/daily-review.md`**:
   - Added a corresponding rule to the markdown template instructions explicitly forbidding technical and developer tasks from being reported.

## ✅ Verification
All future executions of `!brief` or `!review` (and the automated cron jobs) will now instruct the LLM to completely drop these tasks from its generated texts.

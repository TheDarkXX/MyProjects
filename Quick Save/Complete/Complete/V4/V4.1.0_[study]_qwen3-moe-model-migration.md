# 🧠 AI Model Migration Review: ChatGPT Pro → Qwen3 MoE
*Target: Transitioning the "Pinky" router and OpenClaw gateways from a dying GPT Pro subscription to cost-effective OpenRouter payloads.*

## 1. 🔍 Initial System Analysis
The first step was auditing the current architecture to determine which components still depend on the expiring ChatGPT Pro (`gpt-5.4/5.1`) integration and which systems are securely offloaded.

**Critical Findings:**
- **Discord Bot / UI / Doctor Skill / Combat:** Completely reliant on `GEMINI_API_KEY` (Google AI Studio) using `gemini-2.5-flash`. These services remain **100% FREE** and are untouched by the ChatGPT Pro expiration.
- **Credit Card Agent:** Currently uses `qwen2.5-vl-72b-instruct` alongside `nemotron-nano-12b-v2-vl` as a consensus via OpenRouter. Very high accuracy but somewhat expensive at $0.80/$0.80.
- **OpenClaw Gateway (Pinky + Validator):** Telegram AI responses, Morning Briefs, follow-ups, and verifiers heavily rely on the ChatGPT Pro subscription. This requires an immediate replacement.

## 2. 🆚 Model Comparison & Selection
To replace `gpt-5.4`, we needed a model capable of high-logic morning synthesis, direct instruction following, and fast conversational response, without blowing up the budget.

| Model | Architecture | Cost (In/Out per 1M) | Vision? | Reasoning Capability | Verdict |
|-------|--------------|----------------------|---------|----------------------|---------|
| **qwen3.5-9b** | Hybrid 9B | $0.05 / $0.15 | ✅ Yes | ⭐⭐⭐ | Too shallow for Morning Brief synthesis or complex Validator checks. |
| **qwen3-vl-32b-instruct** | Dense 32B | $0.104 / $0.416 | ✅ Yes | ⭐⭐⭐⭐ | Great fallback, but somewhat pricey output for a heavy chat system. |
| **qwen3-235b-a22b-2507 (Instruct)** | MoE (22B active) | **$0.071 / $0.10** | ❌ No | ⭐⭐⭐⭐⭐ | **WINNER**. Massive 235B knowledge base at a fraction of the cost. Extremely deep reasoning compared to 9B. |

### Why Qwen3 235B A22B "Instruct" instead of "Thinking"?
The *Instruct* variant does not output verbose `<think>` internal reasoning tags, saving token bloat, decreasing message latency on Telegram, and ensuring compatibility without having to redesign OpenClaw Gateway's parser.

### Why not swap CC Agent to Qwen3 32B?
Initially considered migrating the CC agent over to `qwen3-vl-32b-instruct` for cost-saving, but was **strictly rejected.** The OCR extraction precision required for Thai Credit Card statements allows exactly 0% error rate. The system must retain `qwen2.5-vl-72b` to guarantee absolute Perfect Math Override matching. Cost is secondary to flawless execution here.

## 3. ⚙️ Implementation & Code Integration
We executed the pre-integration configuration on the codebase directly, enabling us to flip the switch on the VPS Gateway when ChatGPT Pro officially lapses.

**Files modified:**
1. **`brain-app-public/openclaw.js`**: Updated `MODEL_PRICING` with `qwen/qwen3-235b-a22b-2507` specs for the analytics dashboard visualization.
2. **`brain-app-public/ai-manager.js`**:
   - Added pricing object in `AIM_MODEL_PRICING`.
   - Registered the model in `CANONICAL_MODELS` array as an `openrouter` model to render dynamically.
3. **`docs/agent_model_config.md`**: Restructured the documentation table with `qwen3-235b-a22b-2507` and recorded the new Default Model Priority list.

*Note: Changes made to `credit-card-agent.js` to lower its specs were fully reversed off of the commit timeline before deployment.*

## 4. 🔀 Next Steps (Manual Flip)
Once ChatGPT Pro ends, execute the final cutover route:
1. Open the OpenClaw Command Center Dashboard (`ai-manager`).
2. Search for `Qwen3 235B A22B Instruct`.
3. Click **"Set Default Base"**.
4. Restart the PM2 Gateway instances so Telegram/Cron routes traffic directly to Qwen.

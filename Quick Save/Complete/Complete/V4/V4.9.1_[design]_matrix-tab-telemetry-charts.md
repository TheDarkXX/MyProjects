# Matrix Tab & Telemetry Redesign

## Why this was built
The user pointed out that the "Agent Roster" column in the Awakened Protocol dashboard was too static and wasted valuable Ultra-Wide screen real estate. The goal was to make the Awakened Protocol exclusively for **dynamic, real-time data**, while creating a dedicated space for Agent directory and analytics.

## UI/UX Architectural Decisions

### 1. The Awakened Protocol Dashboard
- **Column 1 (30%):** Evolution Pipeline (Kanban)
- **Column 2 (45%):** AI Social Feed (Wider, easier to read logs)
- **Column 3 (25%) [NEW]:** LIVE TELEMETRY & INTEL
  - **Token Telemetry:** Displays cost today in USD directly pulled from `ai_usage_sessions`.
  - **Watchdog Radar:** Red/Green status box reporting critical watchdog failures.
  - **Knowledge Crystals:** Reads from `agent_learnings` to showcase what agents have recently successfully studied/crystallized.
  - **Active Experiments:** Displays testing metrics from `evolution_experiments`.

### 2. The Meta-Agent Matrix Tab
A brand new 3rd tab added to the `.awakened-tab-bar`.
- **Left Column (40%):** The redesigned Agent Roster grid (`repeat(2, 1fr)`).
- **Right Column (60%):** Intelligence Charts
  - Introduced two `Chart.js` canvas instances.
  - **API Token Resource Burn:** Bar Chart displaying top 10 costliest agents over 7 days.
  - **Core System Health Score:** Line Chart mapping the system stability trend (0-100) from `growth_snapshots` over the last 7 days.

## API Additions (`routes/hydra.js`)
- `GET /telemetry`: Pulls immediate daily sums for Token costs, open experiments, recent lessons, and watchdog errors.
- `GET /matrix-stats`: Pulls grouped intelligence data (7-day historical) optimized for Chart.js rendering (labels vs datasets format).

## Deployment status
Deployed successfully. `node bump-cache.js` executed to refresh cache versions for `index.html` and `hydra-evo.js`.

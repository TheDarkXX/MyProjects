# 🛠 V4.8.2 — HYDRA Core Auto Approve & Notification UX

## 1. Context (ที่มา)
User สอบถามเรื่องช่องทางการส่งแจ้งเตือนของ HYDRA Core (ว่าส่งผ่าน hydra-core ช่องทางเดียวใช่ไหม) 
ทำให้พบว่าจริงๆ แล้ว `ideator.js` สมมุติฐานเก่าส่งหา `watchdog` อยู่
เมื่อตรวจเรื่องการ Approve ก็พบว่าการที่มี requirement ว่า `medium` ต้องพิมพ์ \`!evo approve\` ทำให้เป็นการเพิ่มภาระให้ User มากเกินไป 

เป้าหมายคือต้องการ **"โหลดงาน User ให้น้อยที่สุด และลด Technical Jargon ลงให้อ่านง่าย"** ในระดับ Full Auto

## 2. สิ่งที่ตัดสินใจเปลี่ยนแปลง (Decisions & Architecture)

### 2.1 ลดคอขวด Manual Approval (Full Auto สำหรับ Medium Risk)
**เดิม:** `medium` risk proposal จะถูก stage ค้างไว้จนกว่า user จะพิมพ์ \`!evo approve <id>\`
**ใหม่:** ให้ `medium` ควบรวมกับ flow ของ `safe` คือ **"Auto-apply ลง VPS ทันที"** โดยอาศัย Safety Gates ของ Evolver (Whitelist, Circuit Breakers, Dry-runs) ช่วยคัดกรองอยู่แล้ว
**ผลลัพท์:** ตอนนี้ AI จะรันทุกอย่าง ยกเว้นว่าเป็น `high` risk ที่ต้องพึ่งพา decision เชิง architecture จาก User เท่านั้น 

### 2.2 เปลี่ยนการแจ้งเตือนและภาษาของ Notification (No-coder Friendly)
**เดิม:** ส่งไปห้อง `#watchdog` และใช้ภาษาที่บอก Root Cause เป็นโค้ดยาวยืด พร้อม Proposed Changes ระดับระดับ diff
**ใหม่:** 
- เปลี่ยน Channel ให้ยิงตรงไปที่กลุ่ม `'hydra-core'` เป็น Single Source of Truth
- ปรับ Formatting ใหม่ทั้งหมดให้สั้น กระชับ เป็นภาษาที่คนนอกอ่านแล้วเข้าใจทันที พร้อมย้ำว่ามันจะจัดการตัวเองให้แล้ว (✅ Deploy อัตโนมัติ — ไม่ต้องทำอะไร) หรือหากเป็น `high` risk ก็จะเตือนชัดๆ ให้ทำเอง (⚠️ ต้องดำเนินการเอง)

## 3. Code References & implementation

### 3.1 แก้ไข \`discord-bot/scripts/hydra/ideator.js\`

เปลี่ยนจากการยิงแจ้งเตือนที่มี technical jaggon ยืดเยื้อให้กระชับขึ้นและ redirect ไปยัง hydra-core

```javascript
async function notifyDiscord(proposal) {
  try {
    const isHigh = proposal.risk_level === 'high';
    const icon = isHigh ? '🔴' : proposal.risk_level === 'medium' ? '🟡' : '🟢';
    const actionLine = isHigh
      ? \`\\n⚠️ **ต้องดำเนินการเอง** — HYDRA ไม่แตะโค้ดนี้\\nพิมพ์ \\\`!evo reject \${proposal.id}\\\` ถ้าไม่ต้องการ\`
      : \`\\n✅ **Deploy อัตโนมัติ** — ไม่ต้องทำอะไร\`;

    const content = \`🧬 **HYDRA ตรวจพบสิ่งที่ควรปรับปรุง**

**\${proposal.title}**
\${icon} \${proposal.risk_level === 'high' ? 'ความเสี่ยงสูง' : proposal.risk_level === 'medium' ? 'ความเสี่ยงกลาง' : 'ปลอดภัย'} • ID: \\\`\${proposal.id}\\\`

\${proposal.description || proposal.root_cause}
\${actionLine}\`;

    const destinations = await getNotifyDestinations('hydra-core', '1493640968736346152');
    await sendToAllDestinations(destinations, content);
  } catch (error) { ... }
}
```

### 3.2 แก้ไข \`scripts/hydra/evolver.js\`

ปรับ logic mode auto ให้ดึง `medium` นำมา deploy เช่นเดียวกับ `safe`

```javascript
  if (mode === 'auto') {
      // ดึงทั้ง safe และ medium ควบรวมกันนำมา auto-patch ได้เลย
      proposals = db.prepare(\`SELECT * FROM evolution_log WHERE status = 'proposed' AND risk_level IN ('safe', 'medium')\`).all();
  }

  // แจ้งเตือนแค่ High Risk ที่ต้องการให้ User ทราบว่าต้องแก้เอง
  if (mode === 'auto') {
      const highProposals = db.prepare(\`SELECT * FROM evolution_log WHERE status = 'proposed' AND risk_level = 'high'\`).all();
      for (const h of highProposals) {
          ...
          await notifyDiscord(\`🔴 **HYDRA พบปัญหาที่ต้องดูแลเอง**\\n\\n**\${propData.title}**\\nID: \\\`\${propData.id}\\\`\\n\\n\${propData.description || propData.root_cause}\\n\\n⚠️ เรื่องนี้ซับซ้อนเกินไปสำหรับ AI — ต้องการการตัดสินใจจากคุณ\`);
          db.prepare(\`UPDATE evolution_log SET status = 'awaiting_human' WHERE id = ?\`).run(h.id);
      }
  }
```

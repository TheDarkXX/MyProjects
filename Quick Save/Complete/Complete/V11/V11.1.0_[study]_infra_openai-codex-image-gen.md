# 🎨 GPT-Image-2 via Codex OAuth — Parameter & Limits Study

## 📌 Context (Compiled Truth)
We successfully integrated Codex OAuth for OpenAI text models (GPT-5.4/5.5) in `V11.0.0`. The user wanted to know if this Codex OAuth flow also supports image generation (specifically `gpt-image-2` or DALL-E 3) without incurring API costs, allowing the creation of complex Thai infographics.

**Key Discoveries & Architecture:**
1. **Endpoint Identity:** Image generation via Codex OAuth does NOT use the standard `/v1/images/generations` endpoint. Instead, it leverages the same Codex text streaming endpoint (`chatgpt.com/backend-api/codex/responses`).
2. **Tool Injection:** To trigger image generation, we must inject a specific tool object into the text model's request payload: `tools: [{ type: "image_generation", size: "1024x1024" }]`.
3. **Response Handling:** The API responds via SSE (Server-Sent Events). The generated image arrives as base64 encoded data inside the `response.image_generation_call.partial_image` (as `partial_image_b64`) and `response.output_item.done` events.

**Parameter Specifications (Live Tested):**
- **Size / Aspect Ratio:** Width and height MUST both be divisible by 16. Minimum pixel budget is ~768x768. Maximum longest edge is 3840px. (e.g., 1024x1024, 1024x1536, 1920x1088, 3840x2160 are valid).
- **Format:** Supports `png` (default, ~800KB), `webp` (~770KB), and `jpeg` (highly compressed, ~65KB).
- **Quality:** Accepts `low`, `medium`, `high`, or `auto`.
- **Background:** `transparent` is NOT supported (throws an error). Only `opaque` or `auto`.
- **Count:** `n` (multiple images per request) is NOT supported.
- **Limits:** Governed by ChatGPT Plus rolling window (approx. 40-50 images per 3 hours). No strict daily hard cap.

This proves we can completely bypass OpenAI API image generation costs for OpenClaw agents by wrapping this tool-calling mechanism in our `ai-gateway.js`.

# GPT-Image-2 via Codex OAuth — Parameter & Limits Study

## ✅ How It Works

Generate images through the **Codex Responses API** by adding `image_generation` as a tool alongside a text model (GPT-5.4):

```json
{
  "model": "gpt-5.4",
  "tools": [{ "type": "image_generation", "size": "1024x1024" }],
  "input": [{ "role": "user", "content": "Draw a cat" }]
}
```

The model internally calls `gpt-image-2` and returns a base64-encoded image in the SSE stream.

---

## 📐 Size / Aspect Ratio Support

### Constraint Rules
- **Both width and height must be divisible by 16**
- **Minimum pixel budget**: ~768×768 (512×512 rejected)  
- **Maximum edge length**: 3840px

### Tested Results

| Size | Aspect Ratio | Status | Time | File Size |
|------|-------------|--------|------|-----------|
| `1024x1024` | 1:1 (Square) | ✅ | 16.3s | 767 KB |
| `1024x1536` | 2:3 (Portrait) | ✅ | 19.6s | 1,000 KB |
| `1536x1024` | 3:2 (Landscape) | ✅ | 14.4s | 1,033 KB |
| `1280x768` | 5:3 | ✅ | 13.9s | 723 KB |
| `768x1280` | 3:5 (Portrait) | ✅ | — | — |
| `1792x1024` | ~16:9 | ✅ | 17.8s | 1,163 KB |
| `1920x1088` | ~16:9 | ✅ | 16.8s | 1,264 KB |
| `2048x2048` | 1:1 (2K) | ✅ | 23.6s | 2,167 KB |
| `3840x2160` | 16:9 (4K) | ✅ | 30.4s | 3,307 KB |
| `auto` | AI picks | ✅ | ~17s | ~814 KB |

### ❌ Rejected Sizes

| Size | Error |
|------|-------|
| `1920x1080` | Width and height must both be divisible by 16 (1080 ÷ 16 = 67.5) |
| `512x512` | Below the current minimum pixel budget |
| `256x256` | Below the current minimum pixel budget |
| `4096x4096` | Longest edge must be ≤ 3840 |

### Size Formula
```
Valid size = WxH where:
  W % 16 == 0
  H % 16 == 0
  W * H >= ~589,824 (768×768)
  max(W, H) <= 3840
```

## 🎨 Quality Settings

| Quality | Tested | Notes |
|---------|--------|-------|
| `auto` (default) | ✅ | GPT decides, typically maps to `low` for simple prompts |
| `low` | ✅ | Fastest, still good quality |
| `medium` | ✅ | — |
| `high` | ✅ | Consumes more tokens |

> [!NOTE]
> Quality parameter is accepted but the API reports `quality: "low"` in the result for simple prompts even when `high` is requested. GPT-5.4 appears to auto-select quality based on prompt complexity.

## 📦 Output Format

| Format | Status | File Size (same prompt) |
|--------|--------|------------------------|
| `png` | ✅ | 814 KB |
| `jpeg` | ✅ | **65 KB** (12x smaller!) |
| `webp` | ✅ | 770 KB |

> [!TIP]
> Use `jpeg` for massive file size reduction. Use `png` for quality/transparency. Use `webp` for good balance.

## 🖼️ Background

| Background | Status | Notes |
|------------|--------|-------|
| `auto` (default) | ✅ | Typically `opaque` |
| `opaque` | ✅ | Always opaque background |
| `transparent` | ❌ | **Not supported for gpt-image-2** |

> [!IMPORTANT]
> Transparent background is not supported via Codex OAuth / gpt-image-2. Use `gpt-image-1.5` for transparency (requires API key path).

## 🔢 Multi-Image (n parameter)

| Parameter | Status | Notes |
|-----------|--------|-------|

> [!NOTE]
> Multi-image generation per request is **not supported** via the Codex Responses tool. Generate one image per request.

## 🚦 Rate Limits (ChatGPT Plus via Codex OAuth)

> [!IMPORTANT]
> These limits are from ChatGPT Plus subscription, NOT the OpenAI API.

| Metric | Limit |
|--------|-------|
| **Per 3-hour window** | ~40–50 images |
| **Per 24 hours (theoretical)** | ~80–150 images |
| **Daily/Weekly hard cap** | **None** — rolling window system |
| **Throttle behavior** | Temporarily throttled or switched to smaller model |
| **Free tier** | ~2–3 images per 24 hours |

### Factors affecting limits:
- **System demand**: Peak hours = lower throughput
- **Prompt complexity**: Complex prompts + editing consume more quota
- **Mixed usage**: Text conversations share the same message pool

## 🛠️ Full Tool Parameters Reference

```json
{
  "type": "image_generation",
  "size": "1024x1024",          // WxH, div by 16, min ~768², max edge 3840
  "quality": "auto",            // auto | low | medium | high
  "output_format": "png",       // png | jpeg | webp
  "background": "auto",         // auto | opaque (transparent NOT supported)
  "output_compression": 100,    // 0-100, compression level
  "model": "gpt-image-2",       // auto-selected, don't override
  "moderation": "auto"          // content moderation
}
```

### ❌ Parameters NOT supported via Codex Responses:
- `n` (multi-image count)
- `background: "transparent"`
- Sizes below ~768×768
- Sizes with edge > 3840

# GBRAIN Deterministic Knowledge Graph Implementation

## 📌 Context (Compiled Truth)
The OpenClaw workspace memory was a flat-file database. While historically structured, finding relationships between disconnected files required sending large directory trees or file contents into the LLM context window—causing severe token bloat and context drops during deep ideation.

To solve this, we implemented the second pillar of the GBRAIN architecture: **Zero-Token Auto-Links**. By converting the flat `Quick Save` folder into a bidirectional knowledge graph, the AI can simply follow explicit markdown links to navigate the codebase without needing a search agent to grep the entire archive.

### Deep Study: GBRAIN vs Obsidian Backlinks — Merged Strategy
The original GBRAIN Benchmark (Recall@5: 83%→95%) uses 6 layers of matching, including typed links and backlink-boosted ranking. We merged the best parts of GBRAIN with the best parts of Obsidian to create a hybrid architecture for OpenClaw:

| Feature | GBRAIN | Obsidian | Our System (OpenClaw) |
|---|---|---|---|
| **Link Syntax** | `[Title](relative/path.md)` | `[[filename]]` wikilink | Markdown links `[](path)` (for compatibility with VS Code & grep) |
| **Typed Links** | ✅ `depends_on`, `enables` | ❌ ไม่มี native | ✅ Added to `/save` skill (`depends_on`, `enables`, `related_to`) |
| **Bidirectional** | ✅ Auto-generate | ✅ Built-in | ✅ Implemented via Script (A→B and B→A) |
| **Aliases** | ✅ YAML `aliases: [...]` | ✅ YAML `aliases: [...]` | ✅ Added frontmatter requirement for exact keyword matching |
| **MOC (Map of Content)**| ✅ `RESOLVER.md` | ✅ Hub note pattern | ✅ Created 4 Hubs (hydra, ui, infra, payment) to reduce token reads |

**New Standard Format:**
Every file MUST have `aliases: [...]` in YAML frontmatter. Every file MUST end with the `<!-- GBRAIN_BACKLINKS_START -->` block containing bidirectional `- **YYYY-MM-DD HH:MM** | [Title](path) -- reason` strings.

## 🏛️ Implementation Details

### 1. `/save` Skill Modernization
The primary choke-point for data entry into the Second Brain is the `/save` skill. We updated `docs/skills/save.md` to formally adopt the GBRAIN structure:
- **Bidirectional Mandate:** Every new file must link backward to older files, and MUST edit the older files to link forward to the new file.
- **Timestamp Accuracy:** Implemented the `- **YYYY-MM-DD HH:MM** | [page](path) -- context` string layout.
- **Aliases:** Mandated `aliases: [keyword]` arrays in the YAML frontmatter of every file to enable fast lookup.

### 2. Map of Content (MOC) Hubs
To prevent the knowledge graph from becoming a scattered web, we built four centralized nodes (MOCs) in `Quick Save/Complete/`:
1. `MOC_hydra.md` (Evolution, safety gates, memory)
2. `MOC_ui.md` (Dashboard, layout, Bill Manager)
3. `MOC_infra.md` (Gateway, PM2, telemetry)
4. `MOC_payment.md` (Credit cards, deduplication)

These files serve as high-level entry points for future agents.

### 3. Historical Backfill Script (`backlink-generator.js`)
To link the existing 175+ legacy markdown files, we authored a purely deterministic Node.js script.
- **Logic:** Reads all Markdown files. Parses the YAML frontmatter. Matches files if they share a `component` keyword or have overlapping `tags`.
- **Injection:** Automatically appends the `## 🔗 GBRAIN Backlinks` block.
- **Noise Control:** Capped at 8 relationships per file to prevent over-linking.
- **Fixing Aliases:** The script automatically parses the `component` keyword out of the `V{x.y.z}_[type]_component_desc.md` filename and injects it into the YAML `aliases: []` array.

### 4. Graph Traversal Script (`graph-search.js`)
To fully utilize the bidirectional graph without LLM hallucinations or bloated RAG greps, we created a native traversal script: `node scripts/graph-search.js <keyword>`.
- **Functionality:** This script scans `Quick Save/Complete`, `Quick Save/Active`, `docs/`, `agents/`, `brain/`, and `self-improving/`. It matches against filenames, paths, and YAML `aliases`.
- **Zero-Waste Protocol:** The script automatically surfaces the Map of Content (MOC) if it exists, along with the top related primary nodes, extracting their summaries and explicit `1st-Degree Connections`. This gives the AI exact file paths to open without blind guessing.
- **Agent Adoption:** Updated `self-improving/memory.md` and `agents/dev-agent/AGENT.md` with an "Iron Rule" forbidding the initial use of broad `grep` commands for knowledge retrieval, mandating `graph-search.js` instead.
- **Recheck Bugfix:** Fixed a path resolution bug where agent definition files (`AGENT.md`) were missed because the script only evaluated the file basename instead of the full relative path containing the agent's name.

### 5. Systemic Guardrails (Mechanical Override)
During the final `/recheck`, we diagnosed a persistent failure where the AI repeatedly violated the "Conversation Context Rule" by summarizing artifacts instead of pasting their raw contents. This occurs because LLMs naturally summarize when told to "save".
To defeat this instinct, we implemented a mechanical override:
- **Hardcoded Markdown Template:** Updated `docs/skills/save.md` to explicitly embed the structural template:
  `## 📦 RAW ARTIFACT BACKUP (Iron Rule)`
  `(PASTE 100% OF THE ARTIFACT TEXT HERE. DO NOT SUMMARIZE. If > 1000 lines, use <details> tags. Do NOT abbreviate.)`
- **Cognitive Interception Checklist:** Updated `AGENTS.md` and the `/save` skill pre-flight checklist to ask confrontational questions: "Did I slip into my summarizing habit and delete code snippets?" This forces the LLM to second-guess its compression instinct.

## 🔬 Timeline & Debugging Log
- **[Initial Pass]:** Wrote script. Failed execution because `scripts/` folder uses ESM (`type: "module"`). Rewrote script to standard ES6 imports.
- **[Injection Run 1]:** Success. Updated 103 files.
- **[Full Quality Audit / Recheck]:** Discovered 5 critical bugs:
  1. Ghost file overlap (`V7.9.0` existed in both Active and Complete). Deleted the Active version.
  2. `MOC_payment` possessed `2026-xx-xx` placeholder dates. Corrected dates or removed them.
  3. `MOC_payment` referenced files that had been archived into `V3/`. Fixed the relative paths.
  4. Alias extraction flaw: The script extracted full filename segments (e.g., `unified-ai-gateway-migration.md`) instead of meaningful keywords.
- **[Cleanup Script]:** Wrote a secondary one-time use script (`fix-aliases.cjs`) using regex capture groups to strip the `.md` strings and isolate the true component keyword. Applied cleanup to 74 affected files.
- **[Mechanical Override]:** Embedded strict Markdown template in `/save` skill and modified pre-flight checklist to break LLM summarization habits.
- **[Final Run]:** Reran the generator and verified strict system integrity. Emitted skill log to `self-improving/skill-emit.jsonl`.

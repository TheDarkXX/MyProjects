# Study: GBRAIN Architecture for OpenClaw
> **Source:** `garrytan/gbrain` (MIT)

## 📌 Context
The user requested an analysis of Garry Tan's GBRAIN architecture—a memory and orchestration layer for OpenClaw/Hermes agents. Following the `/dev` skill review, the proposal was **APPROVED** for concept adaptation. This file serves as the full deep-study to map GBRAIN's concepts onto our existing ecosystem.

## 🧠 3 Core Pillars of GBRAIN

### 1. The Knowledge Model (Compiled Truth + Timeline)
- **Problem:** AI agents lose context because information is scattered or overwritten destructively.
- **GBRAIN Solution:** Every markdown page has two parts.
  - **Top (Compiled Truth):** The current, best unified understanding.
  - **Bottom (Timeline):** An append-only log of events/evidence.
- **Why it matters:** The agent reads the top to "know" things instantly, and reads the bottom to "prove" things historically.

### 2. Auto-Link Knowledge Graph (Zero LLM)
- **Problem:** Building knowledge graphs using LLMs burns tokens and is slow.
- **GBRAIN Solution:** Uses deterministic regex/pattern matching to extract entities and insert typed links (`works_at`, `invested_in`, `related_to`) *during the file write hook*. 
- **Why it matters:** It builds a relational database inside Markdown without making API calls to an LLM. This enables graph traversal queries ("Who is connected to X via Y?").

### 3. Minions Orchestrator (Deterministic Jobs)
- **Problem:** Orchestrating complex background work via LLM sub-agents causes timeouts, context drops, and high API costs (Sub-agent storm / Gateway wall).
- **GBRAIN Solution:** A `pglite` (Database) active job queue. 
  - **Deterministic Routing:** If a task can be done via code (e.g., fetch API, parse JSON), it runs on a Minion (0 tokens, milliseconds).
  - **Judgment Routing:** Only decisions/creativity go to the LLM.
- **Why it matters:** Resilient against server restarts. 100% execution guarantee.

## 🛠️ Actionable Adaptation Plan for Our Agents

### Phase 1: Memory Restructuring (Second Brain)
- Apply the **Compiled Truth + Timeline** pattern to our `memory/` and `docs/skills/` directories.
- Ensure that `corrections.md` or `memory.md` follow a strict Top/Bottom structure.

### Phase 2: Hydra Engine Optimization
- Replace token-burning sub-agent calls in the Hydra pipeline (e.g., repository fetching, lint checking) with a "Minion-like" deterministic job queue.
- Limit the Evolver/Ideator to purely *Judgment* tasks.

### Phase 3: Hybrid Search Foundation
- Investigate adding a simple Vector + Keyword matching script inside the `dev-agent` tools to mock the GBRAIN Hybrid Search before doing any heavy DB migrations.

# 🧠 `/evo` — Hydra Proposal Review

### Proposal: `EVO-20260421-955` — Add Pre-Deploy Health Check Before Auto-Commit

**Core Idea:** `evolver.js` currently has `DEPLOY_HEALTH_CHECK_THRESHOLD` and `DEPLOY_HEALTH_CHECK_TIMEOUT_MS` in CONFIG, but they're dead config — never actually used. The proposal is to implement a real `deployWithHealthCheck()` wrapper: snapshot error count before deploy → deploy → wait → snapshot after → if delta > threshold, rollback immediately.

## Compiled Truth (Implementation Details)
- **`countCurrentErrors()`**: Added to `evolver.js`. Pings the PM2 log and minion queues for all modified `.js` files.
- **`deployWithHealthCheck()`**: Acts as a safety wrapper. Takes the synchronous pre-deploy snapshot, commits/reloads via pm2, waits `CONFIG.DEPLOY_HEALTH_CHECK_TIMEOUT_MS` (30s), and takes the post-deploy snapshot. If delta exceeds `CONFIG.DEPLOY_HEALTH_CHECK_THRESHOLD` (5 errors), it forcibly `git reset --hard` and routes a warning to Discord.
- **Unicode Safety Note**: Care must be taken not to double-escape unicode literals (`\u`) in `notifyDiscord` templates within `evolver.js`.

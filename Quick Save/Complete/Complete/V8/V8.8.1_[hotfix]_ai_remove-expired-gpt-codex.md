# Remove Expired GPT Codex from Provider Chains

## 🎯 The Problem
The legacy `openai-codex` (gpt-5.1, gpt-5.4) service has expired and is no longer reachable. 
However, several cron jobs (`cron-news`, `cron-gmail-brief`, `cron-gmail-expenses`) had these models hardcoded at the top of their provider arrays in `health-news.js` and `gmail-agent.js`.

Because the models were still in the list, the system would attempt to query them, wait for a timeout (~20 seconds per call), and only then fall back to Kie / Gemini. This resulted in unnecessary delays (~40 seconds total) for every execution.

## 🛠️ The Solution (Priority 1)
Removed the `oc-gpt-5-4` and `oc-gpt-5-1` objects from the provider arrays in:
1. `discord-bot/lib/health-news.js`
2. `discord-bot/lib/gmail-agent.js`

This eliminates the timeout wait, allowing the crons to immediately fallback to the working Kie / Gemini models and restoring their original execution speed.

## 🔍 Note on "Dead Code"
A comprehensive audit revealed that 35 other references to `gpt-5.1` and `gpt-5.4` exist throughout the codebase (e.g., in `channels.js`, `ai.js`, `command-center.js`), but they are practically **dead code**. 
- Discord bots (Tata, Lily, Pinky) are all configured with `useOpenClaw: false` or lack the override, meaning they natively fall back to `gemini-2.5-flash`.
- Database defaults are merely display values.

## 🧹 Priority 2: Dead Code Cleanup
Subsequently cleaned up the identified dead code across 6 files to prevent future confusion:
- **`discord-bot/config/channels.js`**: Removed `openClawModel: 'gpt-5.1'` overrides and the unused `AGENT_MODELS` mapping.
- **`discord-bot/lib/ai.js`**: Removed OpenClaw mapping defaults and fallback behavior.
- **`routes/command-center.js`**: Replaced all `gpt-5.1` defaults in the DB schema and `KNOWN_SCRIPTS` mapping with `gemini-2.5-flash`.
- **`routes/ai.js` & `routes/ai-models-patch.js`**: Removed `gpt-5.1` and `gpt-5.4` from the Brain App UI model lists.
- **`routes/ai-chat.js`**: Removed GPT codex model configurations and API key fallback logic.

## 🚀 Priority 3: DeepSeek V3.2 Migration
Migrated the default AI logic to favor DeepSeek V3.2 as the primary intelligent engine via OpenRouter, replacing the Google AI Studio default.
- **`lib/ai.js`**: Changed default `_provider` from `google` to `openrouter`. Re-engineered the `callAI` function to construct an automatic 3-tier fallback chain when no explicit model override is provided: `deepseek/deepseek-v3.2 (OpenRouter)` → `qwen/qwen3-32b (OpenRouter)` → `gemini-2.5-flash (Kie)`. Retained backward compatibility to route `modelOverride: 'gemini-3.1-pro-preview'` directly to the Kie provider.
- **`config/channels.js`**: Removed hardcoded `modelOverride: 'gemini-2.5-flash'` from `dev-agent` and `hydra-core` so they inherit the new DeepSeek V3.2 fallback chain automatically. Complex agents (`minnie`, `researcher`, `credit-card-agent`) were intentionally left unchanged to preserve their specific vision/creative model requirements.
- **`lib/health-news.js`**: Replaced the hardcoded Kie Gemini Pro provider chain with a smarter sequence: `OpenRouter DeepSeek V3.2` → `OpenRouter Qwen3-32B` → `Kie Gemini 2.5 Flash`.
- **`lib/gmail-agent.js`**: Intentionally untouched. The `cron-gmail-expenses` pipeline works flawlessly with Kie Gemini 2.5 Pro/Flash and does not need modification.

**Phase 2 Extensions (Full System Takeover):**
- **`discord-bot/scripts/cron-lily-weekly.js`**: Re-routed from Kie Gemini Flash to OpenRouter DeepSeek V3.2.
- **`routes/cron-mc-ideator.js`**: Updated `sendPrompt` endpoint to target DeepSeek V3.2 via OpenRouter for much smarter Facebook content ideation.
- **`routes/ai-chat.js` & `routes/ai.js`**: Set `deepseek-v3.2` as the global `default: true` model for the Brain App Chat interface and API endpoints.
- **`routes/command-center.js`**: Updated the DB schema default `ai_model` to `deepseek/deepseek-v3.2` so newly created custom Cron Jobs immediately benefit from Tier 1 logic.

## 🎨 Phase 3: Frontend Sync & AI Manager Update
- **`brain-app-public/openclaw.js`**: Added DeepSeek and Qwen pricing metadata, updated `dcChannels` mapping to reflect that all Discord agents route to DeepSeek V3.2.
- **`brain-app-public/index.html`**: Cleaned up the AI Chat Model Selector dropdown, removing dead GPT/Gemini Pro Free options and defaulting to DeepSeek V3.2.
- **`brain-app-public/chat.js`**: Removed `gpt-5.x` from `FALLBACK_MODELS`.
- **`brain-app-public/ai-manager.js`**: Completely purged all frontend references to `openai-codex/gpt-5.1` and `gpt-5.4`. Added DeepSeek V3.2 and Qwen3 to `CANONICAL_MODELS` and `CRON_REGISTRY` UI definitions to accurately represent the current backend configurations.

// Function to recursively replace model strings
function replaceModel(obj) {
  for (const key in obj) {
    if (typeof obj[key] === 'string') {
      if (obj[key] === 'openai-codex/gpt-5.1') {
        obj[key] = 'openrouter/deepseek/deepseek-v3.2';
      } else if (obj[key] === 'openai-codex/gpt-5.4') {
        obj[key] = 'openrouter/qwen/qwen3-32b';
      }
    } else if (Array.isArray(obj[key])) {
      obj[key] = obj[key].map(item => {
        if (item === 'openai-codex/gpt-5.1') return 'openrouter/deepseek/deepseek-v3.2';
        if (item === 'openai-codex/gpt-5.4') return 'openrouter/qwen/qwen3-32b';
        return item;
      });
      // Further recursion for objects inside arrays
      obj[key].forEach(item => {
        if (typeof item === 'object' && item !== null) {
          replaceModel(item);
        }
      });
    } else if (typeof obj[key] === 'object' && obj[key] !== null) {
      replaceModel(obj[key]);
    }
  }
}

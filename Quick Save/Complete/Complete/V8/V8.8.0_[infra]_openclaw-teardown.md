# V8.8.0 [infra]: OpenClaw Minimal Teardown

## 📝 Background

This plan originally started as `V4.6.0` (in Icebox). Today, the `openclaw-gateway` (PM2: `openclaw` process) entered a rapid crash loop, eating CPU and crashing `discord-bot` due to orphan processes.

The user asked to finally tear it down because the OAuth tokens for OpenClaw GPT have expired anyway.

## 🛠️ Actions Taken

Instead of rewriting the entire AI pipeline, we did a **safe minimal purge** to stop the bleeding while leaving the code intact (relying on its built-in failovers).

### 1. Stopped Crash Loop on VPS
- **Action**: Ran `pm2 delete openclaw && pm2 save`.
- **Action**: Ran `systemctl stop openclaw-gateway && systemctl disable openclaw-gateway`.
- **Why**: Stopped the crash loops every 12 seconds. Saved system resources. 15 internal OpenClaw cron jobs were already failing with `consecutiveErrors` reaching 15.

### 2. Fixed Monitor Crash (`discord-bot/scripts/cron-ai-monitor.js`)
- **Action**: Removed the `execSync('openclaw models status')` logic.
- **Action**: Removed `oc-gpt-5-1` and `oc-gpt-5-4` from the array.
- **Why**: If we deleted the OpenClaw binary, this cron job would crash completely because `execSync` would throw.

### 3. Left Safely Intact
- **`discord-bot/lib/ai.js`, `health-news.js`, `gmail-agent.js`**: Left the code alone. They will try OpenClaw, immediately fail, and auto-fallback to Kie/Gemini.
- **`~/.openclaw/` directory**: Left alone. There's a `gen-news.mjs` script used by Dr.Mike that still lives inside `/workspace/skills/content-news-thai/scripts/`.

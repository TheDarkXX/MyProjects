# 🚀 V8.9.0: Command Center Alert Optimization & Discord Actions

## 1. 💡 Architecture & Reporting Strategy (คำแนะนำเรื่องการรายงาน)

จากที่สอบถามว่าควรรายงานแยกกันไหม (Bill / Subscription) หรือแยกเวลาส่ง:
**คำแนะนำ (Best Practice):** 
1. **ส่งเวลาเดียวกัน (8:00 AM) เหมือนเดิม:** เพื่อให้เห็น "ภาพรวมกระแสเงินสดประจำวัน" ในทีเดียว ไม่กวนสมาธิหลายรอบ
2. **ปรับ Layout ข้อความหลัก (Summary Report):** ไม่แยกก้อนตาม Database Table (Bill vs Sub) แต่ **แยกตาม Action (สิ่งที่ต้องทำ)**:
   - 🔴 **[ACTION REQUIRED]**: บิลที่ Overdue + Subs ที่ต้องกดต่ออายุเอง (และกำลังจะหมด)
   - 💳 **[AUTO-DEDUCTIONS]**: Subs ที่ตัดบัตรอัตโนมัติ + บิลที่หักบัญชี (แจ้ง FYI ให้เตรียมเงิน)
   - 📅 **[UPCOMING MANUAL BILLS]**: บิลที่รอคิวจ่ายใน 7 วัน
3. **✨ Innovation - Action Cards (รายงานแบบแยกการ์ดเฉพาะเรื่องด่วน):**
   - ข้อความสรุปจะส่งไปเป็นก้อนใหญ่ 1 ก้อน (ไม่มีปุ่ม เพื่อไม่ให้รก)
   - แต่สำหรับรายการที่ **"ต้องจัดการด่วน (Action Required)"** หรือ **"ครบกำหนดวันนี้"** บอทจะยิงข้อความตามมาเป็น **"การ์ดแยกทีละใบ"** พร้อม **ปุ่ม (Buttons)** ท้ายการ์ดนั้นๆ
   - *ข้อดี:* ผู้ใช้รู้ทันทีว่าปุ่มนี้คือของบิลไหน และไม่ติดข้อจำกัด 25 ปุ่มของ Discord

## 2. 🛠️ Implementation Steps

### Phase 1: Backend Deduplication & API Enhancement
- แก้ไฟล์ `routes/command-center.js` ที่ `/alerts` endpoint
- ดึงข้อมูล `expiringSubs` (ที่มี `expire_date` อยู่ใน 14 วัน) มารวมไว้ใน API
- **Deduplication:** หาก Sub ใดโผล่ใน `expiringSubs` (เพราะ NO auto-renew) จะต้องถูกหักลบออกจาก `dueSoonSubs` เพื่อไม่ให้แจ้งเตือนเบิ้ล 2 ที่
- เพิ่ม endpoint `/crons/interactions` หรือใช้ webhook เดิมมารับ Event ตอนคลิกปุ่มจาก Discord (ถ้ามี Discord Interaction Webhook อยู่แล้ว) *Note: เราอาจต้องใช้ Discord Interaction endpoint ที่เซิร์ฟเวอร์เปิดไว้ (เช่น `discord-bot/index.js` หรือ `routes/discord-webhook.js`)*

### Phase 2: Refactoring `cron-alerts.js`
- เปลี่ยนกลุ่มการแสดงผลในข้อความหลัก:
  - **Overdue & Expiring (No auto-renew):** -> 🔴 ACTION REQUIRED
  - **Due Soon (Auto-renew):** -> 💳 UPCOMING AUTO-DEDUCTIONS
  - **Due Soon Bills:** -> 📅 UPCOMING BILLS
- ลบการคำนวณ Date ซ้ำซ้อนและใช้ผลลัพธ์จาก API ตรงๆ

### Phase 3: Discord Action Buttons (Action Cards)
- สร้างฟังก์ชันส่ง Interactive Message แยกรายตัว (เฉพาะ item ที่ overdue หรือ due <= 3 วัน)
- **Bill Card:** 
  - 💸 Tiger Insurance (Overdue)
  - Button: `[✅ Mark as Paid]` (เปลี่ยนสถานะ paid=1 ใน DB)
- **Subscription Card:**
  - 🔄 ChatGPT Pro (Due in 1d)
  - Buttons: `[✅ Mark Paid (Next Cycle)]`, `[🚫 Turn off Auto-Renew]`, `[❌ Cancel/Archive]`

# 📊 Command Center Alert Analysis & Optimization Plan

จากการวิเคราะห์ Daily Alert report ของ Lily และโครงสร้างระบบ Backend พบประเด็นปัญหา (Pain Points) และโอกาสในการปรับปรุงระบบ (Opportunities) เพื่อให้ได้ระบบแจ้งเตือนที่เป็น Best Practice ลดความซ้ำซ้อน และกระตุ้นให้เกิด Action อย่างมีประสิทธิภาพ

## 🔍 1. Current Pain Points (ปัญหาที่พบในปัจจุบัน)

1. **ความซ้ำซ้อนของข้อมูล (Data Redundancy):** 
   - `Subscriptions Due Soon` และ `Expiring Subscriptions` แจ้งเตือนรายการเดียวกัน (เช่น ChatGPT Pro) แต่มาจากคนละเงื่อนไข (`next_due` กับ `expire_date`) 
   - *ตัวอย่างที่เห็น:* แจ้งว่า ChatGPT Due Soon (2d left) แต่กลับไปโผล่ใน Expiring ว่า (1d left) ทำให้ผู้ใช้สับสนเรื่องวันที่แท้จริง
2. **ขยะแจ้งเตือนสะสม (Alert Fatigue):** 
   - รายการที่ค้างจ่าย เช่น "test — THB 123" ค้างมา 5-6 วัน ยิ่งปล่อยไว้นาน Alert จะยิ่งบวมขึ้นเรื่อยๆ (จาก 3 -> 5 -> 6 alerts) ทำให้ความน่าเชื่อถือของการแจ้งเตือนลดลง ผู้ใช้จะเริ่มชินชาและละเลย
3. **การจัดการสถานะ NO Auto-renew ที่ไม่สมบูรณ์:**
   - ถ้าระบุว่า `[NO auto-renew]` หมายความว่าตั้งใจให้หมดอายุ แต่พอมันหมดอายุไปแล้ว ระบบอาจจะนำไปโชว์ใน `Overdue Subscriptions` ทั้งที่จริงควรจะเปลี่ยนสถานะเป็น Expired แล้วจบไป
4. **ความไม่ชัดเจนเรื่องสกุลเงิน (Currency Mix):**
   - ยอดรวมระบุเป็นบาท (`💰 Monthly cost: ฿5,689`) แต่รายการย่อยมี `USD 20` ทำให้ผู้ใช้อาจเกิดความไม่มั่นใจว่ายอดรวมครอบคลุม USD ด้วยอัตราแลกเปลี่ยนไหน หรือว่าละเว้นไปเลย

## 🌟 2. Best Practices สำหรับ Subscription & Bill Management

หลักการที่ดีที่สุดคือ **Action-Driven Notification** (แจ้งเตือนตามสิ่งที่ผู้ใช้ "ต้องทำ" ไม่ใช่แค่แยกตามประเภทของรายการ)

### 💡 เสนอปรับโครงสร้างการรายงานใหม่ (New Notification Hierarchy)

แบ่งกลุ่มตาม Action ที่ต้องการจากผู้ใช้:

1. **🔴 ACTION REQUIRED (ต้องดำเนินการ)**
   - **Overdue Bills:** บิลที่เลยกำหนดชำระ
   - **Manual Renewals (Expiring):** Subscription ที่ต้องกดต่ออายุเอง (No auto-renew) และกำลังจะหมดอายุ
2. **💳 UPCOMING DEDUCTIONS (เตรียมเงินให้พร้อม - Auto-renew)**
   - Subscription ที่จะตัดบัตรอัตโนมัติเร็วๆ นี้
   - บิลที่ตั้งหักบัญชีไว้แล้ว
3. **📅 UPCOMING BILLS (ต้องจ่ายเร็วๆ นี้)**
   - บิลที่ต้องจ่ายเอง และใกล้ถึงกำหนดชำระ

## 🚀 3. Proposed Enhancements & Innovations (ระบบที่ควรพัฒนาเพิ่ม)

### A. Optimization (ปรับแก้ระบบปัจจุบัน)
- **Alert Deduplication:** ถ้า Subscription ไหนอยู่ในหมวด Expiring แล้ว ต้องนำออกจากหมวด Due Soon เพื่อไม่ให้โชว์ซ้ำซ้อน
- **Timezone Fix:** การคำนวณ `daysUntil` ระหว่าง Backend (`/alerts`) กับ Script (`cron-alerts.js`) มีความคลาดเคลื่อน 1 วัน ต้องปรับให้ใช้ Timezone `Asia/Bangkok` อย่างเคร่งครัดทั้งตอนดึงข้อมูลและการตัดรอบวันเกิดเที่ยงคืน
- **Currency Conversion Preview:** หากมีรายการสกุลเงินต่างประเทศ (เช่น USD) ควรแสดงยอดเทียบเท่า THB ไว้ในวงเล็บ เช่น `USD 20 (~฿700)` เพื่อให้ผู้ใช้คำนวณงบประมาณได้ง่ายขึ้น

### B. Enhancement (เพิ่มความฉลาดให้ระบบ)
- **Auto-Archive / Graceful Expiration:** 
  - หาก Subscription เป็นแบบ `[NO auto-renew]` เมื่อพ้นกำหนด `expire_date` ไประยะหนึ่ง (เช่น 1-2 วัน) ให้ระบบอัปเดต Status เป็น `expired` หรือ `inactive` อัตโนมัติ เพื่อไม่ให้กลายเป็น Overdue ลอยค้างในระบบ
- **Stale Alert Suppression:** 
  - บิลที่ Overdue เกินกำหนดนานๆ (เช่น เกิน 14 วัน) ควรลดระดับการแจ้งเตือนจากทุกวัน เป็นอาทิตย์ละครั้ง หรือมีปุ่ม `[Snooze]` / `[Ignore]` เพื่อให้หน้า Dashboard และ Alert กลับมาสะอาดอีกครั้ง

### C. Innovation (ก้าวไปสู่ระดับ Automation)
- **Actionable Buttons ใน Discord:**
  - เพิ่มปุ่ม (Button Components ใน Discord Message) เช่น `[✅ Mark as Paid]`, `[⏭️ Skip Cycle]`, หรือ `[⏸️ Pause Alert]` ให้กดสั่งงานจาก Discord ได้โดยไม่ต้องเข้า Web UI
- **Smart Cashflow Insight:**
  - เพิ่ม Context สั้นๆ ท้ายข้อความ เช่น *"สัปดาห์นี้มีบิลรอจ่ายรวม ฿35,290 - กรุณาเตรียมเงินสดสำรองให้เพียงพอ"* เพื่อให้เป็นเหมือนผู้ช่วยทางการเงินจริงๆ

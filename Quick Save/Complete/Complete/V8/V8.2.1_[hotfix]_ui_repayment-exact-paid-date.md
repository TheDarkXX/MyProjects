# Issue
The user noticed that "Repayment" bill still exists in the list after marking as paid, and wanted to see the exact time it was paid from the history. 

# Root Cause
1. "Repayment" persists because the user viewed it via the "All" bills filter (which includes paid bills). 
2. The UI displayed `b.due_date` alongside the "Paid" indicator, which acts misleading because it causes the card to say `Paid 2026-04-03` which was actually the original due date of the bill, not the exactly `paid_date`.
3. The backend was only recording `paid_date` as `YYYY-MM-DD` out of `today()`, dropping the exact time HH:mm precision.

# Resolution Action
1. Updated `PUT /api/cc/bills/:id/paid` in backend `routes/command-center.js` to store full date-time string `YYYY-MM-DD HH:mm` via `new Date().toLocaleString('sv', { timeZone: 'Asia/Bangkok' }).substring(0, 16)`.
2. Updated `brain-app-public/command-center.js` `ccRenderBillsList` and the `ccBillsViewMode === 'grid'` card templates. If the `.is_paid` flag is truthy, we render `.paid_date` instead of `.due_date`, with fallback to `.due_date` for older data.

# Code implementation
**routes/command-center.js**
\`\`\`javascript
const paidDate = new Date().toLocaleString('sv', { timeZone: 'Asia/Bangkok' }).substring(0, 16); // YYYY-MM-DD HH:mm
...
db.prepare('UPDATE cc_bills SET paid = 1, paid_date = ? WHERE id = ?').run(paidDate, id);
\`\`\`

**brain-app-public/command-center.js**
\`\`\`javascript
<span class="cc-grid-due-date">${b.is_paid ? (b.paid_date || b.due_date) : b.due_date}</span>
\`\`\`

# Verification
1. Ensured the rendering logic defaults back to original due_date on older bills that do not precisely track times.
2. Changes applied cleanly without disruption to the underlying React/PlainJS loop.

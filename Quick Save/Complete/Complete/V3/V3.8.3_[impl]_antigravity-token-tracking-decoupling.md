# Antigravity Token Analytics Decoupling 

## 🤔 Context & Problem
We discovered that the reason the "Total Tokens" shot up to 1.2M tokens during an offline 1-hour span in the wee hours was because of **Antigravity**! The proxy server (`kie-proxy.js`) automatically intercepted the IDE's VS Code AI agent payloads (which sends the entire workspace/file context every single prompt) and stacked them perfectly inside `discord-bot/data/usage.json` under the alias `agentId: 'code-gpt'`.

Because these payloads consume around 300k–400k tokens per prompt, it artificially skews all token graphics and makes the "Estimated Cost" counter extremely bloated, simulating millions of tokens passing through what should normally be cheap agent queries.

## 🎯 Objectives
1. **Identifiable Telemetry**: Rename the abstract `code-gpt` telemetry signature to `antigravity`.
2. **Cost Muting**: Skip adding Antigravity records to actual dashboard `$x.xx` arrays (both paid real cost and model market-value simulations).
3. **Local Time Synchronization**: Adjust `kie-proxy.js` time stamps to parse via BKK locale offset, rather than purely `UTC`.
4. **Isolated Chart Dataset**: Prevent the Daily and Hourly volume bars from being overshadowed by Antigravity context tracking. Inject it as a new `Hidden: True` dataset within the chart (same functionality as Cache Context limits) which can be toggled by the user without blowing open the overall visual Y-axis default limits on load.
5. **New Models Cataloguing**: Include definitions for newly identified `kie.ai` models (`gemini-3.1-pro-high`, `claude-sonnet-4.6-thinking`, `gpt-oss-120b`).

## 🛠️ Implementation Steps Completed

### 1. Renaming and Cataloguing
- Updated `kie-proxy.js` and `kie-proxy-local.js` to define `agentId: 'antigravity'` moving forward.
- Inserted global alias `antigravity` into `AGENT_COLORS` inside `openclaw.js` to inherit the dark pink/red tracking icon.
- Hand-mapped newly introduced proxies (`gemini-3.1-pro-high`, `gemini-3-flash`, `claude-sonnet-4.6-thinking`, etc.) inside the `MODEL_PRICING` dictionary.

### 2. Timezone Offset Override
- Fixed `const today = new Date().toISOString().split('T')[0];` within proxy telemetry hooks, adjusting it with `- tzOffset`.
- Ensured future API injections match local boundaries so Day 1 tokens don't leak into Day 2 charts due to UTC conversions overlapping midnight blocks.

### 3. Cost-Muting 
- Set conditionals in `renderOclStats`, `renderOclModelUsage` (Doughnut Center Map), and `renderOclHealth` functions. They will skip traversing `s.agentId === 'antigravity'` records when generating `actualCost`, `freeCost`, and `marketValue`.

### 4. Graph Decoupling & Toggle Generation
- Altered `renderOclDailyBars` and `renderOclHourlyUsage`.
- Imposed an independent aggregator loop array: `agData`.
- Attached to the `Datasets` matrix underneath `Cache Context` natively disabled by default (`hidden: true`).

# Credit Card Raw Table UI Enhancements (V3.2.3)

## Goal Description
Enhance the functionality and usability of the raw data table in the Credit Card module by introducing sorting capabilities and a native full-screen mode to easily interact with massive amounts of data. 

## Full Screen: Best Practice Recommendation
For the raw data table fullscreen mode, the absolute best practice (for both performance and ease of use) is **CSS-Based Viewport Expansion** rather than opening a Modal. 

**Why not a Modal?**
- Moving an active DOM tree (with event listeners and inputs bound) into a Modal often breaks script references or requires a full re-render.
- Modals restrict the view area within a wrapper and frequently encounter `z-index` conflict issues with dropdown menus.

**Why CSS Viewport Expansion?**
- **Sleek & Immediate:** We simply append a `.cc-table-fullscreen` class to the existing wrapper container making it overlay the entire viewport immediately using `position: fixed; inset: 0; z-index: 9999;`.
- **Zero Data Refresh:** It leverages the already rendered Data Table, so no additional network requests or complex data sync is needed. Inline editing and Excel-paste bindings immediately work without touching any additional JS event delegations!
- **Fast:** Feels instantaneous, premium, and reliable. 

## Proposed Changes

### `command-center.js`
- **Global Variables:**
  - Add `ccCardSortField` (default empty or 'date') and `ccCardSortOrder` (default `desc`) handling.
- **`ccCardRenderTable(rows)`:**
  - Update `<th>` structures to become clickable, injecting sorting indicators (`▲` / `▼`) next to column headers (Date, Name, Price, Category, Card, Payee).
  - Add event listeners for triggering arbitrary sorting on column headers inside this function.
  - Insert a `⛶ Fullscreen` toggle button just above the table container.
- **`ccCardApplyFilters()`:**
  - Apply Array sorting to `filtered` rows based on `ccCardSortField` and `ccCardSortOrder` variables before calling `ccCardRenderTable`. 
- **DOM Integration:** Add the event binding on the Full Screen button to toggle `.cc-table-fullscreen` on `#cccard-table-wrap`. 

### `style.css`
- **Fullscreen Rule:**
  - Inject a css rule `.cc-table-fullscreen` assigning `position: fixed; top:0; left:0; width: 100vw; height: 100vh; z-index: 9999; background: var(--bg-main); overflow: hidden; padding: 24px;`
- **Table adjustments:**
  - Increase the max height of the table area when `.cc-table-fullscreen` is applied so it covers the entire screen, giving maximum viewing context.
  - Make `th.cc-sortable` cursor pointer, and highlight on hover.

### 1. ปัญหา Krungsri ยอดไม่ขึ้นในตาราง "เดือนนี้"
- **สาเหตุ:** เทมเพลตไฟล์ PDF ของกรุงศรีในรอบนี้ ฟอนต์วันที่แคบและเบียดกัน เช่นพิมพ์ `(18/03/26)` ทำให้ AI Vision (Kie.ai) อ่านเลขผิดเพี้ยนเป็นปี `2021` แทนที่จะเป็นปี `2026` เมื่อระบบเซฟเป็น `2021-03` ยอดจึงหายไปจากหน้า Filter ของ "เดือนปัจจุบัน"
- **แก้ปัญหาระยะสั้น:** เขียนสคริปต์ `fix_years.js` รันแก้ข้อมูลที่ถูก Insert เข้าไปผิดให้กลับมาเป็นปี 2026
- **แก้ปัญหาถาวร (ระยะยาว 100%):** 
  - [x] เพิ่มคำสั่งเด็ดขาดใน `credit-card-agent.js` (Prompt Rules) ว่า `"The CURRENT YEAR is 2026. ห้ามมโนปี หากภาพเบลอหรือมีแค่เลข 26 ให้ใช้ปี 2026 เสมอ"`
  - [x] เขียน JavaScript Validator ดักจับหลัง AI สกัดเสร็จ `if (date.startsWith('2021') || date.startsWith('2025')) date = '2026...'` 

### 2. ปัญหา BBL ยอดหายไป (ถูกตรวจจับผิดเป็น KBank)
- **สาเหตุ:** ยอดของ BBL ถูกระบบดึงได้ครบ แต่ด้วยโค้ดเวอร์ชันแรกสุด กฎเงื่อนไขการแยกแยะธนาคาร (`detectCCBank`) กว้างเกินไป ทำให้อีเมลของ BBL ถูกจัดหมวดหมู่ผิดเป็นธนาคาร `KBank` ทำให้เวลามาค้นหารายการในหน้า UI ภายใต้ค่าย `BBL` จึงหาไม่พบ
- **แก้ปัญหาระยะสั้น:** เขียนสคริปต์ค้นหายอด PTT จำนวน 1000 และ Cash Rebate จำนวน 5 บาท ของอีเมล BBL แล้วอัปเดตแก้ย้ายธนาคารกลับคืนมาให้ BBL
- **แก้ปัญหาถาวร (ระยะยาว 100%):**
  - [x] รื้อและเขียน Logic ลำดับความสำคัญในการ Regex ค้นหาชื่อธนาคารจาก Subject และ Sender ยึดคีย์เวิร์ด `'bbl'` และ `'bangkok bank'` เป็นอันดับ 1 ก่อน KBank อย่างรัดกุม 100%

อ้างอิงจากเรทราคา: **Input: $0.09 (~18 credits) / 1M Tokens** และ **Output: $0.75 (~150 credits) / 1M Tokens**

การประเมิน Token สำหรับสแกนบัตรทั้ง 4 ค่าย ภายใน 1 รอบบิล (รวม 90 รายการ):
- **CardX (Text):** 68 รายการ | Input ~5,000 Tokens 
- **KBank (Text):** 14 รายการ | Input ~2,500 Tokens
- **BBL & Krungsri (Vision OCR):** 8 รายการ (รวม 4 หน้ากระดาษ) | Input ~1,200 Tokens
- **Prompt พื้นฐานรวม:** ~2,000 Tokens

**คำนวณราคา:**
- **Input (ขาเข้าอ่าน PDF/รูป):** รวมใช้ประมาณ `10,700 Tokens` = **$0.00096 USD**
- **Output (ขาออกที่ AI ตอบกลับมาเป็นตาราง):** 90 รายการ * 50 tokens = ประมาณ `4,500 Tokens` = **$0.00337 USD**

✅ **สรุป:** การตื่นขึ้นมาสแกนบัตร 4 ใบและอิมพอร์ตข้อมูลรายเดือนให้แบบอัตโนมัติ จะใช้ทรัพยากรรวมเพียง **$0.0043 USD** (ราวๆ 0.86 Credits)

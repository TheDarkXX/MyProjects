# V3.2.2 - OpenClaw Native Telemetry & Proxy Interceptor Fix

## Context & Problem
The OpenClaw Analytics dashboard missed significant telemetry data from VSCode (Code GPT) and other historical systems. OpenClaw Gateway natively displayed 3.4M tokens via `http://localhost:18790/usage`, but this UI proxies data directly from the Kie.ai/OpenRouter APIs and restricts data to the last 1,000 sessions.

## Architectural Decisions & Fixes
- **Model Standardization**: Enforced `gpt-5.1` globally across `openclaw.json` for all Pinky interactions.
- **Kie-Proxy Interceptor**: Instead of attempting to parse 5,000+ `.jsonl` OpenClaw archive files that lacked token data, we modified the gateway's `kie-proxy.js` to intercept HTTP response streams directly. We now synchronously parse `prompt_tokens` and `completion_tokens` on the fly and persist them to `discord-bot/data/usage.json`.
- **Dashboard Aggregation**: Modified `brain-app-public/openclaw.js` to ingest and merge both the proxy's `discord-usage.json` and the Native OpenClaw `openclaw-sessions.json`. Added full agent mapping (incl. `Code GPT`) into `AGENT_COLORS`.
- **Cron Rollback**: We reverted an experimental brute-force `.jsonl` parser back to OpenClaw's native active session export (`openclaw sessions --all-agents`), offloading persistence to the interceptor moving forward.

## Implementation Details
1. **Target**: `/root/.openclaw/workspace/scripts/kie-proxy.js`
   `proxyRes.on('data')/on('end')` hooked to extract native token usage.
2. **Target**: `brain-app-public/openclaw.js`
   Merged Discord and OpenClaw sessions and mapped `code-gpt`.
3. **Target**: `/root/brain-app/scripts/cron-openclaw-tracker.sh`
   Cleaned up shell deployment for production stability.

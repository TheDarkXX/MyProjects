# Dev Lab Piggyback Hotfix

> Hotfix for missing Discord Bot generated tasks (Piggyback failed to pull from VPS)

## Context
ผู้ใช้งานแจ้งว่าหลังจากสร้างงานผ่าน Discord (Project = \`dev-lab\`) แล้ว ลองรัน Piggyback (deploy.js pull) แต่งานกลับไม่ถูกดึงลงมา 

**Root Cause:**
เกิดจากความผิดปกติของโครงสร้าง Git Architecture ของระบบ:
1. \`V3.3.0\` วางแผนให้ใช้ Git remote ที่ชื่อว่า \`origin\` (GitHub) เป็นศูนย์กลาง แต่ในระบบจริงทั้งฝั่ง Local PC และ VPS Working Directory ไม่มี Remote ที่ชื่อ \`origin\` ถูกตั้งค่าไว้ (ใช้ \`vps\` ที่เป็น Bare Repository อย่างเดียว)
2. ชุดคำสั่งใน \`discord-bot/lib/task-file-gen.js\` พยายามสร้างไฟล์แล้ว \`git push origin master\` ซึ่งเกิด Error และไปไม่ถึง Bare Repo
3. ชุดคำสั่งใน \`deploy.js\` ถูกตั้งค่าให้ดึงและดันไปที่ \`origin master\` จึงทำให้พังและไม่สามารถรับงานใหม่ลงมาได้

**Solution:**
ปรับทิศทางการ Push / Pull ของระบบ Dev Lab กลับมาหา \`vps\` Bare Repository ทั้งหมด

## Tasks
- [x] Fix \`task-file-gen.js\`: ปรับคำสั่ง git push ให้ยิงเข้า Bare Repo ของเครื่องเซิร์ฟเวอร์โดยตรง (\`git push /root/repos/brain-app.git HEAD:master\`)
- [x] Fix \`deploy.js\`: ซ่อมบรรทัด \`git pull\` และ \`git push\` ให้กลับมาใช้เซฟโซนเดิมคือ \`vps master\` ทิ้งโค้ดที่เรียก \`origin\` ที่ระบบไม่มีตัวตนอยู่ทิ้งไป
- [x] Salvage Stubs: ดึงไฟล์ \`_STUB_\` ที่ค้างอยู่บนเซิร์ฟเวอร์ลงมาแปลงแบบ Manual ให้กลายเป็นแผน Quick Save 3 งานใหม่ที่สมบูรณ์ เรียบร้อย
- [x] Validate Version: อัปเดตแพตช์ระบบเสร็จสมบูรณ์ เตรียมรัน \`deploy.js push\` อัปโค้ดเข้า VPS

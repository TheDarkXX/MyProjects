# V3.8.1 — Credit Card Triple-Guard: 3-Model Consensus Verification

**Date:** 2026-04-10
**Status:** ✅ Implemented (pending deploy)

## 📌 Context
ระบบปัจจุบันใช้ AI ตัวเดียว (Gemini 2.5 Flash via Kie.ai) ในการสกัดรายจ่ายจาก Statement PDF ทำให้ไม่มีทางรู้ว่า AI อ่านผิด/ตก/สลับ — ต้องพึ่ง manual review ทุกครั้ง

## 🏗️ Architecture: 3-Model Consensus

```
PDF/Images ─┬──→ 🔵 Gemini 2.5 Flash (Kie.ai)                 → JSON A
             ├──→ 🟢 Nemotron Nano 12B VL (OpenRouter :free)    → JSON B  
             └──→ 🟡 Gemma 4 26B (OpenRouter :free)             → JSON C
                          │
                    ┌─────┴─────┐
                    │ Consensus │  ← JavaScript (deterministic)
                    │  Engine   │    (1. Perfect Math Override)
                    │           │    (2. Tie-Breaker Vote)
                    └─────┬─────┘
                          │
                  Math ตรง 100% → ✅ Auto-save (Verified by Perfect Math)
                  ไม่ตรง      → ⚠️ HOLD + Discord Alert
```

### 🧠 Model Selection Update (11 Apr 2026 Shift)
- **Primary Anchor**: `qwen/qwen2.5-vl-72b-instruct` (Paid, highly accurate for complex tables).
- **Secondary Wingman**: `nemotron-nano-12b-v2-vl:free` (Free, OpenRouter).
- **Architecture Shift**: Upgraded from strict 2/3 voting to a **"Perfect Math Override"** system. If Qwen 72B extracts items whose sum exactly matches the Statement Total, it bypasses the manual vote tie-breaker. This prevents weaker models from outvoting the superior model when they both make the same hallucination mistakes.

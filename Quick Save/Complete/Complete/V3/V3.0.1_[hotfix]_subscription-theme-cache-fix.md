# Subscription Analytics Theme Refinement & Cache-Busting Automation

## Context & Problem
1. **Neon UI Clash:** The newly integrated Subscription Analytics (V3.0.0) initially launched using the legacy `#1c1e54` neon purple backgrounds. The user requested a shift toward a clean, premium, and sophisticated visual language (similar to Stripe), noting the neon colors were glaring and didn't fit the desired high-end aesthetic.
2. **Ghost Bugs & Styling Stagnation (The Cache Hell):** Despite executing `git push vps` and successfully deploying CSS patch scripts, the user's browser persistently displayed the legacy neon colors. Both the AI's internal string replacements (due to Windows CRLF vs Linux LF differences) and manual omissions in bumping the `v=` cache string in `index.html` resulted in severe UI ghosting where the local representation did not reflect production reality.

## Implementation Strategy / Rationale
1. **Premium Clean Theme (`V3.0.1` UI Polish):**
   - Transformed all cards (`.cc-sub-grid-card`, `.cc-analytics-chart-card`, `.cc-analytics-left`) from static neon colors to `var(--bg-elevated)` combined with subtle 12px border radii, soft transluscent borders (`rgba(255,255,255,0.08)`), and modern shadow depths (`box-shadow: 0 4px...`).
   - Adapted all Chart.js visualizations to utilize a curated "Clean Palette" (Sky Blue, Indigo, Crimson, Deep Gold, Emerald) ensuring harmonious and readable data display.
2. **The "Bump Cache" Automation:**
   - Instead of relying on error-prone manual `v=VXX` incrementations across CSS and JS tags in `index.html`, a dedicated `bump-cache.js` script was authored. This utility automatically injects a UTC timestamp into all asset URIs within `index.html`.
   - Formally updated `AGENTS.md` and `memory.md` to impose an absolute automation mandate: AI must execute `node bump-cache.js` prior to every `git push vps`.
3. **Cross-OS Compatibility Protocol:**
   - Provisioned `.gitattributes` (`* text=auto eol=lf`) to enforce LF carriage returns repository-wide.
   - Banned the usage of fragile regex `sed` or node string replacement scripts for deep CSS modifications, enforcing the use of native `replace_file_content` functionality.

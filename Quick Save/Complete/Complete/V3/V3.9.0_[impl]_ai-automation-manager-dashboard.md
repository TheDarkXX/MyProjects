## Phase 2: Token Analytics & UX Enhancements

### 1. Global Time Range Header (ครอบ 3 tabs)

**แนวคิด**: เพิ่ม row ของปุ่ม time range filter เหมือนหน้า OpenClaw Analysis ไว้ที่ header ใต้ stats bar, เหนือ tabs — effect ต่อทุก tab พร้อมกัน

**Data Source**: Fetch `/data/openclaw-sessions.json` + `/data/discord-usage.json` (same source as OpenClaw Analysis) ครั้งเดียวตอน init แล้ว filter ตาม range

```
┌──────────────────────────────────────────────┐
│ 🧠 AI Model & Automation Manager    [STATUS] │
│ [Stats bar: Models | Agents | Bots | Crons]  │
│ ┌──────────────────────────────────────────┐  │
│ │ Today | Ydy | 7D | 30D | This Mo | ALL  │  │ ← NEW: Global time filter
│ └──────────────────────────────────────────┘  │
│ [AI Models] [Agents & Bots] [Crons & Auto]   │
└──────────────────────────────────────────────┘
```

**Options**: `Today`, `Yesterday`, `7D` (default), `30D`, `This Month`, `ALL`

**Implementation**:
- **ai-manager.js**: Add `aimRange` state var (default 7), `aimSessionsData` to cache fetched sessions
- Fetch sessions once in `fetchAiRegistry()`, store globally
- All render functions receive filtered sessions as parameter
- Reuse `MODEL_PRICING` dict from `openclaw.js` (import or duplicate the essential subset)

### 3. Tab 2: Agents & Bots — List View Toggle + Token Usage

#### 3a. Grid/List View Toggle

เพิ่มปุ่ม toggle ที่มุมขวาบนของ tab:
```
[Grid View ▦] [List View ☰]
```

**List View**: Interactive table with columns:
| Agent | Primary Model | FB1 | FB2 | Crons | Tokens | Sessions | Cost |
|-------|---------------|-----|-----|-------|--------|----------|------|
| main  | gpt-5.1 ▼     | gemini-2.5-flash ▼ | — | 3 | 125.4K | 28 | $0.00 |

- Dropdowns ยังคง interactive เหมือน Grid View (เลือกเปลี่ยน model ได้เลย)
- Sort by column header click
- Compact mode — เห็นทุก agent ในหน้าจอเดียว

**Grid View**: เหมือนเดิม (card layout) แต่เพิ่ม token stats ในแต่ละ card

#### 3b. Token Usage per Agent

ทั้ง Grid และ List view เพิ่มโชว์:
- Total tokens used (filtered by time range)
- Sessions count
- Est. cost

**Implementation**:
- `renderAgentsTab(filteredSessions, viewMode)` where `viewMode` = 'grid' | 'list'
- State var `aimAgentView` default 'grid'
- Token calc: filter sessions by `agentId`

## 🔵 Recommendations for Optimize & Enhance (✅ Phase 2 Extended - Implemented)

- ตรวจจาก sessions ว่า cron ตัวไหน "ไม่ได้ run" ในรอบ 24h ทั้งที่ schedule daily
- แสดง ⚠ badge — ช่วยจับ cron ที่ตายเงียบ

### 10. Quick Model Swap Shortcut ✅

## Architecture Summary (V2)

```
ai-manager.js
├── fetchAiRegistry()        — /api/config/ai-registry (config data)
├── fetchSessionsData()      — /data/openclaw-sessions.json (NEW)
│                            — /data/discord-usage.json (NEW)
├── filterSessionsByRange()  — Shared date filtering logic  
├── renderDashboardHeader(sessions) — Stats + token totals
├── renderModelsTab(sessions)       — Model list + pricing + usage
├── renderAgentsTab(sessions, mode) — Grid/List + token stats
├── renderCronsTab(sessions)        — Cron list + token stats
└── MODEL_PRICING_SUBSET {}  — Subset from openclaw.js
```

**Data flow**:
```
openclaw-sessions.json ──┐
                         ├──→ filterSessionsByRange(range) ──→ All render functions
discord-usage.json ──────┘
                         ↕
/api/config/ai-registry ──→ aiRegistryData (config, models, agents)
```

# Hotfix: Hono Router Wildcard Conflict (`/:scope`)

## 🛑 Problem Statement
After deploying V3.9.0, the AI Manager UI on `brain.doctorbankonline.com` shows a blank/error screen or fails to load data.  
Deep debugging via SSH revealed that the API call to `GET /api/config/ai-registry` fails with a `400 Invalid scope` error (when authenticated) or returns a massive PM2 process chunk instead of a 403 error.

**Root Cause**: 
In `routes/config.js`, the wildcard routing rule `router.get('/:scope', ...)` is intercepting the `/ai-registry` requests because Hono v3 routing sometimes overrides static routes or treats `ai-registry` as a dynamic `:scope` param. Since `ai-registry` is not inside the `ENV_FILES` mapping, it throws the 400 error.

## ✅ Solution Plan (Pending Next Session)

When coming back, perform these exact steps:

1. **Reorder `routes/config.js`**:
   - Locate the wildcard routes:
     - `router.get('/:scope', ...)`
     - `router.get('/:scope/reveal/:key', ...)`
     - `router.put('/:scope/:key', ...)`
   - Move ALL of these wildcard endpoints to the **very bottom** of the router block (just before `export function configRoutes`).
   - This prevents them from preempting static utility routes like `/services/status` and `/cron/list` which were also partially affected by the same bug.

2. **Backend Restart**:
   - Run `pm2 restart brain-app` to apply the routing changes.

3. **Verify API Logic**:
   - Ensure that calling `GET /api/config/ai-registry` returns the correct JSON containing `models` and `agents` without getting trapped by `/:scope`.

4. **Verify UI**:
   - Open browser at `https://brain.doctorbankonline.com/` -> System -> AI Manager
   - Make sure no red UI error boxes appear from our previous defensive catch.
   - Screen must display Data, Tokens and Estimated Costs smoothly.

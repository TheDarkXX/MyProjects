# 🐕 Watchdog Agent — Implementation Plan (V3.2.0)

## Background

ระบบ OpenClaw VPS มี **3 PM2 services** (brain-app, discord-bot, kie-proxy) และ **20+ cron jobs** ที่ทำงานตั้งแต่ Content Pipeline (Doctor Skill FB), Financial Tracking (Lily/Expenses), System Monitoring (Health Check), Briefing (Gmail/Dev/News) ไปจนถึง Discord Bot.

**ปัญหาหลักที่เจอในหน้างานจริง:**
1. ❌ **Doctor Skill ไม่โพส FB** — cron ทำงาน แต่ไม่มี content ใน queue (auto-planner ไม่ได้สร้าง idea → ไม่มีอะไรจะ post) และไม่มีใครแจ้งเตือน
2. ❌ **Cron พังเงียบ** — script crash แล้วไม่มี notification → รู้ตัวอีกทีงานหายไปแล้ว
3. ❌ **Discord Bot สมองตัน** — OpenClaw gateway unknown agent fallback, Brain API 404, syntax error ทำให้ bot reboot ซ้ำ (restart 105-106 ครั้ง)

แนวคิด: สร้าง **Watchdog Agent** ที่เป็น "ผู้ตรวจงาน" ด้วยระบบ **Goal-Based Monitoring** ไม่ใช่แค่ตรวจว่ารันรึยัง แต่ตรวจว่า "บรรลุเป้าหมายของงานนั้นไหม" พร้อมระบบ Auto-Fix ที่คอยกู้คืนระบบแบบอัตโนมัติ

### Schedule (Cron)
- `*/15 * * * *` → quick health check (PM2, API, disk)
- `0 */2 * * *` → full goal-based check (content pipeline, bot health, cron outcomes)
- `0 7 * * *` → morning report (full daily summary to Discord)
- `0 22 * * *` → evening report (what happened today)

## 🛠️ Code Details & Structure

### 1. Goal Registry (`scripts/watchdog/goals.js`)
กำหนดว่าแต่ละ pipeline ต้องบรรลุเป้าอะไร:

```javascript
const GOALS = {
  // === Doctor Skill Content Pipeline ===
  'doctor-skill-pipeline': {
    name: 'Doctor Skill FB Pipeline',
    priority: 'CRITICAL',
    checks: [
      {
        id: 'ideas-in-queue',
        description: 'มี idea ใน backlog พร้อม schedule',
        query: `SELECT COUNT(*) as cnt FROM content_plans WHERE project='doctor-skill' AND status='idea'`,
        expect: 'cnt >= 3',
        severity: 'WARNING',
        autofix: 'alert-only' 
      },
      {
        id: 'today-has-scheduled',
        description: 'วันนี้มีโพสที่ schedule ไว้',
        query: `SELECT COUNT(*) as cnt FROM content_plans WHERE scheduled_at LIKE '{today}%' AND status IN ('scheduled','approved','published')`,
        expect: 'cnt >= 1',
        severity: 'CRITICAL',
        autofix: 'trigger-planner' 
      }
    ]
  },

  // === Discord Bot Health ===
  'discord-bot-health': {
    name: 'Discord Bot Connectivity',
    priority: 'CRITICAL',
    checks: [
      {
        id: 'bot-restart-count',
        description: 'Bot ไม่ crash loop',
        type: 'pm2-check',
        process: 'discord-bot',
        expect: 'restarts < 5 in last hour',
        severity: 'CRITICAL',
        autofix: 'pm2-restart-with-delay'
      }
    ]
  }
};
```

### 2. Auto-Fix Engine (`scripts/watchdog/remediate.js`)
| Action ID | What it does | Safety |
|-----------|-------------|--------|
| `pm2-restart` | `pm2 restart <process>` | ✅ Safe |
| `pm2-restart-with-delay` | Wait 30s → restart (break crash loop) | ✅ Safe |
| `trigger-planner` | Run `cron-auto-planner.js` manually | ✅ Safe (idempotent) |

> **Safety Guardrails:** Max 3 auto-fix attempts per issue per day (circuit breaker). No destructive DB actions without human review.

### 3. Database Schema Update (brain.db)
```sql
CREATE TABLE IF NOT EXISTS watchdog_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  timestamp TEXT DEFAULT (datetime('now','localtime')),
  check_id TEXT NOT NULL,
  goal_name TEXT,
  status TEXT NOT NULL,
  severity TEXT,
  message TEXT,
  autofix_action TEXT,
  autofix_result TEXT,
  raw_data TEXT
);
```

## 🔍 Root Cause Analysis (3 Cascading Bugs)
1. **`reviewed_at = NULL`**: จากการที่ script เวอร์ชั่นเก่ารันเนื้อหาเข้าสู่กระบวนการ combat ก่อนที่ตัวโค้ดเวอร์ชันใหม่จะ deploy เสร็จสิ้น ส่งผลให้ไม่ได้สร้างค่า `reviewed_at` เก็บไว้ในระบบ Database
2. **`cron-content-remind.js` silent skip**: โค้ดก่อนหน้าติดลูปตรงที่ `if (!post.reviewed_at) continue;` เมื่อโค้ดตรวจพบค่า Null โพสจึงถูกข้ามไปและ auto-approve เลยไม่ทำงาน
3. **Pipeline chain break**: เมื่อ 2 เหตุการณ์บนเกิดพร้อมกัน ทำให้โพสค้างในเมนู `review` ไม่มีทีท่าจะอนุมัติ (เพราะบอสก็ไม่เห็น) และ Auto-approve ไม่ถูกเรียก คิวโพสจึงหยุดทำงานไปโดยปริยาย

## 💡 Resolution Details
- ระบบแจ้งเตือนจะใช้ `updated_at` เป็นแผนสำรองถ้าเจอโพสรุ่นเก่าที่ไม่ได้เซฟเวลา review ไว้ได้แล้ว
- ฝัง Safety net ใน DB Query เมื่อ Agent ทำการ combat เสร็จเพื่อป้องกันความผิดพลาดถ้าหาก Parent Script รีเซ็ตค่าอีก
- แก้ไขปัญหาคิวโพสที่ตกค้างทั้งหมด ปลายทางกลับมาทำงานอัตโนมัติตาม Flow เดิมได้สมบูรณ์

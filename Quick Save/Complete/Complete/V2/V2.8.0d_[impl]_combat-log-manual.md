# ⚔️ Agent Combat Pipeline — Accumulative Log Manual
**(Version 2.8.0)**

คู่มือฉบับนี้อธิบายกลไกการเก็บประวัติ (Log) ของระบบตบตีคอนเทนต์ระหว่าง **Doctor Skill** และ **น้องจู้จี้** (Agent Combat) เพื่อให้เห็นว่าระบบบันทึกข้อมูลทุกรอบอย่างสมบูรณ์ และไม่มีการสูญหายของข้อมูลเก่าแม้จะถูกบอสสั่ง Reject ตีกลับมาจาก Discord

---

## 1. การเก็บ Log ต่อเนื่อง (Accumulative Log)

ระบบไม่ได้แค่เปลี่ยนสถานะ (Status) หรือทับข้อมูลเดิมทิ้งเมื่อมีการเริ่มตบตีรอบใหม่ แต่จะใช้วิธี **โหลดประวัติที่มีอยู่เดิมจากฐานข้อมูล** ขึ้นมาก่อนทำงานเสมอ (อ้างอิงจาก `scripts/agent-combat.js`)

```javascript
  let combatLogRaw = post.combat_log || '[]';
  let logs = [];
  try { logs = JSON.parse(combatLogRaw); } catch (e) { }
  let rounds = post.combat_rounds || 0;
```
จากจุดนี้ ทำให้ตัวแปร `rounds` บวกเพิ่มขึ้นไปเรื่อยๆ ตามจำนวนการตบตีจริง (Round 1, Round 2, Round 3...) และ Array `logs` จะเก็บประวัติเชื่อมต่อกันเป็นสายโซ่ยาวๆ 

---

## 2. โครงสร้างข้อมูลใน 1 Round

ในแต่ละรอบที่ตบตีเสร็จ ระบบจะบันทึกชิ้นข้อมูลลงไปต่อท้ายใน `logs` เสมอ โดยโครงสร้างที่บันทึกคือ:

```javascript
  logs.push({
    round: rounds,                       // เลขที่รอบ เช่น 1, 2, 3...
    doctor_skill_output: currentCaption, // ข้อความล่าสุดที่ Doctor Skill เขียน
    juji_feedback: feedbackObj.feedback, // รายละเอียดคำวิจารณ์จากน้องจู้จี้
    juji_approved: feedbackObj.approved  // ผลตรวจ (true = ผ่าน, false = ตีกลับ)
  });
```

และการตัดสินใจเลือกรูปภาพในรอบสุดท้ายก็จะถูก push ตามเข้าไปรวมใน log นี้ด้วย:
```javascript
  logs.push({ 
    image_selection: true, 
    selected_variant: selectedVariant, 
    reason: imageRes.reason 
  });
```

## 3. กลไกเมื่อบอสกดยกเลิก (Reject ❌) ทาง Discord

เมื่อบอสกดยกเลิกใน Discord Bot เมสเซจ:
1. ระบบจะไม่เปลี่ยนสถานะกลับไปเป็น `planned` ให้เสียเวลา
2. Bot จะสั่งรัน script ทันทีด้วย Flag `--rounds 1` เพื่อบีบบังคับให้รัน **Round 2 (หรือรอบถัดไป) เริ่มตบตี ณ ขณะนั้นทันที**

เมื่อเริ่มทำงาน โค้ดจะดึงประวัติล็อกของ Round 1 มาไว้ในหัว ต่อท้ายด้วยการตบตีของเจเนอเรชั่นใหม่ จากนั้นบันทึกเข้า Database ลง `combat_log` เป็นประวัติรวม ทำให้สามารถตรวจสอบย้อนหลังได้ทุกสเต็ปบนคอมแบทล็อกของเบราว์เซอร์

---

## 4. การแสดงผล (UI) ใน Mission Control

ตัว UI ใน Mission Control ถูกตั้งโปรแกรมให้อ่านฟิลด์ `combat_log` แบบเต็มรูปแบบ 
เมื่อเปิด content ใน Editor จะปรากฏหน้าต่าง **"Combat Log"** ที่นำข้อมูล JSON ทั้งหมดนี้มากางเป็นไทม์ไลน์ (Timeline) ให้บอสอ่านได้ตั้งแต่:

- **รอบที่ 1:** Dr.Skill เปิดด้วยข้อความอะไร → น้องจู้จี้ด่ากลับยังไง
- **รอบที่ 2 (ที่บอส Reject):** Dr.Skill แก้ตัวมาใหม่เป็นอย่างไร
- **Final:** น้องจู้จี้ปล่อยผ่าน และฟันธงเลือกรูปภาพใบไหน ด้วยเหตุผลใด

---

## 5. เกราะป้องกัน (Infinite Loop Safety)

ถึงระบบจะรองรับการตีกลับและลูปแบบไม่สิ้นสุด เพื่อเป็นการรักษา Cost และป้องกัน AI เออเร่อ จึงมีตัวแปร `MAX_ROUNDS_SAFETY` กั้นไว้เป็นปราการด่านสุดท้าย เพื่อบังคับให้คอนเทนต์หลุดจากการติดลูปการด่าทอกันเองหากเกินลิมิตที่กำหนด

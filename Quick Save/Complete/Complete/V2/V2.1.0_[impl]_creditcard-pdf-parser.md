# 💳 Phase 2: Credit Card e-Statement PDF Parser

## สรุปปัญหา
บัตรเครดิตจะส่ง e-Statement มาทางอีเมลเป็น **ไฟล์ PDF แนบ** (ไม่ใช่ข้อความในเมล) โดยไฟล์ PDF ทุกฉบับจะ **ล็อกด้วยรหัสผ่าน** ของแต่ละธนาคาร

### เมลที่ต้องจับ (4 แบงก์)

| ธนาคาร | ผู้ส่ง | หัวข้อ | หมายเหตุ |
|--------|--------|--------|----------|
| กสิกร K-Credit | bankxl@hotmail.com (Forward) | `K-Credit Card email statement MM/YYYY` | Forward เข้า Gmail |
| CardX (กรุงศรี) | bankxl@hotmail.com (Forward) | `ใบแจ้งยอดบัตรเครดิต CardX 8292` | Forward เข้า Gmail |
| Krungsri Signature | bankxl@hotmail.com (Forward) | `Krungsri Signature Credit Card of 409344XXXXXX7539` | Forward เข้า Gmail |
| กรุงเทพ BBL | BBL_eStatement@bangkokbank.com | `Bangkok Bank Credit Card e-Statement` | ส่งตรง |

---

## 📐 Architecture

```
Gmail API → ตรวจจับหัวข้อ → ดาวน์โหลด PDF Attachment
  → ปลดล็อก PDF ด้วย Password (จาก config)
  → ดึง Text ดิบ → ส่ง AI สกัดรายการ
  → POST /api/cc/expenses/batch (จัดเก็บ)
  → แจ้ง Discord #expense-tracker (สรุปยอดเดือน)
```

### 3. Gmail Agent: ฟังก์ชันใหม่

#### [MODIFY] `discord-bot/lib/gmail-agent.js`

เพิ่ม 3 ฟังก์ชันใหม่:

**a) `fetchPdfAttachment(email, messageId)`**
- เรียก Gmail API `GET /messages/{id}?format=full`
- วนหา `part` ที่เป็น `application/pdf`
- ดาวน์โหลด Attachment data (base64) → เซฟเป็นไฟล์ชั่วคราวใน `/tmp/`

**b) `decryptAndParsePdf(filePath, bankKey)`**
- อ่าน PDF ด้วย `pdf-parse` + Password จาก config
- ดึง raw text ออกมา (ตัดให้เหลือ ~3000 chars เพื่อประหยัด token)
- Return text ดิบ

**c) `extractCreditCardExpenses(pdfText, bankKey, statementMonth)`**
- ส่ง text ดิบให้ AI พร้อม Prompt เฉพาะทาง:
  - *"นี่คือสรุปยอดบัตรเครดิต ให้ดึงเฉพาะ 'รายการรูดซื้อสินค้า' ข้ามยอดชำระคืน/ดอกเบี้ย แล้ว extract เป็น วันที่, ชื่อร้าน, จำนวนเงิน"*
- Return JSON array เหมือน expense ปกติ

## ⚠️ ข้อควรระวัง

1. **PDF Parser อาจไม่รองรับทุก Format** — บางแบงก์ใช้ PDF แบบ image-based (scan) ถ้า `pdf-parse` ดึง text ไม่ได้ ต้องใช้ OCR (ซับซ้อนขึ้นมาก)
2. **Token Cost** — PDF แต่ละฉบับอาจยาว 3000+ chars → ค่า AI token สูงกว่า expense ปกติ
3. **Statement มาเดือนละ 1 ครั้ง** — ไม่จำเป็นต้องสแกนทุกวัน อาจตั้ง Cron แยกเดือนละครั้ง

# AI Monitor Gateway Fix (V2.6.0)

## ปัญหาปัจจุบัน
- AI Connectivity Matrix ใน Command Center เพิ่งถูกแก้บั๊กไม่ยอมรัน โดยการตั้ง Crontab ให้ `cron-ai-monitor.js` ทำงานทุก 5 นาทีไปแล้วเมื่อสักครู่
- `cron-ai-monitor.js` เจอปัญหาตอนเช็คสถานะฝั่ง GPT 5.1 / 5.4 ว่า "Key missing" (แก้ข้ามด่านแรกด้วย Dummy Token ไปแล้ว เพราะใช้ OAuth)
- ตอนนี้พอยิง request เข้า Local Gateway (`http://127.0.0.1:18789/v1/chat/completions`) ผ่านโมเดล `openai-codex/gpt-5.1` ปรากฏว่า Gateway ตอบกลับมาแจก Error เป็น **404 Not Found**
- มีความเป็นไปได้สูงมากว่า `gmail-agent` และ `health-news` ก็ใช้งาน OpenClaw ไม่ได้และเจอ 404 เช่นกัน แต่ระบบไม่ล่มเพราะโค้ดมี Fallback ให้ไปเรียกใช้งาน Kie.ai (Gemini 2.5) ช่วยประคองชีวิตแทนโดยอัตโนมัติ

## Root Cause & Fix Summary
### สาเหตุ (3 ข้อ)
1. **Endpoint ปิดอยู่** — OpenClaw Gateway ปิด `/v1/chat/completions` by default ต้อง enable ด้วย `gateway.http.endpoints.chatCompletions.enabled = true`
2. **Auth mode ผิด** — `gateway.auth.mode` ตั้งเป็น `"none"` แต่ยังบังคับ auth → แก้เป็น `"token"` + sync token ให้ตรงกับ `.env`
3. **Model naming convention ผิด** — OpenClaw ใช้ agent-target model (`openclaw/main`) ไม่ใช่ provider model (`openai-codex/gpt-5.1`) → ต้องส่ง model เป็น `openclaw/main` แล้วใส่ header `x-openclaw-model: openai-codex/gpt-5.1`

### ไฟล์ที่แก้
- `discord-bot/scripts/cron-ai-monitor.js` — เพิ่ม `x-openclaw-model` header + ใช้ `openclaw/main` เป็น model
- `discord-bot/lib/gmail-agent.js` — เพิ่ม `x-openclaw-model` header + ใช้ `openclaw/main`
- `discord-bot/lib/health-news.js` — เพิ่ม `x-openclaw-model` header + ใช้ `openclaw/main` (2 จุด)
- `.env` — แก้ URL จาก `185.250.38.247` → `127.0.0.1` (gateway bind loopback)
- VPS `/root/discord-bot/.env` — เพิ่ม `GATEWAY_AUTH_TOKEN` + `OPENCLAW_GATEWAY_URL`
- VPS `openclaw.json` — enable chatCompletions + auth.mode=token + sync token

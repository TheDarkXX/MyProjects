# 🚀 V2.9.1 /bypass Command & Bypass-Diff Strategy

## 📝 เหตุผล (Context)
ปัญหาโปรแกรมจัดการโค้ดของหน้าต่างแชทค้างอยู่ที่สถานะ `0+0` (UI Freezing) เมื่อพยายามแก้ไขไฟล์ขนาดใหญ่หรือไฟล์ที่มีโครงสร้างซับซ้อน (เช่น `memory.md` และ `AGENT.md`) ทำให้สูญเสียเวลาและไม่สามารถรันงานต่อได้ การใช้ Powershell ปกติในการทำ String Replace ก็มีความเสี่ยงสูงต่อการลบไฟล์ผิดพลาดหรือติดช่องว่าง (White space) 

## ⚙️ กฎและกลไกกู้ชีพ (New OS-Level Bypass Mechanism)
เมื่อเกิดอาการค้าง ผู้ใช้จะกด `Stop Run` แล้วพิมพ์คำสั่งโค้ดลับตัวนี้ลงมา:
**`/bypass`** (หรือพิมพ์ว่า `bypass`)

ทันทีที่ได้รับคำสั่งนี้ AI จะต้องทิ้งวิธีเดิม แล้วทำงานตามลำดับ Bypass ทันที เพื่อหลีกเลี่ยงการประมวลผลความเปลี่ยนแปลงโค้ด (Diff) บนหน้าต่างแชท

### 📋 สเต็ปการทำงานของ AI เมื่อรันสูตร Bypass (Checklist)
- [x] 1. **ดึงบริบทเดิม Auto:** คว้าเป้าหมายและเนื้อหาบรรทัดล่าสุดที่ต้องการแก้ มาเริ่มทำงานต่อทันที
- [x] 2. **แบนเครื่องมือเก่า:** ห้ามใช้เครื่องมือ Edit ของแชท (`replace_file_content`) กับไฟล์ต้นฉบับตรงๆ โดยเด็ดขาด 
- [x] 3. **โหมดซุ่มเขียน:** ใช้พลัง AI สร้างและเขียนเนื้อหาฉบับอัปเดต 100% ซ่อนไว้ในไฟล์ชั่วคราว (เช่น `/tmp/safe_edit.md`) ผ่าน API โดยตรง เพื่อความแม่นยำขั้นสุด
- [x] 4. **สวมรอยระดับ OS:** ใช้ Powershell ทับไฟล์ต้นฉบับชนิดตาไม่กะพริบด้วยคำสั่ง `Copy-Item /tmp/safe_edit.md TargetFile.md -Force`
- [x] 5. **จบงาน:** แจ้งเตือนผู้ใช้ว่าอัปเดตไฟล์เสร็จสมบูรณ์ โดยไม่ต้องโชว์ Diff ในแชท

> **หมายเหตุ:** ปลอดภัย 100% ไว และป้องกันการชะงักของหน้าต่างแชทได้เด็ดขาด เป็น Best Practice ยืนพื้นสำหรับ AI!

# Bug Report & Fix: AI Monitor Status Not Updating (V2.7.1)

บันทึกสรุปปัญหาและวิธีการแก้ไข กรณีแท่งสถานะ AI ในหน้า Dashboard ไม่ยอมวิ่งเดินหน้า และค้างอยู่ที่เวลาเดิม

## 🐞 อาการของปัญหา (Symptom)
หน้า Command Center > AI Monitor ไม่ยอมอัปเดตข้อมูลแบบ Real-time ขบวนประวัติ (Incident Feed) และหลอดเขียวๆ ค้างอยู่ที่เวลาเดิม ถึงแม้ Cronjob ฝั่งหลังบ้าน (`cron-ai-monitor.js`) จะทำงานยิงเช็คครบทุก 5 นาทีตามปกติแล้วก็ตาม

## 🔍 ต้นตอของปัญหา (Root Cause)
เกิดจาก **2 สาเหตุหลัก** ที่ซ้อนทับกัน:

1. **Path Directory ผิดฝั่ง Production:** ตัว Express Server ฝั่ง VPS (`server.js`) ถูกตั้งค่าให้เสิร์ฟระเบียงหน้าบ้านจากโฟลเดอร์ `/root/brain-app/public/` แต่เดิมตัวสคริปต์ `cron-ai-monitor.js` ดันเขียนข้อมูล Export ออกไปอัปเดตใส่ `/root/brain-app/brain-app-public/` ทำให้ฝั่งหน้าเว็บไซต์เวลาดึง `/ai-status.json` ไม่เคยได้ไฟล์ข้อมูลใหม่เลย 
2. **Git Overwrite (ไฟล์ผีหลอกข้ามมิติ):** ตัวไฟล์ข้อมูลประวัติ AI (`ai-status.json`) ดันไปอยู่ในระบบที่ส่งขึ้น Git ไปด้วย ส่งผลให้ทุกครั้งที่เราทำการผลักโค้ดขึ้นเซิร์ฟเวอร์ (`git push vps`) ตัวระบบ Git จะเอาไฟล์แบบร่างประวัติเก่าๆ จากคอม Local ไป **เซฟทับประวัติของจริงบนเซิร์ฟเวอร์** ทำให้บันทึกเวลาโดนรีเซ็ตย้อนยุคกลับมา 

## 🛠 วิธีการแก้ไข (Resolution)

- [x] **แก้ไข Path การเซฟไฟล์:** แก้ไขโค้ด `cron-ai-monitor.js` ให้ตอนจบงานสั่งเขียนไฟล์ `ai-status.json` แจกจ่ายลงไป **ทั้ง 2 โฟลเดอร์เลย** 
   * `brain-app-public/ai-status.json` (สำหรับ Local)
   * `public/ai-status.json` (เพื่อให้เว็บฝั่ง VPS ดึงไปโชว์ได้จริง)
- [x] **ปลดล็อก Git Tracking (Untrack):** เพิ่มรายชื่อไฟล์ด้านล่างนี้ลงใน `.gitignore` สั่งให้ Git เลิกสนใจไฟล์กลุ่มประวัติเหล่านี้ 
```text
# AI Monitor status (don't overwrite server state)
data/ai-status.json
brain-app-public/ai-status.json
public/ai-status.json
```
- [x] **ลบไฟล์ออกจาก Git Cache:** รัน `git rm --cached` เพื่อปลดแอกไฟล์นี้ออกจากการครอบงำของระบบ Version Control (กันการโดนเด้งทับตอน Deploy)
- [x] **ล้างไพ่เซิร์ฟเวอร์:** ให้พายุของ Cron เริ่มนับและสร้างฐานข้อมูลขึ้นใหม่เองแบบ 100% ทำให้เส้นกราฟหน้าเว็บไหลได้อย่างอิสระไม่ติดบัคเดิมอีกต่อไป

> 💡 **สรุป:** การแก้ปัญหาครั้งนี้จัดการทั้งเรื่อง **"การแชร์โฟลเดอร์ผิดที่"** และ **"ปัญหาระบบเซฟทับข้ามโลก"** ทำให้ Dashboard กลับมาแสดง Real-time ได้ 100% แล้วครับ

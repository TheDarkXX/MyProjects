# V2.9.0 Sync AG Raw Data (PC Cross-Sync Plan)

## 📌 สรุปคอนเซปต์
ดึงข้อมูลจากการแชทแบบดิบๆ (Raw Conversation Data) ที่ AG เก็บไว้ ส่งข้ามเครื่องผ่าน API ไปเก็บลงฐานข้อมูลกองกลาง (`ag_logs`) เพื่อให้ "Doctor Skill" หยิบเอาเนื้อผ้าดิบๆ พวกนี้ไปย่อยเป็น Daily Journal สนุกๆ แล้วส่งทำคอนเทนต์ต่อ โดยไม่ต้องจัดการกับขนาดไฟล์มหาศาล

> `AG Raw Chat` ➔ `ag_logs` (ฐานข้อมูล) ➔ `Doctor Skill` ➔ `journal` (Story telling) ➔ `content_plans` (โพสโซเชียล)

## 📋 Checklist (Implementation)

### Phase 1: Database & API (VPS)
- [x] แก้ไฟล์ `server.js` เพื่อแทรกคำสั่งสร้างตาราง (Migration) `ag_logs` โดยบังคับ `UNIQUE(date, pc_source, conversation_id)` กันการบันทึกซ้ำซ้อน
- [x] สร้างไฟล์ API ใหม่ `routes/ag-logs.js` ทำระบบรับคำสั่ง (POST) เล็กๆ สำหรับให้ AG ยิงสรุปแชทยัดเข้ามา และทำจุดดึงข้อมูล (GET) เพื่อส่งให้ Doctor Skill ดู
- [x] เชื่อม (Mount) API `/api/ag-logs` เข้าในตัว Server ให้พร้อมใช้งาน

### Phase 2: AG Command (สั่งปุ๊บส่งปั๊บ)
- [x] อัปเดตไฟล์ `AGENTS.md` -> สร้างคำสั่งใหม่คือคำว่า `"sync log"`
- [x] ผูกคำสั่งให้เป็น: เมื่อสั่ง `"sync log"` ให้ AG มองย้อนแชทในวันนั้น, สรุปประเด็น, รวบตึงใส่ตัวแปร แล้ว POST ขึ้น API ทันที
- [x] แทรก [PRE-CHECK] ✍️ ในขั้นตอนทำ Daily Journal (สรุปวันนี้) ให้ Agent ดึงข้อมูลจาก API `/api/ag-logs` ก่อนเสมอ เพื่อเอาข้อมูลทั้งจากโฮมและออฟฟิศมากางบนโต๊ะรวมกันก่อนสรุปเรื่อง

### Phase 3: ย้อนหลังเวลา (Retroactive Sync)
- [x] ทดสอบสั่ง `"sync log ย้อนหลัง"` บนเครื่องออฟฟิศ ให้ AG ยิงย้อนไป 1 สัปดาห์เพื่อเป็นระบบจัดสต็อกข้อมูลตั้งต้น

### Phase 4: PC 2 (ฝั่งบ้าน)
- [x] กลับไปที่บ้าน เช็ค Hostname รัน `$env:COMPUTERNAME` แจ้งให้ AG รับรู้
- [x] ทดสอบรันคำสั่ง `"sync log"` จากเครื่องบ้าน เพื่อดูว่าข้อมูลวิ่งเข้าท่อไปรวมกับของฝั่งออฟฟิศได้เนียนกริ๊ป ไม่ทับซ้อนกัน
- [x] **เพิ่มระบบ Idle-First Auto-Sync (Phase พิเศษ)** — ติดตั้ง pre-push hook ให้ทำงานรอ 60s แล้วค่อย sync อัตโนมัติ ป้องกัน process ซ้อนทับ

# V2.9.1 - AI Monitor & Gateway Hardening

## Overview
การปรับปรุงและอุดช่องโหว่ปัญหา OpenClaw Gateway Crash (จากไฟล์ sessions.json บวม) และแก้ไขให้สคริปต์ Monitor การแจ้งเตือนต่างๆ ตรวจจับอาการ Flapping ได้แม่นยำ พร้อมทั้งแจ้งเตือนไปที่ Discord + Telegram

## Task List

- [x] วิเคราะห์หาสาเหตุอาการ OOM ของ Node.js ตอนเปิดโหลด Dashboard (เจอตัวการคือ `sessions.json` บวม 92MB จาก Heartbeat ping สะสม 3,696 แชท)
- [x] ซ่อมแซมระบบ Memory Crash: ทำการ Backup `sessions.json` เป็น `.bak` และลบไฟล์ `.tmp` ขยะทิ้งเพื่อปลดล็อกแรมเซิร์ฟเวอร์
- [x] แก้ไขบั๊ก Monitor 1: `cron-ai-monitor.js` ทำงานผิดพลาดเวลาจับอาการ Flapping โดยเปลี่ยนจากการนับ `fails` เปล่าๆ ไปเป็นการนับ `s === 0` (ขีดแดง) ย้อนหลัง 3 แท่งจากแถว `history` โดยตรง
- [x] แก้ไขบั๊ก Monitor 2: แก้ชื่อตัวแปร `DISCORD_TOKEN` ในสคริปต์ AI Monitor ที่ไม่ตรงกับไฟล์ `.env` (`DISCORD_BOT_TOKEN`) คืนชีพการแจ้งเตือน
- [x] อัปเกรด Alert Multi-channel: เพิ่มให้ `cron-ai-monitor.js` ยิง Alert เข้า Telegram เป็นช่องทางด่วนควบคู่ไปกับ Discord
- [x] ซ่อม Health Check: แก้ไข `health-check.sh` ให้เช็คเป้าที่ถูก (พอร์ต 18790) และลดเวลารอ Rate limit ลงจาก 2 ชม. เป็น 30 นาที
- [x] แจ้งเตือน Token: แก้ไขสคริปต์แจ้งเตือน OpenAI Codex หมดเวลา 7 วัน (`check-oauth-expiry.sh`) ให้แจ้งเตือนลง Discord ด้วย ควบคู่กับวิธีแก้ (Actionable hint)
- [x] Preventative Maintenance: เขียนกฎลงให้ `openclaw.json` ให้ Gateway บังคับ Auto-prune `sessions` (`maintenance.mode: enforce`) หลัง 14 วันเพื่อป้องกันฮาร์ดดิสก์และแรมระเบิดรอบถัดไป
- [x] ลั่นคำสั่ง `git push vps` ส่งโค้ดขึ้นไปบน Production Server อย่างสมบูรณ์

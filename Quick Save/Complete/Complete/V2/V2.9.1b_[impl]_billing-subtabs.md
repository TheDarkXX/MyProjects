# V2.9.1 Billing Sub-Tabs Architecture & Implementation Plan

## Background
Current state: **One "billing" tab** crammed with 3 columns (Upcoming Alerts, Bills, Subscriptions + Category Chart). On ultrawide this works visually, but management is limited — you can't search, filter, sort, or see detail views. As data grows, this layout becomes unmanageable.

**Goal:** Split into **3 sub-tabs** inside the billing section, each optimized for its purpose:

```
┌──────────────────────────────────────────────────────────┐
│  [Dashboard]    [Bills]    [Subscriptions]               │
│  ─────────────────────────────────────────────────────── │
│                    (content area)                         │
└──────────────────────────────────────────────────────────┘
```

## Architecture Decision: Sub-Tab State

**State Design:**
```javascript
// New billing sub-tab state
let ccBillingSubTab = 'dashboard';  // 'dashboard' | 'bills' | 'subscriptions'

// Each sub-tab gets its own filter/sort state
let ccBillsFilter = { status: 'all', sort: 'due_date' };
let ccSubsFilter  = { category: 'all', sort: 'next_due', search: '' };
```

**Benefits:**
- Switching sub-tabs doesn't lose filter/search state.
- URL hash preserves sub-tab (e.g., `#billing`).
- The main `ccLoadTab('billing')` still works — it renders the sub-tab bar + delegates to the active sub-tab renderer. No changes needed in `app.js` or sidebar navigation.

## Implementation Details (Tech Stack)

### Component 1: Tab Router Update (`command-center.js`)
Replace `ccRenderBillingTab()` with a new function that renders the sub-tab bar `ccSwitchBillingSubTab(tab)` and delegates the container to `ccRenderBillingSubContent()`.

### Component 2: Extract Render Functions (`command-center.js`)
- `ccRenderBillingDashboard()`: Uses the existing summary, upcoming alerts, and Pie Chart.
- `ccRenderBillsManager()`: Renders the full-screen table and state management logic.
- `ccRenderSubsManager()`: Renders the subscriptions view with view-mode toggle.
  - `ccRenderSubsListView()`: Master-detail split layout (30/70).
  - `ccRenderSubsGridView()`: Card grid layout (existing design).
  - `ccRenderSubsDetail(subId)`: Right panel detail renderer for list view.
  - `ccRenderSubsPaymentChart(subId)`: Bar chart (Chart.js) for payment history.

### Component 3: New CSS Styles (`style.css`)
- `.cc-bl-subtabs` — sub-tab bar with pill-style buttons
- `.cc-bills-table` — full-width table with proper column sizing
- `.cc-sub-card` — rich subscription card with favicon area, info rows, action bar
- `.cc-sub-grid` — responsive card grid (`grid-template-columns: repeat(auto-fill, minmax(300px, 1fr))`)
- `.cc-sub-split` — master-detail container (`display: grid; grid-template-columns: 30% 70%;`)
- `.cc-sub-list` — left column list styling with selected state glow
- `.cc-sub-list-item` — compact row with favicon, name, price, badge
- `.cc-sub-list-item.selected` — accent glow border for active selection
- `.cc-sub-detail` — right column detail panel with sections
- `.cc-sub-detail-header` — large icon + name + badges
- `.cc-sub-stat-cards` — row of 3 mini stat cards
- `.cc-sub-payment-chart` — payment history chart container
- `.cc-sub-detail-info` — metadata details section
- `.cc-view-toggle` — icon buttons for switching list/grid view

### Component 4: New State Variables (`command-center.js`)
```javascript
let ccSubsViewMode = 'list';        // 'list' | 'grid'
let ccSelectedSubId = null;         // selected subscription ID for detail view
```

### What Stays the Same
- **Modal logic** — `ccShowBillingModal()` remains untouched.
- **Backend API** — Endpoints `/subscriptions` and `/bills` stay exactly as they are.

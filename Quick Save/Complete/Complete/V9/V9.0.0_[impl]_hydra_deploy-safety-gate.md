# Hydra Deploy Safety Gate

This plan implements a strict deploy safety mechanism for the Hydra Evolver to prevent the autonomous deployment of broken code.

## Evaluation & Verdicts

### 1. EVO-20260423-122: Add pre-deploy BLOCKED file validation before auto-deploy
**Core Idea:** Prevent auto-deploy from modifying files classified as BLOCKED (like `scripts/hydra/*`) without human review.
| Dimension | Score | Reasoning |
|-----------|-------|-----------|
| Impact (🚀) | ⭐⭐⭐⭐ | High impact on system stability by preventing autonomous agents from breaking core architecture. |
| Feasibility (⚙️) | ⭐⭐⭐⭐⭐ | Very easy to implement since `classifyFileRisk()` already exists. |
| Urgency (⏱️) | ⭐⭐⭐⭐⭐ | Active regression occurred (9 errors). Needs immediate patching. |
| Alignment (🎯) | ⭐⭐⭐⭐ | Aligns with safety guardrails for autonomous systems. |
**Overall Score:** 4.5
**Recommended Verdict:** ✅ **APPROVE** (Merged)

### 2. EVO-20260423-263: Add deploy-time health check gate (node --check)
**Core Idea:** Run a syntax check (`node --check`) on affected JS files after applying edits but before committing. Rollback if it fails.
| Dimension | Score | Reasoning |
|-----------|-------|-----------|
| Impact (🚀) | ⭐⭐⭐⭐ | Prevents bad syntax/obvious errors from being deployed to production. |
| Feasibility (⚙️) | ⭐⭐⭐⭐⭐ | Running a dry-run/syntax check is standard and reliable. |
| Urgency (⏱️) | ⭐⭐⭐⭐⭐ | Active regression occurred. |
| Alignment (🎯) | ⭐⭐⭐⭐⭐ | Directly enables safe "Awakened" self-healing capabilities. |
**Overall Score:** 4.75
**Recommended Verdict:** ✅ **APPROVE** (Merged)

### 3. EVO-20260421-955: Add pre-deploy health check (Count errors)
**Core Idea:** Count errors before and after deploy, rollback if post-deploy errors exceed threshold.
| Dimension | Score | Reasoning |
|-----------|-------|-----------|
| Impact (🚀) | ⭐⭐⭐ | Good idea but reactive compared to pre-commit checks. |
| Feasibility (⚙️) | ⭐⭐⭐ | Flaky to implement; counting async errors from Discord or logs within a 30s window is prone to race conditions. |
| Urgency (⏱️) | ⭐⭐⭐⭐ | Addresses the same regression, but approach is less optimal. |
| Alignment (🎯) | ⭐⭐⭐⭐ | Aligns well, but implementation is messy. |
**Overall Score:** 3.5
**Recommended Verdict:** 🔄 **REVISE & MERGE** (Discard error-counting, use `node --check` instead).

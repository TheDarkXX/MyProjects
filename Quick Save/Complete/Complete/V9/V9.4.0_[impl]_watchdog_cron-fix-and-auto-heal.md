# 🚀 Implementation Plan: Watchdog Cron False Positive Fix + Auto-Healing

## 📌 Context (Compiled Truth)
**Root Cause:** The `runCronOutcomeCheck` used strict conditions (e.g., `DATE(start_time) = DATE('now')` or a strict 24h window), causing false positives on weekends, temporary gaps, or timezone mismatches. Additionally, there was no auto-healing when the scheduler was legitimately stuck.

**Final Implementation:**
- Modified the time window to 48 hours for schedule checks, adding a 72-hour grace period for weekends.
- Fixed timezone mismatches using `datetime('now', 'localtime')`.
- Added logic to distinguish 'no cron jobs scheduled' vs 'scheduler broken'.
- Added an auto-healing remediation function (`trigger-scheduler-recovery`) that explicitly restarts PM2 scheduler processes or triggers `cron-auto-planner.js` when the system determines the scheduler is stuck.
- **Note:** The core logic was auto-applied by Hydra via background proposals (EVO-189, EVO-952).

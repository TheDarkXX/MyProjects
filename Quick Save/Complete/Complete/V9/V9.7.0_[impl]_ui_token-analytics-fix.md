## 📋 The Problem
The OpenClaw Token Analytics dashboard went completely blank after a major telemetry gateway unification, leaving "0 MODELS" and blank Canvas blocks.

### The Investigation
1. Chart.js was aborting initialization on the Doughnut chart due to an empty dataset array resulting from an empty `intelSessions`.
2. `intelSessions` was empty because `mergedSessions` filtering dropped all valid Discord Gateway `.json` records.
3. The filtering dropped records because `Date.parse` or string date-math `s.date.split('-')` and `m - 1` without numeric coercion caused mismatched temporal overlap logic.
4. On top of that, there was a redundant data aggregation loop `dcSessions` injected explicitly in the `renderOclModelUsage` function from earlier implementations, meaning data for some elements was effectively counted twice before the bug wiped everything out.

## 🛠️ The Implementation
1. **Removed Redundant Aggregation**: Cleaned up legacy `dcSessions.forEach` loops in `renderOclModelUsage`, `renderOclSessionBreakdown`, and `renderOclModelDeepDive`. `mergedSessions` natively brings Discord usage telemetry from the unified gateway now, so duplicate merging corrupts chart integrity.
2. **Hardened Date Parsing**: Enforced `parseInt(y, 10), parseInt(m, 10) - 1, parseInt(d, 10)` in `Date` construction ensuring cross-browser resilience during timezone boundary filtering.

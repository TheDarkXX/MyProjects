# Hydra Evolution Pipeline — UI Enhancement

## Overview
This implementation significantly upgrades the observability and UX of the Hydra Evolution Pipeline dashboard. The ideator can now see short context (in Thai) on each Kanban card directly on the board. Furthermore, clicking on any card opens a rich modal containing the full proposal details (such as root cause, proposed changes table, score breakdown, and rollback plan) parsed directly from the `raw_data` JSON. 

We also implemented vertically stacked, collapsed sections beneath the main Kanban board to present historical and contextual data without cluttering the main view:
- **Deployed / Approved** 
- **Rejected / Rolled Back**
- **Expired / Icebox**
- **Pool Claimed / Processed**
- **AG Feedback History**

## Implementation Details
1. **API Expansion (`routes/hydra.js`)**:
   - Modified `/evolution-pipeline` to include `raw_data`, `pool_item_id`, and created distinct arrays for historical statuses (`deployed`, `rejected`, `expired`).
   - Integrated reading of the `data/hydra-memory/feedback-log.jsonl` into the response.
   - Updated `/pool` endpoint to return `claimed` and `processed` items.
2. **Frontend UI (`hydra-evo.js`, `hydra.css`, `index.html`)**:
   - **Clickable Cards**: Updated `renderEvolutionPipeline` and `fetchPool` to render cards with pointer cursor, hover effects, and a short description text (`.aw-card-desc`).
   - **Modals**: Added `showProposalDetailById()`, `showProposalDetail()`, and `showPoolItemDetail()` functions utilizing the existing `window.showAwakenedModal`. Styled the modal with the `.aw-modal-detail` class for a wider and more structured data view (badges, tables).
   - **Layout Fix for Collapsed Sections**: Separated the collapsed sections from the `.aw-pipeline` flex container by adding a new `<div id="aw-pipeline-collapsed" style="display:flex; flex-direction:column; gap:0; width:100%;"></div>` in `index.html`. This ensures the historical sections stack vertically at full width beneath the horizontally scrolling Kanban board.

## Deployment & Verification
- Incrementally updated layout, bumping the cache version each time via `node bump-cache.js`.
- Pushed updates to the VPS (`git push vps`).
- Ran an automated UI browser test using the browser subagent to verify that the detail modals open properly, Thai descriptions render correctly, and the collapsed history sections stack correctly below the board.

# 🚀 AI Monitor Optimization & System Audit

## 📋 Task Context & Why
The user requested several optimizations for the AI Monitor framework:
1. Ensure the UI strictly respects the requested order of models, instead of the default alphabetical sorting.
2. Reduce the cron schedule from 5 minutes to 15 minutes to prevent rate-limiting from OpenRouter/BytePlus while saving redundant costs.
3. Fix "Bad Request" errors for Qwen 3 32B and Nemotron Nano 12B VL, which were failing because the Kie Proxy did not support routing for those specific models.
4. Execute a full system audit of the entire Hydra + Inspector fleet to ensure 100% coverage.

## 🛠️ Implementation Details

### 1. UI Hardcoded Sorting (`aimonitor.js`)
Previously, the frontend dynamically sorted models by alphabetical name. This was changed to enforce a `DESIRED_ORDER` array.
```javascript
const DESIRED_ORDER = [
  'Kie: Gemini 2.5 Flash',
  'Openrouter: DeepSeek-V3.2',
  'Openrouter: Qwen 3 32B',
  'Openrouter: DeepSeek R1 70B',
  'Openrouter: qwen/qwen2.5-vl-72b-instruct',
  'Openrouter: nvidia/nemotron-nano-12b-2-vl',
  'BytePlus: DeepSeek-V3.2',
  'BytePlus: GLM-4.7'
];
```

### 2. Cron Optimization & Proxy Routing Fix (`cron-ai-monitor.js`)
- **Schedule:** Changed `*/5` to `*/15` in `crontab`. Adjusted `FORCE_CHECK_MS = 840000` (14 minutes).
- **Routing Fix:** Identified that `http://127.0.0.1:18810/openrouter/api/v1/chat/completions` (Kie Proxy) fails with `400 Bad Request` for `qwen/qwen3-32b` and `nvidia/nemotron-nano-12b-v2-vl`. These models were explicitly updated to bypass the proxy and hit `https://openrouter.ai/api/v1/chat/completions` directly.

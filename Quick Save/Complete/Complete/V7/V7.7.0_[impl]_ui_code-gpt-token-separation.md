## Objective
Fix the OpenClaw Dashboard metrics for "Code GPT"/AG. Specifically:
1. Prevent Code GPT generated tokens from inflating the server API/Cron cost metrics shown directly in the Hero Section.
2. Ensure Code GPT's actual underlying model strings (such as `gemini-2.5-pro` payload signatures that represent Gemini 3.1 Pro) are translated into visually accurate labels instead of "Unknown" or just repeating the fallback name.
3. Replace the `(DC)` tags that attached themselves to Code GPT because it was incorrectly grouped inside the Discord usage loop, giving it a proper `(AG)` badge instead.

## Changes Implemented

### 1. Token Isolation (`openclaw.js` / Hero Section)
* Re-wrote the `renderOclHero()` to segregate `sessions` and `allSessions`:
  * Filtered `isIDEAgent(s.agentId)` out of `heroSessions` resulting in accurate total input/output metrics for background system services.
  * Extracted the IDE token sums separately (`agTokensToday`).
* Added UI element `ocl-hero-ag-tokens` directly below the "tokens today" text in `index.html` to clearly render `💻 AG Tokens: {count}` conditionally.

### 2. Tag & Chart Accuracy (`openclaw.js` / Doughnut Charts)
* Segregated `ag` source tags from `dc`.
* Explicitly mapped IDE provider signature combinations back to their human-selected variants used by Antigravity:
  * `gemini-2.5-pro` -> `gemini-3.1-pro` 
  * `claude-opus-4.5` -> `claude-opus-4.6-thinking`
  * `claude-sonnet-4` -> `claude-sonnet-4.6-thinking`
* In the Legend renderer, recognized `ag` mapping and implemented a distinct pink pill `[AG]` to clarify where the data slice originates.

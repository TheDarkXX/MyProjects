# V7.5.1: Hydra Auto-Audit — Watchdog + 3-Day Digest + UI Widget

## Why This Was Built

After V7.5.0 fixed the root causes of Hydra's 14% success rate, the user asked: *"Should we create an agent to check the 3 companions regularly?"*

Decision: **No new agents. Use existing infrastructure instead.** An agent-that-monitors-agents adds unnecessary complexity. Better to wire monitoring into Watchdog (already running every hour), upgrade the existing report script, and surface live stats in the UI.

## What Was Built

### 1. Watchdog Hydra Health Checks (goals.js)

Added a new `hydra-health` group to `scripts/watchdog/goals.js`:

```javascript
'hydra-health': {
    name: 'Hydra Agent System Health',
    priority: 'HIGH',
    checks: [
        {
            id: 'hydra-success-rate',
            // Alerts on Discord if Evolver success rate drops below 20% in 7 days
            query: `SELECT CASE WHEN total = 0 THEN 100 ELSE ROUND(deployed * 100.0 / total, 1) END as cnt ...`,
            expect: 'cnt >= 20',
            severity: 'WARNING'
        },
        {
            id: 'hydra-hallucination-rate',
            // Alerts if Ideator hallucinates fake file paths more than 3 times/day
            query: `SELECT COUNT(*) as cnt FROM evolution_log WHERE deploy_result LIKE '%ideator_hallucination%' AND created_at >= datetime('now', '-24 hours')`,
            expect: 'cnt <= 3'
        },
        {
            id: 'hydra-last-deploy',
            // Alerts if Evolver hasn't deployed anything in 5+ days
            query: `SELECT CAST(julianday('now') - julianday(MAX(deployed_at)) AS INTEGER) as cnt FROM evolution_log WHERE status = 'deployed'`,
            expect: 'cnt <= 5'
        }
    ]
}
```

Watchdog fires Discord alerts when these thresholds are breached.

### 2. 3-Day Evolution Digest (weekly-evolution-report.js)

Changed from weekly (every Monday) to every 3 days (`0 9 1,4,7,10,13,16,19,22,25,28 * *`).

Report now includes:
- `hydra_success_rate_pct`: Percentage of proposals that led to successful deploys
- `proposals_hallucinated`: Count of hallucinated file path rejections
- `top_reject_reasons`: Top 5 rejection reasons (diagnosis or reason text)
- `model_performance`: Which model was responsible for each successful deploy
- `proposals_rejected`: Raw rejected count

System prompt updated to include quality analysis framing:
```
1. 🏥 สุขภาพ Hydra (Success Rate, Hallucination Count)
2. 🚀 Deploy ที่สำเร็จ & สิ่งที่ถูกปรับปรุง
3. 🧠 โมเดลไหนเก่งสุด? (Model Performance)
4. ⚠️ ปัญหาที่พบบ่อย (Top Reject Reasons)
5. 💡 สกิลใหม่ที่เรียนรู้
```

### 3. Audit Mini-Bar Widget (index.html + hydra-evo.js + routes/hydra.js)

New `/api/hydra/audit-stats` endpoint:
```javascript
app.get('/audit-stats', (c) => {
    return c.json({
        success_rate_pct: ...,
        total_proposals_7d: ...,
        deployed_7d: ...,
        hallucination_count_24h: ...,
        days_since_last_deploy: ...
    });
});
```

UI widget renders in Awakened Protocol tab, below the 4 main stat cards:
- **Success Rate**: Green ≥30%, Orange 15-29%, Red <15%
- **Hallucinations**: Green = 0, Orange ≤3, Red >3
- **Last Deploy**: Green ≤2d, Orange ≤5d, Red >5d or Never

Fetched via `fetchHydraAudit()` which is called in `initAwakenedProtocol()`.

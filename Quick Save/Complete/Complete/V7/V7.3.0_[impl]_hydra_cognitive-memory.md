# 🧠 Hydra Cognitive Memory Engine & Cost Optimization (V7.3.0)

## Overview
Transformed the Hydra meta-agent pipeline (`Inspector`, `Ideator`, `Evolver`) from stateless cron scripts into a **self-improving, agentic architecture**. Additionally, drastically optimized the Model Selection Tiers to drop total operating cost from ~$11.34/month to ~$2.00/month by utilizing highly efficient tier structures.

## 🛠 Features Implemented

### 1. Persistent Memory Layer (`lib/hydra-memory.js`)
Created a structured file-based memory system under `data/hydra-memory/` to store:
- `feedback-log.jsonl`
- `learned-rules.md`
- `prompt-patches.md`

### 2. Closed Learning Loop
- **Inspector (Diagnostician):** Loads `suppression-rules.md` to filter out known false positives. Includes GC9 regression tracking.
- **Ideator (Strategist):** Injects `prompt-patches.md` into instructions dynamically and filters proposals against `rejection-patterns.jsonl` to ensure it stops proposing known rejected ideas.
- **Evolver (Executor):** Tracks success rates of generated code per model and dynamically re-orders AI model fallbacks (`stage1Models`) based on historical success rates.

### 3. Weekly Reflection Engine (`scripts/hydra/weekly-reflection.js`)
A cron job running on Sundays at 06:00. This agent acts as the "Dean", reading raw feedback logs and distilling them into compounding wisdom. It updates `prompt-patches.md`, suppression rules, and generates model preference analytics.
- **Engine Model:** Runs exclusively on **Claude 4.6 Sonnet** for top-tier reasoning and distillation ability.

### 4. Model Tier Optimization (-83% Cost Reduction)
Reconstructed the model tiers used by the 3 main Agents to hit the ultimate performance/cost ratio.
> *Note: OpenRouter retired the older Claude 3.5 Sonnet, accelerating this migration.*

**Final Model Tiers (Live):**
- **Inspector** (Log Analysis/SQL):
  - Tier 1: **DeepSeek R1 Llama 70B** ($0.70/m)
  - Tier 2: DeepSeek-V3.2 (BytePlus Free)
  - Tier 3: GLM-4.7 (BytePlus Free)
- **Ideator** (Planning & Architecture):
  - Tier 1: **MiniMax M2.7** ($1.20/m)
  - Tier 2: DeepSeek-V3.2 (BytePlus Free)
  - Tier 3: GLM-4.7 (BytePlus Free)
- **Evolver** (Coding & Execution):
  - Tier 1: **MiniMax M2.7** ($1.20/m) — *Sorted 1st via memory logic*
  - Tier 2: MiniMax M2.5 ($0.99/m)
  - Tier 3: DeepSeek-V3.2 (BytePlus Free)

## ✅ Verification
1. `node scripts/hydra/weekly-reflection.js` tested against Claude 4.6 Sonnet successfully generated prompt patches.
2. `/posttest 3 agent` passed End-to-End on the VPS with zero fallback errors. Total pipeline cost dropped heavily while maintaining execution quality.

## ⛔ Rule Updates & Pointers
- Do not use old generic provider strings. Anthropic models must explicitly use `anthropic/claude-sonnet-4.6`.
- For budget alerts, `lib/budget-guard.js` must strictly monitor these agents' thresholds.
- Ideator uses `isLlmFormat: true` because MiniMax requires correctly mapped prompt completions.

## 📝 Addendum: Non-Core Hydra Scripts Review
Reviewed the remaining 18 Hydra scripts to evaluate if they should also be transformed into full cognitive agents. 
**Decision:** They must **remain stateless cron scripts**.
- **Reasoning:** Cognitive Memory is only valuable for closed feedback loops (`Inspector` -> `Ideator` -> `Evolver` + `Weekly Reflection`). Scripts like `visionary.js`, `dream-mode.js`, `impact-measurer.js` are either one-shot brainstormers (where memory biases creativity) or pure SQL deterministic trackers (where AI memory provides zero ROI). Memory overhead for these scripts would massively inflate cost with no systematic benefit.

# Command Center Height Collapse Bug Fix

## Symptom
The `#page-command` element representing the Command Center ("Plans & Events", etc.) was completely invisible when navigating to it. Despite all JavaScript executing successfully and injecting over 21,000 bytes of HTML into the DOM, the screen remained blank (displaying only the dark blue application background).

## Root Cause Analysis
1. **Initial Misdiagnosis (Opacity & Animations):** Initially, it was suspected that the manual JavaScript `opacity: 0` -> `opacity: 1` transition inside `ccLoadTab()` was responsible for hanging the render flow. While this was removed to improve determinism, it did not solve the visual absence.
2. **True Cause (Flexbox Height Collapse):** The layout hierarchy on `#page-command` is entirely flex-based:
   - `body > .app-layout` has `display: flex; min-height: 100vh;`
   - `.app-layout > .main-content` has `flex: 1`
   - `.main-content > #page-command` has `flex: 1; display: flex; flex-direction: column`
   - `#page-command > .cc-shell` had `height: 100%`

In modern browsers (Chrome/Safari), if a flex child (`.main-content`) lacks an *explicit* `height` property (relying only on flex stretching), any descendant attempting to map its height using percentage-based constraints (`height: 100%`) will fail to resolve the percentage against the parent's block formatting context. Resultantly, `.cc-shell` evaluated to `0px` in height.
Compounding this, `.cc-content` inside the shell had `overflow-y: auto`. A zero-height container hiding its overflow rendered the entire UI completely invisible.

## The Fix
1. **CSS Layout Update:**
   Modified `.cc-shell` in `style.css` to act as a proper flex-growing element instead of depending on percentages:
   ```css
   /* Before */
   .cc-shell { display: flex; flex-direction: column; height: 100%; gap: 0; }

   /* After */
   .cc-shell { display: flex; flex-direction: column; flex: 1; min-height: 0; gap: 0; }
   ```
2. **Animation Stability:**
   Removed the legacy `pageEnter` fade-up animation from `.page-content` elements, allowing navigation between tabs to snap cleanly without risking CSS `transition-end` race conditions.

## Takeaway
Never use `height: 100%` inside a flex container that itself is stretched via `flex: 1` unless the container has a defined pixel or viewport-relative height. Always use `flex: 1; min-height: 0` for nested vertical stretching.

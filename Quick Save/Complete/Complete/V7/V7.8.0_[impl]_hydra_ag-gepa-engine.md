# AG-GEPA: Custom Self-Evolution Engine

## Overview

Based on the Hermes Agent's GEPA algorithm, we successfully designed and implemented **AG-GEPA (Autonomous Genetic-Pareto Prompt Evolution)** inside Hydra. It leverages our existing infrastructure to drive self-improvement via Reflective Mutation and Pareto Selection without the heavy overhead of DSPy or separate Python services.

## What Was Built

1. **`scripts/hydra/ag-gepa.js`**: Core evolution engine doing the 4 phases:
   - **Harvesting:** Collects feedback from `skill-emit.jsonl`, `corrections.md`, and execution traces.
   - **Reflective Mutation:** Generates improved rules via MiniMax.
   - **Evaluation:** Scores generated variants on Correctness, Specificity, and Safety using Qwen QWQ-32B via AI Gateway.
   - **Pareto Selection & Deployment:** Uses Risk Classification (`SAFE`, `MEDIUM`, `HIGH`). Drops safe items directly into `prompt-patches.md` and generates `/evo` stubs for human review for the rest.
2. **`lib/hydra-memory.js` Updates**:
   - Tracking `evolution-history.jsonl`
   - Added APIs: `readEvolutionHistory`, `appendEvolution`, `getWorstPerformingTarget`
3. **Web UI & Route (`/api/hydra/gepa-history`)**:
   - Added a new `🧬 GEPA` tab inside the Awakened Protocol UI (`brain-app-public/index.html`, `hydra-evo.js`)
   - Presents historical evolution runs and radar chart/eval scores.
4. **Cron Job Integration**:
   - Scheduled AG-GEPA to run every 3 days (e.g. Wednesday, creating rhythm with Sunday weekly reflection)
   - Cost optimized natively down to ~$0.01 per run.

## Why this Architecture?

- By utilizing the localized `evolution-history.jsonl`, the OpenClaw AI Gateway `18810`, and `hydra-memory.js`, we bypassed all dependencies heavily needed by Hermes Agent's original algorithm.
- Cross-model evaluations limit self-bias per cycle.
- Enacts human checks automatically via `/evo`.

# Deep Root Cause Analysis: Awakened Protocol Stuck Items

## Problem Statement
The Awakened Protocol dashboard has persistent issues:
1. **"✅ Mark Done" buttons** keep appearing on items that should have been cleared
2. **"⏳ Awaiting Deployment..."** badge permanently stuck
3. Symptomatic fixes applied 1-2 times already but issues recur

## Root Cause Analysis
1. **Bug 1: "⏳ Awaiting Deployment..." Permanently Stuck**
   - **Cause:** Evolver cron's auto-deploy query filtered `risk_level IN ('safe', 'medium')` and ignored `'low'` risk items.
2. **Bug 2: "Mark Done" Items Keep Reappearing (Recycling Problem)**
   - **Cause:** Both Inspector and Ideator used exact `title` matching for deduplication. Inspector titles contained highly dynamic data (e.g. "GC8: Evolution rate 13%" vs "14%"). Ideator LLM generated slightly different wording each run. The deduplicator logic failed, creating redundant items.
3. **Bug 3: Zombie Items**
   - **Cause:** Items sitting in `ideated`, `duplicate`, `failed_doctor`, and aging `awaiting_human` state had no automated lifecycle cleanup to clear them from DB.
4. **Bug 4: Auto-Markdone Script Format Mismatch**
   - **Cause:** Script `vps-auto-markdone.js` matched on `EVO-...` prefixes, but UI approved items had `EDIT-...` prefixes format.

## Architectural Changes (Deployed V7.0.4)

### Component 1: Deterministic `dedup_key` System (via PagerDuty pattern)
Instead of matching dynamic strings in titles, each anomaly check now enforces a deterministic key (e.g. `gc8_evolution_rate`). 
- Inspector queries by `dedup_key`.
- Added **Exponential Backoff**: If same `dedup_key` alerts ≥3 times in 30 days, the suppression window is extended from 7 days to 14 days.

### Component 2: Category Lock & Inheritance (via Renovate pattern)
Ideator prevents generating duplicate proposals targeting the same system error by checking the active category before LLM execution.
- If an active proposal exists with same category/dedup_key, it halts and marks it as `duplicate`.
- New proposals now inherit `dedup_key` from Inspector findings.
- **Auto-Supersede** mechanism implemented: A new completed proposal supersedes any older uncompleted ones targeting the same `dedup_key`.

### Component 3: Lifecycle Management (via Kubernetes pattern)
Inserted `lifecycleCleanup()` hook at the start of `evolver.js`.
- Automatically expires stale anomalies and zombie proposals:
  - `awaiting_human` > 7 days → `expired`
  - `ideated` > 3 days → `expired` 
  - `finding` > 3 days → `expired`
- Corrected risk filter bug to handle `'low'` risk payload for `evolver.js`.

### Component 4: Pipeline GUI & Audit Trails
- Updated `hydra.js` stats correctly evaluating actionable items.
- Added visible manual "Mark Done" buttons for `approved` items on the Frontend.
- **Dedup Badge (`🔑 Key: ...`)** added to every card allowing visual inspection of AI grouping logic.
- **🧹 Auto-Cleaned History (Max 30)** Dropdown appended to the Findings column. It stores locally removed UI items (`superseded`, `expired`, `duplicate`) allowing humans to audit AI deletion choices without visual clutter.

### Component 5: Database Migration & Storage Sweep
- Added `dedup_key TEXT` column onto `evolution_log`.
- Triggered aggressive sweep of all local database `ideated`, `duplicate`, `approved` and stale `awaiting_human` statuses on the production VPS.
- Configured a scheduled **Hard Delete (14-day limit)** for unviable records to conserve backend SQL storage.

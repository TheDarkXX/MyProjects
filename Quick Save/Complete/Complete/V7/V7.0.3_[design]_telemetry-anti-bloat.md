# Design: Telemetry Anti-Bloat Sweep

## Context & Decision
After completing the OpenClaw Dashboard UI Revamp (V7.0.0) and hotfixing its missing CSS classes (V7.0.2), the user observed that the "Telemetry Coverage" banner was static and taking up unnecessary vertical space. Because we had indeed achieved **100% coverage** via the Unified AI Gateway (`V7.0.0_[impl]_unified-ai-gateway-migration.md`), the banner no longer served a purpose as a "visual reminder to migrate".

The user correctly invoked the **Anti-Bloat Principle** (from the Karpathy ruleset):
> "if it 100% , will we nedd it , it just static value . or we removed it or not"

**Decision:**
Remove the oversized Banner HTML blocks entirely. Provide the tracking validation (100%) as a single line in the existing `💡 System Health` widget. Clean up all associated dead CSS classes.

## Implementation Details
1. **HTML Removal:** Deleted `<div class="ocl-tracking-banner" id="ocl-tracking-status">` and its nested properties (progress bar, 22% static values, legends) from `index.html`.
2. **CSS Purge:** Ran a thorough sweep of `style.css` and deleted 60 lines of tracking banner definitions (`.ocl-tracking-banner`, `.ocl-tracking-bar-wrap`, `.ocl-tracking-header`, etc.).
3. **JS Injection:** Updated `renderOclHealth` in `openclaw.js` to insert a static system status check:
   ```javascript
   <div class="ocl-health-item">
     <span class="ocl-hi-icon" style="color:#10b981">📡</span>
     <span>Telemetry</span>
     <span class="ocl-hi-val ocl-hi-green">100% Tracked</span>
   </div>
   ```

## Post-Implementation Verification
1. **Cache Busting:** Cache string bumped to `v=20260419_124543` to ensure immediate UI update.
2. **VPS Deployment:** Standard PM2 rolling restart via `git push vps`.
3. **Browser Smoke Test (`/webtest`):** Confirmed the giant banner is completely gone from the hero section, yielding a much cleaner dashboard aesthetic. The 📡 icon safely renders alongside standard widgets in the System Health section.

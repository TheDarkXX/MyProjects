# Telemetry Hotfix: Gateway Stream Flags, Git Overwrite Bug, and Atomic Writes

## 📌 Context
User reported that AI Gateway metrics spanning from 00:00 to 14:47 today completely vanished from the UI, despite yesterday's metrics correctly populating. Initial investigation into `ai-gateway.js` revealed that `stream: true` completions were not emitting tokens due to missing OpenAI parameter requirements, but further Deep Dives exposed an overlying, larger, non-Gateway related persistence bug affecting the legacy system.

## 🚀 Root Cause Analysis
1. **The Git Push Overwrite Bug (Primary Cause):**
   - The `/root/brain-app/discord-bot/data/` folder was still being tracked by the local Git repository.
   - During a deployment at `14:47` today, a `git push vps` execution restored the local stale snapshot of `usage.json` (which ended at exactly yesterday 23:59), obliterating the entire morning's telemetry data running dynamically on the VPS. This completely explains why yesterday's data was perfectly intact while the morning's records were cleanly wiped.
2. **Missing `include_usage` Flag for Streams (Secondary Bug):**
   - Inside the newly deployed (V7.0.0) `ai-gateway.js` proxy hook, OpenAI-compatible APIs (OpenRouter, Kie) were sent `stream: true` requests but without `stream_options: {"include_usage": true}`. 
   - This caused backend endpoints to terminate streams without emitting the required usage blocks, resulting in undefined token totals.
3. **Race Condition during I/O:**
   - With both PM2 nodes (`ai-gateway` & `discord-bot`) writing asynchronously to a shared telemetry flat file, multi-read/write block I/O corruptions were theoretically possible via race conditions.

## 🛠️ Execution & Fixes

### 1. Data Recovery & File Isolation (Zero Deployment Wipeout)
- `git rm --cached` was executed to decouple `discord-bot/data` from Git.
- Added `discord-bot/data/` to `.gitignore`.
- Recovered the wiped Morning telemetry data using archived raw logs from `~/.pm2/logs/discord-bot-out.log` and cleanly injected it directly back into `usage.json`, saving roughly 30,000 lost tokens.

### 2. Stream Generation Tracking Emitted
- Rewrote the proxy filter inside `ai-gateway.js`. Every `req.method === 'POST'` execution capturing `parsedBody.stream === true` now forcefully mutates the body to append `parsedBody.stream_options = { include_usage: true }`. This guarantees API providers emit the trailing telemetry schema block for parsing.

### 3. Atomic Write Implementation (Race-Condition Hardening)
- Refactored `fs.writeFileSync()` logic across `ai-gateway.js` and `ai.js` into Atomic Writes utilizing `.tmp` intermediary files and synchronous OS-level `renameSync()`.
```javascript
// Implementation snippet for atomic write
const tmp = USAGE_FILE + '.tmp';
writeFileSync(tmp, JSON.stringify(data, null, 2));
renameSync(tmp, USAGE_FILE);
```

## 🔄 Deployment & Validation
- Executed `scp` manually pushing patches.
- Hot-reloaded processes via `pm2 restart ai-gateway && pm2 restart discord-bot`.
- Ran `git push vps` to sync the `.gitignore` safely.
- Verified dashboard visually reflects recovered data and handles new generation loads identically to proxy schemas.

## Feature Details

The user requested an automation tool to pull recorded credit card statement transactions (already aggregated by month in the `cc_credit_cards` database) into the `cc_bills` system so that they don't have to manually input the total amounts every month.

The user opted to sum up all payees (Personal, Ashira, etc.) together per bank, allowing them to manually assign default recurring due dates directly in the Bills once imported.

### Implementation Checklist
1. **Backend Integration** (`routes/command-center.js`):
    - Added `GET /api/cc/credit-cards/statement-summary` to query distinct statement months and compute sum totals of prices, grouped by `credit_card`. Also includes a check if a bill containing the specific bank name and month already exists.
    - Added `POST /api/cc/bills/generate-from-cc` which receives array elements of totals and banks, and iteratively creates `cc_bills` rows. Included a deduplication safeguard that skips generation if the `existingBills` query matches.

2. **Frontend UI** (`brain-app-public/command-center.js`):
    - Appended a new ✨"Import CC Bills" button with purple gradient styling to differentiate it from standard item creation.
    - Added `ccShowImportCCBillsModal()` which constructs an elaborate modal similar to `ccShowBillingModal`, but displaying a preview table listing: `[Bank Name, Transaction Count, Total THB Amount, Due Date input]`.
    - Integrated with bank-specific UI accent colors.

## Changelog
- **Auto-pull Trigger (`discord-bot/scripts/cron-cc-statements.js`)**: Updated the system to completely eliminate the need for manual clicking via the UI! Now, whenever the AI finishes scanning the statements and parsing them locally successfully, the cron script iterates and queries `http://127.0.0.1:3001/api/cc/bills/generate-from-cc` automatically.
- **Smart Due Date Pre-filling**: Added intelligent offset default dates based on user requests natively within the `cron-cc-statements.js` loop *and* `brain-app-public/command-center.js` frontend table:
  - BBL: 3rd of the next month
  - Krungsri: 8th of the next month
  - KBank: 10th of the next month
  - SCB: 12th of the next month

### Architectural Overhaul (Phase 2) — 03:30
- **Credit Card Exception Rule:** CC Bills logic reverted to a literal "New Card" approach every month. Modified the logic to enforce assigning `cycle: monthly` natively to indicate nature, while adding an exception in `PUT /api/cc/bills/:id/paid` to process it like a `once` payment mechanically (bypassing generic auto-spawning) since the AI already handles monthly spawns dynamically.
- **Uncluttering Standard Monthly Bills:** Rewrote the overarching standard `monthly` bills architecture! When marked Paid, they will immediately spawn a "New Card" for the adjacent month in the DB (preserving full `paid = 1` legacy history!).
- **T-20 Hiding Logic:** Implemented an implicit `GET` filter to intentionally obliterate new spawned cards from the UI *unless* their `due_date` drifts within <= 20 days. This flawlessly solves the "look ahead clutter" problem allowing variable expense estimates to gracefully pop into view just in time for editing and payment. 
- **Production Migration:** Built and launched `discord-bot/scripts/fix-cc-cycles.js` directly against the live `db/brain.db` infrastructure on the VPS to bulk update legacy cards overriding them to exactly fit perfectly within the new `monthly` ecosystem. Fix validated.

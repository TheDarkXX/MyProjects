# V7.1.0 [impl] AG-Skills — Hybrid Versioning & Session Lock

## Context & Motivation
The user noted that the previous versioning strategy caused version numbers to inflate rapidly (e.g., V6 reaching 29 files up to V6.9.3 within a short period) due to subsequent saves in the same chat session generating completely new version bumps. Additionally, filenames like `V6.5.3_[hotfix]_discord-tasks-cutoff-sync.md` lacked discoverability for AI search, making it difficult to locate files related to specific components.

## Implementation Details

### Hybrid Naming Convention
A new filename format was introduced to make historical records easily discoverable via `grep` and system-wide searches:
`V{x.y.z}_[type]_component_clear-description.md`
- **`component`**: A mandatory keyword (e.g., `hydra`, `watchdog`, `ui`, `ag-skills`) used to group tasks visually and programmatically.

### Intelligent Session Lock
The `save.md` skill logic was updated to prevent version inflation within the same session.
- **Batched Updates**: Any subsequent `/save` calls occurring in the exact same `conversation` ID will not bump the version or create a new file. Instead, the `save.md` skill will append a `## Changelog` section to the existing document, tracking updates chronologically.

### Rule Enforcement
Rules were modified in:
1. `AGENTS.md`: Formalized the **Session Lock & Batching Rule** as an Iron Rule.
2. `docs/skills/save.md`: Refactored **Phase 1 (Intelligent Context Saving)** to scan both `Complete/` and `Active/` for an existing file with the current `conversation` ID before creating a new active document. 

### Why No Retroactive Updates Were Made
Old Quick Save files (V6, V5, etc.) were left unchanged to preserve Git history and cross-references. The new rules apply exclusively to post-V7.1.0 files to ensure zero-regression stability.

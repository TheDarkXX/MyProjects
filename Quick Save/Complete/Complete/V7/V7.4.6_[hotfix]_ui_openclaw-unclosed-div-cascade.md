## Issue
Several modules under the `Life` and `Workspace` categories (notably Journal, Team, Agents, Command Center) were rendering as completely "blank" or pitch-black screens. Pages rendered prior in the DOM flow, such as `#notes` and `#tasks`, functioned perfectly. 

Earlier investigations mistakenly attributed the bug to CSS flexbox height collapses and stale animation opacity states (because the affected elements did report `opacity: 0` or 0-height during inspection). After an aggressive rollback to commit `068d93b`, the issue persisted.

## Investigation & Root Cause
By analyzing the DOM depth sequentially, we discovered a mismatch:
- `page-openclaw` started at depth **1**.
- The next page, `page-health`, started at depth **2**, and all subsequent pages started at depth **2** or **3**.
- At the end of the file, the depth dropped to **-1** indicating an extra `</div>` at the bottom which acted as a balancing tag for the missing one.

The actual missing `</div>` belonged to `<div class="ocl-hero">` inside the `#page-openclaw` container. Because the div was never closed, the `#page-openclaw` wrapper consumed the remainder of the document, nesting all subsequent pages inside it. 

When navigating to `#journal` (or any other page after openclaw), the `navigateTo` script correctly added `page-hidden` to `#page-openclaw`. Because `#page-journal` was physically nested inside `#page-openclaw`, the `display: none!important` rule on the parent overrode everything, causing the target pages to be invisible.

## Fix
- In `brain-app-public/index.html`, inserted the missing `</div>` immediately following the `ocl-hero-chips` block, successfully closing the `ocl-hero` section.
- Executed `node bump-cache.js` to ensure the HTML cache structure was bumped dynamically.
- Deployed via `git push vps`.

## Verification
- Sent a browser subagent to visually verify `#journal`, `#expenses`, `#agents`, and `#team`.
- All pages are now confirmed to be visible, correctly sized, and fully functional, proving that the DOM structure has been restored and siblings are no longer accidentally nested under the `openclaw` page container.

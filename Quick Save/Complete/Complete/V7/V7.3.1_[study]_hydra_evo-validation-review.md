# 🧠 Hydra /evo Review — Evolver Safety Gates (2026-04-19)

## Context

All 4 proposals share the **same root incident**: a bad watchdog-fix proposal was auto-deployed by Evolver without proper validation, generating **21 runtime errors** in production.
They attack the same problem from 4 different angles — all in `scripts/hydra/evolver.js`.

---

## Proposal Evaluations

### 1. EVO-20260419-145 — Pre-deploy Validation Gate + Post-deploy Auto-Rollback

**Core Idea:** Add `validateEditsAndSyntax()` before deploy; add `postDeployHealthCheck()` after push that rolls back if error count > threshold (5).

| Dimension | ⭐ Score | Reasoning |
|---|---|---|
| 🚀 Impact | 5/5 | Auto-rollback is a true safety net — the exact gap that caused the incident |
| ⚙️ Feasibility | 4/5 | `node --check` + git rollback is straightforward; async fire-and-forget is safe |
| ⏱️ Urgency | 5/5 | Active bleed — without this, any bad patch repeats the 21-error incident |
| 🎯 Alignment | 5/5 | Core self-healing property of Awakened architecture |
| **Overall** | **4.75 / 5** | |

**Verdict: ✅ APPROVE**

---

### 2. EVO-20260419-192 — Block Auto-Deploy for `scripts/hydra/` (BLOCKED tier)

**Core Idea:** Move `scripts/hydra/` from `HIGH` → `BLOCKED` risk tier so ALL Hydra self-modifications require manual review.

| Dimension | ⭐ Score | Reasoning |
|---|---|---|
| 🚀 Impact | 4/5 | Eliminates the entire class of "Hydra accidentally breaks itself" bugs |
| ⚙️ Feasibility | 5/5 | Single line change — trivially safe |
| ⏱️ Urgency | 5/5 | Direct response to the exact incident; stops the bleeding immediately |
| 🎯 Alignment | 4/5 | Trades some autonomy for safety — appropriate tradeoff for meta-agent self-edits |
| **Overall** | **4.5 / 5** | |

**Verdict: ✅ APPROVE**

> ⚠️ **Note:** This is complementary to EVO-145, not a replacement. EVO-145 handles *all* deploys; EVO-192 adds a hard block specifically for Hydra's own self-modification code path. Both should ship together.

---

### 3. EVO-20260419-615 — Pre-deploy Syntax Validation Before applyEdits + `vm.compile`

**Core Idea:** 1) Replace `split/join` in `applyEdits` with regex+word-boundary. 2) After each edit, call `vm.compile` to validate JS syntax before writing.

| Dimension | ⭐ Score | Reasoning |
|---|---|---|
| 🚀 Impact | 4/5 | Catches syntax errors *before* file is touched — earlier than EVO-145's `node --check` |
| ⚙️ Feasibility | 3/5 | `vm.compile` has nuances (modules vs CommonJS); regex word-boundary on code is tricky |
| ⏱️ Urgency | 4/5 | Addresses the root cause of the 21-error incident (bad replacement logic) |
| 🎯 Alignment | 4/5 | Hardens the core edit engine — structural improvement |
| **Overall** | **3.75 / 5** | |

**Verdict: 🔄 REVISE & MERGE** into EVO-145 plan

> The `vm.compile` idea is valuable but should be **merged into EVO-145's validation pipeline** instead of being a standalone plan. The regex word-boundary approach also needs refinement — occurrence counting (from EVO-781) is more practical than regex word boundary for code strings.

---

### 4. EVO-20260419-781 — Edit Disambiguation (Count Occurrences Before Replace)

**Core Idea:** In `validateEdits()`, count how many times the `find` string appears. If > 1, reject with an error explaining the string is ambiguous.

| Dimension | ⭐ Score | Reasoning |
|---|---|---|
| 🚀 Impact | 4/5 | Directly prevents the multi-replacement bug that caused the incident |
| ⚙️ Feasibility | 5/5 | Pure JS, no dependencies — regex occurrence count is trivial and safe |
| ⏱️ Urgency | 5/5 | Patches the exact mechanism that allowed 21 overwrites |
| 🎯 Alignment | 4/5 | Makes Hydra self-aware about edit precision — core autonomy property |
| **Overall** | **4.5 / 5** | |

**Verdict: ✅ APPROVE** (merge into EVO-145 as a validation sub-step)

---

## Recommended Merge Strategy

Ship as **2 Active plans**:

1. **`V7.4.0_[impl]_hydra_evolver-safety-gate.md`** — Lead plan combining EVO-145 + EVO-615 + EVO-781:
   - Add `validateEditsAndSyntax()` (node --check on modified JS files)
   - Add occurrence-count check in `validateEdits()` (EVO-781)
   - Add `vm.compile` as secondary syntax check (EVO-615, simplified)
   - Add `postDeployHealthCheck()` with auto-rollback (EVO-145)
   - Add `CONFIG` object with thresholds

2. **`V7.4.1_[hotfix]_hydra_block-self-modify-deploy.md`** — Companion plan for EVO-192:
   - Single line: move `scripts/hydra/` to `BLOCKED` tier
   - Fast-ship this one first (zero risk, zero dependencies)

## Deep Context & Investigation Matrix

**The Bug Report:** The User initially reported that the "Today" and historical OpenClaw CLI data in the AI Management Dashboard were not plotting (0/0) or blank. 

**Investigative Pathway:**
1. **TypeError Crash (Deduplication):** During chronological merging, we encountered a crash: JS evaluating `s.key.includes(':run:')` on newer temporal schema objects that had no `key` at all (only `date`). Fixed by adding `(s.key || '').includes`.
2. **Temporal String Conversion:** Realized Discord logs output `"date": "YYYY-MM-DD"`, while local scripts parsed `updatedAt` timestamps. Rewrote the `monthTok` and `weekTok` iteration to properly intercept string bounds via `new Date(tsRaw).getTime()`.
3. **Data Overwrite via Cron:** Confirmed that `generate-ocl-sessions-full.cjs` outputting `0 tokens` inside `openclaw-sessions.json` across all lines occurred because legacy `inputTokens` are NOT natively stored in the raw `.jsonl` files by `kie-proxy/lib/logUsage.js` — they must mathematically bubble up. 
4. **Cloudflare/Auth Redirection (Ghost SyntaxError):** Tested the `/data/openclaw-sessions.json` endpoint locally and hit a `<!DOCTYPE html>` response `SyntaxError()`. Mapped it down through the NextJS fallback and discovered that `authMiddleware` inside `routes/auth.js` strictly blocks unauthenticated static paths. But the Browser Agent had cookies so this turned out to be an orchestration red herring.
5. **The Missing Function (Root Cause):** The total dashboard apocalypse was caused because during an earlier timeline mapping refinement, I accidentally completely deleted `const sumToks = (arr) => arr.reduce((a, s) => a + (s.inputTokens || 0) + (s.outputTokens || 0), 0);`. This caused a synchronous `ReferenceError` inside `renderOclHero()`, completely halting execution *before* the DOM manipulated any integers.

**Resolution:**
The mathematical reducer was safely restored and deployed to the VPS. Smoke tests verify exact historical numbers (36.7M context, 10.6K today) correctly binding and rendering properly without halting at all.

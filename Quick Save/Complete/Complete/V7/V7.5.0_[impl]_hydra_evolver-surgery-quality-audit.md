# V7.5.0: Hydra Evolver Surgery & Quality Audit

## Situation

The user noticed the Hydra agents (Inspector, Ideator, Evolver) had been "silent" and only achieved a 14% success rate recently. Initially, we suspected the file whitelist (`classifyFileRisk`) in `evolver.js` was too strict and causing legitimate code proposals to be blocked. 

## Deep Quality Audit Findings

We ran a deep quality audit on the VPS database against `/root/brain-app/db/brain.db` to literally read the `proposed_changes` of rejected proposals. 

**Discoveries:**
1. **Safety Gate is Perfect:** The whitelist correctly blocked terrible ideas (e.g. adding empty comments, vague non-actions, modifying `evolver.js` itself). Pre-deploy validation (`vm.compile`, `node --check`) and dynamic-import smoke testing were functioning immaculately and rolling back broken diffs.
2. **The Real Culprit - Ideator Hallucinations:** The Ideator would propose optimizations like "add semantic caching" but point to fake file paths (e.g., `src/services/cache/semanticCache.ts`). 
3. **Silent Loading Failures:** `loadContextForPatching` in `evolver.js` would skip non-existent files. If all files were fake, it returned `{}`. Evolver then failed the proposal with `"Could not load context source files"`, never explicitly telling the Ideator *why* it failed, meaning the Cognitive Feedback Loop learned nothing about hallucination.
4. **Model Reference Bug:** Stage 2 diffing referenced `MODELS.gemini`, which did NOT exist in the `MODELS` constant.
5. **Goal Loop:** There were only 2 Growth Goals (`api_latency_ms` and `daily_token_cost`), causing the Ideator to propose the exact same hallucinated token-cost ideas on a loop.

## The Surgical Implementation

Instead of recklessly removing the whitelist, we implemented 3 precise fixes:

### 1. Injected Real Codebase Tree (Prevention)
Modified `scripts/hydra/ideator.js` to run a Linux `find` command dynamically before prompt injection.
```javascript
  let fileTreeInjection = '';
  try {
      const tree = execSync(`find /root/brain-app/scripts /root/brain-app/lib /root/brain-app/discord-bot /root/brain-app/routes -name "*.js" -type f 2>/dev/null | sed "s|/root/brain-app/||"`, { encoding: 'utf-8', timeout: 5000 }).trim();
      fileTreeInjection = `\n\n=== REAL FILES (เสนอเฉพาะไฟล์ในลิสต์นี้เท่านั้น ห้ามเดาชื่อไฟล์ใน proposed_changes.file เด็ดขาด!) ===\n${tree}\n`;
  } catch(e) {}
```
This forces the AI to only select files that *actually exist*.

### 2. Added Code Diagnosis for Hallucination (Detection & Feedback)
Modified `loadContextForPatching` in `scripts/hydra/evolver.js`:
```javascript
} else {
    console.warn(`[Evolver] File NOT FOUND: ${change.file} — Ideator hallucinated path`);
    sourceTexts['__hallucinated__'] = (sourceTexts['__hallucinated__'] || '') + change.file + '\n';
}
```
Modified the main loop to explicitly reject the proposal:
```javascript
if (sourceTexts['__hallucinated__']) {
    const hallucinated = sourceTexts['__hallucinated__'];
    delete sourceTexts['__hallucinated__'];
    if (Object.keys(sourceTexts).length === 0) {
        updateStmt.run('rejected', JSON.stringify({ 
            reason: `All proposed files do not exist: ${hallucinated.trim()}`,
            diagnosis: 'ideator_hallucination'
        }), null, null, null, row.id);
        continue;
    }
}
```
This ensures the rejection reason is logged so cognitive memory can learn from it.

### 3. Fixed Model Reference
Added `gemini-2.5-flash` to the `MODELS` dictionary inside `evolver.js` to restore Stage 2 fallback patching.
```javascript
gemini: { name: 'Gemini 2.5 Flash (Kie)', model: 'gemini-2.5-flash',
          url: 'http://127.0.0.1:18800/gemini-2.5-flash/v1/chat/completions', cost: 0.15, auth: () => process.env.KIE_API_KEY }
```

### 4. Diversified Growth Goals
Injected 4 new goals directly into the `brain.db` database on the VPS:
- `deploy_success_rate` (increase to 50.0)
- `cron_error_rate` (decrease to 5.0)
- `watchdog_pass_rate` (increase to 95.0)
- `agent_uptime_hours` (increase to 23.0)

## Conclusion
Code changes pushed to VPS and successfully deployed. We safely hardened the Evolver's architecture and fixed the true root conditions without compromising the protective whitelist barriers.

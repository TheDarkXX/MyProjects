# Unified AI Gateway Migration (Phase 1 & 2)

## 📌 Context
The OpenClaw system relied on scattered direct Axios and Fetch requests (Google, Kie.ai, OpenRouter, BytePlus) embedded across over 30+ files. This architecture resulted in blind spots for token accounting (`discord-usage.json`), particularly for Hydra meta-agents and isolated cron jobs.

We decided to build a standalone **Unified Proxy Gateway (`ai-gateway.js`)** running locally on port 18810 via PM2.

## 🚀 Execution Summary
- **Phase 1: Infrastructure:** Created `ai-gateway.js` combining the legacy `kie-proxy/kie-proxy-local` sanitization capabilities with brand-new real-time chunked token extraction (supporting both OpenAI schema and Google's `usageMetadata`).
- **Phase 2: Blanket Migration (38 Files):**
  1. **Hydra Core (15 files):** Batch migrated using `migrate-hydra-urls.js`. Injected `'X-Agent-Id': 'evolver'` and routing through `127.0.0.1:18810`.
  2. **Scripts & Bots (10 files):** Migrated isolated modules (e.g. `gmail-agent`, `credit-card-agent`).
  3. **Backend Routes (5 files):** Updated `routes/ai-chat.js`, `routes/ai.js`, `lib/brain-ai.js` to route backend HTTP requests securely through the gateway while preserving real-time streams to the user interface.
- **Critical Hotfixes:**
  - Resolved double trailing commas `,,` generated during RegExp header injection.
  - Resolved crash inside `ai-gateway.js` where `require` was invoked inside an ESM target repo (swapped to ESM `import`).
  - Corrected `callAIDirect()` within `discord-bot/lib/ai.js` to prevent bypassing the proxy on fallback triggers.

## 🛠 Architectural Snippets (For Memory)

### Routing Architecture:
```
http://127.0.0.1:18810/openrouter/... -> https://openrouter.ai/...
http://127.0.0.1:18810/kie/...        -> https://api.kie.ai/...
http://127.0.0.1:18810/google/...     -> https://generativelanguage.googleapis.com/...
http://127.0.0.1:18810/byteplus/...   -> https://ark.ap-southeast.bytepluses.com/...
```

### Telemetry Header Standard:
All proxied endpoints MUST specify the following headers to allow `usage.json` attribution:
```json
{
  "X-Claw-Cron": "cron-ai-monitor",
  "X-Agent-Id": "ai-monitor"
}
```

### Group C Disclaimer:
`discord-bot/lib/ai.js`'s main `chat()` loop still calls direct proxy-less providers. This is **INTENTIONAL** as `ai.js` implements its own isolated `logUsage()` routine. Routing it through the gateway would lead to double-counting tokens.

## 🔄 Deployment & Rollout
- Fully merged into `master` via `git push vps`.
- Running on PM2: `pm2 start /root/brain-app/ai-gateway.js --name ai-gateway`
- **Grace Period Mechanism:** `kie-proxy` remains active on port 18800. It will be scheduled for decommission strictly after 24 hours of stable gateway operation (approx. 2026-04-20).

# V12.9.14 - Reel Pipeline: Prompt Formatting & Discord Automation Tags

## Issue
1. **Missing Panel References**: Gemini was ignoring the standard prefix rule and generating raw tags like `[ Scene 1 ]` instead of the expected `C-XXX-NN` prefix. It was also not referencing the Storyboard Panel number in the Image Prompts, which is critical for downstream manual review and automation.
2. **Missing Image References for Automation**: For automated image generation tools (like Midjourney bot setups via Discord), the prompts were missing the placeholder tags (`@image1`, `@image2`) required to inject the actual image URLs (Storyboard Grid and Character Reference).

## Resolution
1. **Strict Panel Reference Rule**: Re-wrote Phase 2 prompt rules in `blueprint.py` to enforce a strict format:
   - Every single image prompt MUST start with the C-XXX-NN ID followed by the panel reference.
   - Format: `C-XXX-NN: (Reference Panel N of the Storyboard @image1 ) [Main subject and dynamic action...]`
2. **Character Reference Tag Injection**: 
   - Added a critical automation rule: If the Holy Man is present in the scene, the AI MUST add the `@image2` tag IMMEDIATELY after the words "Holy Man" in the sentence.
   - Example output: `C-055-02: (Reference Panel 2 of the Storyboard @image1 ) A Holy Man @image2 stands with arms slightly outstretched...`
   - This ensures character reference mapping locks onto the correct subject locally within the prompt syntax.
3. **Hook Variations Enforcement**: Updated the Phase 2 `hook_variations` instruction to explicitly enforce the new `C-XXX-NN: (Reference Panel 1 of the Storyboard @image1)` format, preventing Gemini from dropping the prefix on hook alternatives.

## Conclusion
The `blueprint.py` logic now cleanly prepares image prompts for 1-Click integration into Veo or Midjourney Discord bots, properly injecting reference placeholders (`@image1`, `@image2`) and firmly locking the prompt prefixes against LLM hallucination.

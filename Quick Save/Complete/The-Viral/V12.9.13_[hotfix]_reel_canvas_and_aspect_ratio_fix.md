# V12.9.13 - Reel Pipeline: Canvas, Grid, and Aspect Ratio Hotfix

## Issue
1. **Wrong Aspect Ratio Generated**: The pipeline (`blueprint.py`) accidentally generated `9:16` vertical storyboards instead of `16:9` for 4-scene videos (like C-054). The old logic `aspect_ratio = "9:16" if "9:16" in storyboard_prompt else "16:9"` was flawed because the string `"Vertical 9:16"` is injected into *every single panel's* description based on Director's Eye rules. Thus, it always forced `--aspect_ratio 9:16` when calling Higgsfield CLI.
2. **Incorrect Canvas/Grid Logic**: The grid logic for 1-4 scenes, 5-6 scenes, and 7-8 scenes was not adhering to the proper horizontal vs vertical ratios.
3. **No Holy Man in Hook Failure**: The `NO HOLYMAN` rule for the Hook (Panel 1) was buried under the `hook_variations` block in the Gemini prompt, causing Gemini to only apply it to variations but still place the Holy Man in the main Panel 1 `image_prompt`.
4. **Agentic Misunderstanding**: The Antigravity agent previously misunderstood the architecture, thinking `extract_and_sync.py` was the core orchestrator and that Higgsfield ran on the VPS. In reality, `blueprint.py` is the Auto-Pipeline orchestrator and it calls `higgsfield.cmd` locally via `subprocess`.

## Resolution
1. **Fixed Aspect Ratio Parsing**: Updated `blueprint.py` to prioritize the Canvas instructions:
   ```python
   if "3:4" in storyboard_prompt:
       aspect_ratio = "3:4"
   elif "16:9" in storyboard_prompt:
       aspect_ratio = "16:9"
   else:
       aspect_ratio = "9:16"
   ```
2. **Updated Canvas & Grid Rules**:
   - 1-4 Scenes: `16:9 horizontal canvas split vertically into 4 panels (4x1 grid)`
   - 5-6 Scenes: `3:4 vertical canvas split into a 3x2 grid`
   - 7-8 Scenes: `3:4 vertical canvas split into a 4x2 grid`
   - *Tested & Verified*: Higgsfield CLI `gpt_image_2` natively supports `--aspect_ratio "3:4"` and `--aspect_ratio "16:9"` without errors.
3. **Global Hook Rule**: Extracted the `NO HOLY MAN IN HOOK (CRITICAL)` rule to be a top-level instruction in Phase 2 of `blueprint.py`, strictly forbidding Gemini from inserting the Holy Man in the Scene 1 `image_prompt`.
4. **Skill Update**: Completely rewrote the `/reel` AG skill (`ag_skills_backup/reel.md`) to point back to the fully-automated `python blueprint.py` workflow, eliminating any manual intervention.
5. **Cleaned Up Processed Folder**: Refactored `blueprint.py` so that it no longer uses the `Storyboards/` subfolder. All extracted grids, videos, storyboards, and JSONs are now directly bundled into `Processed/` with unified prefixes (e.g. `C-054_..._storyboard.png`).

## Testing

- Verified that Holy Man was successfully removed from C-053's hook.

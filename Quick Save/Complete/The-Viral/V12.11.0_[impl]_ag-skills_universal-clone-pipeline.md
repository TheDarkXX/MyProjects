# Universal Viral Video Cloning Skill & ViralCheck

## 📌 Context (Compiled Truth)
The project required a systematic, universal way to reverse-engineer and clone viral videos across multiple genres (The Girl, Jesus, Animal Fight, etc.). 
- We developed the `/clone` skill, a 7-Phase universal pipeline.
- The pipeline introduces the "Why It Viral" phase and a "Viral Scorecard" distinguishing PRIMARY vs SECONDARY keys.
- We adapted the old global `/recheck` skill into a localized `/viralcheck` to specifically perform mandatory post-generation verification for The Viral (checking DNA alignment, Rule 15 Uncanny Valley, and character count limits).

## ★ Phase 2: 💡 WHY IT VIRAL (ใหม่ — หัวใจของระบบ)

### วัตถุประสงค์
ก่อนจะโคลนอะไร ต้อง **เข้าใจก่อนว่าทำไมมันถึงไวรัล** เพื่อให้ระบบรู้ว่าต้อง **เน้นส่วนไหนเป็นหลัก ส่วนไหนเป็นรอง** ไม่เสียเวลาปั้นสิ่งที่ไม่ใช่ปัจจัยหลัก

### Process

**Step 1: AI Brainstorm — วิเคราะห์จากข้อมูล Phase 1**

จาก Grid, Transcript, และ Overview ที่ได้ → AI ระบุ **สมมติฐาน** ว่าทำไมวิดีโอเหล่านี้ถึงไวรัล:

```markdown

จากการวิเคราะห์ [X] คลิปต้นฉบับ ปัจจัยที่ทำให้ช่องนี้ไวรัล:

1. 🔑 [Key Factor 1] — อธิบายสั้นๆ
2. 🔑 [Key Factor 2] — อธิบายสั้นๆ
3. 🔑 [Key Factor 3] — อธิบายสั้นๆ
4. 🔑 [Key Factor 4] — อธิบายสั้นๆ (optional)
5. 🔑 [Key Factor 5] — อธิบายสั้นๆ (optional)
```

**Step 2: User Review & Brainstorm ร่วมกัน**

AI นำเสนอสมมติฐาน → User เห็นด้วย/ไม่เห็นด้วย/เพิ่มเติม → สรุปร่วมกัน

## 🚀 Innovation Enhancements (5 ข้อเสนอเพิ่มเติม)

### 1. 📡 Multi-Source Intelligence (วิเคราะห์หลายช่องพร้อมกัน)

แทนที่จะโคลนจากช่องเดียว → **วิเคราะห์ 3-5 ช่องที่ดังในประเภทเดียวกัน** → หา Common Pattern → โคลนจาก "ค่ากลาง" ที่ดีที่สุด

```
┌─────────────────────────────────────────────┐
│  Multi-Source Intelligence                   │
│                                              │
│  ช่อง A (10M views) ──┐                     │
│  ช่อง B (8M views)  ──┼── หา Common Pattern │
│  ช่อง C (5M views)  ──┘                     │
│                          ↓                   │
│              📊 MASTER VIRAL DNA             │
│        (สูตรที่ดีที่สุดจากหลายช่อง)          │
└─────────────────────────────────────────────┘
```

**ผลลัพธ์:** ได้ DNA ที่แข็งแกร่งกว่าการโคลนจากช่องเดียว เพราะกรองเอาเฉพาะ Pattern ที่ **ทุกช่องมีเหมือนกัน** = พิสูจน์แล้วว่าได้ผลจริง

> ใช้เมื่อ: เริ่มโปรเจกต์ Genre ใหม่ที่ยังไม่มี DNA

---

### 2. 📐 Clone Fidelity Score (คะแนนความเหมือน)

สร้างระบบให้คะแนนอัตโนมัติว่า **คลิปที่โคลนมา "เหมือน" ต้นฉบับแค่ไหน**

```markdown
## Clone Fidelity Report

| Dimension | Score | หมายเหตุ |
|-----------|-------|----------|
| Visual Match | 8/10 | ฉากใกล้เคียง แสงยังต่างนิดหน่อย |
| Timing Match | 9/10 | Duration ตรง, Pacing ตรง |
| Audio Match | 7/10 | BGM mood ตรง แต่ Volume ต่างไป |
| PRIMARY Keys | 3/3 ✅ | ครบทุกข้อ |
| SECONDARY Keys | 1/2 | ขาด Rural Authenticity |
| **Overall** | **82%** | ✅ ผ่าน (Threshold: 75%) |
```

**ผลลัพธ์:** มีตัวเลขวัดคุณภาพชัดเจน → ไม่ต้องเดาว่า "ดีพอหรือยัง"

> ใช้เมื่อ: Phase 5-6 ตรวจสอบก่อนไปผลิตจริง

---

### 3. 🔄 Post-Publish Feedback Loop (วงจรปรับปรุงหลังเผยแพร่)

หลังเผยแพร่คลิปจริง → เก็บ Metrics → ป้อนกลับเข้า DNA → ปรับปรุงคลิปถัดไป

```
Publish คลิป → 48 ชั่วโมง → เก็บ Metrics
    ↓
📊 Performance Check:
- Views vs ค่าเฉลี่ยช่อง?
- Comment Rate สูง/ต่ำ?
- Share Rate สูง/ต่ำ?
- Watch Time เท่าไหร่?
    ↓
🧬 DNA Update:
- คลิปไหนดี → เสริม Pattern นั้น
- คลิปไหนแย่ → วิเคราะห์ว่า PRIMARY Key ไหนขาดไป
    ↓
✍️ ปรับสคริปต์/ฉาก คลิปถัดไป
```

**ผลลัพธ์:** DNA "วิวัฒนาการ" ไปเรื่อยๆ ดีขึ้นทุกรอบ ไม่ใช่ Static

> ใช้เมื่อ: หลังโพสต์ไปแล้ว 3-5 คลิปแรก

---

### 4. 🧪 Cross-Pollination Engine (ยืม Pattern ข้าม Genre)

ยืม Viral Key จาก Genre หนึ่งไปใช้กับอีก Genre → สร้างความแตกต่างจากคู่แข่ง

| ยืมจาก | นำไปใช้กับ | ตัวอย่าง |
|--------|-----------|---------|
| The Girl (มุกทะลึ่ง 2 แง่) | Jesus | ใส่ Twist ending แบบ "ตีความได้ 2 แบบ" |
| Jesus (Epic BGM Sync) | The Girl | ใส่เสียงดนตรีเข้ากับจังหวะอ้อน |
| Animal Fight (Shock Factor) | The Girl | เซอร์ไพรส์ตอนจบ (เช่น ล้ม, ของตก, สัตว์วิ่งผ่าน) |

**ผลลัพธ์:** คลิปที่ "โคลน" แต่มีรสชาติใหม่ → ไม่ซ้ำใคร → Algorithm ให้คะแนนสูง

> ใช้เมื่อ: หลังจากโคลนสำเร็จแล้ว ต้องการสร้างความแตกต่าง

---

### 5. 📉 Viral Decay Detection (ตรวจจับว่า Pattern เริ่มเก่า)

ทุก Viral Pattern มี "อายุขัย" — Pattern ที่เคยดังอาจหมดมุกในอีก 2-3 เดือน

```markdown
## Decay Signals (สัญญาณว่า Pattern เริ่มเก่า)
- Views ลดลง ≥30% ใน 2 สัปดาห์ติดกัน
- Comment เปลี่ยนจาก "สมัครครับ!" เป็น "ซ้ำอีกแล้ว"
- ช่องต้นฉบับเริ่มเปลี่ยน Pattern ตัวเอง
```

**เมื่อตรวจพบ Decay:**
1. แจ้ง User ทันที
2. เสนอ: กลับไป Phase 1-2 ใหม่ (หาช่องต้นฉบับใหม่ + แกะ Viral Keys ใหม่)
3. หรือใช้ Cross-Pollination ยืม Pattern ใหม่มาเสริม

> ใช้เมื่อ: โพสต์ไปแล้ว 20+ คลิป หรือ 2+ เดือน

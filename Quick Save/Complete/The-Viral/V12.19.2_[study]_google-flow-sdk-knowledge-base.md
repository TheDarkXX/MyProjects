# สรุปโครงสร้างและองค์ความรู้ (Knowledge Base) สำหรับการพัฒนาแอปบน Google Flow

## 1. ภาพรวมสถาปัตยกรรม (Architecture Overview)
Flow Tool คือ Frontend-only React Web Application ที่รันอยู่ภายใน Sandbox Iframe ของ Google Flow
- **No Backend**: ทุกอย่างทำงานที่ Client-side (เบราว์เซอร์ของผู้ใช้)
- **Host Communication**: เชื่อมต่อกับระบบหลัก (Host) ผ่าน `flow-sdk` เท่านั้น เพื่อดึงข้อมูลสื่อ (Media), สั่งงาน AI, หรือบันทึกไฟล์
- **Tech Stack**: React 19, TypeScript, Tailwind CSS v4, และ ESM Modules (นำเข้าไลบรารีผ่าน `esm.sh`)

## 2. โครงสร้างไฟล์มาตรฐาน (Standard Code Structure)
เพื่อให้โค้ดอ่านง่ายและดูแลรักษาได้ในระยะยาว ควรแบ่งไฟล์ดังนี้:

1. **`App.tsx` (The Brain)**:
   - เป็นศูนย์กลางของ State Management (เช่น ใช้ `useState` เก็บสถานะการทำงาน)
   - รวบรวม Logic การทำงานหลัก และเรียกใช้ Flow SDK
2. **`types.ts` (The Contract)**:
   - นิยาม Interface สำหรับข้อมูลต่างๆ เช่น `AppState`, `MediaData`, `Settings`
   - ช่วยให้ TypeScript ตรวจสอบข้อผิดพลาดของโค้ดได้แม่นยำ
3. **`components/` (The UI)**:
   - แยก UI เป็นส่วนย่อยๆ (Modularity) เช่น `Header.tsx`, `Sidebar.tsx`, `MediaPreview.tsx`
   - **กฎเหล็ก**: อย่าเขียนทุกอย่างในไฟล์เดียว เพื่อประหยัด Token และลดความซับซ้อนเวลา AI แก้ไขโค้ด
4. **`index.tsx`**: จุดเริ่มต้น (Entry Point) ที่เชื่อม React เข้ากับ DOM (`root`)

## 3. การเชื่อมต่อกับสื่อ (Image & Video Workflow)
การจัดการสื่อใน Flow จะทำงานผ่านวงจร Select → Process → Save

### A. การนำเข้า (Input)
ใช้ `Flow.media.select()` หรือ `selectMultiple()`
- ระบบจะคืนค่าเป็น `MediaItem` ซึ่งมี `mediaId` (สำหรับส่งให้ AI) และ `base64` (สำหรับแสดงผลในแอป)
- **ข้อควรระวัง**: เวลาแสดงผล preview ให้ใช้ `data:${item.mimeType};base64,${item.base64}` เสมอ (ห้าม Hardcode MIME type)

### B. การประมวลผลด้วย AI (Generation)
ใช้ `Flow.generate.image()` หรือ `video()`
- **Model Selection**: ต้องระบุ `modelDisplayName` ให้ตรงกับรุ่นที่ต้องการเสมอ (เช่น `🍌 Nano Banana Pro` หรือ `Omni Flash`)
- **Chaining**: ผลลัพธ์จากการ Generate จะได้ `mediaId` มาด้วย ซึ่งคุณสามารถส่งต่อให้โมเดลอื่นได้ทันที (เช่น Generate รูปภาพก่อน แล้วส่ง `mediaId` นั้นไปทำ Video ต่อ)

### C. การบันทึกและส่งออก (Output)
- `Flow.save()`: บันทึกรูปภาพหรือวิดีโอกลับเข้าไปใน Media Gallery ของผู้ใช้
- `Flow.download()`: เป็น **วิธีเดียว** ในการดาวน์โหลดไฟล์ลงเครื่องคอมพิวเตอร์ (เนื่องจากระบบ Sandbox บล็อกวิธีดาวน์โหลดแบบปกติ) ใช้ได้กับทุกไฟล์รวมถึง ZIP, MP3 หรือ GIF

## 4. การออกแบบ UI/UX (Aesthetics & Design System)
Google Flow มี Design Language เฉพาะตัวที่เน้นความพรีเมียมสำหรับ Creative:
- **Typography**: ใช้ `Google Sans Text` หรือ `Google Sans Flex`
- **Icons**: ใช้ `Material Symbols Outlined` เท่านั้น (ห้ามเดาชื่อ icon ต้องใช้จาก Allowlist)
- **Colors**: เน้น Dark Mode (`0e0e0e`) และใช้ Border โปร่งแสง (`rgba(218,220,224,0.15)`)
- **Components**: ใช้ Pill-shaped buttons (มุมมน `rounded-xl` หรือ `rounded-full`)

## 5. ข้อจำกัดสำคัญ (Sandbox Constraints)
- **No external APIs**: ไม่สามารถเรียก API ภายนอกได้โดยตรง (ต้องใช้ผ่าน SDK หรือ Library ที่รันบน Browser เท่านั้น)
- **No `window.open()`**: ไม่สามารถเปิด Tab ใหม่ หรือ Popup ได้
- **No Standard Forms**: ห้ามใช้แท็ก `<form>` ที่มีการ Submit จริง ให้ใช้ React State และปุ่ม `onClick` แทน
- **Persistence**: ข้อมูลในแอปจะหายไปเมื่อปิดแอป (ยกเว้นจะบันทึกผ่าน `Flow.save()`)

---

## 6. ข้อมูลเชิงลึกเพิ่มเติมจากแพลตฟอร์ม (Advanced Insights)
ข้อมูลส่วนนี้ได้จากการยืนยันสเปคปัจจุบันของ Google Flow:
- **Data Persistence**: Sandbox ใช้ Opaque Origin ทำให้ `localStorage` หรือ `IndexedDB` ไม่คงอยู่ถาวร (มักถูกล้างเมื่อปิด Iframe) เน้นให้ผู้ใช้ `Flow.save()` ผลลัพธ์ลง Gallery แทน ไม่มี API สำหรับดึง User ID (แอปทำงานแบบ Stateless)
- **Networking & CSP**: บล็อกการทำ External Fetch 100% ผ่าน Content Security Policy (CSP) ข้อมูลภายนอกต้อง bundle มากับแอปหรือใช้ CDN ที่ได้รับอนุญาต
- **Background Execution**: คำสั่ง `Flow.generate` ทำงานบนเซิร์ฟเวอร์ แม้ผู้ใช้สลับแท็บงานจะยังรันต่อจนจบ
- **Error Handling**: ควบคุม Quota ที่ฝั่ง Host หาก Error (เช่น ละเมิด Policy หรือ Quota เต็ม) SDK จะโยน Error Object ให้ดักด้วย `try/catch` และนำ `error.message` มาแสดงใน UI
- **Project & Folder Management**: SDK ไม่อนุญาตให้เลือกระบุ Project หรือสร้าง Folder ทางโค้ด ระบบ Flow จะจัดการยัดไฟล์ลง Gallery และโปรเจกต์ที่ Active อยู่โดยอัตโนมัติ

## 7. SDK API Reference แบบละเอียด (The Ultimate Cheat Sheet)

### A. Image Generation (`Flow.generate.image`)
- **Prompt**: ยอมรับ Text ยาว บรรยายรายละเอียดได้ละเอียด (1,000+ ตัวอักษร)
- **AspectRatio**: `'1:1'`, `'16:9'`, `'9:16'`, `'4:3'`, `'3:4'`
- **Count**: 1 รูปต่อ 1 Request
- **Model Names**: `🍌 Nano Banana Pro`, `🍌 Nano Banana 2`, `Imagen 4`
- **Reference Images**: ส่งรูปอ้างอิงผ่าน `referenceImageMediaIds` (Array ของ `mediaId`) โดย `🍌 Nano Banana Pro` รับได้ถึง 10 รูป ใช้ทำ Character Lock ได้ (ส่งรูปหน้าตัวละคร + สไตล์ไปพร้อมกันได้)
- **Advanced Params**: ยังไม่รองรับ `negativePrompt`, `seed`, `guidanceScale`
- **Return**: `{ base64, mimeType, mediaId }`

### B. Video Generation (`Flow.generate.video`)
- **First/Last Frames**: ใช้ `firstFrameImageMediaId` และ `lastFrameImageMediaId` เพื่อเชื่อม Scene แบบต่อเนื่องหรือทำ Video Interpolation
- **AspectRatio**: `'16:9'`, `'9:16'`
- **Duration**: `durationSeconds` ค่าที่แนะนำ `4`, `6`, `8` (Veo) และสูงสุด `10` (Omni Flash)
- **Model Names**: `Omni Flash`, `Veo 3.1 - Lite`, `Veo 3.1 - Fast`, `Veo 3.1 - Quality`
- **Return**: `{ base64, mimeType, mediaId }`

### C. Text Generation (`Flow.generate.text`)
- **Model Selection**: ใช้ Property `thinkingLevel`: `'high' | 'medium' | 'low'` ในการกำหนดระดับโมเดล
- **System Prompt**: ใช้ Key `systemInstruction`
- **Vision Input**: ส่งผ่าน Array `images` หรือ `videos` (ส่งเป็น object `{ base64, mimeType }`)
- **Return**: `{ text: string }`

### D. Advanced Manipulations & Callbacks
- **Parallelism**: รองรับการเรียก `Promise.all` ได้ แต่ระบบหลังบ้านอาจจะ Queue การทำงาน
- **Audio & Edits**: ตอนนี้ไม่มี `Flow.generate.audio()` หรือ `edit() / upscale()` ถ้าจะทำวิดีโอแบบต่อกัน (Concatenate) หรือใส่ลายน้ำ (Overlay) ต้องใช้ Library เสริมพวก `mediabunny`, `ffmpeg` หรือทำผ่าน HTML Canvas

## 8. ระบบอัตโนมัติและข้อจำกัดของระบบ (Automation & System Limits)

### A. การจัดการไฟล์และการดาวน์โหลด
- **Programmatic Upload**: สามารถใช้ `Flow.upload({ base64, mimeType, name })` อัปโหลดไฟล์ (เช่น `.png`, `.jpeg`, `.mp4`) ขึ้น Gallery ได้โดยตรงแบบเงียบๆ ไม่ต้องเปิดหน้าต่าง File Picker
- **Batch Download**: การสั่งรัน `Flow.download()` รัวๆ ในเสี้ยววินาทีจะถูกบล็อกโดย Popup Blocker ทางแก้คือต้องใช้ Library เช่น `jszip` รวมไฟล์ให้เป็น `.zip` ก่อนแล้วสั่งโหลดครั้งเดียว
- **Auto-Download**: ตาม Security Policy ของเบราว์เซอร์ การดาวน์โหลดต้องมาจาก User Gesture (การคลิกปุ่มของคนจริงๆ) ไม่สามารถสั่งรันเมื่อ AI ประมวลผลเสร็จอัตโนมัติแบบ 100% ได้ ควรทำปุ่ม "Download All" รอไว้เมื่อจบกระบวนการ

### B. ระบบการทำงานลูปยาวข้ามคืน (Long-running Loops)
- **Memory Management**: ห้ามเก็บข้อมูลวิดีโอเป็น `base64` จำนวนมากๆ ลงใน React State เด็ดขาด เพราะจะทำให้ Browser Out of Memory (OOM) ทันที **วิธีแก้**: ให้เซฟลง Gallery (`Flow.upload()`) ทันทีที่เจนเสร็จ แล้วเก็บแค่ `mediaId` (String สั้นๆ) ไว้ใน State เพื่อนำไปอ้างอิง
- **Iframe & Tab Priority**: หากเปิดแอปทิ้งไว้แล้วสลับแท็บไปทำงานอื่น เบราว์เซอร์จะลด Priority การประมวลผลของ Iframe ลง ซึ่งอาจทำให้สคริปต์หยุดทำงานกลางคัน แนะนำให้เปิดแท็บทิ้งไว้ด้านหน้าถ้ารันงานหนักๆ
- **Rate Limit & Cooldown**: ถ้าชนลิมิต Quota แพลตฟอร์มจะคืนค่า Error สามารถเขียนโค้ดเพื่อจับ Error นี้และทำ Cooldown Timer แช่ไว้สักพักก่อนลุยต่อได้

## สรุปขั้นตอนการสร้างแอปใหม่ (Checklist)
1. **Define Types**: ออกแบบข้อมูลใน `types.ts`
2. **Setup Layout**: สร้างโครงสร้าง UI ใน `components/`
3. **State Logic**: เขียนฟังก์ชันจัดการข้อมูลใน `App.tsx`
4. **SDK Integration**: เชื่อมต่อปุ่มต่างๆ เข้ากับ `Flow.media` หรือ `Flow.generate`
5. **Polishing**: ใส่ Loading states และ Error handling ให้สวยงาม

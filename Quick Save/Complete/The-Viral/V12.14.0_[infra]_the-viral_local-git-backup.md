# V12.14.0 The Viral Local Git Backup Setup & Skill Recovery

## 📌 Context (Compiled Truth)
The project `P:\AI\The Viral\` experienced a loss of critical skill files (`fah.md` and `pd.md`) due to a server restart disrupting the active workspace. To prevent future loss and establish a robust backup strategy, a dual-backup architecture was implemented.

**Key Architectural Changes:**
1. **Dual Backup System:**
   - **Knowledge/Brain Backup:** Guideline `.md` files continue to be saved to `C:\My Claw\Openclaw-VPS\Quick Save\Complete\The-Viral\` and pushed to the `vps` remote.
   - **Code/Skill Backup:** A new local Git repository was initialized in `P:\AI\The Viral\` and connected to a GitHub Private Repository (`https://github.com/TheDarkXX/The-Viral`).
2. **Media Ignored:** A `.gitignore` was aggressively configured to exclude all large media files (`.mp4`, `.jpg`, `.zip`, etc.) ensuring only code (`.py`, `.js`), skills (`.md`), and config (`.json`) are tracked (approx. 550MB).
3. **Auto-Backup via `/save`:** The `save.md` skill was upgraded. Running `/save` now automatically commits and pushes both the VPS documentation and the local project codebase.

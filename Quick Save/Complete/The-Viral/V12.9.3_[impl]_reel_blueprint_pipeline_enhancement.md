# 🎬 Reel Blueprint Pipeline Enhancements

## 📌 Context (Compiled Truth)
- **Problem:** The original pipeline (`blueprint.py`) generated Image and Motion Prompts but didn't cleanly separate the `Veo Sequence Mapping`, making the web UI cluttered. We also lacked a way to auto-rename files after extraction and lacked a strict Canvas/Grid rule for 1-prompt AI Storyboard generation.
- **Solution:** 
  1. Extracted Veo Sequence into its own `veo_sequence` JSON field.
  2. Implemented Auto-Renaming (`C-XXX_slug_YYYYMMDD.mp4/jpg`) and archiving to `Processed/`.
  3. Defined strict AI Storyboard Canvas rules (4x1=16:9, 3x2=9:16) to ensure single-image generation for 100% character consistency.
  4. Implemented a **Double Recheck** rule for Holy Man (Red shoulder cloth OUTSIDE waist sash) and First Frame Hook.
  5. Designed a 6-feature plan to bring Auto-Storyboard generation, Bulk Excel-like Copy buttons, and A/B Hook variations into the Node.js Backend and Frontend UI.

## 🔴 Pain Points / AI Failures

### 1. 🔁 Bump Cache ล้มเหลว — Push โค้ดไปแต่ไม่ได้ Push Cache Version
> **ผู้ใช้:** _"Bump Cache หรือยัง ไม่เห็นความเปลี่ยนแปลง"_ (MSG #76)

**Root Cause:** AI ทำการ `git push` แค่ไฟล์ JS แต่ลืม Push ไฟล์ `index.html` (ที่เก็บ cache version string `?v=YYYYMMDD_HHMMSS`) ขึ้นไปด้วย ทำให้บราวเซอร์ยังจำไฟล์เก่าอยู่

**Impact:** ผู้ใช้ต้องรอและถามซ้ำ เสียเวลา 5+ นาที

---

### 2. 🖼️ รันวิเคราะห์ผิดภาพ — ส่ง Storyboard Image ไปแทน Extract Grid
> **ผู้ใช้:** _"ตอนคุณรันแกะ prompt ต่างๆ ควรรันจาก Extract Grid image นะ ไม่ใช่ storyboard image"_ (MSG #81)

**Root Cause:** AI สร้าง `run_storyboard.py` (manual override script) โดยดัน Hardcode path ไปชี้ `C-37_Storyboard.png` แทน Extract Grid Image ทำให้ Gemini Vision วิเคราะห์จากภาพผิด ผลลัพธ์ทั้ง Blueprint จึงเพี้ยนหมด

**Impact:** C-37 ถูก generate ใหม่ 2+ ครั้ง ผลลัพธ์เพี้ยนทุกครั้ง เพราะ input ผิดตั้งแต่ต้น

---

### 3. 📐 เข้าใจผิดเรื่อง Scope — ไปยุ่งกับ `/raw` เดิมแทนที่จะสร้างงานใหม่
> **ผู้ใช้:** _"revert ก่อน คุณเข้าใจผิดไปใหญ่ อันนี้งานใหม่นะ คนละอันกับ /raw เดิม"_ (MSG #13)

**Root Cause:** AI อ่าน implementation plan แล้วเข้าใจผิดว่างานนี้คือ modification ของระบบเก่า (`/raw`) จึงไปแก้ไฟล์เดิมโดยไม่ถาม

**Impact:** ต้อง Revert โค้ดกลับ เสียเวลา ~15 นาที และทำให้ผู้ใช้เสียความมั่นใจ

### 6. 🔤 ภาษาไทยไม่ครบ — สั่งให้แปลแต่ไม่ทำ

## 🛡️ Proposed Iron Rules (Memory)

### IR-1: 🚨 ALWAYS Bump Cache After VPS Push
```
เมื่อทำ git push vps master สำเร็จ → MUST run `node bump-cache.js` 
→ MUST commit & push index.html ที่มี cache version ใหม่อีกรอบ
ห้าม Push JS/CSS โดยไม่ Push index.html ที่มี cache string ใหม่เด็ดขาด
```

### IR-2: 🖼️ NEVER Hardcode Image Paths in Manual Scripts
```
เมื่อสร้าง manual/override script → ห้าม Hardcode path
→ ต้องใช้ argument หรือ config file เสมอ
→ ต้อง validate ว่า path ที่ส่งไปตรงกับ purpose (Extract Grid ≠ Storyboard ≠ Thumbnail)
```

### IR-3: 🔀 NEVER Touch Existing Code Without Explicit "Merge" Instruction
```
เมื่อ user ขอ "งานใหม่" → สร้างไฟล์ใหม่เสมอ
→ ห้ามแก้ไขหรือ Merge เข้าไฟล์เดิมจนกว่า user จะสั่ง "merge" หรือ "integrate" โดยตรง
```

### IR-4: 🎯 Apply Changes Consistently Across ALL Views
```
เมื่อแก้ logic ในหน้า Detail View → MUST check & apply หน้า List View ด้วยเสมอ
(เช่น Score rescaling, Thai localization, label naming)
→ ต้อง grep หา usage ทุกจุดก่อน commit
```

### IR-5: 🎬 Director's Eye — Always Analyze Video Structure First
```
เมื่อวิเคราะห์ Extract Grid → ต้องระบุ video structure ก่อน:
- "Long take" (ลากยาว ไม่ตัด)
- "Switch scenes / Cut scenes" (มีการตัดสลับมุมกล้อง)
จะได้ไม่ประเมิน continuity ผิด
```

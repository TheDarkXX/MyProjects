# V12.9.12 [hotfix] Reel Blueprint - Fix Copy Prompts Scene Sorting

## 1. Context & Problem
The user reported that testing the **Copy All Image** and **Copy All Motion** buttons resulted in prompt data that appeared "incomplete" and lacked scenes: "ลอง test ปปุ่ม copy all image and motion แล้วข้อมูล prompt มาไม่ครบทุกซีน เชคหน่อย".

Upon investigation, `b.scenes` from the backend might arrive unsorted. The original UI code mapped `b.scenes` directly and simply joined the text with `\n`, making it impossible to tell which prompt belonged to which scene, and causing them to appear out of order or seemingly missing (e.g. seeing Scene 4 before Scene 2).

## 2. Solution Implemented
1. **Scene Sorting**: Cloned and sorted `b.scenes` by `scene_number` so the prompts always flow chronologically from Scene 1 to the end.
2. **Explicit Labeling**: Prefixed each mapped prompt with `[ Scene N ]\n` before joining with `\n\n`.
3. **Hook Variation Cleanup**: The `[Generated Image: URL]` markdown tags are now stripped from Hook Variations before they are copied to the clipboard.

## 3. Evidence
**Code Modified (`brain-app-public/mission-control.js`):**
```javascript
const sortedScenesForCopy = [...(b.scenes || [])].sort((a,b) => (a.scene_number||0) - (b.scene_number||0));

const allImagePrompts = sortedScenesForCopy.map((s, i) => `[ Scene ${s.scene_number || (i+1)} ]\n${(s.image_prompt || '-').replace(/C-(XXX|001)/gi, formattedId)}`).join('\n\n');
const allMotionPrompts = sortedScenesForCopy.map((s, i) => `[ Scene ${s.scene_number || (i+1)} ]\n${(s.motion_prompt || '-').replace(/C-(XXX|001)/gi, formattedId)}`).join('\n\n');

let allImagePromptsWithVariations = '';
if (b.hook_variations && b.hook_variations.length > 0) {
  allImagePromptsWithVariations += `[ HOOK VARIATIONS (Scene 1) ]\n` + b.hook_variations.map((h, i) => {
    const cleanHook = h.replace(/\n\n\\[Generated Image:.*\\]/s, '').trim();
    return `Variation ${String.fromCharCode(65+i)}:\n${cleanHook}`;
  }).join('\n\n') + '\n\n';
}
allImagePromptsWithVariations += allImagePrompts;
```

## Changelog
**Fix #2 (Quote Cut-off Bug)**: The user found that the text was getting cut off at the first double quote (`"`). This happened because `escapeHtml` converted `"` to `&quot;`, but when injected via `innerHTML`, the browser parsed the `&quot;` back into a literal `"` which prematurely closed the `data-text="..."` attribute.
**Solution**: Replaced all `escapeHtml` in `data-text` attributes across `mission-control.js` with `encodeURIComponent`, and decoded it using `decodeURIComponent` in the `onclick` handler. This guarantees 100% safety for any character (newlines, double quotes, single quotes, backticks).

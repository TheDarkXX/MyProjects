# V12.15.2_[hotfix]_mission-control_send-all-error-visibility

## 📌 Context (Compiled Truth)
The `mcPlannerSendAll()` function in the frontend was previously hiding explicit error messages returned from the backend (e.g., missing video files, FB preflight failures) when batch sending. It also incorrectly queued jobs for all pages rather than the currently filtered page tab.

This hotfix updates both the frontend and backend:
1. **Frontend (`brain-app-public/mission-control.js`)**: Filters `readyJobs` strictly by `vpProjectFilter` and surfaces the `result.error` message to the UI.
2. **Backend (`routes/mission-control.js`)**: Fixes the Data Source Mismatch where the UI saved FB tokens to the SQLite DB (`fb_channels`) but the backend read from `fb-sync-config.json`. Backend now prioritizes reading from the DB.

## 📦 RAW ARTIFACT BACKUP (Iron Rule)

N/A - Direct hotfix based on user feedback.

## 🔬 Timeline & Debugging Log
- Initially updated the frontend `mcPlannerSendAll()` to stop swallowing the 400 error message so the user could see the real error.
- Discovered the root cause: The "Facebook & Instagram Auto-Post" UI saves tokens to the `fb_channels` SQLite table, but `routes/mission-control.js` was strictly reading from a hardcoded `config/fb-sync-config.json` file.
- **Fix:** Modified `routes/mission-control.js` (`send-to-facebook` and `sync-analytics`) to query the `fb_channels` DB first for the `page_id` and `page_token`, and only fallback to `fb-sync-config.json` if not found.
- Executed `/reviewchat` to extract systemic failures, resulting in 3 new Iron Rules (Two-Source Data Consistency, UI State Per-Instance, Surface Real Errors) added directly to `Openclaw-VPS/self-improving/memory.md`.

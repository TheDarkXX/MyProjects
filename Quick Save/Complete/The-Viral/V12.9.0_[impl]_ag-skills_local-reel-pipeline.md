# V12.9.0 [impl] AG Skills Local Reel Pipeline

## 📌 Context (Compiled Truth)
This implementation creates a new local AG Skill (`/reel`) to completely reverse-engineer viral videos into Veo 3.1 cloning blueprints. Previously, reel blueprints were processed asynchronously via a VPS off-peak discord bot. 

The new approach operates locally, leveraging:
1. **Local Grid Extraction**: `blueprint.py` scans a drop-in folder (`Reel Original/`), uses OpenCV (`cv2`) to capture dynamically sampled video frames (up to 36 panels/6 columns), and extracts a thumbnail.
2. **AI Vision Agent**: The script submits the image directly to Gemini's Vision API (`gemini-1.5-pro` / `gemini-2.5-flash`), applying the 'Holy Man' character rules and returning a strict 5-Phase JSON schema (Video DNA, Image/Motion Prompts, Viral Grading, and Clone Template).
3. **Automated VPS Syncing**: The `routes/reel-blueprints.js` route on the VPS Hono backend was upgraded with a `POST /import` endpoint. The local script directly POSTs multipart data (metadata JSON + Grid Image + Thumbnail) into the VPS, seamlessly exposing the clones on the Mission Control Web UI without requiring Git commits for media.
4. **Google Sheets Extension**: Pending addition to also mirror the database append directly into a Google Sheet via `gspread` or VPS `sheets-output.js`.

The user can choose whether to run `blueprint.py` in the background for batch processing or to paste links directly into the AG chat for interactive agent-assisted extraction (using Gemini 3.1 Pro via the AG context).

# 🎞️ Walkthrough: Local Reel Blueprint Pipeline (`/reel`)

I have successfully designed, built, deployed, and tested the **Local Reel Blueprint Pipeline** (`/reel`). The pipeline extracts storyboard grids locally using OpenCV and prompts Gemini Vision API to reverse-engineer viral reels into cloning instructions. The output is directly synced to the VPS Web UI.

## 🛠️ Changes Implemented

### 1. Local Pipeline Setup
- **New Folders**:
  - `P:\AI\The Viral\Reel Original\` (Video input inbox folder)
  - `P:\AI\The Viral\Reel Original\Processed\` (Successfully processed video archive folder)
  - `P:\AI\The Viral\Blueprints\prompts\` (Local storage of prompt templates)
- **Local Prompts**: Copied the master templates `master_instruction.txt` and `knowledge_base_v3.txt` to the local prompt directory.

### 2. Main Script
- **blueprint.py**:
  - Scans `Reel Original/` for `.mp4` video files.
  - Automatically calculates video duration and sampling fps (1 frame/sec for duration ≤ 36s, max 36 panels for longer videos).
  - Uses OpenCV (`cv2`) to capture frames and stitch them into a dynamic 6-column storyboard grid (matching the VPS grid styling).
  - Encodes the storyboard grid to Base64 and executes a structured JSON prompt using **Gemini 2.5 Flash** (via direct REST API, making it extremely fast).
  - Multi-part POSTs the parsed blueprint metadata along with the grid image and first-frame thumbnail to the VPS backend.
  - Moves processed videos to the `Processed/` folder upon completion.

### 3. VPS Web UI Integration
- **routes/reel-blueprints.js**:
  - Added a `POST /import` endpoint that parses multipart form data (text fields + binary images).
  - Saves the grid and thumbnail images to the VPS public folder (`public/blueprints/`).
  - Inserts the completed blueprint into the VPS SQLite database (`reel_blueprints` table), making it instantly available on the Web UI.
- **Git Push Deployment**: Pushed the route updates to the `vps` remote. The deployment hook validated node syntax and restarted `brain-app` and `discord-bot` in PM2 successfully.

### 4. Skill Registration
- **AG_SKILLS_INDEX.md**: Registered `/reel` skill.
- **reel.md**: Created skill instruction file outlining pipeline mechanics and how to execute it.

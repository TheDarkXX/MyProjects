# V12.19.4: Prompt Assembly Bug Fixes (Recovery)

## 📌 Context (Compiled Truth)
The user experienced a power outage and lost critical instructions for fixing Prompt Assembly in the new God Flow App. We recovered the exact chat history where two major bugs were identified:
1. **Variable Replacement Failure:** The God Flow App was not replacing literal placeholder tags like `{V-Tilt}`, `{H-Angle}`, and `{Zoom}` with computed strings. We must implement `.replace()` logic before passing prompts to the API.
2. **Deep Compose (Step 3) Contradiction:** Gemini Vision incorrectly analyzed camera angles from flat 2D images, overriding the calculated 3D parameters. The Step 3 Deep Compose prompt must be explicitly instructed to IGNORE 'Camera Angle' and 'Perspective' entirely.

The Master Prompt Template for the 3D View Tab was also successfully recovered.

## 📦 RAW ARTIFACT BACKUP (Iron Rule)

### The Recovered Master Prompt & Bug Fix Instructions
```text
CRITICAL BUG FIXES FOR PROMPT ASSEMBLY:

1. VARIABLE REPLACEMENT FAILURE:
The Master Prompt is currently displaying and sending literal placeholder tags (e.g. {V-Tilt}, {H-Angle}, {Zoom}) instead of replacing them with the actual generated strings from the Perspective Converter logic.
FIX: Implement a `.replace()` logic to dynamically swap all `{Placeholders}` in the Master Prompt Template with their corresponding calculated strings (hAngleDescription, vTiltDescription, zoomDescription, etc.) before displaying it in the Modal and before sending it to the API.

2. DEEP COMPOSE (STEP 3) CONTRADICTION:
Gemini Vision is currently analyzing the Camera Angle from the 3D snapshot. Because the original flat 2D image has its own baked-in perspective, Gemini incorrectly reports the baked-in angle (e.g. reporting "High angle" when the 3D viewport is actually tilted to a "Low angle"). This directly contradicts and breaks our V-Tilt math.
FIX: Update the Gemini Vision prompt for Step 3 (Deep Compose) to COMPLETELY IGNORE camera angles. 

Use this new Step 3 Gemini Prompt:
"Analyze this image and describe its composition for use in an AI image generation prompt. Be precise. Describe ONLY:
1. FRAMING: What part of the body is visible? (e.g. face and upper chest)
2. CROP EDGES: What is cut off at each edge?
3. ELEMENTS: List all visible elements
4. BACKGROUND: What is visible in background?
CRITICAL: DO NOT describe or mention the 'Camera Angle', 'Perspective', 'High Angle', or 'Low Angle' at all."
```

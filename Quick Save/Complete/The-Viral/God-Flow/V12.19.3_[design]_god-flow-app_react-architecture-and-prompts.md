# God Flow App — React Architecture & Prompts

## 📌 Context (Compiled Truth)
This session involves designing the architecture and generating a series of phased prompts for a Standalone React App (God Flow App) built inside the Google Flow SDK environment. It transitions the previous God Flow Studio Chrome Extension into a native Flow Sandbox application.

Key features designed:
- **3-Panel UI Layout**: Main workspace with Left Panel for settings and Right Panel for Media Inspector.
- **Batch Parallel Pipeline (Phase-Based)**: Replaced sequential execution with `Promise.allSettled` batching (4 concurrent). Explicit synchronization enforces Phase 1 (Image Gen) to 100% completion before Phase 3 (Video Gen).
- **Auto-Group by Image Prompt**: Optimizes API usage by generating shared image prompts only once, then distributing the resulting `mediaId` to multiple video motion prompts.
- **Error Handling Engine**: Robust normalization, Canonical Model Mapping (stripping `[Lower Priority]`), and Error Queue UI.
- **Native PC File Import/Export**: Transitioned project persistence from `Flow.media.select()` (which confuses users as an image picker) to native HTML5 `<input type="file" accept=".json">`.
- **Test Mode (Dry Run)**: Simulated execution bypassing SDK calls for rapid UI testing.

### Component: Error Handling Engine

#### [NEW] `utils/errorEngine.ts`

**Error Classification + Recovery Strategy:**

```
┌──────────────────────────────────────────────────────────────────┐
│                   ERROR HANDLING FLOW                             │
│                                                                  │
│  Flow.generate() throws                                         │
│         │                                                        │
│         ▼                                                        │
│  classifyError(error) ──────────────────────────────────────┐    │
│         │                                                    │    │
│    ┌────┴────┬────────┬──────────┬──────────┐               │    │
│    ▼         ▼        ▼          ▼          ▼               │    │
│  POLICY   RATE     TIMEOUT   SERVER    UNKNOWN              │    │
│    │      LIMIT      │         │          │                 │    │
│    ▼         ▼        ▼          ▼          ▼               │    │
│  AI       Expo.    Retry     Retry      Log +              │    │
│  Rewrite  Backoff  1x       2x (10s)   Skip               │    │
│  Prompt   30→60→   then     gap then                       │    │
│  → Retry  120s     skip     skip                           │    │
│    │         │        │          │          │                │    │
│    ▼         ▼        ▼          ▼          ▼               │    │
│  Success? ──Yes──→ Continue                                 │    │
│    │                                                         │    │
│    No ────────────→ ErrorQueue (Manual Retry UI)            │    │
└──────────────────────────────────────────────────────────────────┘
```

**5 Error Strategies:**

| # | Error Type | Detection | Auto Recovery | Fallback |
|---|---|---|---|---|
| 1 | 🔴 **Policy Violation** | `"policy"`, `"safety"`, `"blocked"` in error msg | **AI Prompt Rewrite** → Gemini rewrites prompt to be policy-safe → retry 1x | → Error Queue |
| 2 | 🟠 **Rate Limit** | `"quota"`, `"rate"`, `"limit"` in error msg | **Exponential Backoff**: 30s → 60s → 120s → 240s (max 4 attempts) | → Pause pipeline + ask user |
| 3 | 🟡 **Timeout** | `Promise.race` 120s timeout | Retry 1x | → Error Queue |
| 4 | ⚫ **Server Error** | Generic/unknown error | Retry 2x with 10s gap | → Error Queue |
| 5 | 🔵 **Batch Partial** | `Promise.allSettled` → `status: 'rejected'` | Isolate failed prompt → retry individually with appropriate strategy | → Error Queue |

**AI Prompt Rewrite (Policy Recovery):**
```typescript
async function rewritePrompt(original: string): Promise<string> {
  const result = await Flow.generate.text({
    prompt: `This image prompt was blocked by content safety policy.
Rewrite it to convey the SAME visual concept while being safe.
Keep artistic intent. Remove any potentially sensitive elements.
Return ONLY the rewritten prompt.

Original: "${original}"`,
    thinkingLevel: 'low'
  });
  return result.text.trim();
}
```

**Exponential Backoff (Rate Limit Recovery):**
```typescript
function calculateCooldown(attempt: number, base: number = 30): number {
  // 30s → 60s → 120s → 240s (capped)
  return Math.min(base * Math.pow(2, attempt), 300);
}
```

**Batch Safety (Promise.allSettled):**
```typescript
// ใช้ allSettled แทน all เพื่อไม่ให้ 1 fail ทำทั้ง batch พัง
const results = await Promise.allSettled(
  batch.map(p => withTimeout(
    Flow.generate.image({ prompt: p, ... }),
    120_000 // 2 min timeout per image
  ))
);

// แยก success vs fail
const successes = results.filter(r => r.status === 'fulfilled');
const failures = results.filter(r => r.status === 'rejected');

// failures → classify + handle individually
for (const fail of failures) {
  const errorType = classifyError(fail.reason);
  await handleError(errorType, fail, ...);
}
```

**Pipeline Execution Logic — 4 Phases:**

เมื่อกด Start:

**Phase 1: Batch Parallel Image Generation**
1. Group rows by imagePrompt (rows ที่มี imagePrompt เหมือนกัน gen ครั้งเดียว)
2. สำหรับแต่ละ group:
   a. สร้าง array of promises ขนาด imagesPerPrompt (e.g. 4)
   b. ทุก promise เรียก: Flow.generate.image({
        prompt: imagePrompt,
        modelDisplayName: imageSettings.model (default '🍌 Nano Banana Pro'),
        aspectRatio: imageSettings.aspectRatio (default '9:16'),

   c. const results = await Promise.allSettled(promises)
   d. fulfilled → Flow.upload() ทันที → เก็บ mediaId ใน row.generatedImageIds
   e. rejected → classify error:
      - Policy: ใช้ Flow.generate.text() rewrite prompt → retry 1x
      - Rate limit: cooldown exponential backoff (30→60→120s)
      - Timeout/Server: retry 1-2x → skip
   f. Update row status + progress bar
   g. Log: "[Group X] Generated Y/Z images"
3. Inter-batch delay between groups

**Phase 2: AI Image Selection (Optional)**
1. ถ้า pipelineSettings.aiSelect === true:

        1. Visual quality and aesthetics
        2. Relevance to this prompt: "${imagePrompt}"
        
        Return ONLY a JSON object: { "score": number, "reasoning": string }`,

2. ถ้า aiSelect === false:
   ใช้รูปแรกของแต่ละ group

**Phase 4: Auto-Save & Report**

4. Log summary:

   "═══════════════════════════════"

   "✅ Videos: A/B generated"

## Phase 6 — Error Handling Engine + Error Queue UI

```
สร้างระบบ Error Handling:

**1. Error Classification Function:**

สร้าง utility function classifyError(error: any): ErrorType:
- ถ้า error.message มี "policy", "safety", "blocked", "inappropriate" → return 'policy'
- ถ้า error.message มี "quota", "rate", "limit", "too many" → return 'rateLimit'
- ถ้า error.message มี "timeout", "timed out" → return 'timeout'
- ถ้า error.message มี "server", "internal", "500" → return 'server'
- อื่นๆ → return 'unknown'

**2. Error Recovery Functions:**

a) handlePolicyError(prompt, settings):
   - เรียก Flow.generate.text({
       prompt: "This image generation prompt was blocked by content safety policy. Rewrite it to convey the same visual concept while being completely safe and policy-compliant. Keep the artistic intent and visual description intact but remove any potentially sensitive elements. Return ONLY the rewritten prompt, nothing else.\n\nOriginal prompt: \"" + prompt + "\"",
       thinkingLevel: 'low'
     })
   - ได้ rewritten prompt → retry Flow.generate.image() 1 ครั้ง
   - Log: "🔄 AI Rewrite: [original 40 chars]... → [rewritten 40 chars]..."
   - ถ้า retry สำเร็จ → return result
   - ถ้า retry fail → add to errorQueue

b) handleRateLimitError(attempt, cooldownBase):
   - Calculate cooldown: min(cooldownBase * 2^attempt, 300) seconds
   - Log: "⏳ Rate limit — cooling down Xs..."
   - แสดง countdown ใน StatusBar
   - await sleep(cooldown * 1000)
   - return → caller จะ retry

c) withTimeout(promise, ms):
   - Promise.race([promise, new Promise((_, reject) => setTimeout(() => reject(new Error('timeout')), ms))])
   - Default timeout: 120000ms (2 นาที)

**3. Batch Error Handling:**

ปรับ execution logic ใน ImageGenTab, VideoTab, PipelineTab:
- เปลี่ยนจาก Promise.all → Promise.allSettled
- สำหรับ results ที่ rejected:
  1. classifyError()
  2. ถ้า policy → handlePolicyError()
  3. ถ้า rateLimit → handleRateLimitError() → re-add to next batch
  4. ถ้า timeout/server → retry 1-2x → ถ้า fail → add to errorQueue
  5. Update row.status = 'error', row.errorInfo = errorEntry

**4. ErrorQueue Component** (floating panel):

- แสดงเมื่อ errorQueue.length > 0
- Position: fixed, bottom 60px (above StatusBar), right 16px
- Width: 380px, max-height 300px, overflow-y auto
- Background: rgba(15,15,35,0.95), border: 1px solid rgba(239,68,68,0.3), border-radius 12px
- Header: "⚠️ Error Queue (X items)" + ปุ่ม Close

แต่ละ error entry:
- Icon ตาม errorType: 🔴 policy, 🟠 rateLimit, 🟡 timeout, ⚫ server
- Row number + prompt (truncated 50 chars)
- Badge: [Policy] / [Rate Limit] / [Timeout] / [Server]
- ถ้ามี rewrittenPrompt → แสดง "Rewritten: [text]"
- 3 action buttons:
  - "🔄 Retry" → retry ด้วย rewritten prompt (ถ้ามี) หรือ original
  - "✏️ Edit" → เปิด ExpandModal ให้ user แก้ prompt → retry
  - "⏭️ Skip" → remove จาก queue

Footer: "🔄 Retry All" + "⏭️ Skip All" buttons

สร้าง React App ชื่อ "God Flow Studio" สำหรับ batch generate ภาพและวิดีโอแบบอัตโนมัติ

เริ่มจากสร้างโครงสร้าง types และ App shell:

1. สร้างไฟล์ types.ts กำหนด TypeScript interfaces ทั้งหมด:

- AppState: 'idle' | 'running' | 'paused' | 'complete' | 'error'
- RowStatus: 'pending' | 'running' | 'success' | 'error' | 'skipped' | 'retrying'
- TabId: 'imageGen' | 'video' | 'pipeline' | 'manual'
- ErrorType: 'policy' | 'rateLimit' | 'timeout' | 'server' | 'unknown'

- ErrorEntry: { rowId, errorType, message, originalPrompt, rewrittenPrompt?, attempts, timestamp }
- MediaItem: { mediaId, base64, mimeType, name }

- PromptRow: { id, prompt, usePerson: boolean, useProduct: boolean, status: RowStatus, errorInfo? }
- VideoRow: { id, imageMediaId, imageThumbnail, fileName, prompt, status, errorInfo? }
- KeywordVdoRow: { id, imagePrompt, motionPrompt, status, generatedImageId?, generatedVideoId?, errorInfo? }
- PipelineRow: { id, imagePrompt, motionPrompt, status, generatedImageIds: string[], selectedImageId, generatedVideoIds: string[], errorInfo? }
- ManualRow: { id, prompt }

- ReferenceImages: { person: MediaItem[], product: MediaItem[] }

- ImageSettings: { model: string (default '🍌 Nano Banana Pro'), aspectRatio: string (default '9:16'), batchSize: number (default 4), interPromptDelay: number (default 5), batchDelay: number (default 10) }
- VideoSettings: { model: string (default 'Veo 3.1 - Lite'), aspectRatio: string (default '9:16'), duration: number (default 8), batchSize: number (default 2), interPromptDelay: number (default 5), batchDelay: number (default 10) }
- PipelineSettings: { imagesPerPrompt: 1|2|4 (default 4), videosPerImage: 1|2 (default 1), aiSelect: boolean (default true), refKeyword: string, maxRetries: number (default 3), cooldownBase: number (default 30), enablePromptRewrite: boolean (default true) }

- ProjectFile: { version: '1.0', savedAt: string, activeTab, imageGenRows, videoRows, kwVdoRows, pipelineRows, manualRows, referenceImages, personKeyword, productKeyword, imageSettings, videoSettings, pipelineSettings, errorQueue: ErrorEntry[] }

2. สร้าง App.tsx เป็น main component:
- ใช้ useState เก็บ state ทั้งหมดตาม types ด้านบน
- เพิ่ม state: activeTab, runState, logs[], progress: { current, total, percent }, errorQueue[]
- สร้าง helper functions: generateId() สำหรับ unique row ids, addLog(msg, level) สำหรับ logging
- Render placeholder div สำหรับแต่ละ section: header (TabBar), main content area, settings sidebar, status bar
- ใช้ dark theme: background #0f0f23, text #e2e8f0
- Import Google Font "Inter" สำหรับ typography ทั้ง app
```

## Phase 4 — Pipeline Tab: Full Auto 4-Phase Batch Parallel

```
สร้าง PipelineTab component — นี่คือ core feature หลัก:

     - pending: transparent

   - Background: rgba(255,255,255,0.05), border-radius 8px

3. ปุ่ม "+ Add Prompt" / "🗑️ Clear All"

4. **Excel Paste (TSV) — เหมือน VideoTab Keyword mode:**
   - onPaste บน table container
   - ตรวจ Tab character → split columns → Image Prompt + Motion Prompt
   - เพิ่ม rows จาก parsed data

**Settings Sidebar:**
- Text input "Reference Keyword" (optional)
- Dropdown "Images per Prompt": [1, 2, 4], default 4
- Dropdown "Videos per Image": [1, 2], default 1
- Checkbox "🧠 AI เลือกรูป (Gemini Vision)" — default checked
- Section "Error Recovery":
  - Number "Max Retries": default 3
  - Number "Cooldown Base (s)": default 30
  - Checkbox "AI Prompt Rewrite": default checked

**Start Button:** gradient สีส้ม-แดง, text "▶️ Start Auto Pipeline"

---

**Pipeline Execution Logic — 4 Phases:**

เมื่อกด Start:

**Phase 1: Batch Parallel Image Generation**
```
1. Group rows by imagePrompt (rows ที่มี imagePrompt เหมือนกัน gen ครั้งเดียว)
2. สำหรับแต่ละ group:
   a. สร้าง array of promises ขนาด imagesPerPrompt (e.g. 4)
   b. ทุก promise เรียก: Flow.generate.image({
        prompt: imagePrompt,
        modelDisplayName: imageSettings.model (default '🍌 Nano Banana Pro'),
        aspectRatio: imageSettings.aspectRatio (default '9:16'),
        referenceImageMediaIds: (ถ้ามี refKeyword → lookup mediaIds)
      })
   c. const results = await Promise.allSettled(promises)
   d. fulfilled → Flow.upload() ทันที → เก็บ mediaId ใน row.generatedImageIds
   e. rejected → classify error:
      - Policy: ใช้ Flow.generate.text() rewrite prompt → retry 1x
      - Rate limit: cooldown exponential backoff (30→60→120s)
      - Timeout/Server: retry 1-2x → skip
   f. Update row status + progress bar
   g. Log: "[Group X] Generated Y/Z images"
3. Inter-batch delay between groups
```

**Phase 2: AI Image Selection (Optional)**
```
1. ถ้า pipelineSettings.aiSelect === true:
   สำหรับแต่ละ group ที่มี > 1 generated image:
   a. Batch evaluate ด้วย Promise.allSettled:
      Flow.generate.text({
        prompt: `You are an image quality evaluator. Rate this image on a scale of 1-10 for:
        1. Visual quality and aesthetics
        2. Relevance to this prompt: "${imagePrompt}"
        
        Return ONLY a JSON object: { "score": number, "reasoning": string }`,
        images: [{ base64: imageBase64, mimeType: imageMimeType }],
        thinkingLevel: 'low'
      })
   b. Parse scores → sort descending → เลือกรูปคะแนนสูงสุด → set row.selectedImageId
   c. ถ้า evaluate fail → fallback ใช้รูปแรก
   d. Log: "[Group X] AI selected image with score Y/10"

2. ถ้า aiSelect === false:
   ใช้รูปแรกของแต่ละ group
```

**Phase 3: Batch Parallel Video Generation**
```
1. สำหรับแต่ละ row ที่มี selectedImageId:
   a. สร้าง array of promises ขนาด videosPerImage
   b. ทุก promise: Flow.generate.video({
        prompt: motionPrompt,
        firstFrameImageMediaId: selectedImageId,
        modelDisplayName: videoSettings.model (default 'Veo 3.1 - Lite'),
        aspectRatio: videoSettings.aspectRatio (default '9:16'),
        durationSeconds: videoSettings.duration (default 8)
      })
   c. Promise.allSettled → fulfilled → Flow.upload() → เก็บ mediaId ใน row.generatedVideoIds
   d. rejected → same error handling (policy rewrite, backoff, retry)
   e. Delay 10s between video batches (video gen ใช้ resources เยอะกว่า image)
```

**Phase 4: Auto-Save & Report**
```
1. Loop ทุก generated mediaIds → Flow.save({ mediaId }) — save ลง Gallery
2. Auto-save project file
3. Collect error queue → แสดงใน ErrorQueue component
4. Log summary:
   "═══════════════════════════════"
   "📊 PIPELINE SUMMARY"
   "═══════════════════════════════"
   "✅ Images: X/Y generated"
   "✅ Videos: A/B generated"
   "🔴 Errors: Z (see Error Queue)"
   "⏱️ Total: Xm Ys"
5. Set runState = 'complete'
```

## Phase 6 — Error Handling Engine + Error Queue UI

```
สร้างระบบ Error Handling:

**1. Error Classification Function:**

สร้าง utility function classifyError(error: any): ErrorType:
- ถ้า error.message มี "policy", "safety", "blocked", "inappropriate" → return 'policy'
- ถ้า error.message มี "quota", "rate", "limit", "too many" → return 'rateLimit'
- ถ้า error.message มี "timeout", "timed out" → return 'timeout'
- ถ้า error.message มี "server", "internal", "500" → return 'server'
- อื่นๆ → return 'unknown'

**2. Error Recovery Functions:**

a) handlePolicyError(prompt, settings):
   - เรียก Flow.generate.text({
       prompt: "This image generation prompt was blocked by content safety policy. Rewrite it to convey the same visual concept while being completely safe and policy-compliant. Keep the artistic intent and visual description intact but remove any potentially sensitive elements. Return ONLY the rewritten prompt, nothing else.\n\nOriginal prompt: \"" + prompt + "\"",
       thinkingLevel: 'low'
     })
   - ได้ rewritten prompt → retry Flow.generate.image() 1 ครั้ง
   - Log: "🔄 AI Rewrite: [original 40 chars]... → [rewritten 40 chars]..."
   - ถ้า retry สำเร็จ → return result
   - ถ้า retry fail → add to errorQueue

b) handleRateLimitError(attempt, cooldownBase):
   - Calculate cooldown: min(cooldownBase * 2^attempt, 300) seconds
   - Log: "⏳ Rate limit — cooling down Xs..."
   - แสดง countdown ใน StatusBar
   - await sleep(cooldown * 1000)
   - return → caller จะ retry

c) withTimeout(promise, ms):
   - Promise.race([promise, new Promise((_, reject) => setTimeout(() => reject(new Error('timeout')), ms))])
   - Default timeout: 120000ms (2 นาที)

**3. Batch Error Handling:**

ปรับ execution logic ใน ImageGenTab, VideoTab, PipelineTab:
- เปลี่ยนจาก Promise.all → Promise.allSettled
- สำหรับ results ที่ rejected:
  1. classifyError()
  2. ถ้า policy → handlePolicyError()
  3. ถ้า rateLimit → handleRateLimitError() → re-add to next batch
  4. ถ้า timeout/server → retry 1-2x → ถ้า fail → add to errorQueue
  5. Update row.status = 'error', row.errorInfo = errorEntry

async function generateThumbnail(base64: string, mimeType: string): Promise<string> {
  return new Promise((resolve) => {
    const img = new Image();
    img.onload = () => {
      const canvas = document.createElement('canvas');
      const SIZE = 64;
      canvas.width = SIZE;
      canvas.height = SIZE;
      const ctx = canvas.getContext('2d')!;
      // cover crop to square
      const scale = Math.max(SIZE / img.width, SIZE / img.height);
      const x = (SIZE - img.width * scale) / 2;
      const y = (SIZE - img.height * scale) / 2;
      ctx.drawImage(img, x, y, img.width * scale, img.height * scale);
      resolve(canvas.toDataURL('image/jpeg', 0.6));
    };
    img.src = `data:${mimeType};base64,${base64}`;
  });
}

const filteredItems = useMemo(() => {
  let items = [...galleryItems];
  
  // Type filter
  if (filter.mediaType !== 'all') {
    items = items.filter(i => i.mediaType === filter.mediaType);
  }
  
  // Source filter
  if (filter.source !== 'all') {
    items = items.filter(i => i.source === filter.source);
  }
  
  // Date filter
  if (filter.dateRange !== 'all') {
    const now = new Date();
    items = items.filter(i => {
      const d = new Date(i.createdAt);
      if (filter.dateRange === 'today') return d.toDateString() === now.toDateString();
      if (filter.dateRange === 'week') return (now.getTime() - d.getTime()) < 7 * 86400000;
      if (filter.dateRange === 'month') return (now.getTime() - d.getTime()) < 30 * 86400000;
      return true;
    });
  }
  
  // Favorites
  if (filter.favorites) items = items.filter(i => i.isFavorite);
  
  // Search
  if (filter.searchQuery.trim()) {
    const q = filter.searchQuery.toLowerCase();
    items = items.filter(i => 
      i.prompt.toLowerCase().includes(q) ||
      i.model.toLowerCase().includes(q) ||
      i.tags.some(t => t.toLowerCase().includes(q))
    );
  }
  
  // Sort
  items.sort((a, b) => {
    if (filter.sortBy === 'newest') return new Date(b.createdAt).getTime() - new Date(a.createdAt).getTime();
    if (filter.sortBy === 'oldest') return new Date(a.createdAt).getTime() - new Date(b.createdAt).getTime();
    return a.prompt.localeCompare(b.prompt);
  });
  
  return items;
}, [galleryItems, filter]);

// Scene
const scene = new THREE.Scene();
scene.background = new THREE.Color(0x0f0f23);

// Camera
const camera = new THREE.PerspectiveCamera(fov, width/height, 0.1, 100);
camera.position.set(0, 0, 2);

// Studio Lighting Rig (ported from Pro)
const ambient = new THREE.AmbientLight(0xffffff, 0.4);
const spotLight = new THREE.SpotLight(0xffffff, 0.8);
spotLight.position.set(2, 3, 2);
spotLight.castShadow = true;
const pointLight = new THREE.PointLight(0x8b5cf6, 0.3);
pointLight.position.set(-2, 1, 1);
scene.add(ambient, spotLight, pointLight);

// Ground shadow plane
const shadowGeo = new THREE.PlaneGeometry(10, 10);
const shadowMat = new THREE.ShadowMaterial({ opacity: 0.15 });
const shadowPlane = new THREE.Mesh(shadowGeo, shadowMat);
shadowPlane.receiveShadow = true;
shadowPlane.rotation.x = -Math.PI / 2;
shadowPlane.position.y = -0.8;
scene.add(shadowPlane);

// Subdivisions based on depthQuality
const subdivisions = { low: 64, medium: 128, high: 256 }[depthQuality];
const displacementScale = { low: 0.08, medium: 0.12, high: 0.18 }[depthQuality];

// Plane geometry for image
const imageTexture = new THREE.TextureLoader().load(`data:${imageMimeType};base64,${imageBase64}`);
const geometry = new THREE.PlaneGeometry(1, aspectRatio, subdivisions, subdivisions);
const material = new THREE.MeshStandardMaterial({
  map: imageTexture,
  side: THREE.DoubleSide,
});

// If depthMap provided: apply displacement for 2.5D effect
if (depthMapBase64) {
  const depthTexture = new THREE.TextureLoader().load(`data:image/png;base64,${depthMapBase64}`);
  material.displacementMap = depthTexture;
  material.displacementScale = displacementScale;
  material.displacementBias = -displacementScale / 2;
}

const mesh = new THREE.Mesh(geometry, material);
mesh.castShadow = true;
scene.add(mesh);

// Renderer
const renderer = new THREE.WebGLRenderer({ antialias: true, canvas: canvasRef.current });
renderer.setSize(width, height);
renderer.setPixelRatio(window.devicePixelRatio);
renderer.shadowMap.enabled = true;

// OrbitControls (import from 'three/addons/controls/OrbitControls.js' or CDN)
const controls = new OrbitControls(camera, renderer.domElement);
controls.enablePan = false;
controls.enableDamping = true;
controls.dampingFactor = 0.08;
controls.minDistance = 0.5;
controls.maxDistance = 5;
controls.autoRotate = autoRotate;
controls.autoRotateSpeed = 2.0;

// Sync controls → state on user drag
controls.addEventListener('change', () => {
  // Convert camera position → euler angles for sliders
  const euler = new THREE.Euler().setFromQuaternion(camera.quaternion, 'XYZ');
  onRotationChange(
    THREE.MathUtils.radToDeg(euler.x),
    THREE.MathUtils.radToDeg(euler.y),
    THREE.MathUtils.radToDeg(euler.z)
  );
  onZoomChange(camera.position.length() / 2); // normalized zoom
});

// Animation loop
let animId: number;
const animate = () => {
  animId = requestAnimationFrame(animate);
  controls.update();
  renderer.render(scene, camera);
};
animate();

// Cleanup
return () => {
  cancelAnimationFrame(animId);
  renderer.dispose();
  geometry.dispose();
  material.dispose();
};

```typescript
export interface Viewport3DState {
  rotationX: number;
  rotationY: number;
  rotationZ: number;
  zoom: number;
  fov: number;
}

export function convertPerspectiveToPrompt(state: Viewport3DState): string {
  const parts: string[] = [];

  // Yaw → horizontal direction
  const y = state.rotationY;
  if (y >= -22 && y <= 22) parts.push('front-facing');
  else if (y > 22 && y <= 67) parts.push('three-quarter angle from the left');
  else if (y > 67 && y <= 112) parts.push('side profile from the left');
  else if (y > 112 || y < -112) parts.push('from behind, back view');
  else if (y < -22 && y >= -67) parts.push('three-quarter angle from the right');
  else if (y < -67 && y >= -112) parts.push('side profile from the right');

  // Pitch → vertical angle
  const x = state.rotationX;
  if (x > 30) parts.push("bird's eye view looking down");
  else if (x > 10) parts.push('slightly elevated high angle');
  else if (x < -30) parts.push("worm's eye view looking up");
  else if (x < -10) parts.push('low angle looking up');
  else parts.push('at eye level');

  // Roll → Dutch angle
  if (Math.abs(state.rotationZ) > 10) {
    parts.push(`Dutch angle tilted ${state.rotationZ > 0 ? 'right' : 'left'}`);
  }

  // Zoom → shot type
  if (state.zoom > 2.0) parts.push('extreme close-up');
  else if (state.zoom > 1.5) parts.push('close-up shot');
  else if (state.zoom < 0.7) parts.push('wide shot');
  else parts.push('medium shot');

  return parts.join(', ');
}
```

export async function generateDepthMap(
  subjectImage: MediaItem,
  quality: 'low' | 'medium' | 'high'
): Promise<MediaItem | null> {
  try {
    // Step 1: Generate depth map image via Imagen 4
    const depthPromptText = await Flow.generate.text({
      prompt: `Analyze the depth/distance of elements in this image. 
List the main elements from closest (foreground) to farthest (background).
Keep your answer brief — just list 3-5 elements with their relative depth: NEAR, MID, FAR.`,
      images: [{ base64: subjectImage.base64 ?? '', mimeType: subjectImage.mimeType }],
      thinkingLevel: 'low'
    });
    
    // Step 2: Generate actual grayscale depth map image
    const subdivisions = quality === 'low' ? 'rough' : quality === 'medium' ? 'smooth' : 'very detailed and smooth';
    const depthMapResult = await Flow.generate.image({
      prompt: `Create a ${subdivisions} grayscale depth map for the following scene:
${depthPromptText.text}
Rules: Pure white = closest elements (face, foreground hands, nearby objects).
Pure black = background (sky, distant walls). Mid-grey = middle distance.
Smooth gradients, no hard edges. No text, labels, or artifacts.
Output ONLY the depth map — not the original image.`,
      modelDisplayName: 'Imagen 4',
      aspectRatio: subjectImage.aspectRatio ?? '9:16',
    });
    
    const depthMediaId = await Flow.upload({
      base64: depthMapResult.base64,
      mimeType: depthMapResult.mimeType,
      name: `depth_map_${Date.now()}.png`
    });
    
    return {
      mediaId: depthMediaId.mediaId,
      base64: depthMapResult.base64,
      mimeType: depthMapResult.mimeType,
      name: `depth_map_${Date.now()}.png`
    };
  } catch (err) {
    console.error('Depth map generation failed, using flat 2D mode:', err);
    return null;
  }
}

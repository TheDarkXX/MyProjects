# Phase 11 — Native Project File System (Main DB)

เพิ่มระบบ "Native Project File System (Main DB)" โดยใช้ File System Access API เพื่อแก้ปัญหา Sandbox รีเซ็ตข้อมูล และทำระบบ Auto-save แบบเงียบๆ ลงเครื่องผู้ใช้ โดยไม่ต้องกดดาวน์โหลดใหม่ทุกครั้ง

ให้ดำเนินการสร้างและแก้ไขไฟล์ตามนี้:

1. สร้างไฟล์ `hooks/useFileSystemDB.ts`:
- สร้าง Custom Hook ชื่อ `useFileSystemDB`
- มี State เก็บ `fileHandle` (ชนิด FileSystemFileHandle | null) ไว้ในหน่วยความจำ
- มี State เก็บ `fileName` (ชื่อไฟล์ที่กำลังเปิดอยู่) และ `lastSaved` (timestamp ล่าสุด)
- สร้างฟังก์ชัน `createNewProject(defaultState)`: เรียกใช้ `window.showSaveFilePicker({ suggestedName: 'workspace.gfs.json', types: [...] })` เพื่อสร้างไฟล์ใหม่และเก็บ handle ไว้ จากนั้นเขียน defaultState ลงไป
- สร้างฟังก์ชัน `loadProject()`: เรียกใช้ `window.showOpenFilePicker({ types: [...] })` เพื่อเลือกไฟล์เดิม ดึงข้อมูลมา parse เป็น JSON และเก็บ handle ไว้
- สร้างฟังก์ชัน `autoSave(currentState)`: ตรวจสอบว่าถ้ามี `fileHandle` อยู่ ให้สร้าง `createWritable()`, เขียน JSON string ลงไป, และ `close()` โดยฟังก์ชันนี้ต้องทำงานเงียบๆ (Background) ห้ามเด้ง popup กวนผู้ใช้
- Return { fileHandle, fileName, lastSaved, createNewProject, loadProject, autoSave }

2. อัปเดตไฟล์ `components/ProjectBar.tsx` (หรือสร้างใหม่):
- สร้างปุ่ม "📄 New Project" (เรียก createNewProject)
- สร้างปุ่ม "📂 Load Project" (เรียก loadProject)
- แสดงชื่อไฟล์ที่กำลัง Sync อยู่ และเวลาที่เซฟล่าสุด (เช่น "Connected: workspace.gfs.json | Last saved: 14:30")
- ถ้าเชื่อมต่อไฟล์สำเร็จแล้ว ให้เปลี่ยนสีปุ่มหรือแสดงสถานะสีเขียวว่า "🟢 DB Linked"

3. อัปเดต `App.tsx`:
- Import `useFileSystemDB` เข้ามา
- เมื่อผู้ใช้ Load Project สำเร็จ ให้นำ JSON ที่ได้มา Hydrate (แทนที่) State ทั้งหมดของ App ทันที
- ใช้ `useEffect` เพื่อทำ "Auto-Save Engine": ให้จับตาดู State หลัก (เช่น imageGenRows, videoRows, settings) ถ้ามีการเปลี่ยนแปลง ให้เรียก `autoSave(currentState)` โดยอาจจะครอบด้วย `setTimeout` (Debounce ประมาณ 2 วินาที) เพื่อลดภาระการเขียนไฟล์รัวเกินไป

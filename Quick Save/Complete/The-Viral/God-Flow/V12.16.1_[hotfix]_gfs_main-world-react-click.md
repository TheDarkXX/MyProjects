# V12.16.1 - GFS Send Button Main World React Fiber Bridge

## đź“Ś Context (Compiled Truth)
The "Send" button in God Flow Studio (GFS) was failing to trigger despite receiving native `.click()` and mouse events from the Isolated World (content.js). Also, using CDP (Chrome DevTools Protocol) caused Google Flow to display a yellow "Debugging" banner that froze the UI.

The root cause was that `content.js` runs in an Isolated World and cannot access React's internal properties (e.g. `__reactFiber$`). To bypass this, we implemented a CustomEvent bridge.
1. `flow-inspector.js` runs in the MAIN World (injected at `document_start`). It listens for `__gfs_click_request__`.
2. `content.js` calculates the button's coordinates using `getBoundingClientRect()` and dispatches `__gfs_click_request__` to the window.
3. `flow-inspector.js` receives the event, uses `document.elementFromPoint(x, y)` to find the button.
4. It walks up the React Fiber tree (`__reactFiber$`) up to 20 levels to find the `memoizedProps.onClick` handler.
5. It invokes the `onClick` handler directly with a Synthetic Event.
6. It also falls back to `__reactProps$.onClick`, native `.click()`, and mouse events.
7. Both Image and Video flows (`trySendPrompt` and `FLOW_KW_TYPE_AND_SEND`) now use this CustomEvent bridge to achieve instant, reliable button clicks.

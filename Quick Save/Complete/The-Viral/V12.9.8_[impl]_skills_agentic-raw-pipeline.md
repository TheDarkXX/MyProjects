# V12.9.8 [impl] skills: Agentic raw.py Pipeline Refactor

## 📌 Context (Compiled Truth)
- **Why:** The previous `/raw` pipeline implementation used an automated script (`raw.py`) that called `gemini-2.5-flash` directly via the Gemini API to evaluate video content, generate captions, and assign viral tiers. The user pointed out that this API-driven approach is less powerful and more prone to errors than having Antigravity (AG, powered by Gemini 3.1 Pro/Claude) orchestrate and visually grade the videos natively inside the workspace.
- **What was decided:** Refactor `/raw` into an **Agentic Workflow**:
  1. The Python script (`raw.py`) is stripped of its hardcoded API keys and internal Gemini API calls. It is refactored to support command-line arguments:
     - `python raw.py extract`: Scans `Raw VDO` folder, extracts First Frame, Thumbnail, and a 4x2 Grid of uniformly distributed frames. Prints a JSON list of extracted files.
     - `python raw.py finalize --file <filename> --id <id> --caption <caption> --tier <tier>`: Receives AG's assessment results, renames/moves the video and previews to the corresponding tier folder under `REVIEW/`, logs the entry to `Content_Tracker.md`, and uploads everything to the VPS dashboard.
  2. The AG Skill instruction [raw.md](file:///p:/AI/The%20Viral/ag_skills_backup/raw.md) is updated to guide AG through acting as the central orchestrator (calling `extract`, visually inspecting `_grid.jpg` using the `view_file` tool, deducing Story IDs and emotional captions, and executing `finalize`).

### 🛡️ Iron Rules Update (Namespace Separation)
Following a `/reviewchat` regarding save file cross-contamination, `P:\AI\The Viral\memory.md` was updated with Iron Rules 12, 13, and 14. These enforce strict directory separation for `/save` (Local goes to `The-Viral/`, Cloud goes to `V12/`), mandate artifact relevance filtering, and demand explicit scope reporting to alleviate user paranoia during file batch modifications.

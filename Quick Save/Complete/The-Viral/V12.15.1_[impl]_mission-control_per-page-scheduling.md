# V12.15.1_[impl]_mission-control_per-page-scheduling

## 📌 Context (Compiled Truth)
The Viral Planner Pro UI previously used a global schedule setting (`viral-schedule.json` as a flat array `[12, 16, 22]`) where changing the auto-schedule times would overwrite the settings for all Facebook pages. The user requested this to be per-page.

The backend was updated to migrate the `viral-schedule.json` file from a flat array to a page-specific JSON object map `{ "Page A": [x], "Page B": [y] }`. The `getScheduleTimes` function was refactored to read by `pageName`, and the UI was updated to enforce selecting a page before editing schedule times. `POST /planner/schedule-times` and `/planner/auto-schedule` were also updated to process times per-page.

## Proposed Changes

---

### Backend: `routes/mission-control.js`

แก้ไขระบบการจัดเก็บและดึงข้อมูลเวลา ให้ผูกกับชื่อเพจ (`page_name`)

1. **`getScheduleTimes(pageName)`**: เปลี่ยนโครงสร้าง `viral-schedule.json` จากเดิมที่เป็นแค่ Array `[x,y,z]` ให้เป็น Object ที่แยกตามเพจ เช่น:
   ```json
   {
     "The Reach of Jesus": [12, 16, 22],
     "ฟ้า นภัสสร": [3, 10, 13]
   }
   ```
2. **`GET /planner/schedule-times`**: ดึงข้อมูลตามชื่อเพจที่ส่งมาจาก UI (`?page=...`)
3. **`POST /planner/schedule-times`**: รับ Payload ที่ประกอบด้วย `{ page, times }` เพื่อไปอัปเดตเฉพาะคีย์ของเพจนั้นใน JSON โดยไม่ทับของเพจอื่น
4. **`getNextAvailableSlots(count, pageName)`**: ปรับให้ดึงเวลาอ้างอิงจาก `pageName` ที่ส่งเข้ามา
5. **`POST /planner/auto-schedule`**: ปรับลอจิกตอนวนลูปสร้างคิว ให้จัดกลุ่ม Job ตามเพจก่อน แล้วค่อยดึง `getNextAvailableSlots` ของแต่ละเพจแยกกัน
6. **`GET /planner/next-slots`**: เพิ่มการรับค่า `?page=...`

---

### Frontend: `brain-app-public/mission-control.js`

ปรับ UI ให้การแก้ไขเวลาอิงตามแท็บเพจที่เลือกอยู่

1. **`vpEditScheduleTimes()`**: 
   - เช็คก่อนว่าคุณเลือกแท็บเพจหรือยัง (ถ้าเลือกแท็บ "All Pages" ระบบจะแจ้งเตือนให้กดเลือกเพจใดเพจหนึ่งก่อนตั้งเวลา)
   - ดึงเวลาของเพจที่กำลังใช้งาน (`?page=` + `vpProjectFilter`) มาแสดงผล
   - ตอนกดบันทึก ให้ส่ง Payload ไปแบบระบุชื่อเพจ: `{ page: vpProjectFilter, times: [...] }`
2. **`mcShowPlanner()`**: ตอนโหลดข้อมูลเบื้องต้นให้ส่ง `?page=` แนบไปด้วยเพื่อให้ได้ `nextSlots` ที่ถูกต้อง

# 📝 สรุปการอัปเดตระบบแยกเวลาโพส Viral Planner Pro

ระบบได้ถูกอัปเกรดให้สามารถแยก **Schedule Times** ตามแต่ละเพจ (Page-Specific) เป็นที่เรียบร้อยแล้วครับ 🎉 การทำงานของระบบต่อจากนี้จะเป็นอิสระต่อกันโดยสมบูรณ์

---

## สิ่งที่ถูกปรับปรุง

### 1. ระบบจัดเก็บ (Backend Database)
- **จากเดิม**: บันทึกข้อมูลเป็นแบบ Global Array เดียวแชร์กันทุกเพจ (เช่น `[12, 16, 22]`)
- **ใหม่**: บันทึกแยกเป็น Object ผูกกับชื่อเพจ เพื่อไม่ให้ทับซ้อนกัน:
  ```json
  {
    "The Reach of Jesus": [12, 16, 22],
    "ฟ้า นภัสสร": [3, 10, 13]
  }
  ```
- เพิ่มลอจิกรองรับระบบ Auto-Migration ให้เปลี่ยนของเก่ามาเป็นฟอร์แมตใหม่ได้อัตโนมัติ

### 2. หน้าจอผู้ใช้งาน (UI / Frontend)
- เมื่อกดปุ่ม ⚙️ **Edit Schedule Times** ระบบจะบังคับให้คุณคลิกเลือกแท็บชื่อเพจที่ต้องการด้านซ้ายบน (เช่น 'ฟ้า นภัสสร') ก่อนเข้าสู่โหมดแก้ไข
- ถ้ายังอยู่หน้า "🌍 All Pages" ระบบจะมีหน้าต่างแจ้งเตือนทันที เพื่อป้องกันการแก้ผิดเพจ
- เวลาใน Modal จะดึงมาเฉพาะเวลาของเพจที่คุณกำลังกดดูอยู่เท่านั้น
- ระบบจะคำนวณรอบ Post (Next Slots) แยกกันตามเพจอย่างถูกต้อง แม้ในหน้า Dashboard รวม

### 3. ระบบ Auto-Schedule
- การกดปุ่ม **⏱️ Auto Schedule** หรือการส่งมาจาก `/rawgirl` ตอนนี้ระบบจะทำการจัดกลุ่ม (Group) วิดีโอตามหน้าเพจก่อน จากนั้นจึงค่อยหา Slot เวลาที่ว่างของแต่ละเพจแยกกัน แล้วจึงบันทึกผล ทำให้สามารถตั้งโพสต์อัตโนมัติพร้อมกันหลายๆ เพจใน Batch เดียวได้โดยที่คิวไม่ตีกัน

# V12.19.0: Mission Control Cross-Page Repost

## 1. Problem Statement
The existing "Repost" feature in Viral Planner Pro only allowed reposting (or sharing) the reel back to its original page. The user requested the ability to cross-post successful reels to multiple other pages simultaneously, with the option to customize captions per page and auto-stagger schedules.

## 2. Implementation Timeline
1. **New Database Column**: 
   - Added `repost_source_id` (INTEGER) to `mc_jobs` to track the original job ID the repost was derived from.
2. **Backend Adjustments (`routes/mission-control.js`)**:
   - Updated `POST /planner/:id/repost` to accept `target_page_name`, overriding the original page.
   - Enforced validation: "Share" type is strictly limited to the original page (Facebook API constraint). Cross-page posts are forced to be "Upload as New Reel".
   - Added `GET /planner/:id/repost-history` to fetch all jobs derived from a specific `repost_source_id`.
3. **Frontend UI Redesign (`mission-control.css` & `mission-control.js`)**:
   - Replaced the single-column modal with a `.vp-repost-split` layout featuring a 260px left sidebar for selecting target pages.
   - Populated the sidebar dynamically using `mcFbChannels`.
   - Built a dropdown "Caption Tab" switcher to edit captions individually for each selected page.
   - Implemented `vpRepostSubmit()` which iterates over selected pages, auto-staggers the `scheduled_at` times by +5 minutes per page, and displays a dynamic progress bar replacing the footer buttons during submission.

## 3. Key Decisions & "No Magic" Declarations
- **Auto-Stagger Logic**: Automatically adds 5 minutes sequentially to each selected page's post time, starting from the base selected time.
- **Cross-page Sharing Constraints**: As per Facebook API limits, we cannot share a reel to a page different from the original publisher. The UI logic gracefully downgrades cross-page reposts to "New Reel" upload mode even if "Share" is selected globally.

## 4. Post-Implementation Verification
- Code successfully updated and cache bumped.
- VPS Deployment completed (`git push vps`).
- The modal correctly displays a multi-select sidebar and supports per-page caption edits.

> [!IMPORTANT]
> **Cross-Page Reel Sharing Limitation:**
> Facebook Graph API does not natively allow page A to organically "share" page B's reel directly onto its own reels feed in the same way a user shares a post.

> Therefore, for **cross-page** reposting, the system MUST use "Upload as New Reel" (new_reel). If you select multiple pages, the system will upload the video as a fresh reel for the other pages, rather than "sharing" the existing link. The original page can still use the "Share" option.

# Goal: Cross-Posting (Multi-Page Post) for Unposted Content

The user wants to take content that **has never been posted before** and distribute it across multiple pages, similar to the existing "Repost" feature, while keeping the UX intuitive and easy to understand.

## Proposed Features & UX

To make it **ไม่ งง (not confusing)** and **ใช้ง่าย (easy to use)**, we should treat "Reposting an old video" and "Cross-posting a new video" as the **same core action**: Duplicating a job to other pages. 

Here is the proposed feature set:

### 1. Context Menu Integration
- **If the video is already Posted:** The menu shows `🔁 Repost (รีโพสต์)`
- **If the video is New/Unposted:** The menu shows `👯 Duplicate to Pages (ก๊อปปี้ไปเพจอื่น)`
- Both buttons will open the **same powerful modal**.

### 2. Smart Modal (Adaptive UI)
We will reuse the existing Repost Modal, but it will adapt its text based on the job's status:
- **Title for New Videos:** "Duplicate to Other Pages"
- **Title for Posted Videos:** "Repost to Other Pages"
- **Target Pages:** A checkbox list of all pages (excluding the current page).
- **Caption:** Allows you to tweak the caption before duplicating.
- **Schedule & Auto-Stagger:** This is the killer feature. You can select "Schedule First Post At..." and "Stagger by 1 Hour". The system will automatically calculate the queue for all selected pages!

### 3. Workflow Example for New Content
1. You have a brand new video `Cat_Funny.mp4` on "Page A".
2. You right-click -> `👯 Duplicate to Pages`.
3. You select "Page B" and "Page C".
4. You set the start time for tomorrow at 12:00 PM, stagger by 2 hours.
5. Click **Duplicate**.
6. **Result:** The system creates a job for Page B at 12:00 PM, and a job for Page C at 02:00 PM. (Your original job on Page A remains untouched, you can post it whenever you want).

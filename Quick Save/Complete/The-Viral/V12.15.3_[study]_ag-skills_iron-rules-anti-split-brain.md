# 🧠 Reviewchat: Iron Rules 17, 18, 19 (Viral Planner)

## 📌 Context (Compiled Truth)
Following the debugging session for Viral Planner's per-page scheduling and FB Send All feature, a `/reviewchat` was executed to identify AI missteps. Three critical failure patterns were found:
1. **Architecture Mismatch:** UI wrote FB Tokens to DB, but backend read them from a JSON config.
2. **State Bleed:** Per-page schedule state was stored globally, causing cross-tab bleed.
3. **Silent Failures:** Batch API failures in `Send All` were hidden behind a generic catch block.

These were formulated into 3 new Iron Rules and appended to `P:\AI\The Viral\memory.md`:
- **Iron Rule 17: Two-Source Data Consistency Check (Anti-Split-Brain)**
- **Iron Rule 18: UI State Must Be Per-Instance, Never Global (Anti-Bleed)**
- **Iron Rule 19: Surface Real Error Messages to UI (Anti-Silent-Fail)**

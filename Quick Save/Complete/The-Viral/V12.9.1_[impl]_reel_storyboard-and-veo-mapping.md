# V12.9.1 [impl] Reel Pipeline: Storyboard Auto-Gen & Veo 3.1 Mapping

## 📌 Context (Compiled Truth)
This implementation expands upon the Local Reel Pipeline (V12.9.0) by fully integrating Phase 6 (Automated Storyboard Image Generation) and Phase 7 (Veo 3.1 I2V Sequence Mapping), as well as finalizing the missing data columns in the Google Sheet sync.

The user desired a more end-to-end "one-click" experience. After extracting a reel's blueprint JSON, the system must visually generate a 9:16 (2x2) high-quality cinematic storyboard to visualize the scene composition. It also generates the exact execution tasks for Veo 3.1 using advanced First-Frame / End-Frame continuity locks to guarantee seamless character motion.

### Key Upgrades:
1. **Google Sheets Sync Perfection**:
   - `routes/reel-blueprints.js` on the VPS was updated to push two previously missing JSON fields: `short_story` and `story_scene_th`.
   - A new column (Column P) `AI Storyboard URL` was added to the Google Sheet.
2. **Phase 6 - Storyboard Generation & Storage**:
   - The AG Native `generate_image` tool (or Codex GPT) is now used to generate a 9:16 layout grid depicting the scenes (maintaining strict character consistency like "Holy Man").
   - Images are saved locally to `P:\AI\The Viral\Reel Original\Processed\Storyboards\`.
   - VPS Route `POST /upload-storyboard/:id` was created to accept this image, save it securely to `public/blueprints/`, and update a newly added `ai_storyboard_image_path` column in the SQLite DB.
   - The Google Sheet sync automatically computes a deterministic URL (`https://brain.doctorbankonline.com/blueprints/storyboard_${id}.jpg`) for the image at import time.
3. **Phase 7 - Veo 3.1 Dynamic Sequence Mapping**:
   - Instead of a rigid template, the pipeline dynamically analyzes the original clip's continuity.
   - For standard cutaways: `First Frame -> Video`.
   - For continuous interacting actions (e.g., reaching out and holding): `First Frame -> End Frame -> Video` (Interpolation).

<summary>routes/reel-blueprints.js (Storyboard Upload Endpoint Snippet)</summary>

```javascript
  // Upload generated storyboard image
  app.post('/upload-storyboard/:id', async (c) => {
    try {
      const id = c.req.param('id');
      const formData = await c.req.formData();
      const file = formData.get('storyboard');

      if (!file || file.size === 0) {
        return c.json({ error: 'No file uploaded' }, 400);
      }

      const publicDir = path.resolve(process.cwd(), 'public', 'blueprints');
      if (!fs.existsSync(publicDir)) fs.mkdirSync(publicDir, { recursive: true });
      
      const fileName = `storyboard_${id}.jpg`;
      const filePath = path.join(publicDir, fileName);
      fs.writeFileSync(filePath, Buffer.from(await file.arrayBuffer()));

      db.prepare('UPDATE reel_blueprints SET ai_storyboard_image_path = ? WHERE id = ?').run(filePath, id);

      return c.json({ success: true, path: filePath, url: `/blueprints/${fileName}` });
    } catch (e) {
      console.error('[Upload Storyboard Error]', e);
      return c.json({ error: e.message }, 500);
    }
  });
```

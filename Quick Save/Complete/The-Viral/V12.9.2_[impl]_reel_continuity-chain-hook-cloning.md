# V12.9.2 [impl] Reel Pipeline: Continuity Chain & 100% Hook Cloning

## 📌 Context (Compiled Truth)
This implementation focuses on refining the prompting logic and execution constraints of the Local Reel Pipeline (V12.9.0 & V12.9.1). The user observed two critical flaws in the AI's processing of viral blueprints:
1. **Weak Hooks (Over-summarization):** The AI was summarizing extreme visual pain points (e.g., "An old man struggling without arms") instead of extracting the exact raw details (e.g., "A man missing both arms, using his mouth to bite a rope to fetch water, with a face of extreme despair").
2. **Jump-Cut Generation (Mid-Action Flaws):** The generated storyboards were mid-action, and feeding them to Veo 3.1 created jump cuts because they lacked true continuity anchors.

To solve this, we implemented the **Continuity Chain (Anchor Method)** and strict **100% Visual Clone** mandates into the extraction prompt.

### Key Upgrades:
1. **100% Visual Clone for Hook (Scene 1):**
   - Modified `master_instruction.txt` and `blueprint.py` to forbid loose summarization.
   - The AI must now extract 100% of the physical struggle, raw facial emotions, and grim atmosphere from the original video's first frame to ensure maximum "thumb-stopping" power.
2. **Pre-Action Storyboard Frames:**
   - Enforced a rule that every panel in the 6-scene storyboard must depict the *setup* of the scene (Pre-Action), not a mid-action frame. This serves as a pure First Frame anchor for Veo 3.1.
3. **The Continuity Chain (Cross-Scene Anchor Method):**
   - Transformed the `veo_sequence` JSON logic. Instead of a generic `First Frame -> Video` output, the AI now uses frames from previous or subsequent scenes to act as anchors for interpolation.
   - Example (The Miracle Bridge): Scene 4 uses `First (Image 3) -> End (Image 5) -> Video`. This forces Veo 3.1 to retain the exact background and character positions from Image 3, animate the miracle, and land perfectly on Image 5, preventing any hallucinated environment changes.
4. **Flexible Narrative Structure:**
   - Removed the rigid 6-scene template requirement (e.g., forcing a Flashback in S3). The prompt now instructs the AI to follow the exact narrative progression of the original video (e.g., replacing Flashback with Entrance if the source video dictates).

```text
PHASE 2: 🖼️ SCENE → IMAGE PROMPT
═══════════════════════════════════════════
For EACH scene, write an Image Prompt describing the FIRST FRAME (Pre-Action).
- The image MUST be the literal starting setup (Pre-Action), not mid-action.
- FOR SCENE 1 (HOOK): You MUST CLONE 100% of the extreme visual details from the video grid. Do NOT summarize loosely. Describe the exact physical struggle (e.g. biting rope, no arms) and extreme facial sorrow to guarantee maximum hook power.
- Must lock Holy Man's appearance according to character rules.
- Must be pure visual description (no motion verbs like "walking", "talking").
- No camera instructions (put those in Phase 3).
- Strict prefix: C-XXX-NN (e.g., C-001-01, C-001-02).

═══════════════════════════════════════════
PHASE 3: 🎥 IMAGE → MOTION PROMPT
═══════════════════════════════════════════
For EACH scene, write the Motion Prompt extending from the Image Prompt.
- State duration in seconds.
- Define camera movement (e.g., Static, Slow Pan Right).
- Define subject motion (e.g., Holy Man gently touches eyes).
- Continuity check: verify it flows from previous scene.
- Veo Sequence Mapping (veo_sequence): Use the 'Cross-Scene Anchor Method'. To ensure 100% continuity, define which Keyframes (Images) to use as First and End frames.
  * For standard cutaways: `First (Image X) -> Video`
  * For continuous connecting actions (e.g., connecting Scene 1's pain to Scene 3's entrance, or Scene 3's touch to Scene 5's restored embrace): `First (Image X) -> End (Image Y) -> Video`. 
  * Example Miracle Sequence: S1=`First (Image 1) -> Video`, S2=`First (Image 2) -> Video`, S3=`First (Image 1) -> End (Image 3) -> Video`, S4=`First (Image 3) -> End (Image 5) -> Video`, S5=`First (Image 5) -> Video`, S6=`First (Image 6) -> Video`.
- Same C-XXX-NN prefix as Phase 2.
```

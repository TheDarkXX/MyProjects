# V1.1.0 HyperCut: Cut Lab Silence Detection & UI Polish

## 📌 Context (Compiled Truth)
The user required a professional "Timebolt-style" silence removal lab within HyperCut. 
Several critical refinements were made to ensure pixel-perfect accuracy and high-fidelity visualization:
1. **Waveform Fidelity**: Switched from drawing blocky rectangular max-peaks to drawing true high-fidelity min-max paths (`lineTo`), creating a detailed squiggle waveform.
2. **A/V Sync Drift Fix**: Locked `samplesPerPixel` directly to `videoDuration` rather than array length, fixing minor drift issues caused by variable framerates.
3. **Padding Logic Correction**: 
   - Fixed the Pre-pad and Post-pad visual representation. 
   - `postPad` expands the right edge of the speech (shrinking the left edge of silence: `actualStart = s.start + postPad`).
   - `prePad` expands the left edge of the speech (shrinking the right edge of silence: `actualEnd = s.end - prePad`).
   - Mapped these correctly to `auto-editor --margin A,B` (where A = prePad, B = postPad).
4. **Visual Aids**: Added a dashed yellow horizontal **Threshold Line** to visually prove to the user why certain "flat" areas are not cut (because background noise exceeds the low `-35dB` threshold).
5. **Waveform Zoom**: Added a "Waveform Zoom" slider (1x - 10x) to visually multiply the vertical scale, making it easy to inspect quiet clips.
6. **Audio Normalize (EBU R128)**: Added a toggle to append `--audio-normalize ebu` to `auto-editor`, ensuring the final exported video has broadcast-standard volume.

```markdown
## Round 2: Fixes Applied

### 🐛 Bug Fix: Video & Waveform Sync Issue (Root Cause)
**Problem:** Waveform was drawn BEFORE `videoDuration` was available from the `<video>` element. This caused `getDynamicWidth()` to return the wrong width, so the waveform was drawn at the wrong scale — making it look like a completely different file.

**Fix:** Implemented a `tryDrawAll()` gate that waits for BOTH:
1. Audio peaks data (from IPC call)
2. Video metadata (from `<video>` loadedmetadata event)

Only when both are ready does it draw the waveform and silence map, ensuring perfect alignment.

### 🎯 Auto-Threshold Suggestion
Added `autoSuggestThreshold()` that:
- Calculates the RMS (Root Mean Square) of all audio peaks
- Converts to dB scale
- Sets a smart threshold at `RMS - 6dB`
- Automatically updates the Threshold slider when video loads

This means the silence detection threshold adapts to each video's audio level.

### 📐 Waveform Height
Reduced from 150px → 110px to better fit the waveform content.

### 🔄 Scan → Redraw Both
After scanning silence, both waveform AND silence map are redrawn together to ensure they always use the same canvas dimensions.
```

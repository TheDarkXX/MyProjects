# 💾 Quick Save: V1.3.2 - Session 0 Isolation & UpNote Script Formatting

## 🎯 วัตถุประสงค์ (Objective)
สรุปบทเรียนสำคัญจากการทดสอบเชื่อมต่อระบบอัตโนมัติ (Automation) ระหว่าง AI (Antigravity IDE) กับแอปพลิเคชันบนเครื่องผู้ใช้ (UpNote) เพื่อใช้อ้างอิงในการออกแบบ Workflow การส่งมอบสคริปต์ในอนาคต

## 💡 สิ่งที่ค้นพบและบทเรียน (Key Discoveries & Limitations)

### 1. ระบบ Session 0 Isolation ของ Windows
* **ปัญหา:** ความพยายามในการสั่งให้คอมพิวเตอร์เปิดแอป UpNote เพื่อสร้างโน้ตใหม่ (ทั้งผ่าน URL Scheme `x-callback-url` และการใช้บอท `pyautogui`) ไม่สามารถแสดงผลให้ผู้ใช้เห็นบนหน้าจอได้
* **สาเหตุ:** ระบบ AI (Antigravity IDE) ทำงานในรูปแบบ **Headless Background Service** ซึ่งถูกจัดให้อยู่ใน **Session 0** ของ Windows OS
* **ผลลัพธ์จาก Session 0:** Windows ตัดขาด Session 0 ออกจากหน้าจอ Desktop จริงของผู้ใช้ (Session 1) เพื่อความปลอดภัย (Session 0 Isolation) ทำให้:
    * การรัน `os.startfile('upnote://...')` จะไปเปิดแอปในหน้าจอจำลอง (Invisible Desktop)
    * การใช้ `pyautogui` ก็จะขยับเมาส์และกดคีย์บอร์ดในหน้าจอจำลองเช่นกัน
    * แม้จะใช้โค้ดระดับลึก (ctypes `AttachThreadInput` / `SetForegroundWindow`) เพื่อกระชากหน้าต่าง ก็ไม่สามารถข้ามกำแพง Session 0 มาโผล่บน Session 1 ได้
* **แนวทางแก้ไขที่ดีที่สุด (Best Practice):** การพยายามเจาะข้าม Session อัตโนมัติมีความเสี่ยงสูงที่จะถูก Windows Defender บล็อก จึงเปลี่ยนมาใช้ท่า **1-Click Button** โดยให้ AI เขียนบอทแพ็คใส่ไฟล์ `Auto-Send-To-UpNote.bat` แล้วให้มนุษย์ (ผู้มีสิทธิ์เต็มใน Session 1) เป็นคนดับเบิ้ลคลิกรันบอทด้วยตัวเอง ซึ่งทำงานได้เสถียรและปลอดภัยกว่า 100%

### 2. ข้อจำกัดของ URL Scheme (`x-callback-url`) กับภาษาไทย
* การส่งสคริปต์ความยาว 1,500 ตัวอักษร (ภาษาไทย) ผ่าน URL ส่งผลให้เมื่อแปลงเป็น URL Encoding (เช่น `%E0%B8%...`) ความยาวจะพุ่งทะลุ 13,000 ตัวอักษร
* Windows มีขีดจำกัดความยาวของ URL ในการรันคำสั่ง (ประมาณ 2,048 ตัวอักษร) ทำให้คำสั่งถูกตีตกและแอปไม่เปิด (Fail Silently)

### 3. การปรับรูปแบบสคริปต์ให้รองรับระบบ Infinity
* ได้ทำการปรับเปลี่ยนตารางเปรียบเทียบในเอกสารวิดีโอ (จากรูปแบบ Markdown Table) ให้กลายเป็น **Plain Text Bullet Points** 
* **ประโยชน์:** ป้องกันปัญหาเลย์เอาต์แตกเมื่อผู้ใช้นำไปคัดลอกลงใน Description Column ของระบบจัดการโปรเจกต์ที่อาจไม่รองรับ Markdown Render (เช่น Infinity)

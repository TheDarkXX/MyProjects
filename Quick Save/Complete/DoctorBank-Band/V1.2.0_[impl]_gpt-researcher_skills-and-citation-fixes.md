# Deep Medical Research Citation Fix & AG Skills

## 📌 Context (Compiled Truth)
The user required "Perplexity-style" deep medical research with at least 15-30 unique academic references. Previously, `gpt-researcher` was consolidating citations resulting in only 8-9 references despite retrieving 80+ URLs. Furthermore, attempting to increase `MAX_ITERATIONS` caused silent 38-minute hangs.

### Root Cause Analysis
1. **The 38-Minute Hang:** A hardcoded `os.environ["CURATE_SOURCES"] = "True"` in `medical_research.py` was silently overriding the `.env` configuration. When `MAX_ITERATIONS` was increased to 12, this curation process was fed 130+ URLs, blowing past the LLM context window and timing out.
2. **The Low Citation Count:** The LLM was summarizing too heavily. The `MEDICAL_ROLE` prompt did not have a strict minimum threshold for references, allowing it to merge distinct papers into single citations.
3. **Verbosity vs Concision:** The report hit 4,600+ words because of a `TOTAL_WORDS=3000` target, leading to repetition and "word salad". 

### Implementation Strategy
1. **Removed the Hardcoded Override:** Deleted the `os.environ["CURATE_SOURCES"]` line in `medical_research.py` so it properly respects the `.env` value (which must be `False` for large queries).
2. **Strict Minimum Requirements:** Injected `MINIMUM 15 UNIQUE REFERENCES REQUIRED` into the `MEDICAL_ROLE` prompt, forcing the LLM to separate out references.
3. **Conciseness Protocol:** Dropped target word count to 2,000-2,500 and added anti-hedging/anti-verbosity rules, successfully reducing the final word count from 4,600 to 2,900 while maintaining 16 references and 126 inline citations.
4. **Systemic Defenses (Iron Rules):** Added Rule B (Env Override Audit) to check for hardcoded overrides before `.env` changes, and Rule C (Proportional Scaling) to calculate total URLs before increasing batch sizes.
5. **Skill Automation:** Created `/web`, `/res`, and `/deep` skills to allow the user to easily trigger different tiers of research depth natively from AG.

## Proposed Changes

### 1. Configuration Changes (`.env`)
แก้ไขไฟล์ `.env` เพื่อเพิ่มขีดจำกัดการค้นหาและบังคับรูปแบบรายงาน:

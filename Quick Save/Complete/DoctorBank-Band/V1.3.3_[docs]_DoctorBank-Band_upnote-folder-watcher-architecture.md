# 💾 Quick Save: V1.3.3 - UpNote Folder Watcher Architecture

## 🎯 วัตถุประสงค์ (Objective)
ออกแบบสถาปัตยกรรมระบบ 100% Full Auto สำหรับการ Push สคริปต์วิดีโอไปยัง 2 แพลตฟอร์มพร้อมกัน (Infinity & UpNote) โดยก้าวข้ามข้อจำกัดของ Windows Session 0 Isolation

## 🏗️ สถาปัตยกรรม "สายลับคนกลาง" (Folder Watcher Pattern)

เพื่อให้ AI (ที่อยู่ใน Session 0) สามารถสั่งงาน UpNote (ที่อยู่ใน Session 1) ได้แบบอัตโนมัติ 100% โดยที่ผู้ใช้ไม่ต้องคลิกปุ่มใดๆ จะใช้สถาปัตยกรรมดังนี้:

### 1. โฟลเดอร์พักข้อมูล (Queue Directory)
* **Path:** `C:\My Claw\Openclaw-VPS\Quick Save\Upnote_Sync_Queue\`
* ทำหน้าที่เป็นจุดส่งมอบข้อมูล (Hand-off point) ระหว่างระบบ 2 Session

### 2. บอทเฝ้าโฟลเดอร์ (Session 1 Watcher)
* **เครื่องมือ:** Python Script (`watch_upnote.py`) คู่กับไลบรารี `watchdog`, `pyautogui`, `pyperclip`
* **การติดตั้ง:** วาง Shortcut ของ Script ไว้ในโฟลเดอร์ `shell:startup` ของ Windows เพื่อให้โปรแกรมรันอัตโนมัติเป็น Background Process บนหน้าจอหลัก (Session 1) ทันทีที่ผู้ใช้เปิดเครื่อง
* **พฤติกรรม:** 
  1. เฝ้าดูโฟลเดอร์ `Upnote_Sync_Queue`
  2. เมื่อพบไฟล์ `.md` หรือ `.txt` ถูกสร้างขึ้นใหม่
  3. ดึงข้อความในไฟล์มาไว้ใน Clipboard
  4. เรียกหน้าต่าง UpNote ขึ้นมา (Force Foreground)
  5. สั่งจำลองคีย์บอร์ด `Ctrl+N` และ `Ctrl+V` เพื่อสร้างโน้ตใหม่
  6. ลบไฟล์ทิ้งหลังจาก Sync เสร็จสิ้น

### 3. กระแสข้อมูลจากฝั่ง AI (Session 0 Trigger)
เมื่อ AI ร่างสคริปต์เสร็จสิ้น จะทำการสั่ง Action คู่อัตโนมัติ:
1. **API Call:** ยิงข้อมูลสคริปต์ผ่าน Webhook/API เข้าไปบันทึกใน Board ของระบบ **Infinity** เพื่อการบริหารจัดการโปรเจกต์
2. **File Write:** สร้างไฟล์ข้อความและบันทึกลงในโฟลเดอร์ `Upnote_Sync_Queue` เพื่อเป็นการกระตุก (Trigger) ให้ Watcher Script ในฝั่งผู้ใช้ทำงาน

## 📌 แผนงานขั้นต่อไป (Next Steps @ Office)
1. เขียนสคริปต์ `watch_upnote.py` แบบสมบูรณ์ พร้อมระบบป้องกันข้อผิดพลาด (Fail-safes)
2. สร้างและทดสอบโฟลเดอร์คิว
3. เซ็ตอัปการทำงานกับ Infinity API
4. ตั้งค่าให้สคริปต์สายลับรันอัตโนมัติตอนเปิดเครื่อง (Startup)

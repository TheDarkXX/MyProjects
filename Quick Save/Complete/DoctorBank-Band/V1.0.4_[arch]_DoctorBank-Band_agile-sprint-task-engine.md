# V1.0.4 [arch] DoctorBank-Band Agile Sprint Task Engine

## 📌 Context (Compiled Truth)
- **Problem**: The original "Road to 1M" dashboard hardcoded tasks in `doctorbank.js`. Day 4+ simply repeated the exact same routine tasks with no variation. W1+ overdue checking logic was broken.
- **Strategic Pivot (Agile Sprint)**: Instead of attempting to hardcode the entire 12-week schedule, the project pivots to a **Rolling 2-Week Sprint** methodology.
- **Database-Driven Tasks**: Tasks will no longer live in JS code. They will be written directly to the database. This allows AG to dynamically schedule tasks without requiring a VPS code deployment.
- **Sunday Review System**: Every Sunday, a review session takes place. The user and AG will review the previous week's performance, tackle obstacles, and plan out the detailed content calendar for the upcoming sprint.
- **Task Flexibility**: Added a `[❌ Skip]` button per task so uncompleted tasks don't pile up as "Overdue". Additionally, macro-features (like LIVE, Lazada, Ads) will be toggled via a Settings panel.

## Proposed Changes

### 1. Refactor `getDbWins()` Architecture

แทนที่จะ hardcode tasks ลงใน function ตรงๆ → แยก data layer ออกมาเป็น 3 ส่วน:

```
┌─────────────────────────────────────┐
│  DATA: Content Calendar Registry    │ ← W1-W12 หัวข้อคลิปตาม execution plan
│  DATA: Daily Routine Template       │ ← Morning Ops, ถ่ายคลิป, ตัดต่อ, etc.
│  DATA: Weekly/Phase Tasks           │ ← Affiliate push, Analytics review, etc.
└──────────┬──────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│  getDbWins(dayNum)                  │
│  1. คำนวณ weekNum + dayOfWeek      │
│  2. ดึง Content Calendar ของ week  │
│  3. สร้าง tasks จาก Daily Routine  │
│  4. เพิ่ม Weekly/Phase tasks       │
│  5. ใช้ dayNum-based ID (ไม่ใช่    │
│     todayKey) → Overdue ทำงานถูก   │
└─────────────────────────────────────┘
```

### 2. แก้ Task ID ให้ใช้ `dayNum`

**ปัจจุบัน (bug):**
```js
id: `${dbTodayKey()}-1`  // e.g. "2026-6-25-1" ← ใช้วันจริงของ "วันนี้" เสมอ
```

**แก้เป็น:**
```js
id: `d${dn}-1`  // e.g. "d8-1", "d9-1" ← ใช้ dayNum ที่ส่งเข้ามา
```

### 4. Daily Routine Tasks (ทุกวัน จ.-ศ.)

สร้าง tasks ตาม Daily Routine จากแผน:

```js
function getDailyRoutineTasks(dn, dow, weekNum) {
  const tasks = [];
  
  // ทุกวัน: Morning Ops
  tasks.push({
    id: `d${dn}-ops`,
    text: '☀️ Morning Ops: เช็คออเดอร์ + แพ็คส่ง + ตอบ Chat',
    sub: '08:30 — 45 นาที',
    impact: 'routine', quadrant: 'do',
    tasks: [
      'เช็คออเดอร์ทุก platform (TikTok + Shopee + Lazada)',
      'แพ็คสินค้า + พิมพ์ใบจ่าหน้า + ส่งขนส่ง',
      'ตอบแชทลูกค้า (ตอบใน 1 ชม.)',
    ]
  });

  // ทุกวัน: ถ่าย+ตัด Content (หัวข้อดึงจาก Calendar)
  const calEntry = getCalendarEntry(weekNum, dow);
  tasks.push({
    id: `d${dn}-content`,
    text: `🎬 ถ่าย+ตัดคลิป: ${calEntry.topic}`,
    sub: calEntry.detail,
    impact: 'revenue', frog: true, quadrant: 'do',
    tasks: [...calEntry.steps]
  });

  // ทุกวัน: Night Check
  tasks.push({
    id: `d${dn}-night`,
    text: '🌙 Night Check: analytics + ตอบ comment',
    sub: '22:00 — 30 นาที',
    impact: 'routine', quadrant: 'do',
    tasks: []
  });

  // วันพุธ: LIVE
  if (dow === 3) {
    tasks.push({
      id: `d${dn}-live`,
      text: '🔴 LIVE ขาย 20:00 (2 ชม.)',
    });
  }

  // วันจันทร์: Weekly Review
  if (dow === 1) {
    tasks.push({
      id: `d${dn}-review`,
      text: '📊 Weekly Review: สรุปสัปดาห์ที่ผ่านมา',
    });
  }

  // วันศุกร์: Plan next week + ตั้งเวลาคลิปวีคเอนด์
  if (dow === 5) {
    tasks.push({
      id: `d${dn}-plan`,
      text: '📋 Plan สัปดาห์หน้า + ตั้งเวลาคลิป ส.-อา.',
    });
  }

  return tasks;
}
```

## 🔄 แผนใหม่: Rolling 2-Week Sprint

### แนวคิดหลัก

```
          สัปดาห์นี้           สัปดาห์หน้า         สัปดาห์ถัดไป
         (กำลังทำ)            (วางแผนแล้ว)         (ร่างคร่าวๆ)
      ┌────────────┐       ┌────────────┐       ┌────────────┐
      │ W1 จ.-ศ.   │       │ W2 จ.-ศ.   │       │ W3 จ.-ศ.   │
      │ ทำงานจริง   │       │ detail ครบ  │       │ draft      │
      └─────┬──────┘       └────────────┘       └────────────┘
            │
            │  📅 วันอาทิตย์ = Review Day
            │
            ▼
      ┌──────────────────────────────────────────┐
      │  🧠 Sunday Review Session                │
      │                                          │
      │  1. สรุปยอด: ขายได้เท่าไร? คลิปไหนปัง?   │
      │  2. ปัญหา/อุปสรรค อะไรค้าง?              │
      │  3. ข้อมูลใหม่: trend? ลูกค้าถามอะไร?    │
      │  4. ปรับแผน W2 (ที่วางไว้แล้ว)           │
      │  5. ร่างแผน W3 (สัปดาห์ใหม่)             │
      └──────────────────────────────────────────┘
```

**หลักการ:**
- **ไม่ hardcode 12 สัปดาห์ล่วงหน้า** → วางแค่ 2 สัปดาห์ข้างหน้า
- **ทุกวันอาทิตย์** คุณแบงค์แวะมาคุย → อัปเดตสถานการณ์จริง
- AG ช่วยสรุป + เสนอแผนสัปดาห์ใหม่ → คุณแบงค์ approve/ปรับ
- Tasks ถูกเขียนลง **database** (ไม่ใช่ hardcode ใน JS อีกต่อไป)

### ระบบจะทำงานยังไง

**1. Tasks เก็บในฐานข้อมูล (ไม่ใช่ hardcode)**
- tasks อยู่ใน database → AG เขียนเข้าได้เลย ไม่ต้อง deploy

**2. Sunday Review = คุยกับ AG ปรับแผน**
- ทุกวันอาทิตย์ AG จะ: ดึงข้อมูลสัปดาห์ที่ผ่านมา ถามคำถามเป็นระบบ เสนอแผนสัปดาห์หน้า และเขียนลง DB

**3. Cron Reminder วันอาทิตย์**
- ตั้ง cron แจ้งเตือนผ่าน Discord ทุกวันอาทิตย์

**4. Settings & Skip**
- มีปุ่ม `[❌ ข้าม]` นอกเหนือจาก `[✅ เสร็จ]`
- มี toggle เปิด/ปิดฟีเจอร์ที่ยังไม่พร้อมใน Settings เพื่อไม่ให้ Task รกกระดาน (เช่น LIVE, Lazada Ads)

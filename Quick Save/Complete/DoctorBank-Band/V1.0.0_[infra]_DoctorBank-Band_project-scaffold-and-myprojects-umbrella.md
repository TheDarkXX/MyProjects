# Project Scaffold & MyProjects Umbrella Architecture

## 📌 Context (Compiled Truth)
- **Goal**: Initialize the `DoctorBank-Band` project (a Brand/Business project) with no initial Git intent, but later the user realized they need to sync across 2 PCs (Home & Office).
- **The Problem**: A `git init` per small project creates Git repository bloat (e.g. 5 projects = 5 repos to push/pull).
- **The Solution**: An "Umbrella Repository" pattern (`c:\My Claw\MyProjects\`). All small projects live inside this single Git repo as subfolders.
- **Architectural Shift**: 
  - Global files (like `.code-workspace`, `docs`, `scratch`, `Quick Save` root, and `ag_skills_backup`) were moved to the `MyProjects` root.
  - Per-project files (`AGENT_CONTEXT.md`, `memory.md`, and business folders like `Brand`, `Products`, `Sales`) stay in their respective subfolders.
  - A **Focus Lock Protocol** was established via the root `AGENT_CONTEXT.md` Router. The agent must lock onto a specific project upon opening the workspace, ensuring tools like `/save` and `/qc` execute within the correct scope without writing to the wrong directory.
- **Skill Updates**: 
  - `save.md` was rewritten to be Focus Lock-aware and includes all 5 Iron Rules from Openclaw's `/save` (YAML, RAW BACKUP, etc.). It saves locally, syncs to `Openclaw-VPS/Quick Save` (the headquarters), and does a single `git push` for the entire `MyProjects` repo.
  - `newpj.md` was updated to scaffold only per-project directories/files instead of duplicating global skills/folders.
